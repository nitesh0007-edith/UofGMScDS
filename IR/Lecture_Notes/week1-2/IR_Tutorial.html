1. Introduction to Information Retrieval

What is Information Retrieval?

Information Retrieval (IR) is the science of searching for information in documents, searching for documents themselves, searching for metadata that describes documents, or searching within databases, whether relational stand-alone databases or hypertext networked databases such as the World Wide Web.

1.1 Core Concepts

Information Need vs Query

Understanding the distinction between what users actually want and what they search for is crucial:

  • Information Need: The actual information requirement that exists in the user's mind
  • Query: The translation of the information need into the formal language of the retrieval system

📝 Real-World Example

Information Need: "I want to learn about how climate change affects polar bear populations in the Arctic"

Query: "climate change polar bears arctic"

Notice how the query is a simplified, keyword-based version of the actual information need!

Relevance: The Heart of IR

Relevance is the key concept in information retrieval. We distinguish between two types:

Type Definition Example
Topical Relevance Document is about the same topic as the query Query: "machine learning" → Document about neural networks
User Relevance Document satisfies the user's actual information need User wants beginner tutorial, gets introductory guide (not research paper)

A document d is represented as:

where \(t_i\) is a term and \(f_i\) is its frequency

2. Information Retrieval Architecture

Overview

An IR system's architecture determines how efficiently it can store and retrieve information. The core component is the inverted index, which maps terms to documents.

2.1 The Inverted Index

An inverted index is the fundamental data structure in IR systems. Unlike a forward index (document → terms), an inverted index maps terms to documents.

The inverted index allows us to quickly find all documents containing a specific term!

Components of an Inverted Index

Component Description Example
Lexicon (Dictionary) List of all unique terms ["apple", "banana", "cherry"]
Postings List For each term, list of documents containing it "apple" → [doc1, doc3, doc7]
Document Index Metadata about documents doc1: {title: "...", length: 500}

Building an Inverted Index: Step-by-Step

📝 Complete Example: Building an Inverted Index

Step 1: Start with Documents

  • Doc1: "The cat sat on the mat"
  • Doc2: "The dog sat on the log"
  • Doc3: "Cats and dogs are friends"

Step 2: Process Each Document (tokenize, remove stopwords, stem)

  • Doc1: ["cat", "sat", "mat"]
  • Doc2: ["dog", "sat", "log"]
  • Doc3: ["cat", "dog", "friend"]

Step 3: Create Term-Document Pairs

(cat, Doc1), (sat, Doc1), (mat, Doc1)
(dog, Doc2), (sat, Doc2), (log, Doc2)

2.2 Document Representation

Documents are represented as vectors in a high-dimensional space, where each dimension corresponds to a term.

For a collection with vocabulary size \(|V|\), each document \(d\) is a vector:

$$\vec{d} = (w_{1}, w_{2}, ..., w_{|V|})$$

where \(w_i\) is the weight of term \(i\) in the document

📝 Document Vector Example

Vocabulary: [cat, dog, friend, log, mat, sat]

Doc1: "cat sat mat"

Vector representation (using binary weights):

Doc1 = (1, 0, 0, 0, 1, 1)

This means Doc1 contains "cat", "mat", and "sat", but not "dog", "friend", or "log".

2.4 Boolean Retrieval

Boolean retrieval uses Boolean operators (AND, OR, NOT) to combine search terms:

Boolean Operations on Postings Lists:

• AND: Intersection of postings lists

$$\text{cat AND dog} = \text{postings}(\text{cat}) \cap \text{postings}(\text{dog})$$

• OR: Union of postings lists

$$\text{cat OR dog} = \text{postings}(\text{cat}) \cup \text{postings}(\text{dog})$$

• NOT: Complement of postings list

$$\text{NOT cat} = \text{all\_docs} - \text{postings}(\text{cat})$$

📝 Boolean Query Example

Index:

  • cat → [Doc1, Doc3, Doc5]
  • dog → [Doc2, Doc3, Doc4]

Query: "cat AND dog"

Result: [Doc3] (only Doc3 contains both terms)

Query: "cat OR dog"

Result: [Doc1, Doc2, Doc3, Doc4, Doc5]

Query: "cat AND NOT dog"

Result: [Doc1, Doc5] (documents with cat but not dog)

3. Term Weighting and Statistical Information

Why Term Weighting?

Not all terms are equally important! Some terms are more discriminative than others. Term weighting assigns numerical weights to terms based on their importance in documents and the collection.

3.1 Statistical Laws in IR

Zipf's Law

Zipf's Law describes the frequency distribution of words in natural language:

Zipf's Law:

$$r \times f \approx k$$

where:

  • \(r\) = rank of the word (most frequent word has rank 1)
  • \(f\) = frequency of the word
  • \(k\) = a constant

Alternative formulation:

$$f \propto \frac{1}{r}$$

📝 Zipf's Law Example

In a corpus of 1 million words:

Rank (r) Word Frequency (f) r × f
1the70,00070,000
2of35,00070,000
10from7,00070,000
100software70070,000
1000algorithm7070,000

Notice how r × f stays approximately constant!

Heap's Law

Heap's Law predicts vocabulary growth as more text is processed:

Heap's Law:

$$V = K \cdot n^{\beta}$$

where:

  • \(V\) = vocabulary size (number of unique words)
  • \(n\) = total number of words in the collection
  • \(K\) = constant (typically 10-100)
  • \(\beta\) = constant (typically 0.4-0.6)

📝 Heap's Law Example

Assuming K = 50 and β = 0.5:

Collection Size (n) Vocabulary Size (V)
1,000 words50 × √1,000 ≈ 1,581
10,000 words50 × √10,000 = 5,000
100,000 words50 × √100,000 ≈ 15,811
1,000,000 words50 × √1,000,000 = 50,000

3.2 Term Frequency (TF)

Term Frequency measures how often a term appears in a document.

Raw Term Frequency:

$$\text{tf}(t, d) = \text{count of term } t \text{ in document } d$$

Normalized Term Frequency:

$$\text{tf}(t, d) = \frac{\text{count of } t \text{ in } d}{\text{total terms in } d}$$

Log-scaled Term Frequency:

$$\text{tf}(t, d) = \begin{cases} 1 + \log(\text{count of } t \text{ in } d) & \text{if count} > 0 \\ 0 & \text{otherwise} \end{cases}$$

📝 Term Frequency Example

Document: "information retrieval retrieval systems information"

Total terms: 5

Term Raw TF Normalized TF Log TF
information 2 2/5 = 0.4 1 + log(2) ≈ 1.301
retrieval 2 2/5 = 0.4 1 + log(2) ≈ 1.301
systems 1 1/5 = 0.2 1 + log(1) = 1.0

3.3 Inverse Document Frequency (IDF)

IDF measures how rare or common a term is across all documents. Rare terms are more discriminative!

Inverse Document Frequency:

$$\text{idf}(t) = \log\left(\frac{N}{df_t}\right)$$

where:

  • \(N\) = total number of documents in the collection
  • \(df_t\) = number of documents containing term \(t\)
High IDF = Rare term = More discriminative = More valuable for retrieval!

📝 IDF Calculation Example

Collection: N = 10,000 documents

Term Documents Containing (df) IDF Interpretation
the 9,000 log(10,000/9,000) ≈ 0.046 Very common, low value
information 1,000 log(10,000/1,000) = 1.0 Moderately common
algorithm 100 log(10,000/100) = 2.0 Fairly rare, good discriminator
quantum 10 log(10,000/10) = 3.0 Very rare, excellent discriminator

3.4 TF-IDF Weighting

TF-IDF combines both term frequency and inverse document frequency to give a balanced weight:

TF-IDF Formula:

$$\text{tf-idf}(t, d) = \text{tf}(t, d) \times \text{idf}(t)$$

Complete Formula (log-scaled TF):

$$\text{tf-idf}(t, d) = (1 + \log(\text{tf}(t, d))) \times \log\left(\frac{N}{df_t}\right)$$

📝 Complete TF-IDF Example

Scenario:

  • Collection size: N = 10,000 documents
  • Document D1 contains 100 words
Term TF in D1 DF Log TF IDF TF-IDF
information 5 1,000 1.699 1.0 1.699
retrieval 3 500 1.477 1.301 1.922
quantum 2 10 1.301 3.0 3.903

Notice: "quantum" has the highest TF-IDF because it's rare (high IDF) even though it appears less frequently in the document!

3.5 Resolving Power

The resolving power of a term measures its ability to discriminate between relevant and non-relevant documents.

Terms with medium frequency have the best resolving power - not too common, not too rare!

4. Vector Space Model

The Geometric View of IR

The Vector Space Model (VSM) represents documents and queries as vectors in a high-dimensional space. Documents similar to the query are "close" in this space!

4.1 Document and Query Vectors

In the VSM, both documents and queries are represented as vectors in a t-dimensional space, where t is the number of unique terms.

Document Vector:

$$\vec{d} = (w_{1,d}, w_{2,d}, ..., w_{t,d})$$

Query Vector:

$$\vec{q} = (w_{1,q}, w_{2,q}, ..., w_{t,q})$$

where \(w_{i,d}\) is the weight of term \(i\) in document \(d\) (often TF-IDF)

📝 Vector Representation Example

Vocabulary: [information, retrieval, database, query, search]

Document 1: "information retrieval and search"

Document 2: "database query search"

Query: "information search"

Term Doc1 TF-IDF Doc2 TF-IDF Query TF-IDF
information2.50.02.5
retrieval2.00.00.0
database0.02.30.0
query0.02.10.0
search1.51.51.5

Vectors:

  • Doc1 = (2.5, 2.0, 0.0, 0.0, 1.5)
  • Doc2 = (0.0, 0.0, 2.3, 2.1, 1.5)
  • Query = (2.5, 0.0, 0.0, 0.0, 1.5)

4.2 Cosine Similarity

Cosine similarity measures the angle between two vectors. It's the most popular similarity measure in IR!

Cosine Similarity Formula:

$$\text{sim}(\vec{q}, \vec{d}) = \frac{\vec{q} \cdot \vec{d}}{|\vec{q}| \times |\vec{d}|}$$

Expanded Form:

$$\text{sim}(\vec{q}, \vec{d}) = \frac{\sum_{i=1}^{t} q_i \times d_i}{\sqrt{\sum_{i=1}^{t} q_i^2} \times \sqrt{\sum_{i=1}^{t} d_i^2}}$$

where:

  • \(\vec{q} \cdot \vec{d}\) = dot product of query and document vectors
  • \(|\vec{q}|\) = magnitude (length) of query vector
  • \(|\vec{d}|\) = magnitude (length) of document vector
Cosine similarity ranges from 0 to 1:
• 1 = vectors point in same direction (very similar)
• 0 = vectors are orthogonal (completely different)
Cosine similarity is independent of document length!

📝 Cosine Similarity Calculation - Step by Step

Given:

  • Query: \(\vec{q}\) = (2, 0, 1)
  • Document: \(\vec{d}\) = (3, 1, 2)

Step 1: Calculate Dot Product

$$\vec{q} \cdot \vec{d} = (2 \times 3) + (0 \times 1) + (1 \times 2) = 6 + 0 + 2 = 8$$

Step 2: Calculate Query Magnitude

$$|\vec{q}| = \sqrt{2^2 + 0^2 + 1^2} = \sqrt{4 + 0 + 1} = \sqrt{5} \approx 2.236$$

Step 3: Calculate Document Magnitude

$$|\vec{d}| = \sqrt{3^2 + 1^2 + 2^2} = \sqrt{9 + 1 + 4} = \sqrt{14} \approx 3.742$$

Step 4: Calculate Cosine Similarity

$$\text{sim}(\vec{q}, \vec{d}) = \frac{8}{2.236 \times 3.742} = \frac{8}{8.367} \approx 0.956$$

Interpretation: Very high similarity (0.956) means the query and document are very similar!

4.3 Other Similarity Measures

Dice Coefficient

Dice Coefficient:

$$\text{Dice}(X, Y) = \frac{2|X \cap Y|}{|X| + |Y|}$$

For vectors:

$$\text{Dice}(\vec{q}, \vec{d}) = \frac{2 \sum_{i=1}^{t} q_i \times d_i}{\sum_{i=1}^{t} q_i^2 + \sum_{i=1}^{t} d_i^2}$$

Jaccard Coefficient

Jaccard Coefficient:

$$\text{Jaccard}(X, Y) = \frac{|X \cap Y|}{|X \cup Y|}$$

For binary vectors (presence/absence of terms):

$$\text{Jaccard}(\vec{q}, \vec{d}) = \frac{\text{number of common terms}}{\text{total unique terms in both}}$$

📝 Comparing Similarity Measures

Query terms: {information, retrieval, search}

Document terms: {information, search, engine}

Jaccard Similarity:

Common terms: {information, search} → |X ∩ Y| = 2

All unique terms: {information, retrieval, search, engine} → |X ∪ Y| = 4

$$\text{Jaccard} = \frac{2}{4} = 0.5$$

Dice Coefficient:

$$\text{Dice} = \frac{2 \times 2}{3 + 3} = \frac{4}{6} = 0.667$$

🏆 Challenge 2: Complete TF-IDF Ranking

Given a collection of 1000 documents and query "neural networks":

  • Doc A: "neural" appears 5 times, "networks" appears 3 times
  • Doc B: "neural" appears 2 times, "networks" appears 8 times
  • "neural" appears in 50 documents total
  • "networks" appears in 100 documents total

Task: Which document ranks higher using TF-IDF cosine similarity?

Step-by-step Solution:

Calculate IDFs:

  • IDF(neural) = log(1000/50) = log(20) ≈ 1.301
  • IDF(networks) = log(1000/100) = log(10) = 1.0

Calculate TF-IDFs for Doc A:

  • TF-IDF(neural) = (1 + log(5)) × 1.301 ≈ 1.699 × 1.301 ≈ 2.210
  • TF-IDF(networks) = (1 + log(3)) × 1.0 ≈ 1.477 × 1.0 ≈ 1.477
  • Doc A vector = (2.210, 1.477)

Calculate TF-IDFs for Doc B:

  • TF-IDF(neural) = (1 + log(2)) × 1.301 ≈ 1.301 × 1.301 ≈ 1.693
  • TF-IDF(networks) = (1 + log(8)) × 1.0 ≈ 1.903 × 1.0 ≈ 1.903
  • Doc B vector = (1.693, 1.903)

Query vector (both terms equal weight): (1.301, 1.0)

Cosine similarity for Doc A:

Dot product = 2.210×1.301 + 1.477×1.0 = 2.875 + 1.477 = 4.352

|Q| = sqrt(1.301² + 1²) = sqrt(2.693) ≈ 1.641

|A| = sqrt(2.210² + 1.477²) = sqrt(7.065) ≈ 2.658

cos(Q,A) = 4.352 / (1.641 × 2.658) ≈ 0.998

Cosine similarity for Doc B:

Dot product = 1.693×1.301 + 1.903×1.0 = 2.202 + 1.903 = 4.105

|B| = sqrt(1.693² + 1.903²) = sqrt(6.489) ≈ 2.547

cos(Q,B) = 4.105 / (1.641 × 2.547) ≈ 0.983

Answer: Doc A ranks higher (0.998 > 0.983)
