Question 1: Regression (20 marks)

(a) Basis Functions [4 marks]

What is the purpose of using basis functions in regression?

Solution:

Basis functions allow us to transform the input features into a higher-dimensional space. [2 marks]

This enables linear regression methods to fit non-linear relationships in the data by learning linear combinations of the transformed features. [2 marks]

Example: For polynomial basis functions, we transform \(x\) into \([1, x, x^2, x^3, ...]\). For RBF basis functions, we use \(\phi_j(x) = \exp(-\frac{(x-\mu_j)^2}{2s^2})\). This way, a linear model in the transformed space can represent complex non-linear patterns.

(b) Overfitting and Regularization [6 marks]

Explain what overfitting is and how regularization helps prevent it.

Solution:

Overfitting:

Overfitting occurs when a model learns the training data too well, including noise and random fluctuations, resulting in poor generalization to new data. [2 marks]

The model has low training error but high test error.

How Regularization Helps:

Regularization adds a penalty term to the loss function that discourages large parameter values. [2 marks]

This constrains the model complexity and reduces overfitting by preventing the model from fitting noise in the training data. [2 marks]

Regularized Loss Function:

\[L_{regularized} = L_{data} + \lambda \cdot \text{Penalty}(w)\]

where \(\lambda\) controls the strength of regularization.

Key Insight: Regularization forces the model to find a balance between fitting the training data well and keeping the model simple. This improves generalization to unseen data.

(c) Cross-Validation [5 marks]

Describe the k-fold cross-validation procedure and explain why it is useful.

Solution:

K-fold Cross-Validation Procedure:

  1. Divide the dataset into k equal-sized folds [1 mark]
  2. For each fold (i = 1 to k):
    • Use fold i as the validation set
    • Use the remaining k-1 folds as the training set
    • Train the model and evaluate on the validation fold
    [2 marks]
  3. Average the k validation scores to get the final performance estimate [1 mark]

Why It Is Useful:

Cross-validation provides a more reliable estimate of model performance by using all data for both training and validation. It reduces the variance in performance estimates and helps in model selection and hyperparameter tuning. [1 mark]

Common Value: k=5 or k=10 are commonly used. When k equals the dataset size, it's called Leave-One-Out Cross-Validation (LOOCV).

(d) Ridge Regression Calculation [5 marks]

Given the following:

  • Design matrix \(\Phi\) is 3×2
  • Target vector \(y = [1, 2, 3]^T\)
  • Regularization parameter \(\lambda = 0.1\)

Write the equation for computing the Ridge regression weights.

Solution:

The Ridge regression weights are computed using: [2 marks]

\[w = (\Phi^T\Phi + \lambda I)^{-1}\Phi^T y\]

Where:

  • \(\Phi\) is the design matrix (3×2) [1 mark]
  • \(I\) is the identity matrix (2×2 in this case) [1 mark]
  • \(\lambda = 0.1\) is the regularization parameter [1 mark]
Important: The identity matrix \(I\) has the same dimensions as \(\Phi^T\Phi\). Since \(\Phi\) is 3×2, \(\Phi^T\Phi\) is 2×2, so \(I\) is also 2×2.

Question 2: Classification (20 marks)

(a) Logistic Regression Output [4 marks]

What does the output of logistic regression represent?

Solution:

The output of logistic regression represents the probability that a given input belongs to the positive class (class 1). [2 marks]

It is computed using the sigmoid function: \(P(y=1|x) = \sigma(w^Tx) = \frac{1}{1+e^{-w^Tx}}\), which maps any real-valued input to a value between 0 and 1. [2 marks]

Decision Rule: We typically classify as class 1 if \(P(y=1|x) \geq 0.5\), and class 0 otherwise. However, this threshold can be adjusted based on the application requirements.

(b) SVM Margin [6 marks]

Explain what the margin is in Support Vector Machines and why maximizing it is important.

Solution:

Definition of Margin:

The margin is the distance between the decision boundary (hyperplane) and the nearest data points from either class, called support vectors. [2 marks]

It is defined as \(\frac{2}{\|w\|}\), where \(w\) is the weight vector perpendicular to the hyperplane.

Why Maximizing the Margin Is Important:

  • A larger margin provides better generalization to new data [2 marks]
  • It makes the classifier more robust to noise and variations in the data [1 mark]
  • It leads to a unique optimal solution [1 mark]
Intuition: Think of the margin as a "safety zone" around the decision boundary. A wider safety zone means the classifier is more confident and less likely to make mistakes on similar but slightly different data points.

(c) Confusion Matrix Analysis [6 marks]

Given the following confusion matrix for a binary classifier:

Predicted Positive Predicted Negative
Actual Positive 80 20
Actual Negative 10 90

Calculate: (i) Precision, (ii) Recall, (iii) Accuracy

Solution:

Given:

  • True Positives (TP) = 80
  • False Negatives (FN) = 20
  • False Positives (FP) = 10
  • True Negatives (TN) = 90

(i) Precision: [2 marks]

\[\text{Precision} = \frac{TP}{TP + FP} = \frac{80}{80 + 10} = \frac{80}{90} = 0.889 \text{ or } 88.9\%\]

(ii) Recall: [2 marks]

\[\text{Recall} = \frac{TP}{TP + FN} = \frac{80}{80 + 20} = \frac{80}{100} = 0.8 \text{ or } 80\%\]

(iii) Accuracy: [2 marks]

\[\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN} = \frac{80 + 90}{80 + 90 + 10 + 20} = \frac{170}{200} = 0.85 \text{ or } 85\%\]

Interpretation:
  • Precision (88.9%): When the model predicts positive, it's correct 88.9% of the time
  • Recall (80%): The model correctly identifies 80% of all actual positive cases
  • Accuracy (85%): Overall, the model makes correct predictions 85% of the time

(d) Multi-class Classification [4 marks]

Briefly explain the "one-vs-all" (one-vs-rest) approach for multi-class classification.

Solution:

In the one-vs-all approach, we train K separate binary classifiers for K classes. [2 marks]

Each classifier learns to distinguish one class from all other classes combined. To make a prediction, we run all K classifiers and choose the class whose classifier gives the highest confidence score. [2 marks]

Example: For 3 classes (A, B, C), we train:
  • Classifier 1: A vs (B and C)
  • Classifier 2: B vs (A and C)
  • Classifier 3: C vs (A and B)
For a new data point, we apply all three classifiers and select the class with the highest score.

Question 3: Unsupervised Learning (20 marks)

(a) K-means Initialization [4 marks]

Why is initialization important in K-means clustering, and what problems can poor initialization cause?

Solution:

K-means is sensitive to initial centroid positions because it uses an iterative optimization process that can get stuck in local minima. [2 marks]

Poor initialization can lead to:

  • Suboptimal clustering results (finding local minima instead of global minimum)
  • Empty clusters (no points assigned to a centroid)
  • Slow convergence
[2 marks]

Solution: Use K-means++ initialization, which chooses initial centroids that are far apart from each other. This typically leads to better and more consistent clustering results.

(b) EM Algorithm Steps [6 marks]

Describe the two main steps of the EM (Expectation-Maximization) algorithm used in Gaussian Mixture Models.

Solution:

E-Step (Expectation Step): [3 marks]

Calculate the responsibility (posterior probability) that each Gaussian component has for each data point, given the current parameters.

\[r_{ik} = \frac{\pi_k \mathcal{N}(x_i|\mu_k, \Sigma_k)}{\sum_{j=1}^{K} \pi_j \mathcal{N}(x_i|\mu_j, \Sigma_j)}\]

This tells us how likely each data point belongs to each cluster.

M-Step (Maximization Step): [3 marks]

Update the model parameters (means \(\mu_k\), covariances \(\Sigma_k\), and mixing coefficients \(\pi_k\)) to maximize the likelihood, using the responsibilities calculated in the E-step.

\[\mu_k = \frac{\sum_{i=1}^{N} r_{ik} x_i}{\sum_{i=1}^{N} r_{ik}}\]

\[\pi_k = \frac{1}{N}\sum_{i=1}^{N} r_{ik}\]

Iteration: These two steps alternate until convergence (when parameters stop changing significantly). The algorithm is guaranteed to increase the likelihood at each iteration.

(c) PCA Dimensionality Reduction [6 marks]

A dataset has 10 features. After performing PCA, you obtain the following eigenvalues:

[25, 18, 12, 8, 5, 3, 2, 1, 0.5, 0.5]

(i) How much variance is explained by the first principal component?

(ii) How many principal components are needed to retain at least 80% of the variance?

Solution:

Total variance: Sum of all eigenvalues = 25 + 18 + 12 + 8 + 5 + 3 + 2 + 1 + 0.5 + 0.5 = 75

(i) Variance explained by first PC: [2 marks]

\[\text{Variance explained} = \frac{\lambda_1}{\sum \lambda_i} = \frac{25}{75} = 0.333 \text{ or } 33.3\%\]

(ii) Components needed for 80% variance: [4 marks]

Calculate cumulative variance:

  • PC1: 25/75 = 33.3%
  • PC1-2: (25+18)/75 = 43/75 = 57.3%
  • PC1-3: (25+18+12)/75 = 55/75 = 73.3%
  • PC1-4: (25+18+12+8)/75 = 63/75 = 84.0% ✓

Answer: 4 principal components are needed to retain at least 80% of the variance.

Benefit: By using 4 components instead of 10 features, we reduce dimensionality by 60% while retaining 84% of the variance. This can significantly speed up subsequent learning algorithms.

(d) Hierarchical Clustering [4 marks]

What is the difference between agglomerative and divisive hierarchical clustering?

Solution:

Agglomerative (Bottom-up): [2 marks]

Starts with each data point as its own cluster and iteratively merges the closest pairs of clusters until all points are in a single cluster or a stopping criterion is met.

Divisive (Top-down): [2 marks]

Starts with all data points in one cluster and iteratively splits clusters into smaller clusters until each point is in its own cluster or a stopping criterion is met.

Common Practice: Agglomerative clustering is more commonly used because it is computationally more efficient. The results can be visualized using a dendrogram, which shows the hierarchy of cluster merges/splits.
