Question 1: Regression (20 marks)

(a) Linear Regression Assumptions [6 marks]

(i) State two key assumptions made in linear regression. [4 marks]

(ii) What happens if these assumptions are violated? [2 marks]

Solution:

(i) Key Assumptions: [4 marks]

  1. Linearity: The relationship between features and the target is linear (or can be made linear through basis function transformation). [2 marks]
  2. Gaussian Noise: The errors (residuals) are normally distributed with mean zero and constant variance: ε ~ N(0, σ²) (homoscedasticity). [2 marks]

Other valid assumptions include:

  • Independence of errors (no autocorrelation)
  • No multicollinearity (features are not perfectly correlated)
  • Errors have constant variance across all levels of predictors

(ii) Consequences of Violation: [2 marks]

  • If linearity is violated: The model will have high bias and poor predictions
  • If noise assumptions are violated: Parameter estimates may be inefficient, and confidence intervals/hypothesis tests may be invalid
  • The model may produce biased or unreliable predictions
Diagnostic Tools: Use residual plots to check assumptions. Plot residuals vs. fitted values (should show random scatter for homoscedasticity), and Q-Q plots to check normality of residuals.

(b) RBF Basis Functions [7 marks]

Consider RBF (Radial Basis Function) regression with basis functions:

\[\phi_j(x) = \exp\left(-\frac{(x-\mu_j)^2}{2s^2}\right)\]

(i) What do the parameters μ_j and s represent? [4 marks]

(ii) How would you choose the number of basis functions? [3 marks]

Solution:

(i) Parameter Meanings: [4 marks]

μ_j (centers): [2 marks]

  • Represents the center/location of the j-th basis function
  • Each RBF is centered at a specific point μ_j in the input space
  • Typically chosen as evenly spaced points across the range of training data, or using clustering (e.g., K-means)

s (width/scale): [2 marks]

  • Controls the width or spread of each basis function
  • Larger s means wider, smoother basis functions (more global influence)
  • Smaller s means narrower, more localized basis functions
  • Often chosen as the average distance between centers

(ii) Choosing Number of Basis Functions: [3 marks]

  • Use cross-validation to select the number that gives best validation performance [1 mark]
  • More basis functions = more complex model (risk of overfitting) [1 mark]
  • Fewer basis functions = simpler model (risk of underfitting) [1 mark]
  • Can also use regularization to allow more basis functions while controlling complexity
Practical Approach: Start with M = √N basis functions where N is the number of training samples. Then tune using cross-validation. Place centers using K-means clustering on the training data.

(c) Lasso Feature Selection [7 marks]

Consider a dataset with 20 features. After applying Lasso regression with λ=0.5:

  • 12 features have weights equal to zero
  • 8 features have non-zero weights

(i) Explain what happened during the Lasso optimization. [4 marks]

(ii) How does this compare to Ridge regression? [3 marks]

Solution:

(i) Lasso Optimization Process: [4 marks]

Lasso performed automatic feature selection by shrinking 12 feature weights to exactly zero. [2 marks]

Explanation:

  • Lasso minimizes: Sum of squared errors + λ × (sum of absolute values of weights)
  • The L1 penalty (|w_j|) has a "corner" at zero that encourages sparse solutions
  • As λ increases, more weights are pushed to exactly zero
  • The 8 remaining features are the most important for prediction
[2 marks]

(ii) Comparison with Ridge: [3 marks]

Aspect Lasso (L1) Ridge (L2)
Penalty λΣ|w_j| λΣw_j²
Feature Selection Yes (sets weights to 0) No (only shrinks weights)
Result Sparse solution (8/20 features) All 20 features kept with small weights
[3 marks]
When to Use: Use Lasso when you believe only a subset of features are truly relevant. Use Ridge when all features contribute but you want to prevent any single feature from dominating. Elastic Net combines both L1 and L2 penalties.

Question 2: Classification (20 marks)

(a) k-NN Algorithm [7 marks]

Consider a 2D dataset with the following points:

  • Class A: (1,1), (2,2), (2,3)
  • Class B: (6,6), (7,7), (8,8)

Using k-NN with k=3 and Euclidean distance:

(i) Classify the test point (3,3). Show your working. [5 marks]

(ii) What would happen if we used k=1? [2 marks]

Solution:

(i) Classification with k=3: [5 marks]

Step 1: Calculate distances from (3,3) to all training points: [3 marks]

Class A points:

d((3,3), (1,1)) = √[(3-1)² + (3-1)²] = √[4+4] = √8 ≈ 2.83

d((3,3), (2,2)) = √[(3-2)² + (3-2)²] = √[1+1] = √2 ≈ 1.41

d((3,3), (2,3)) = √[(3-2)² + (3-3)²] = √[1+0] = 1.0

Class B points:

d((3,3), (6,6)) = √[(3-6)² + (3-6)²] = √[9+9] = √18 ≈ 4.24

d((3,3), (7,7)) = √[(3-7)² + (3-7)²] = √[16+16] = √32 ≈ 5.66

d((3,3), (8,8)) = √[(3-8)² + (3-8)²] = √[25+25] = √50 ≈ 7.07

Step 2: Find 3 nearest neighbors: [1 mark]

  1. (2,3) from Class A: distance = 1.0
  2. (2,2) from Class A: distance = 1.41
  3. (1,1) from Class A: distance = 2.83

Step 3: Majority vote: [1 mark]

All 3 nearest neighbors belong to Class A

Prediction: Class A

(ii) With k=1: [2 marks]

The nearest neighbor is (2,3) from Class A (distance = 1.0), so the prediction would still be Class A. [1 mark]

However, k=1 is more sensitive to noise and outliers. A single mislabeled training point could cause incorrect predictions. Using k=3 provides more robust predictions through majority voting. [1 mark]

Choosing k: Odd values of k avoid ties in binary classification. Small k (e.g., 1, 3) captures local structure but is sensitive to noise. Large k creates smoother decision boundaries but may oversmooth. Use cross-validation to find optimal k.

(b) SVM Decision Boundary [6 marks]

For a linear SVM:

(i) Write the equation of the decision boundary. [2 marks]

(ii) What are support vectors? [2 marks]

(iii) If we remove a non-support vector from the training set, what happens to the decision boundary? [2 marks]

Solution:

(i) Decision Boundary Equation: [2 marks]

\[w^Tx + b = 0\]

where w is the weight vector (perpendicular to the boundary) and b is the bias term.

(ii) Support Vectors: [2 marks]

Support vectors are the training data points that lie closest to the decision boundary (on the margin boundaries). [1 mark]

These points are critical for defining the decision boundary - they "support" the maximum margin hyperplane. They satisfy |w^Tx_i + b| = 1. [1 mark]

(iii) Removing a Non-Support Vector: [2 marks]

The decision boundary remains unchanged. [1 mark]

Only support vectors determine the decision boundary. Non-support vectors lie outside the margin and do not affect the optimization. Removing them does not change w or b. [1 mark]

Key Property: This is a major advantage of SVM - the model depends only on a subset of training points (support vectors), making it memory-efficient for prediction and robust to non-support vectors that might be noisy.

(c) ROC Analysis [7 marks]

Given two classifiers with the following performance:

  • Classifier 1: AUC = 0.95, operates at TPR=0.90, FPR=0.10
  • Classifier 2: AUC = 0.85, operates at TPR=0.95, FPR=0.20

(i) Which classifier has better overall performance? Explain. [3 marks]

(ii) In which scenario might you prefer Classifier 2? [4 marks]

Solution:

(i) Better Overall Performance: [3 marks]

Classifier 1 has better overall performance [1 mark]

Reasoning:

  • Higher AUC (0.95 vs 0.85) indicates better discrimination ability across all thresholds [1 mark]
  • AUC represents the probability that a randomly chosen positive example is ranked higher than a randomly chosen negative example
  • AUC = 0.95 is considered excellent performance, while 0.85 is good but not as strong [1 mark]

(ii) Scenario Preferring Classifier 2: [4 marks]

You might prefer Classifier 2 in applications where high recall (TPR) is critical: [2 marks]

Examples:

  • Medical screening: Missing a disease (false negative) is more costly than a false alarm. Classifier 2 catches 95% of positive cases vs. 90% for Classifier 1 [1 mark]
  • Fraud detection: Better to flag more transactions for review (higher FPR) than miss fraudulent transactions
  • Security screening: Missing a threat is more dangerous than extra security checks
[1 mark]
Trade-off: Classifier 2 achieves higher recall (95% vs 90%) at the cost of higher false positive rate (20% vs 10%). Whether this trade-off is worthwhile depends on the relative costs of false negatives vs false positives in your application.

Question 3: Unsupervised Learning (20 marks)

(a) K-means Convergence [5 marks]

(i) How do you know when K-means has converged? [2 marks]

(ii) Is K-means guaranteed to converge? [3 marks]

Solution:

(i) Convergence Criteria: [2 marks]

K-means has converged when:

  • The cluster assignments no longer change between iterations, OR [1 mark]
  • The centroids no longer move (or move less than a small threshold ε), OR [0.5 marks]
  • The objective function (within-cluster sum of squares) stops decreasing [0.5 marks]

(ii) Convergence Guarantee: [3 marks]

Yes, K-means is guaranteed to converge. [1 mark]

Reasoning:

  • Each iteration (assignment + update) decreases or maintains the objective function value [1 mark]
  • There are only a finite number of possible cluster assignments
  • The algorithm must eventually reach a state where no improvement is possible [1 mark]
Important Caveat: While convergence is guaranteed, it only converges to a local minimum, not necessarily the global minimum. The final result depends on initialization. This is why we run K-means multiple times with different initializations and choose the best result.

(b) Gaussian Mixture Model Properties [8 marks]

Compare K-means and GMM:

(i) What type of assignment does each method use? [4 marks]

(ii) Which method is more flexible for modeling different cluster shapes? Explain. [4 marks]

Solution:

(i) Assignment Types: [4 marks]

Method Assignment Type Explanation
K-means Hard Assignment Each point is assigned to exactly one cluster (binary: belongs or doesn't belong)
GMM Soft Assignment Each point has a probability of belonging to each cluster (values between 0 and 1)
[4 marks: 2 marks per method]

Example:

  • K-means: Point x belongs to cluster 2 (100% assignment)
  • GMM: Point x has 70% probability of cluster 1, 25% cluster 2, 5% cluster 3

(ii) Flexibility for Cluster Shapes: [4 marks]

GMM is more flexible [1 mark]

Explanation:

K-means limitations:

  • Assumes spherical clusters of roughly equal size [1 mark]
  • Uses Euclidean distance, which creates circular/spherical boundaries
  • Cannot handle elongated or elliptical clusters well

GMM advantages:

  • Each component has its own covariance matrix Σ_k [1 mark]
  • Can model elliptical clusters with different orientations and sizes [1 mark]
  • Covariance matrix determines the shape and orientation of each cluster

GMM allows:

  • Diagonal Σ: Elliptical clusters aligned with axes
  • Full Σ: Elliptical clusters at any orientation
  • Different Σ for each component: Each cluster can have unique shape
Practical Consideration: GMM is more flexible but also more computationally expensive and requires more data to estimate parameters reliably. Use K-means for quick clustering of well-separated spherical clusters; use GMM when clusters have different shapes or sizes.

(c) PCA for Visualization [7 marks]

You have a dataset with 50 features and want to visualize it in 2D using PCA.

(i) Describe the steps to perform PCA. [4 marks]

(ii) How do you determine if 2 principal components are sufficient? [3 marks]

Solution:

(i) PCA Steps: [4 marks]

  1. Center the data: Subtract the mean from each feature to center the data at the origin

    \[X_{centered} = X - \bar{X}\]

    [1 mark]
  2. Compute covariance matrix: Calculate C = (1/N)X^T X (for centered data) [1 mark]
  3. Eigendecomposition: Find eigenvalues λ and eigenvectors v of the covariance matrix. The eigenvectors are the principal components (directions of maximum variance) [1 mark]
  4. Project data: Select the 2 eigenvectors with largest eigenvalues, form matrix U (50×2), and project: Z = XU to get 2D representation [1 mark]

(ii) Determining if 2 Components are Sufficient: [3 marks]

Calculate the proportion of variance explained: [2 marks]

\[\text{Proportion of variance} = \frac{\lambda_1 + \lambda_2}{\sum_{i=1}^{50} \lambda_i}\]

Decision criteria:

  • If the proportion is ≥ 0.8 (80%), 2 components capture most information [1 mark]
  • Can also use a scree plot: plot eigenvalues and look for an "elbow" where adding more components gives diminishing returns
  • For visualization specifically, 2 or 3 components are standard even if variance explained is lower
Example Interpretation: If λ₁=15, λ₂=10, and sum of all 50 eigenvalues is 40, then variance explained = (15+10)/40 = 0.625 or 62.5%. This means 2 PCs capture about 63% of the original information - reasonable for visualization but might lose important structure.
Visualization Trade-off: For visualization purposes, 2D or 3D is necessary even if variance explained is lower (e.g., 60-70%). The goal is to gain insights and identify patterns, not perfect reconstruction. Higher variance explained is better but not always achievable in 2D.
