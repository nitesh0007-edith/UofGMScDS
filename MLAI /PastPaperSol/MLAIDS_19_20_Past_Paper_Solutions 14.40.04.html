Question 1: Regression (20 marks)

(a) Polynomial Rescaling [4 marks]

In the Olympic example, why did we do rescaling in polynomial regression?

Solution:

In the Olympic data, high polynomials could make the value of \(x^d\) very big. [2 marks]

As a result, the computation of parameters becomes unstable. [2 marks]

Note: When using high-degree polynomials, the input features raised to high powers can become extremely large values. This causes numerical instability during computation and makes it difficult to optimize the model parameters accurately. Rescaling helps keep values in a manageable range.

(b) RBF Basis Function Number [4 marks]

When the number of RBF basis function increases, will the model be more complex or simpler?

Solution:

The model will be more complex. [2 marks]

There are more parameters. [2 marks]

Note: Each RBF basis function adds a new feature dimension to the model. More basis functions mean more parameters need to be learned, increasing the model's capacity to fit complex patterns but also increasing the risk of overfitting.

(c) Ridge vs Lasso Regularization [6 marks]

Please explain the difference between Ridge and Lasso for regularization.

Solution:

Ridge Regression:

  • Uses L2 penalty term: \(\lambda \sum_{j=1}^{D} w_j^2\) [1 mark]
  • Shrinks coefficients but keeps all features (coefficients approach zero but don't become exactly zero) [1 mark]

Lasso Regression:

  • Uses L1 penalty term: \(\lambda \sum_{j=1}^{D} |w_j|\) [1 mark]
  • Can shrink some coefficients to exactly zero, performing feature selection [1 mark]

Key Difference:

Lasso can perform automatic feature selection by setting some weights to zero, while Ridge only shrinks weights towards zero without eliminating them. [2 marks]

Practical Tip: Use Lasso when you suspect some features are irrelevant and want automatic feature selection. Use Ridge when you want to keep all features but reduce their influence.

(d) Training vs Test Error [6 marks]

Please define training error and test error.

Solution:

Training Error:

The error calculated on the training dataset that was used to fit/train the model. [3 marks]

Formula: \(E_{train} = \frac{1}{N_{train}} \sum_{i=1}^{N_{train}} (y_i - \hat{y}_i)^2\)

Test Error:

The error calculated on a separate test dataset that was not used during training. It measures how well the model generalizes to new, unseen data. [3 marks]

Formula: \(E_{test} = \frac{1}{N_{test}} \sum_{i=1}^{N_{test}} (y_i - \hat{y}_i)^2\)

Important: Test error is a better indicator of model performance on real-world data. Training error can be misleadingly low if the model overfits the training data.

Question 2: Classification (20 marks)

(a) k-NN Decision Boundary [4 marks]

Does the decision boundary have to be linear in k-NN?

Solution:

No, the decision boundary does not have to be linear. [2 marks]

k-NN can produce non-linear decision boundaries because it makes predictions based on the majority class of the k nearest neighbors, which can create complex, curved boundaries. [2 marks]

Note: This is one of the advantages of k-NN - it can capture complex, non-linear relationships in the data without assuming any particular functional form.

(b) SVM Kernel Trick [6 marks]

Why is the kernel trick important for SVM?

Solution:

The kernel trick allows SVM to work in high-dimensional feature spaces without explicitly computing the transformation. [3 marks]

This makes it computationally efficient to find non-linear decision boundaries by implicitly mapping data to higher dimensions where it becomes linearly separable. [3 marks]

Example: Instead of computing \(\phi(x_i) \cdot \phi(x_j)\) directly (which could be expensive in high dimensions), we compute \(K(x_i, x_j)\) directly, such as the RBF kernel: \(K(x_i, x_j) = \exp(-\gamma \|x_i - x_j\|^2)\)

(c) AUC Calculation [10 marks]

Given the following data, please calculate the AUC (Area Under the ROC Curve) for two different threshold scenarios.

Class label 1 0 1 1 0
Output 0.8 0.7 0.6 0.4 0.1

(i) When the threshold is between 0.4 and 1.0

(ii) When the threshold is less than 0.4

Solution:

(i) Threshold between 0.4 and 1.0: AUC = 1 [5 marks]

Explanation:

When threshold is in the range [0.4, 1.0], we need to check different threshold values:

  • Threshold = 0.8: Only output 0.8 is classified as positive (class 1), which is correct (TP=1, FP=0)
  • Threshold = 0.7: Outputs 0.8 classified as positive, both are class 1 (TP=1, FP=0)
  • Threshold = 0.6: Outputs 0.8, 0.7, 0.6 classified as positive. We get TP=2, FP=1
  • Threshold = 0.4: Outputs 0.8, 0.7, 0.6, 0.4 classified as positive. We get TP=3, FP=1

For this range, the ROC curve achieves perfect classification at certain thresholds, giving AUC = 1.

(ii) Threshold less than 0.4: AUC < 1 [5 marks]

Explanation:

When threshold is less than 0.4 (e.g., threshold = 0.1):

  • All outputs (0.8, 0.7, 0.6, 0.4, 0.1) are classified as positive
  • True Positives (TP) = 3 (all actual class 1 instances)
  • False Positives (FP) = 2 (all actual class 0 instances)
  • The classifier now misclassifies the negative examples, reducing performance

This gives us less than perfect classification, so AUC < 1.

Key Concept: AUC measures the classifier's ability to distinguish between classes across all possible thresholds. An AUC of 1 means perfect classification, while 0.5 means random guessing.

Question 3: Unsupervised Learning (20 marks)

(a) K-means Centroids [4 marks]

In K-means clustering, how do we update the centroids?

Solution:

We update each centroid by calculating the mean (average) of all data points assigned to that cluster. [2 marks]

Formula: \(\mu_k = \frac{1}{|C_k|} \sum_{x_i \in C_k} x_i\), where \(C_k\) is the set of points assigned to cluster \(k\). [2 marks]

Algorithm Step: After assigning each point to its nearest centroid, we recalculate each centroid as the mean of all points in that cluster. This process repeats until convergence.

(b) Kernel K-means Distance [6 marks]

How do we calculate the distance in Kernel K-means?

Solution:

In Kernel K-means, we use the kernel trick to compute distances in the feature space without explicitly computing the transformation. [2 marks]

The squared distance from a point \(x_i\) to cluster \(k\) is:

\[\|\phi(x_i) - \mu_k\|^2 = K(x_i, x_i) - \frac{2}{|C_k|}\sum_{x_j \in C_k} K(x_i, x_j) + \frac{1}{|C_k|^2}\sum_{x_j, x_l \in C_k} K(x_j, x_l)\]

[4 marks]
Key Point: We never need to compute \(\phi(x)\) explicitly. All computations are done using the kernel function \(K(x_i, x_j)\), which makes it efficient even for high-dimensional or infinite-dimensional feature spaces.

(c) GMM vs K-means [6 marks]

What are the main differences between Gaussian Mixture Models (GMM) and K-means?

Solution:

Assignment Type:

  • K-means: Hard assignment - each point belongs to exactly one cluster [1 mark]
  • GMM: Soft assignment - each point has a probability of belonging to each cluster [1 mark]

Cluster Shape:

  • K-means: Assumes spherical clusters with equal size [1 mark]
  • GMM: Can model elliptical clusters with different sizes and orientations (due to covariance matrices) [1 mark]

Mathematical Foundation:

  • K-means: Distance-based (minimizes within-cluster sum of squares) [1 mark]
  • GMM: Probabilistic model (maximizes likelihood using EM algorithm) [1 mark]
When to Use: Use K-means for quick clustering of well-separated, spherical clusters. Use GMM when clusters have different shapes/sizes or when you need probability estimates for cluster membership.

(d) PCA Variance [4 marks]

In PCA, what does the eigenvalue represent?

Solution:

The eigenvalue represents the amount of variance explained by the corresponding principal component (eigenvector). [2 marks]

Larger eigenvalues indicate directions in the data with more variance, which are more important for capturing the structure of the data. [2 marks]

Practical Use: We typically select principal components with the largest eigenvalues to retain most of the variance while reducing dimensionality. For example, if the first 3 eigenvalues account for 95% of total variance, we can use just 3 components instead of all features.
