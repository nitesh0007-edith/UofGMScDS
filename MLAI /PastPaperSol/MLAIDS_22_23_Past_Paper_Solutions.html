Question 1: Regression (20 marks)

(a) Polynomial Basis Functions [6 marks]

Consider fitting a polynomial regression model to predict house prices based on the size (in square meters).

(i) Write the polynomial basis function expansion for a cubic polynomial (degree 3). [2 marks]

(ii) How many parameters need to be learned? [2 marks]

(iii) What problem might occur if we use a very high degree polynomial without regularization? [2 marks]

Solution:

(i) Cubic Polynomial Basis Function: [2 marks]

\[\phi(x) = [1, x, x^2, x^3]^T\]

The model is: \(y = w_0 + w_1x + w_2x^2 + w_3x^3\)

(ii) Number of Parameters: [2 marks]

Answer: 4 parameters (w₀, w₁, w₂, w₃)

For a polynomial of degree d, we need (d+1) parameters including the bias term.

(iii) Problem with High Degree Polynomial: [2 marks]

Overfitting: The model will fit the training data very closely, including noise and random fluctuations. This leads to:

  • Very low training error but high test error (poor generalization)
  • The curve will oscillate wildly between data points
  • The model becomes too sensitive to small variations in the training data
Solution: Use regularization (Ridge or Lasso) to constrain the magnitude of the polynomial coefficients, or use cross-validation to select an appropriate degree.

(b) Ridge Regression Matrix Form [8 marks]

Given a dataset with N=4 training examples and D=2 features (plus bias), the design matrix is:

\[\Phi = \begin{bmatrix} 1 & 1 & 2 \\ 1 & 2 & 3 \\ 1 & 3 & 4 \\ 1 & 4 & 5 \end{bmatrix}, \quad y = \begin{bmatrix} 3 \\ 5 \\ 7 \\ 9 \end{bmatrix}\]

(i) What are the dimensions of Φᵀ Φ? [2 marks]

(ii) Write the Ridge regression solution for weights w. [3 marks]

(iii) If λ=0, what solution do we get? [3 marks]

Solution:

(i) Dimensions of Φᵀ Φ: [2 marks]

  • Φ has dimensions 4×3 (N=4 rows, D+1=3 columns including bias)
  • Φᵀ has dimensions 3×4
  • Φᵀ Φ has dimensions 3×3

(ii) Ridge Regression Solution: [3 marks]

\[w = (\Phi^T\Phi + \lambda I)^{-1}\Phi^T y\]

Where:

  • λ is the regularization parameter [1 mark]
  • I is the 3×3 identity matrix [1 mark]
  • The solution requires inverting a 3×3 matrix [1 mark]

(iii) When λ = 0: [3 marks]

When λ = 0, the Ridge regression becomes Ordinary Least Squares (OLS): [1 mark]

\[w = (\Phi^T\Phi)^{-1}\Phi^T y\]

[2 marks]

This is the standard least squares solution with no regularization penalty.

Important: When λ=0, if ΦᵀΦ is not invertible (singular), the solution doesn't exist. Ridge regression with λ>0 ensures ΦᵀΦ + λI is always invertible.

You are comparing three models on a dataset:

Select Model C [1 mark]

Reasoning:

  • It also shows a small gap between training (0.15) and validation error (0.18), indicating good balance [1 mark]

Model B shows clear signs of overfitting [1 mark]

Explanation:

Question 2: Classification (20 marks)

(a) Logistic Regression Gradient [8 marks]

For logistic regression with sigmoid function σ(z) = 1/(1+e⁻ᶻ):

(i) Write the prediction function for a single data point x. [2 marks]

(ii) Write the negative log-likelihood (loss) for a single data point (x, y). [3 marks]

(iii) State the derivative of the sigmoid function. [3 marks]

Solution:

(i) Prediction Function: [2 marks]

\[p(y=1|x) = \sigma(w^Tx) = \frac{1}{1+e^{-w^Tx}}\]

(ii) Negative Log-Likelihood (Cross-Entropy Loss): [3 marks]

\[L(w) = -[y \log(\sigma(w^Tx)) + (1-y)\log(1-\sigma(w^Tx))]\]

Where:

  • y is the true label (0 or 1) [1 mark]
  • σ(wᵀx) is the predicted probability [1 mark]
  • The formula handles both classes: when y=1, only the first term matters; when y=0, only the second term matters [1 mark]

(iii) Derivative of Sigmoid: [3 marks]

\[\frac{d\sigma(z)}{dz} = \sigma(z)(1-\sigma(z))\]

[3 marks]
Useful Property: The derivative of the sigmoid has a simple form that only depends on σ(z) itself. This makes gradient computation efficient. At z=0, σ(0)=0.5, so the derivative is 0.5×0.5 = 0.25 (maximum value).

(b) SVM with RBF Kernel [6 marks]

Consider an SVM with RBF (Gaussian) kernel: K(x, x') = exp(-γ||x-x'||²)

(i) What does the parameter γ control? [3 marks]

(ii) What happens when γ is very large vs. very small? [3 marks]

Solution:

(i) Role of Parameter γ: [3 marks]

γ (gamma) controls the "reach" or influence of individual training examples: [1 mark]

  • It determines how far the influence of a single training example extends [1 mark]
  • It controls the smoothness of the decision boundary (related to the width of the Gaussian) [1 mark]

(ii) Effect of γ Values: [3 marks]

Large γ (e.g., γ=100):

  • Each training example has a narrow area of influence (only nearby points matter)
  • Creates complex, wiggly decision boundaries
  • Risk of overfitting - model is too sensitive to individual points
[1.5 marks]

Small γ (e.g., γ=0.01):

  • Each training example has a wide area of influence (distant points matter)
  • Creates smooth, simple decision boundaries
  • Risk of underfitting - model may be too simple to capture patterns
[1.5 marks]
Practical Tip: Use cross-validation to find the optimal γ. Common values to try: [0.001, 0.01, 0.1, 1, 10, 100]. γ = 1/(n_features) is often a good starting point.

(c) Evaluation Metrics [6 marks]

You are building a spam email classifier. You have 1000 emails: 100 are spam, 900 are not spam.

Your classifier predicts:

  • 80 spam emails correctly as spam
  • 20 spam emails incorrectly as not spam
  • 50 non-spam emails incorrectly as spam
  • 850 non-spam emails correctly as not spam

(i) Calculate Precision and Recall. [4 marks]

(ii) Which metric is more important for this application? [2 marks]

Solution:

Given Information:

  • True Positives (TP) = 80 (spam correctly identified)
  • False Negatives (FN) = 20 (spam missed)
  • False Positives (FP) = 50 (non-spam incorrectly marked as spam)
  • True Negatives (TN) = 850 (non-spam correctly identified)

(i) Precision and Recall Calculation: [4 marks]

Precision: [2 marks]

\[\text{Precision} = \frac{TP}{TP + FP} = \frac{80}{80 + 50} = \frac{80}{130} \approx 0.615 \text{ or } 61.5\%\]

Interpretation: When the classifier predicts spam, it's correct 61.5% of the time.

Recall: [2 marks]

\[\text{Recall} = \frac{TP}{TP + FN} = \frac{80}{80 + 20} = \frac{80}{100} = 0.8 \text{ or } 80\%\]

Interpretation: The classifier correctly identifies 80% of all spam emails.

(ii) More Important Metric: [2 marks]

Precision is more important for spam filtering [1 mark]

Reasoning: False positives (legitimate emails marked as spam) are more costly than false negatives (spam in inbox). Users would be more frustrated if important emails are incorrectly sent to spam folder than if some spam reaches their inbox. [1 mark]

Trade-off: In practice, spam filters are often tuned to favor precision over recall. It's better to let some spam through than to risk losing important emails. However, the exact balance depends on user preferences and the application context.

Question 3: Unsupervised Learning (20 marks)

(a) K-means Objective Function [6 marks]

(i) Write the objective function that K-means minimizes. [3 marks]

(ii) Explain why K-means is not guaranteed to find the global optimum. [3 marks]

Solution:

(i) K-means Objective Function: [3 marks]

\[J = \sum_{k=1}^{K} \sum_{x_i \in C_k} \|x_i - \mu_k\|^2\]

Where:

  • K is the number of clusters [1 mark]
  • C_k is the set of data points assigned to cluster k [1 mark]
  • μ_k is the centroid (mean) of cluster k [1 mark]

This is the within-cluster sum of squares, measuring how compact the clusters are.

(ii) Why Not Guaranteed Global Optimum: [3 marks]

K-means uses an iterative optimization procedure (Lloyd's algorithm) that is guaranteed to converge, but only to a local minimum, not necessarily the global minimum. [1.5 marks]

The final result depends on the initial centroid positions. Different initializations can lead to different local minima with different objective function values. [1.5 marks]

Solution: Run K-means multiple times with different random initializations and select the result with the lowest objective function value. K-means++ initialization also helps by choosing initial centroids that are far apart.

(b) EM Algorithm for GMM [8 marks]

For a Gaussian Mixture Model with K=2 components in 1D:

(i) List all the parameters that need to be estimated. [3 marks]

(ii) In the E-step, what quantity do we calculate? [2 marks]

(iii) In the M-step, how do we update the mean μ_k? [3 marks]

Solution:

(i) Parameters to Estimate: [3 marks]

For K=2 components in 1D, we need to estimate:

  • Means: μ₁, μ₂ (2 parameters) [1 mark]
  • Variances: σ₁², σ₂² (2 parameters) [1 mark]
  • Mixing coefficients: π₁, π₂ where π₁ + π₂ = 1 (effectively 1 free parameter) [1 mark]

Total: 5 parameters (though only 4 are free due to the constraint on mixing coefficients)

(ii) E-step Calculation: [2 marks]

In the E-step, we calculate the responsibilities r_ik: [1 mark]

\[r_{ik} = \frac{\pi_k \mathcal{N}(x_i|\mu_k, \sigma_k^2)}{\sum_{j=1}^{2} \pi_j \mathcal{N}(x_i|\mu_j, \sigma_j^2)}\]

These represent the posterior probability that point x_i belongs to component k. [1 mark]

(iii) M-step Mean Update: [3 marks]

\[\mu_k = \frac{\sum_{i=1}^{N} r_{ik} x_i}{\sum_{i=1}^{N} r_{ik}}\]

[2 marks]

This is a weighted average of all data points, where the weights are the responsibilities. Points with higher responsibility for cluster k contribute more to its mean. [1 mark]

Intuition: Unlike K-means which uses hard assignments (each point fully belongs to one cluster), GMM uses soft assignments (each point partially belongs to all clusters). The mean is updated using a weighted average where weights reflect how much each point "belongs" to that cluster.

(c) PCA Computation [6 marks]

Given a centered data matrix X (N×D) where N=100 samples and D=5 features:

(i) How do you compute the covariance matrix? What are its dimensions? [3 marks]

(ii) After PCA, you want to project the data onto the first 2 principal components. What is the dimension of the projected data? [3 marks]

Solution:

(i) Covariance Matrix Computation: [3 marks]

Since the data is already centered (mean subtracted), the covariance matrix is: [1.5 marks]

\[C = \frac{1}{N}X^TX = \frac{1}{100}X^TX\]

Dimensions: [1.5 marks]

  • X is 100×5
  • Xᵀ is 5×100
  • C is 5×5 (square matrix with dimensions D×D)

(ii) Projected Data Dimensions: [3 marks]

To project onto the first 2 principal components:

  • We select the 2 eigenvectors corresponding to the 2 largest eigenvalues [1 mark]
  • Form a projection matrix U with dimensions 5×2 (original dimension × number of components) [1 mark]
  • Projected data: Z = XU has dimensions 100×2 (N × number of components) [1 mark]
Dimensionality Reduction: We've reduced from 5 dimensions to 2 dimensions (60% reduction) while retaining the directions of maximum variance in the data. This can be useful for visualization and computational efficiency.
