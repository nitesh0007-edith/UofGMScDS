Question 1: Regression (20 marks)

(a) Maximum Likelihood Estimation [8 marks]

Consider a linear regression model where we assume the targets are generated by:

\[y = w^T\phi(x) + \epsilon\]

where \(\epsilon \sim \mathcal{N}(0, \sigma^2)\) is Gaussian noise.

(i) Write the likelihood function for a single data point \((x_i, y_i)\). [3 marks]

(ii) Write the log-likelihood for the entire dataset. [3 marks]

(iii) What loss function does maximizing this log-likelihood correspond to? [2 marks]

Solution:

(i) Likelihood for a single data point: [3 marks]

Given that \(\epsilon \sim \mathcal{N}(0, \sigma^2)\), the target \(y_i\) follows:

\[p(y_i|x_i, w) = \mathcal{N}(y_i|w^T\phi(x_i), \sigma^2) = \frac{1}{\sqrt{2\pi\sigma^2}} \exp\left(-\frac{(y_i - w^T\phi(x_i))^2}{2\sigma^2}\right)\]

(ii) Log-likelihood for entire dataset: [3 marks]

Assuming N independent data points, the log-likelihood is:

\[\log p(y|X, w) = \sum_{i=1}^{N} \log p(y_i|x_i, w)\]

\[= -\frac{N}{2}\log(2\pi\sigma^2) - \frac{1}{2\sigma^2}\sum_{i=1}^{N}(y_i - w^T\phi(x_i))^2\]

(iii) Corresponding loss function: [2 marks]

Maximizing the log-likelihood is equivalent to minimizing the sum of squared errors (SSE) or Mean Squared Error (MSE):

\[L(w) = \sum_{i=1}^{N}(y_i - w^T\phi(x_i))^2\]

Key Insight: The Gaussian noise assumption directly leads to the least squares loss function. This shows why MSE is a natural choice for regression problems.

(b) Regularization Comparison [6 marks]

Consider the following three objective functions:

  1. Ordinary Least Squares: \(L_1 = \sum_{i=1}^{N}(y_i - w^T\phi(x_i))^2\)
  2. Ridge: \(L_2 = \sum_{i=1}^{N}(y_i - w^T\phi(x_i))^2 + \lambda\sum_{j=1}^{D}w_j^2\)
  3. Lasso: \(L_3 = \sum_{i=1}^{N}(y_i - w^T\phi(x_i))^2 + \lambda\sum_{j=1}^{D}|w_j|\)

Explain which method you would use if you suspect that only a small subset of features are relevant.

Solution:

Answer: Use Lasso (L3) [2 marks]

Explanation:

Lasso is the best choice when only a small subset of features are relevant because: [4 marks]

  • The L1 penalty (\(\lambda\sum|w_j|\)) encourages sparsity by shrinking some coefficients exactly to zero [2 marks]
  • This performs automatic feature selection, effectively identifying and keeping only the relevant features [1 mark]
  • Ridge regression would shrink all coefficients but not eliminate any, keeping all features (even irrelevant ones) with small weights [1 mark]
Practical Example: If you have 100 features but suspect only 5 are truly important, Lasso will identify those 5 by setting the other 95 weights to exactly zero. Ridge would give small weights to all 100 features.

(c) Bias-Variance Trade-off [6 marks]

Explain the bias-variance trade-off in machine learning. Include in your answer:

  • What bias and variance represent
  • How model complexity affects both
  • Why there is a trade-off

Solution:

Bias: [2 marks]

Bias measures how much the average prediction differs from the true value. High bias means the model makes strong assumptions and underfits the data (too simple to capture the true pattern).

Variance: [2 marks]

Variance measures how much predictions vary for different training sets. High variance means the model is too sensitive to training data and overfits (captures noise as if it were signal).

The Trade-off: [2 marks]

  • Simple models (low complexity): High bias, low variance - underfits
  • Complex models (high complexity): Low bias, high variance - overfits
  • We need to find the right balance: a model complex enough to capture true patterns but not so complex that it fits noise

Expected Test Error:

\[\text{Expected Error} = \text{Bias}^2 + \text{Variance} + \text{Irreducible Error}\]

Key Point: As we increase model complexity, bias decreases but variance increases. The optimal model complexity minimizes the sum of bias² and variance.

Question 2: Classification (20 marks)

(a) k-NN Distance Metrics [4 marks]

For the k-Nearest Neighbors algorithm:

(i) Give the formula for Euclidean distance between two points \(x_i\) and \(x_j\). [2 marks]

(ii) Why might Euclidean distance not always be appropriate? [2 marks]

Solution:

(i) Euclidean Distance Formula: [2 marks]

\[d(x_i, x_j) = \sqrt{\sum_{k=1}^{D}(x_{ik} - x_{jk})^2} = \|x_i - x_j\|_2\]

(ii) Limitations of Euclidean Distance: [2 marks]

  • It assumes all features have the same scale and importance. If features have different scales (e.g., age in years vs. income in dollars), features with larger scales will dominate the distance calculation
  • It may not be meaningful for categorical features or when features have different units
  • It assumes features are equally important, which may not be true in practice
Solution: Normalize/standardize features before using Euclidean distance, or use alternative distance metrics like Manhattan distance or Mahalanobis distance that account for feature correlations.

(b) SVM Soft Margin [8 marks]

In Support Vector Machines:

(i) Explain why we need soft margin SVM (as opposed to hard margin). [3 marks]

(ii) What role does the parameter C play in soft margin SVM? [3 marks]

(iii) What happens when C is very large vs. very small? [2 marks]

Solution:

(i) Why Soft Margin SVM is Needed: [3 marks]

  • Hard margin SVM requires data to be perfectly linearly separable, which is rare in real-world problems
  • Hard margin SVM is sensitive to outliers - a single misplaced point can dramatically affect the decision boundary
  • Soft margin allows some misclassifications, making the model more robust and generalizable

(ii) Role of Parameter C: [3 marks]

C controls the trade-off between:

  • Maximizing the margin (having a wider separation between classes)
  • Minimizing the classification errors on the training data

It penalizes misclassification: higher C means higher penalty for errors.

Soft Margin Objective:

\[\min_{w,b,\xi} \frac{1}{2}\|w\|^2 + C\sum_{i=1}^{N}\xi_i\]

where \(\xi_i\) are slack variables allowing misclassification

(iii) Effect of C Values: [2 marks]

  • Large C: Strong penalty for errors → tries to classify all training points correctly → smaller margin, risk of overfitting
  • Small C: Weak penalty for errors → allows more misclassifications → larger margin, risk of underfitting
Practical Tip: Use cross-validation to find the optimal C value that balances training accuracy with generalization to test data.

(c) ROC Curve and AUC [8 marks]

Consider a binary classifier with the following predictions on a test set:

Sample True Label Predicted Score
1 1 0.9
2 1 0.8
3 0 0.7
4 1 0.6
5 0 0.3

(i) What do the axes of an ROC curve represent? [2 marks]

(ii) Calculate TPR and FPR when the threshold is 0.65. [4 marks]

(iii) Approximately what is the AUC for this classifier? [2 marks]

Solution:

(i) ROC Curve Axes: [2 marks]

  • X-axis: False Positive Rate (FPR) = \(\frac{FP}{FP + TN}\) [1 mark]
  • Y-axis: True Positive Rate (TPR) or Recall = \(\frac{TP}{TP + FN}\) [1 mark]

(ii) TPR and FPR at threshold = 0.65: [4 marks]

When threshold = 0.65, we classify as positive if score ≥ 0.65:

  • Sample 1 (score 0.9, true label 1): Predicted Positive ✓ → TP
  • Sample 2 (score 0.8, true label 1): Predicted Positive ✓ → TP
  • Sample 3 (score 0.7, true label 0): Predicted Positive ✗ → FP
  • Sample 4 (score 0.6, true label 1): Predicted Negative ✗ → FN
  • Sample 5 (score 0.3, true label 0): Predicted Negative ✓ → TN

Results: TP = 2, FP = 1, TN = 1, FN = 1 [2 marks]

\[\text{TPR} = \frac{TP}{TP + FN} = \frac{2}{2 + 1} = \frac{2}{3} \approx 0.67\]

\[\text{FPR} = \frac{FP}{FP + TN} = \frac{1}{1 + 1} = \frac{1}{2} = 0.5\]

[2 marks]

(iii) AUC Estimate: [2 marks]

To calculate AUC, we need to check all thresholds. Looking at the ranking:

  • Total positives: 3 (samples 1, 2, 4)
  • Total negatives: 2 (samples 3, 5)

The classifier ranks samples in order: 1(+), 2(+), 3(-), 4(+), 5(-)

Positive samples 1 and 2 are ranked above both negatives → 2×2 = 4 correct pairs

Positive sample 4 is ranked above 1 negative (sample 5) → 1 correct pair

Total correct pairs: 5 out of 3×2 = 6 possible pairs

AUC ≈ 5/6 ≈ 0.83

Interpretation: An AUC of 0.83 means the classifier has an 83% chance of ranking a randomly chosen positive sample higher than a randomly chosen negative sample. This indicates good classification performance.

Question 3: Unsupervised Learning (20 marks)

(a) K-means Algorithm [6 marks]

Given the following 5 data points in 1D: [2, 4, 10, 12, 3]

Perform ONE iteration of K-means with K=2, starting with initial centroids at μ₁=3 and μ₂=11.

(i) Assign each point to the nearest centroid. [3 marks]

(ii) Calculate the new centroids. [3 marks]

Solution:

(i) Assignment Step: [3 marks]

Calculate distance from each point to both centroids:

  • Point 2: |2-3|=1, |2-11|=9 → Assign to μ₁ [0.5 marks]
  • Point 4: |4-3|=1, |4-11|=7 → Assign to μ₁ [0.5 marks]
  • Point 10: |10-3|=7, |10-11|=1 → Assign to μ₂ [0.5 marks]
  • Point 12: |12-3|=9, |12-11|=1 → Assign to μ₂ [0.5 marks]
  • Point 3: |3-3|=0, |3-11|=8 → Assign to μ₁ [0.5 marks]

Cluster assignments:

  • Cluster 1 (μ₁): {2, 4, 3}
  • Cluster 2 (μ₂): {10, 12}
[0.5 marks]

(ii) Update Step: [3 marks]

Calculate new centroids as the mean of assigned points:

\[\mu_1^{new} = \frac{2 + 4 + 3}{3} = \frac{9}{3} = 3\]

\[\mu_2^{new} = \frac{10 + 12}{2} = \frac{22}{2} = 11\]

New centroids: μ₁ = 3, μ₂ = 11

Observation: The centroids haven't changed, which means the algorithm has converged after just one iteration! This happens when the initial centroids were already optimal for the data.

(b) Kernel K-means [6 marks]

(i) Explain why we might use Kernel K-means instead of standard K-means. [3 marks]

(ii) What is the kernel trick and how does it help? [3 marks]

Solution:

(i) Why Use Kernel K-means: [3 marks]

  • Standard K-means assumes clusters are spherical and uses Euclidean distance, which only works well for linearly separable clusters [1 mark]
  • Kernel K-means can find non-linear cluster boundaries by implicitly mapping data to a higher-dimensional space [1 mark]
  • This allows it to handle complex cluster shapes (e.g., concentric circles, interleaved patterns) that standard K-means cannot separate [1 mark]

(ii) The Kernel Trick: [3 marks]

The kernel trick allows us to compute distances in a high-dimensional (or infinite-dimensional) feature space without explicitly computing the transformation φ(x). [1.5 marks]

Instead of computing φ(x) and then the inner product φ(x_i)·φ(x_j), we directly compute the kernel function K(x_i, x_j) = φ(x_i)·φ(x_j). [1.5 marks]

Example - RBF Kernel:

\[K(x_i, x_j) = \exp\left(-\frac{\|x_i - x_j\|^2}{2\sigma^2}\right)\]

This implicitly maps to infinite dimensions without ever computing φ(x)!

Key Benefit: Computational efficiency - we can work in very high-dimensional spaces at the cost of computing pairwise kernel values, which is much cheaper than explicit feature mapping.

(c) Gaussian Mixture Models [8 marks]

(i) What is the main advantage of GMM over K-means? [2 marks]

(ii) Write the probability density function for a Gaussian Mixture Model with K components. [3 marks]

(iii) In the EM algorithm for GMM, what do the responsibilities represent? [3 marks]

Solution:

(i) Main Advantage of GMM: [2 marks]

GMM provides soft (probabilistic) cluster assignments rather than hard assignments. Each data point has a probability of belonging to each cluster, which better captures uncertainty and overlapping clusters.

(ii) GMM Probability Density Function: [3 marks]

\[p(x) = \sum_{k=1}^{K} \pi_k \mathcal{N}(x|\mu_k, \Sigma_k)\]

Where:

  • \(\pi_k\) are mixing coefficients (weights) with \(\sum_{k=1}^{K}\pi_k = 1\) [1 mark]
  • \(\mathcal{N}(x|\mu_k, \Sigma_k)\) is the Gaussian density for component k [1 mark]
  • \(\mu_k\) and \(\Sigma_k\) are the mean and covariance of component k [1 mark]

(iii) Responsibilities in EM: [3 marks]

Responsibilities \(r_{ik}\) represent the posterior probability that data point \(x_i\) belongs to cluster k, given the current model parameters. [2 marks]

\[r_{ik} = \frac{\pi_k \mathcal{N}(x_i|\mu_k, \Sigma_k)}{\sum_{j=1}^{K} \pi_j \mathcal{N}(x_i|\mu_j, \Sigma_j)}\]

They range from 0 to 1, and for each data point, \(\sum_{k=1}^{K}r_{ik} = 1\). [1 mark]

Interpretation: If r_{i3} = 0.7, it means point x_i has a 70% probability of belonging to cluster 3. This is more informative than K-means' binary assignment, especially for points near cluster boundaries.
