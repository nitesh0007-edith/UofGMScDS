Question 1: Regression (20 marks)

(a) Basis Function Transformation [5 marks]

(i) What is the purpose of using basis functions in regression? [2 marks]

(ii) Give two examples of basis functions. [3 marks]

Solution:

(i) Purpose of Basis Functions: [2 marks]

Basis functions transform the input features into a higher-dimensional space, allowing linear models to capture non-linear relationships in the data. [2 marks]

By learning linear combinations of these transformed features, we can fit complex patterns while still using linear regression techniques.

(ii) Examples of Basis Functions: [3 marks]

Example 1: Polynomial Basis Functions [1.5 marks]

\[\phi(x) = [1, x, x^2, x^3, ..., x^d]^T\]

Transforms a single input x into polynomial terms up to degree d.

Example 2: Radial Basis Functions (RBF/Gaussian) [1.5 marks]

\[\phi_j(x) = \exp\left(-\frac{(x-\mu_j)^2}{2s^2}\right)\]

Creates localized "bump" functions centered at μ_j with width s.

Other valid examples: Fourier basis (sin, cos functions), Sigmoid basis, Wavelet basis

(b) Regularization Methods [8 marks]

Compare Ridge and Lasso regularization:

(i) Write the objective function for each method. [4 marks]

(ii) What is the key difference in their behavior? [4 marks]

Solution:

(i) Objective Functions: [4 marks]

Ridge Regression (L2 regularization): [2 marks]

\[L_{Ridge}(w) = \sum_{i=1}^{N}(y_i - w^T\phi(x_i))^2 + \lambda\sum_{j=1}^{D}w_j^2\]

Penalizes the sum of squared weights.

Lasso Regression (L1 regularization): [2 marks]

\[L_{Lasso}(w) = \sum_{i=1}^{N}(y_i - w^T\phi(x_i))^2 + \lambda\sum_{j=1}^{D}|w_j|\]

Penalizes the sum of absolute values of weights.

(ii) Key Difference in Behavior: [4 marks]

Aspect Ridge (L2) Lasso (L1)
Effect on weights Shrinks weights toward zero but never exactly to zero Can shrink weights to exactly zero
Feature selection No - keeps all features Yes - performs automatic feature selection
Solution Dense (all non-zero weights) Sparse (many zero weights)
When to use When all features are relevant When only subset of features are relevant
[4 marks: 2 marks for Ridge description, 2 marks for Lasso description]
Geometric Interpretation: The L1 penalty has "corners" at axes where weights can be exactly zero, while the L2 penalty is smooth and only asymptotically approaches zero.

(c) Cross-Validation [7 marks]

You have 100 data points and want to use 5-fold cross-validation to select the best regularization parameter λ.

(i) How many data points are in each fold? [2 marks]

(ii) Describe the complete validation process. [5 marks]

Solution:

(i) Data Points per Fold: [2 marks]

With 100 data points divided into 5 folds: 100 ÷ 5 = 20 data points per fold [2 marks]

(ii) Complete Validation Process: [5 marks]

  1. Divide data: Split the 100 data points into 5 equal folds of 20 points each [1 mark]
  2. For each candidate λ value: [3 marks]
    • For each fold i (i=1 to 5):
      • Use fold i as validation set (20 points)
      • Use remaining 4 folds as training set (80 points)
      • Train model with parameter λ on training set
      • Evaluate performance on validation fold
    • Average the 5 validation scores to get mean performance for this λ
  3. Select best λ: Choose the λ value that gives the best average validation performance [1 mark]
  4. Final model: Retrain on all 100 data points using the selected λ
Why 5-fold? It's a good balance:
  • Uses 80% of data for training (good for model learning)
  • Uses 20% for validation (reasonable estimate of performance)
  • Computationally efficient (only 5 training runs per λ)
  • Every data point is used for validation exactly once

Question 2: Classification (20 marks)

(a) Logistic Regression Basics [6 marks]

(i) Write the logistic regression prediction formula. [3 marks]

(ii) Why do we use the sigmoid function? [3 marks]

Solution:

(i) Prediction Formula: [3 marks]

\[P(y=1|x) = \sigma(w^Tx + b) = \frac{1}{1 + e^{-(w^Tx + b)}}\]

Where:

  • σ is the sigmoid function [1 mark]
  • w^Tx + b is the linear combination of features [1 mark]
  • Output is a probability between 0 and 1 [1 mark]

(ii) Why Sigmoid Function: [3 marks]

  • Maps to probability range: Sigmoid transforms any real-valued input to the range (0,1), which can be interpreted as a probability [1 mark]
  • Smooth and differentiable: Has a nice derivative (σ'(z) = σ(z)(1-σ(z))), making gradient-based optimization efficient [1 mark]
  • Interpretable: When w^Tx+b = 0 (decision boundary), σ = 0.5. Positive values give probability > 0.5 (class 1), negative values give probability < 0.5 (class 0) [1 mark]
Properties of Sigmoid:
  • σ(0) = 0.5 (midpoint)
  • σ(∞) → 1 (approaches 1 for large positive values)
  • σ(-∞) → 0 (approaches 0 for large negative values)
  • Symmetric around 0.5: σ(-z) = 1 - σ(z)

(b) Support Vector Machines [7 marks]

(i) What is the margin in SVM? [3 marks]

(ii) Explain the difference between hard margin and soft margin SVM. [4 marks]

Solution:

(i) The Margin: [3 marks]

The margin is the perpendicular distance between the decision boundary (hyperplane) and the closest data points from either class. [2 marks]

It equals 2/||w||, where w is the weight vector. SVM finds the hyperplane that maximizes this margin. [1 mark]

For a point x_i with label y_i ∈ {-1, +1}:

\[\text{Distance to hyperplane} = \frac{|w^Tx_i + b|}{\|w\|}\]

\[\text{Margin} = \frac{2}{\|w\|}\]

(ii) Hard vs Soft Margin: [4 marks]

Hard Margin SVM: [2 marks]

  • Assumes data is linearly separable
  • No training points allowed inside the margin
  • All constraints must be satisfied: y_i(w^Tx_i + b) ≥ 1
  • Problem: Fails if data is not perfectly separable or if there are outliers

Soft Margin SVM: [2 marks]

  • Allows some misclassifications and margin violations
  • Introduces slack variables ξ_i ≥ 0 to permit violations
  • Constraint: y_i(w^Tx_i + b) ≥ 1 - ξ_i
  • Objective: minimize (1/2)||w||² + C∑ξ_i (balance margin vs errors)
  • Advantage: More robust to outliers and works with non-separable data
Parameter C: Controls the trade-off
  • Large C: Strong penalty for errors → tries to classify all points correctly → smaller margin
  • Small C: Weak penalty for errors → allows more violations → larger margin

(c) Performance Metrics [7 marks]

Given a confusion matrix:

Predicted Positive Predicted Negative
Actual Positive 60 40
Actual Negative 20 80

Calculate: (i) Accuracy [2 marks], (ii) Precision [2 marks], (iii) Recall [2 marks], (iv) F1-score [1 mark]

Solution:

Given values:

  • True Positives (TP) = 60
  • False Negatives (FN) = 40
  • False Positives (FP) = 20
  • True Negatives (TN) = 80
  • Total = 200

(i) Accuracy: [2 marks]

\[\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN} = \frac{60 + 80}{200} = \frac{140}{200} = 0.70 = 70\%\]

(ii) Precision: [2 marks]

\[\text{Precision} = \frac{TP}{TP + FP} = \frac{60}{60 + 20} = \frac{60}{80} = 0.75 = 75\%\]

Interpretation: When the model predicts positive, it's correct 75% of the time.

(iii) Recall: [2 marks]

\[\text{Recall} = \frac{TP}{TP + FN} = \frac{60}{60 + 40} = \frac{60}{100} = 0.60 = 60\%\]

Interpretation: The model correctly identifies 60% of all actual positive cases.

(iv) F1-Score: [1 mark]

\[F1 = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}} = 2 \times \frac{0.75 \times 0.60}{0.75 + 0.60} = 2 \times \frac{0.45}{1.35} = \frac{0.90}{1.35} \approx 0.667 = 66.7\%\]

Summary of Results:
  • Accuracy: 70% (overall correct predictions)
  • Precision: 75% (quality of positive predictions)
  • Recall: 60% (coverage of actual positives)
  • F1-score: 66.7% (harmonic mean of precision and recall)

The model has moderate performance with higher precision than recall, suggesting it's conservative in predicting positives.

Question 3: Unsupervised Learning (20 marks)

(a) K-means Clustering [8 marks]

Given 1D data points: [1, 2, 5, 8, 9, 10]

Perform ONE iteration of K-means with K=2, starting with centroids μ₁=2 and μ₂=9.

(i) Assign points to clusters. [4 marks]

(ii) Calculate new centroids. [4 marks]

Solution:

(i) Assignment Step: [4 marks]

Calculate distance from each point to both centroids and assign to nearest:

Point 1: |1-2|=1, |1-9|=8 → Assign to μ₁

Point 2: |2-2|=0, |2-9|=7 → Assign to μ₁

Point 5: |5-2|=3, |5-9|=4 → Assign to μ₁

Point 8: |8-2|=6, |8-9|=1 → Assign to μ₂

Point 9: |9-2|=7, |9-9|=0 → Assign to μ₂

Point 10: |10-2|=8, |10-9|=1 → Assign to μ₂

[4 marks]

Result:

  • Cluster 1: {1, 2, 5}
  • Cluster 2: {8, 9, 10}

(ii) Update Step: [4 marks]

Calculate new centroids as the mean of assigned points:

\[\mu_1^{new} = \frac{1 + 2 + 5}{3} = \frac{8}{3} \approx 2.67\]

\[\mu_2^{new} = \frac{8 + 9 + 10}{3} = \frac{27}{3} = 9.0\]

[4 marks: 2 marks per centroid]

New centroids: μ₁ = 2.67, μ₂ = 9.0

Next Iteration: We would repeat the process using the new centroids (2.67 and 9.0) until convergence. In this case, the assignments would likely remain the same, indicating convergence has been reached.

(b) Gaussian Mixture Models [6 marks]

(i) What is the main advantage of GMM over K-means? [3 marks]

(ii) What are the two steps of the EM algorithm? [3 marks]

Solution:

(i) Main Advantage of GMM: [3 marks]

Soft probabilistic assignments vs hard assignments [1.5 marks]

  • GMM assigns each point a probability of belonging to each cluster (soft assignment)
  • K-means assigns each point to exactly one cluster (hard assignment)
  • GMM provides uncertainty estimates - useful when cluster boundaries are unclear

More flexible cluster shapes [1.5 marks]

  • GMM can model elliptical clusters with different sizes and orientations
  • K-means assumes spherical clusters of similar size
  • GMM uses covariance matrices to capture cluster shape

(ii) EM Algorithm Steps: [3 marks]

E-Step (Expectation): [1.5 marks]

Calculate responsibilities r_ik - the posterior probability that point x_i belongs to component k:

\[r_{ik} = \frac{\pi_k \mathcal{N}(x_i|\mu_k, \Sigma_k)}{\sum_{j=1}^{K} \pi_j \mathcal{N}(x_i|\mu_j, \Sigma_j)}\]

M-Step (Maximization): [1.5 marks]

Update parameters (μ_k, Σ_k, π_k) using the responsibilities to maximize likelihood:

\[\mu_k = \frac{\sum_{i} r_{ik} x_i}{\sum_{i} r_{ik}}, \quad \pi_k = \frac{1}{N}\sum_{i} r_{ik}\]

Iteration: These steps alternate until convergence. Each iteration is guaranteed to increase (or maintain) the log-likelihood of the data.

(c) Principal Component Analysis [6 marks]

You have a dataset with eigenvalues: [50, 30, 15, 5]

(i) What is the total variance in the data? [1 mark]

(ii) How much variance is explained by the first principal component? [2 marks]

(iii) How many components are needed for 90% variance? [3 marks]

Solution:

(i) Total Variance: [1 mark]

\[\text{Total Variance} = \sum \lambda_i = 50 + 30 + 15 + 5 = 100\]

(ii) Variance Explained by PC1: [2 marks]

\[\text{Variance Explained} = \frac{\lambda_1}{\text{Total}} = \frac{50}{100} = 0.50 = 50\%\]

The first principal component explains half of the total variance.

(iii) Components for 90% Variance: [3 marks]

Calculate cumulative variance:

PC1: 50/100 = 50%

PC1-2: (50+30)/100 = 80/100 = 80%

PC1-3: (50+30+15)/100 = 95/100 = 95% ✓

[2 marks for calculations]

Answer: 3 principal components are needed to explain at least 90% of variance (specifically, 95%). [1 mark]

Dimensionality Reduction:
  • Original data has 4 dimensions
  • Using 3 PCs captures 95% of variance
  • Reduction: 25% fewer dimensions with only 5% information loss
  • This can significantly speed up subsequent algorithms while retaining most information
