Classification Problem

Part (a): k-Nearest Neighbors Classification [10 marks]

Consider the following 2D dataset:

Point x₁ x₂ Class
A 1 2 Red
B 2 3 Red
C 3 3 Blue
D 6 5 Blue
E 7 6 Blue

(i) Using k-NN with k=3, classify the test point T=(4,4). Show all distance calculations. [7 marks]

(ii) How would the classification change if k=1? [3 marks]

Solution:

(i) Classification with k=3: [7 marks]

Step 1: Calculate Euclidean distances from T=(4,4) to all training points: [5 marks]

Distance to A (1,2):

\[d(T,A) = \sqrt{(4-1)^2 + (4-2)^2} = \sqrt{9 + 4} = \sqrt{13} \approx 3.61\]

Distance to B (2,3):

\[d(T,B) = \sqrt{(4-2)^2 + (4-3)^2} = \sqrt{4 + 1} = \sqrt{5} \approx 2.24\]

Distance to C (3,3):

\[d(T,C) = \sqrt{(4-3)^2 + (4-3)^2} = \sqrt{1 + 1} = \sqrt{2} \approx 1.41\]

Distance to D (6,5):

\[d(T,D) = \sqrt{(4-6)^2 + (4-5)^2} = \sqrt{4 + 1} = \sqrt{5} \approx 2.24\]

Distance to E (7,6):

\[d(T,E) = \sqrt{(4-7)^2 + (4-6)^2} = \sqrt{9 + 4} = \sqrt{13} \approx 3.61\]

Step 2: Identify the 3 nearest neighbors: [1 mark]

Sorted by distance:

  1. C (Blue): 1.41
  2. B (Red): 2.24
  3. D (Blue): 2.24

Step 3: Majority vote among k=3 neighbors: [1 mark]

  • Blue class: 2 votes (C and D)
  • Red class: 1 vote (B)

Prediction: BLUE class

(ii) Classification with k=1: [3 marks]

With k=1, we only consider the single nearest neighbor. [1 mark]

The nearest neighbor is C (Blue) with distance 1.41. [1 mark]

Prediction: BLUE class (same result as k=3 in this case) [1 mark]

Key Observations:
  • Point T=(4,4) is geometrically closer to the Blue cluster
  • Both k=1 and k=3 give the same classification, indicating T is clearly in the Blue region
  • Using k=3 is generally more robust as it reduces sensitivity to outliers

Part (b): Decision Tree Construction [10 marks]

Given the following dataset for weather prediction:

Outlook Temperature Humidity Play Tennis?
SunnyHotHighNo
SunnyHotHighNo
OvercastHotHighYes
RainMildHighYes
RainCoolNormalYes
OvercastCoolNormalYes

(i) Calculate the entropy of the target variable (Play Tennis). [4 marks]

(ii) Which feature would you choose as the root node? Explain your reasoning. [6 marks]

Solution:

(i) Entropy of Target Variable: [4 marks]

Count class frequencies: [1 mark]

  • Yes: 4 instances
  • No: 2 instances
  • Total: 6 instances

Calculate probabilities: [1 mark]

  • P(Yes) = 4/6 = 2/3
  • P(No) = 2/6 = 1/3

Apply entropy formula: [2 marks]

\[H(S) = -\sum_{i} p_i \log_2(p_i)\]

\[H(\text{Play Tennis}) = -\frac{2}{3}\log_2\left(\frac{2}{3}\right) - \frac{1}{3}\log_2\left(\frac{1}{3}\right)\]

\[= -\frac{2}{3}(-0.585) - \frac{1}{3}(-1.585)\]

\[= 0.390 + 0.528 = 0.918 \text{ bits}\]

(ii) Choosing Root Node: [6 marks]

Calculate Information Gain for each feature:

Feature: Outlook [2 marks]

  • Sunny: 2 instances (0 Yes, 2 No) → Entropy = 0
  • Overcast: 2 instances (2 Yes, 0 No) → Entropy = 0
  • Rain: 2 instances (2 Yes, 0 No) → Entropy = 0

\[\text{Weighted Entropy} = \frac{2}{6}(0) + \frac{2}{6}(0) + \frac{2}{6}(0) = 0\]

\[\text{Info Gain(Outlook)} = 0.918 - 0 = 0.918\]

Feature: Temperature [2 marks]

  • Hot: 3 instances (1 Yes, 2 No)
  • Mild: 1 instance (1 Yes, 0 No)
  • Cool: 2 instances (2 Yes, 0 No)

Entropy for Hot = -1/3·log₂(1/3) - 2/3·log₂(2/3) ≈ 0.918

Weighted Entropy ≈ 3/6(0.918) + 1/6(0) + 2/6(0) ≈ 0.459

Info Gain(Temperature) ≈ 0.918 - 0.459 = 0.459

Feature: Humidity [1 mark]

  • High: 4 instances (2 Yes, 2 No) → Entropy = 1.0
  • Normal: 2 instances (2 Yes, 0 No) → Entropy = 0

Weighted Entropy = 4/6(1.0) + 2/6(0) ≈ 0.667

Info Gain(Humidity) ≈ 0.918 - 0.667 = 0.251

Decision: [1 mark]

Choose "Outlook" as the root node because it has the highest information gain (0.918), which means it provides the most information about the target variable. In fact, it perfectly separates the classes!

Summary of Information Gains:
  • Outlook: 0.918 (highest - best choice) ✓
  • Temperature: 0.459
  • Humidity: 0.251

Outlook creates pure leaf nodes (all Sunny→No, all Overcast→Yes, all Rain→Yes), making it the ideal root split.

Part (c): Performance Evaluation [10 marks]

A binary classifier is tested on 100 samples with the following results:

  • 40 actual positives: 35 correctly predicted, 5 missed
  • 60 actual negatives: 50 correctly predicted, 10 incorrectly predicted as positive

(i) Construct the confusion matrix. [2 marks]

(ii) Calculate Accuracy, Precision, Recall, and F1-score. [6 marks]

(iii) Is this classifier better at identifying positives or negatives? Explain. [2 marks]

Solution:

(i) Confusion Matrix: [2 marks]

Predicted Positive Predicted Negative Total
Actual Positive 35 (TP) 5 (FN) 40
Actual Negative 10 (FP) 50 (TN) 60
Total 45 55 100

(ii) Performance Metrics: [6 marks]

Accuracy: [1.5 marks]

\[\text{Accuracy} = \frac{TP + TN}{\text{Total}} = \frac{35 + 50}{100} = \frac{85}{100} = 0.85 = 85\%\]

Precision: [1.5 marks]

\[\text{Precision} = \frac{TP}{TP + FP} = \frac{35}{35 + 10} = \frac{35}{45} \approx 0.778 = 77.8\%\]

When the classifier predicts positive, it's correct 77.8% of the time.

Recall (Sensitivity): [1.5 marks]

\[\text{Recall} = \frac{TP}{TP + FN} = \frac{35}{35 + 5} = \frac{35}{40} = 0.875 = 87.5\%\]

The classifier correctly identifies 87.5% of all actual positive cases.

F1-Score: [1.5 marks]

\[F1 = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}}\]

\[= 2 \times \frac{0.778 \times 0.875}{0.778 + 0.875} = 2 \times \frac{0.681}{1.653} \approx 0.824 = 82.4\%\]

(iii) Performance Analysis: [2 marks]

The classifier is better at identifying positives. [1 mark]

Evidence: [1 mark]

  • For positives: Recall = 87.5% (catches 35 out of 40 positives, only misses 5)
  • For negatives: Specificity = TN/(TN+FP) = 50/60 = 83.3% (correctly identifies 50 out of 60 negatives)
  • The recall for positives (87.5%) is higher than the specificity for negatives (83.3%)
  • The classifier has lower false negative rate (5/40 = 12.5%) compared to false positive rate (10/60 = 16.7%)
Summary:
Metric Value Interpretation
Accuracy 85% Overall correct predictions
Precision 77.8% Reliability of positive predictions
Recall 87.5% Coverage of actual positives
F1-Score 82.4% Harmonic mean of precision & recall

This is a good classifier with high recall (few false negatives) but moderate precision (some false positives).
