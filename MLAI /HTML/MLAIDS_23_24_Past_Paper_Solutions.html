Question 1: Regression - ML Training Data Size (20 marks)

Context: Consider using regression to predict the size of training data used in machine learning models from 1950-2023.

(a) Identifying Rescaling Method [4 marks]

A rescaling method was used to rescale the years to values displayed in Figure 1.2. Describe which rescaling method was used (with enough details of the procedure), and why you think it was applied.

Solution:

Rescaling Method Used: [1 mark] Whitening (or equivalent answer such as standardization/z-score normalization)

Procedure: [1 mark]

\[\text{Year}_{scaled} = \frac{\text{Year} - \text{mean}(\text{Year})}{\text{std}(\text{Year})}\]

Evidence from Figure 1.2: [2 marks total]

  • [1 mark] The mean of rescaled years in Figure 1.2 is around 0, indicated by a roughly equal number of data points around 0.
  • [1 mark] The rescaled years range from around -2.5 to around 1.5, mimicking the unit variance (equals to 1) expected with standard normal distribution.
Why Whitening? The distribution of rescaled values centered at 0 with symmetric spread is characteristic of standardization. This is different from min-max scaling (which would give [0,1]) or log transform (which would preserve positive values).

(c) RBF vs Polynomials [4 marks]

Consider fitting the data in Figure 1.1 with a regression using radial basis functions:

\[h_{n,k} = \exp\left(-\frac{(x_n - \mu_k)^2}{2s^2}\right), \quad n = 1, \ldots, N; \quad k = 1, \ldots, K\]

Outline one advantage and disadvantage of using this radial basis function over polynomials with the data in Figure 1.1.

Solution:

Advantage: [2 marks total]

  • [1 mark] The data is not equally distributed across x values - there is a clear gap (1960-1980).
  • [1 mark] Using location-specific basis functions such as the radial basis functions can model this localized effect by placing basis functions around regions where we have strong data support.

Disadvantage: [2 marks total]

  • [1 mark] Need to choose hyperparameters \(\mu_k\) (centers) and \(s\) (bandwidth).
  • [1 mark] A bad choice of \(\mu_k\) could lead to very bad prediction in the gap (1960-1980) where there's no data to constrain the model.
Key Trade-off: RBF's locality is both a strength (can ignore irrelevant regions) and weakness (requires careful placement of centers, especially in data-sparse regions).

Solution:

Figure 1.3A: Ridge Regression [3 marks total]

  • [1 mark] Reasoning: The fitted model is a curve that is close to a flat straight line. This happens when most of the basis functions have close to zero coefficients.
  • [1 mark] This also suggests that the hyperparameter controlling the strength of L2 regularization is set to be too strong (large λ value).

Figure 1.3B: Lasso [3 marks total]

  • [1 mark] Identified as Lasso.

Ridge (L2): \(\lim_{\lambda \to \infty} \mathbf{w} \to \mathbf{0}\) (all weights → 0, flat prediction)

Lasso (L1): \(\lim_{\lambda \to \infty}\) Most \(w_i = 0\) exactly (sparse solution, linear if only bias survives)

Training data in the two-dimensional plane of \(X_1\) and \(X_2\):

(a)(i) k-NN Decision Boundary Equations [6 marks]

Design a k-NN classifier with k=1 and write down the equations that specify the decision boundary between the two classes.

Solution:

Decision Boundary Equations:

[2 marks] In region \(X_1 > 0.5\) and \(X_2 > 0.5\):

\[X_1 - X_2 = 0\]

[2 marks] In region \(X_1 > 0.5\) and \(X_2 < -0.5\):

\[X_1 + X_2 = 0\]

[2 marks] In region \(-0.5 < X_2 < 0.5\):

\[X_1 - 0.5 = 0 \quad \text{or} \quad X_1 = 0.5\]

Explanation: With k=1, the decision boundary consists of the perpendicular bisectors between nearest neighbors of opposite classes. The boundary forms piecewise linear segments separating the Voronoi regions:
  • Between (2,0) and (0,2): line \(X_1 - X_2 = 0\)
  • Between (2,0) and (0,-2): line \(X_1 + X_2 = 0\)
  • Between (2,0) and (-1,0): vertical line \(X_1 = 0.5\)

(b)(i) SVM Support Vectors and Predictor [4 marks]

In the same dataset, we apply a linear SVM model with predictor \(y(X_1, X_2)\) for classification. Which data points are the support vectors? Write down the equation for \(y(X_1, X_2)\).

Solution:

Support Vectors: [1 mark]

The support vectors are: (2,0), (0,2), and (0,-2)

Predictor Equation: [3 marks]

Explanation:

The decision boundary is the vertical line \(X_1 = 1\) (midpoint between red points at x=2,3 and blue points at x=0,-1). The SVM predictor is:

(b)(ii) Lagrange Multipliers [5 marks]

Specify the Lagrange multipliers \(\alpha_1, \alpha_2, \alpha_3, \alpha_4, \alpha_5\) for each of the data points in the training data: (2,0), (0,2), (0,-2), (3,0), and (-1,0), respectively.

Solution:

Lagrange Multipliers: [5 marks]

\[\alpha_1 = 0.5, \quad \alpha_2 = \alpha_3 = 0.25, \quad \alpha_4 = \alpha_5 = 0\]

Verification:

The weight vector is computed as:

\[\mathbf{w} = \sum_{i=1}^{5} \alpha_i y_i \mathbf{x}_i\]

\[= 0.5(+1)\begin{bmatrix}2\\0\end{bmatrix} + 0.25(-1)\begin{bmatrix}0\\2\end{bmatrix} + 0.25(-1)\begin{bmatrix}0\\-2\end{bmatrix} + 0(+1)\begin{bmatrix}3\\0\end{bmatrix} + 0(-1)\begin{bmatrix}-1\\0\end{bmatrix}\]

\[= \begin{bmatrix}1\\0\end{bmatrix} + \begin{bmatrix}0\\-0.5\end{bmatrix} + \begin{bmatrix}0\\0.5\end{bmatrix} = \begin{bmatrix}1\\0\end{bmatrix}\]

This gives \(\mathbf{w} = [1, 0]^T\), confirming \(y(X_1, X_2) = X_1 - 1\) with appropriate bias.

Interpretation: Only support vectors have non-zero α values. Point (2,0) has the largest α because it alone represents the positive class on the margin, while the two blue support vectors share the responsibility for the negative class.

(b)(iii) Comparing k-NN vs SVM Accuracy [2 marks]

Which classifier (k-NN or SVM designed above) will be more accurate? Explain your answer in up to two sentences.

Solution:

Answer: [2 marks] k-NN will work better as a nonlinear classifier.

Explanation:

The data distribution has a nonlinear structure - the blue class surrounds the red class in a non-convex pattern (points at (0,2), (0,-2), and (-1,0) form a "C" shape around the red points). Question 3: Unsupervised Learning (20 marks)

Context: Clustering Gaussian Distributions

Consider using K-means to perform clustering on data in Figure 3.1A, where we expect to form three clusters as shown in Figure 3.1B. The data appears to follow Gaussian distributions with anisotropic (elongated) shapes.

(a) K-means on Anisotropic Gaussian Clusters [3 marks]

Outline what would happen if we directly apply K-means with Euclidean distance to this data. Can it achieve the clustering objective? How will it split/group the data and why?

Solution:

Analysis:

[1 mark] The clusters seem to follow Gaussian distributions and some of them are anisotropic (elongated in certain directions).

[1 mark] K-means is not able to encode anisotropy. Therefore, K-means would be partially effective.

[1 mark] They will center the three clusters well but they won't be able to capture well the more distant points (at the extremes of the elongated clusters).

Detailed Explanation: K-means assumes isotropic (spherical) clusters with equal variance in all directions. When clusters are elongated (anisotropic), K-means will:
  • Correctly identify cluster centers
  • Misclassify points at the elongated ends of clusters
  • Potentially split elongated clusters if they're very stretched
  • Merge nearby elongated clusters if their ends overlap

(b) Kernel K-means for Anisotropic Clusters [2 marks]

An alternative approach is to use Kernel K-means. Would kernel K-means help in this dataset and why?

Solution:

Answer: Yes, a kernel would help in this case.

[1 mark] A Gaussian kernel would be able to encode the anisotropy in the data.

[1 mark] In this case, the recovered clusters would have a very close fit with the actual ground truth.

How It Works: Kernel K-means with an appropriate kernel (e.g., RBF/Gaussian kernel) can implicitly map the data to a higher-dimensional space where the anisotropic clusters become more separable with spherical boundaries. The kernel parameters can be tuned to capture the scale and orientation of each cluster's elongation.

(c) Mixture Models for Gaussian Clusters [3 marks]

An alternative approach is to use mixture models. Would mixture models help to better classify this dataset than K-means and why?

Solution:

[1 mark] Mixture models is the ideal method to detect clusters generated via a Gaussian model.

[1 mark] Mixture models assume a Gaussian distribution of the underlying clusters and they capture different variations based on the parameters of the distribution (mean, covariance).

[1 mark] In this case, the clusters obtained will be a close fit and you get the additional advantage of recovering the parameters of the generative process (means, covariances, mixing proportions).

GMM Advantage:

Each cluster k has its own covariance matrix \(\Sigma_k\) that can be:

  • Full: \(\Sigma_k\) = general matrix (captures any ellipsoid orientation)
  • Diagonal: Different variances per dimension
  • Spherical: Isotropic (like K-means)

For anisotropic Gaussian clusters, full covariance GMM is optimal.

Perfect Match: Since the data is actually generated from Gaussian distributions, GMM is the ideal choice - it's fitting the exact generative model that produced the data.

(f) Choosing Number of Clusters in GMM [3 marks]

Explain how you would choose the number of clusters in an application of mixture models.

Solution:

Method:

[1 mark] Use k-fold cross-validation.

[1 mark] Estimate the log-likelihood across an increasing number of clusters, K.

[1 mark] Decide the number of clusters based on the extreme/maximum point of the curve.

Algorithm:

For K = 1 to K_max:
    For each fold in cross-validation:
        Train GMM with K components on training folds
        Compute log-likelihood on validation fold
    Average log-likelihood across folds → LL(K)

Select K* = argmax_K LL(K)
Alternative Methods:
  • BIC (Bayesian Information Criterion): Penalizes model complexity: BIC = -2·log(L) + k·log(n)
  • AIC (Akaike Information Criterion): AIC = -2·log(L) + 2k
  • Silhouette Score: Measures cluster cohesion and separation
  • Elbow Method: Plot log-likelihood vs K and look for "elbow"

(g) Outlier Detection with Mixture Models [5 marks]

Explain how you could detect an outlier point with mixture models. Write a high-level pseudo code and describe each step. (Hint: Start with the expectation-maximization algorithm and how it can facilitate the detection)

Solution:

Algorithm Steps:

Step 1: Apply Gaussian Mixture Models for Clustering [2 marks]

Use Gaussian Mixture Models for clustering:

1. Decide on the number of clusters K
2. Use expectation-maximization algorithm:
   - theta = {π, μ, Σ}: parameters of the Gaussian distribution
     encoding each of the K clusters
   - Initialize parameters (e.g., use k-means to get approximate answer)

   - Expectation step:
     Estimate the probability to assign data to each component q_nk
     (responsibilities)

   - Maximization step:
     Update μ_k, Σ_k, and π_k

   - Repeat until convergence

Step 2: Use Responsibilities for Outlier Detection [1 mark]

The probability assigned to each data point (q_nk - responsibilities) reflects its distance from the cluster distribution.

Step 3: Threshold-Based Detection [2 marks]

Outlier Detection:

1. For each data point n:
   - Compute max_k(q_nk) = maximum responsibility across all clusters

2. Use a threshold to highlight points unlikely to belong to any cluster:
   - Set threshold: pval_threshold

3. Identify outliers:
   - Outliers O = {points where max_k(q_nk) < pval_threshold}

Interpretation:
- If max_k(q_nk) is small for all k, the point doesn't fit any cluster well
- Such points are likely outliers
Alternative Approach: Compute the log-likelihood of each point under the fitted GMM. Points with very low likelihood (below a threshold) are outliers:

\[\log p(x_n) = \log\left(\sum_{k=1}^K \pi_k \mathcal{N}(x_n|\mu_k, \Sigma_k)\right)\]

Outliers: \(\{x_n : \log p(x_n) < \text{threshold}\}\)

Complete Pseudo-code:

OUTLIER_DETECTION_GMM(data, K, threshold):
    # Fit GMM
    params = INITIALIZE_GMM(data, K)  # Use k-means

    converged = False
    while not converged:
        # E-step
        responsibilities = COMPUTE_RESPONSIBILITIES(data, params)

        # M-step
        params = UPDATE_PARAMETERS(data, responsibilities)

        converged = CHECK_CONVERGENCE(params)

    # Detect outliers
    outliers = []
    for n in 1 to N:
        max_responsibility = max(responsibilities[n, :])
        if max_responsibility < threshold:
            outliers.append(n)

    return outliers, params
