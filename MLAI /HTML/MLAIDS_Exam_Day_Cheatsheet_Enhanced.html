1. REGRESSION

🔹 Linear Regression

🎯 Why We Use Linear Regression:

  • Simplicity: Easy to understand, interpret, and explain to non-technical stakeholders
  • Closed-form solution: Can find optimal weights directly without iterative optimization
  • Baseline model: Serves as a benchmark to compare against more complex models
  • Low computational cost: Fast to train, even on large datasets
  • Interpretability: Each weight shows the direct impact of that feature on the output
  • Well-understood: Extensive theory, known statistical properties, confidence intervals available

📅 When to Use Linear Regression:

  • Relationship between features and target is approximately linear
  • You need interpretable results (e.g., "each year of education increases salary by $X")
  • As a baseline model before trying complex methods
  • When you have limited data (low risk of overfitting)
  • Real-time predictions needed (very fast inference)
  • Examples: predicting house prices, sales forecasting, temperature prediction

Model Equation:

\[y = w^T x + b = w_0 + w_1x_1 + w_2x_2 + ... + w_Dx_D\]

What this means: The output y is a weighted sum of input features. Each weight w_j tells us how much feature x_j contributes to the prediction. The bias b shifts the prediction up or down. This creates a hyperplane in D-dimensional space.

Loss Function (Mean Squared Error):

\[L(w) = \frac{1}{N}\sum_{i=1}^{N}(y_i - \hat{y}_i)^2 = \frac{1}{N}\sum_{i=1}^{N}(y_i - w^Tx_i)^2\]

What this means: MSE measures the average squared difference between true values (y_i) and predictions (ŷ_i). We square the errors to: (1) make all errors positive, (2) penalize large errors more heavily than small ones, (3) make the function differentiable everywhere. Minimizing MSE finds the "best fit" line that's closest to all data points.

Closed-Form Solution (Normal Equation):

\[w = (\Phi^T\Phi)^{-1}\Phi^T y\]

What this means: This gives the optimal weights directly without iterative optimization! Φ is the design matrix (N×D) where each row is a data point. Φ^TΦ is D×D (features × features), and (Φ^TΦ)^(-1) exists when columns are linearly independent. This formula comes from setting ∇L(w) = 0 and solving for w.

Computational note: Works well for small/medium D, but expensive for large D (O(D³) due to matrix inversion). Also fails if Φ^TΦ is singular (use Ridge regression instead).

🧠 Geometric Intuition:

Linear regression finds a hyperplane that minimizes the sum of squared vertical distances from all points to the plane. In 2D (one feature), this is a line. In 3D (two features), it's a plane. In higher dimensions, it's a hyperplane.

Key Assumptions (and why they matter):
  • Linearity: Relationship is linear (if violated → use basis functions or non-linear models)
  • Independence: Observations are independent (if violated → use time series methods or mixed effects models)
  • Homoscedasticity: Constant variance of errors (if violated → weighted least squares or transform target)
  • Normality: Errors are normally distributed ε ~ N(0, σ²) (if violated → robust regression or bootstrap for inference)
  • No multicollinearity: Features not too correlated (if violated → use Ridge/Lasso or PCA)

🔹 Basis Functions

🎯 Why We Use Basis Functions:

  • Capture non-linearity: Transform features so linear regression can fit non-linear relationships
  • Flexibility: Can approximate any continuous function (universal approximation)
  • Keep simplicity: Still use linear regression machinery (closed-form solutions, interpretability)
  • Feature engineering: Create informative features from raw inputs
  • Domain knowledge: Encode known relationships (e.g., polynomial for growth, periodic for seasons)

📅 When to Use Each Basis Function:

  • Polynomial: When relationship has global trends (curves, parabolas). Good for: growth curves, physics equations. Bad for: local features, oscillations
  • RBF (Gaussian): When relationship has local patterns or bumps. Good for: spatial data, local anomalies. Bad for: extrapolation, interpretability
  • Sigmoid: When relationship has transitions or thresholds. Good for: S-shaped growth, saturation effects
  • Fourier: When data has periodic patterns. Good for: seasonal data, time series, signals

Polynomial Basis:

\[\phi(x) = [1, x, x^2, x^3, ..., x^d]^T\]

What this means: Transform single input x into d+1 features. Model becomes: y = w₀ + w₁x + w₂x² + ... + wₐx^d. This creates polynomial curves. Degree d controls flexibility: d=1 is linear, d=2 is parabola, higher d captures more complex curves but risks overfitting.

RBF (Gaussian) Basis:

\[\phi_j(x) = \exp\left(-\frac{(x-\mu_j)^2}{2s^2}\right)\]

What this means: Creates "bump" functions centered at μ_j. Value is 1 at center, decays to 0 away from center. Width s controls spread: small s = narrow bumps (local features), large s = wide bumps (smooth). Place M centers across data range to cover space.

📝 Concrete Example:

Task: Fit Olympic birth rate data (1909-2008) with multiple ups and downs

  • Polynomial (degree 2): Too simple, misses baby boom and later drops. Forms single parabola.
  • Polynomial (degree 10): Without rescaling, year^10 ~ 10^33 causes numerical overflow!
  • RBF basis: With centers at each year and small width, can capture all local changes perfectly but may overfit noise.
  • Solution: Use moderate degree polynomial OR RBF with regularization
Rescaling is Critical! When using high-degree polynomials (d≥5), values like 2000^10 are astronomical. This causes:
  • Numerical overflow (numbers too large to store)
  • Matrix inversion instability (Φ^TΦ becomes ill-conditioned)
  • Loss of precision in floating-point arithmetic
Solutions: (1) Standardization: x' = (x-μ)/σ, (2) Min-max: x' = (x-min)/(max-min), (3) Logarithm: x' = log(x)

🔹 Regularization (Ridge & Lasso)

🎯 Why We Use Regularization:

  • Prevent overfitting: Constrains model complexity even with many features
  • Handle p > n: Works when features > samples (ordinary regression fails)
  • Multicollinearity: Stabilizes estimates when features are correlated
  • Feature selection: Lasso automatically identifies important features
  • Improve generalization: Better test performance by trading training error for lower complexity
  • Encode prior beliefs: We expect most weights should be small (smoothness prior)

📅 When to Use Ridge vs Lasso:

Use Ridge (L2) when:

  • All features are potentially relevant
  • Features are correlated (Ridge handles better than Lasso)
  • You want smooth, stable coefficient estimates
  • Need fast computation (closed-form solution available)
  • Example: Image reconstruction, signal denoising

Use Lasso (L1) when:

  • Only subset of features are relevant (sparse ground truth)
  • You need interpretability (which features matter?)
  • You have many features, limited samples (p >> n)
  • Want automatic feature selection
  • Example: Gene expression analysis (1000s genes, 100s samples)

Use Elastic Net when:

  • You're unsure which is better
  • Want both feature selection and grouping
  • Correlated features but need sparsity

Objective \(\sum(y_i - w^T\phi(x_i))^2 + \lambda\sum w_j^2\) \(\sum(y_i - w^T\phi(x_i))^2 + \lambda\sum|w_j|\)

Solution Dense (all non-zero) Sparse (many zeros)
Closed Form? Yes: w = (Φ^TΦ + λI)^(-1)Φ^Ty No (use coordinate descent, LARS)
Geometry Circular constraint (L2 ball) Diamond constraint (L1 ball with corners)

Ridge Regression Solution:

\[w = (\Phi^T\Phi + \lambda I)^{-1}\Phi^T y\]

What this means: Similar to ordinary regression but adds λI to the matrix before inverting. This has two effects: (1) Ensures invertibility - even if Φ^TΦ is singular, adding λI makes it positive definite and invertible. (2) Shrinks weights - larger λ pulls weights toward zero. The identity matrix I has same dimensions as Φ^TΦ (D×D). As λ→0, recovers ordinary regression; as λ→∞, weights→0.

🧠 Geometric Intuition - Why Lasso Creates Zeros:

Imagine plotting the loss function contours (ellipses) and the constraint region (diamond for Lasso, circle for Ridge). The optimal solution is where they touch:

  • Lasso (diamond): Has sharp corners on axes where weights are zero. Contours often hit these corners first → many weights become exactly zero!
  • Ridge (circle): Smooth, no corners. Contours touch at non-axis points → weights are small but non-zero.

📝 Real Example from Past Papers:

Birth rate data with sigmoid basis functions:

  • Too strong Lasso (λ large): Created three flat regions where ALL basis function weights = 0. Model can't respond to data there.
  • Too strong Ridge (λ large): All basis function weights very small (~0 but not exactly). Model barely follows trend, nearly flat line.
  • Lesson: λ controls trade-off. Use cross-validation to find sweet spot!
Choosing λ (Regularization Strength):
  • λ = 0: No regularization, ordinary regression (may overfit)
  • Small λ: Weak penalty, nearly same as ordinary regression
  • Optimal λ: Found by cross-validation - balances bias-variance trade-off
  • Large λ: Strong penalty, very small weights (may underfit)
  • λ → ∞: All weights → 0, model predicts mean of y

🔹 Bias-Variance Trade-off

🎯 Why Understanding Bias-Variance Matters:

  • Diagnostic tool: Helps identify if model is too simple or too complex
  • Model selection: Guides choice of model complexity
  • Explains generalization: Why training error ≠ test error
  • Guides fixes: High bias → add features/complexity; High variance → add data/regularization
  • Fundamental trade-off: Can't minimize both simultaneously - must find balance

Expected Test Error Decomposition:

\[\text{Expected Error} = \text{Bias}^2 + \text{Variance} + \text{Irreducible Error}\]

What each term means:
  • Bias²: How far average prediction is from true value (systematic error). Measures underfitting.
  • Variance: How much predictions change with different training sets (instability). Measures overfitting.
  • Irreducible Error: Noise in data that no model can capture (σ² in y = f(x) + ε).
These three components are independent! Reducing one doesn't affect others (except bias-variance trade-off).

🔴 High Bias (Underfitting):

Characteristics:

  • Model too simple for the data
  • High training error
  • High test error (similar to training)
  • Cannot capture true pattern
  • Makes strong assumptions

Examples:

  • Linear model for non-linear data
  • Low-degree polynomial (d=1 or 2)
  • Too strong regularization (λ too large)

Fix:

  • Increase model complexity
  • Add more features
  • Reduce regularization
  • Use more powerful model

🔵 High Variance (Overfitting):

Characteristics:

  • Model too complex
  • Very low training error
  • High test error (gap!)
  • Fits noise as signal
  • Very sensitive to training data

Examples:

  • High-degree polynomial (d=10+)
  • Too many RBF centers
  • No regularization with many features

Fix:

  • Add more training data
  • Use regularization (Ridge/Lasso)
  • Reduce model complexity
  • Use cross-validation

📝 Concrete Example: Polynomial Degree Selection

  • d=1 (linear): High bias, low variance. Straight line can't capture curves. Consistent but wrong.
  • d=2-3: Balanced. Captures main trends without fitting noise.
  • d=10: Low bias, high variance. Perfect on training data, terrible on test (wiggles wildly between points).

Lesson: Optimal d minimizes bias² + variance. Use cross-validation to find it!

The Trade-off Explained:
  • As model complexity ↑: Bias ↓ (can fit better), Variance ↑ (more sensitive)
  • As model complexity ↓: Bias ↑ (can't fit well), Variance ↓ (more stable)
  • Sweet spot: Model complexity that minimizes their sum
  • Can't have both low: Fundamental trade-off in learning theory
  • More data helps: Can support more complex models (reduces variance)

🔹 Cross-Validation

🎯 Why We Use Cross-Validation:

  • Better performance estimate: Uses all data for both training and validation
  • Reduces variance: Averages over multiple splits → more reliable estimate
  • Maximum data usage: No data wasted (unlike single train/val split)
  • Model selection: Choose best hyperparameters (λ, degree d, k for k-NN)
  • Fair comparison: Compare different models on same folds
  • Detects overfitting: Large gap between train and CV error indicates overfitting

📅 When to Use Cross-Validation:

  • Always when selecting hyperparameters (never tune on test set!)
  • When you have limited data and can't afford a large validation set
  • When comparing multiple models fairly
  • To get confidence intervals on performance (from variance across folds)
  • Don't use when: Data is huge (expensive), time series (use time-based splits), data has structure (use stratified splits)

K-Fold Cross-Validation Algorithm:

  1. Split: Randomly divide data into K equal folds
    Each fold has N/K samples. Common: K=5 (20% validation) or K=10 (10% validation)
  2. Loop: For each fold i (i=1 to K):
    • Use fold i as validation set
    • Use remaining K-1 folds as training set (80-90% of data)
    • Train model on training set
    • Evaluate error on validation fold → Error_i
    Each data point is used for validation exactly once
  3. Average: Compute mean performance across all K folds
    \(\text{CV Error} = \frac{1}{K}\sum_{i=1}^{K}\text{Error}_i\)
  4. Select: Choose model/hyperparameters with best average CV error
    This model likely generalizes well to new data
  5. Retrain: Train chosen model on ALL data for final model
    Uses maximum data since hyperparameters already selected

📝 Example: Choosing λ for Ridge Regression

Given 100 data points, want to find best λ from [0.01, 0.1, 1, 10, 100]:

  1. Split into 5 folds (20 points each)
  2. For λ = 0.01:
    • Fold 1 validation: Train on folds 2-5 (80 points), test on fold 1 → Error = 2.5
    • Fold 2 validation: Train on folds 1,3-5 (80 points), test on fold 2 → Error = 2.3
    • ... (3 more folds)
    • Average error = 2.4
  3. Repeat for all λ values
  4. Choose λ with lowest average error (say λ=1 gives 1.8)
  5. Retrain with λ=1 on all 100 points → final model
K-Fold Variants:
  • K=5: Common choice. Good bias-variance trade-off. Fast.
  • K=10: More stable estimates, slightly more expensive.
  • K=N (LOOCV): Unbiased estimate but very expensive. Each training uses N-1 points.
  • Stratified K-Fold: For classification - ensures each fold has same class distribution.
  • Time Series CV: For temporal data - use rolling window or expanding window (no random splits!).

2. CLASSIFICATION

🔹 k-Nearest Neighbors (k-NN)

🎯 Why We Use k-NN:

  • Simplicity: No training phase! Just store data and classify on demand
  • Non-parametric: Makes no assumptions about data distribution (works for any pattern)
  • Intuitive: "You are like your neighbors" - easy to explain
  • Flexible decision boundaries: Can form arbitrarily complex boundaries (not just linear)
  • Naturally multi-class: Handles multiple classes without modification
  • Good baseline: Surprisingly effective, especially with good distance metrics

📅 When to Use k-NN:

  • Small to medium datasets (computational cost grows with data size)
  • When decision boundary is complex and non-linear
  • When you have good distance metric for your domain
  • For lazy learning - can update instantly by adding new data points
  • As a baseline before trying complex methods
  • Examples: Handwriting recognition, recommendation systems, anomaly detection

DON'T use when:

  • Dataset is very large (slow predictions)
  • Features have different scales (must normalize!)
  • High dimensionality (curse of dimensionality - distances become meaningless)
  • Need fast real-time predictions

k-NN Algorithm:

  1. Store: Keep all training data (x_i, y_i)
    This is the entire "training" - just memorize!
  2. Predict: Given new point x*:
    • Compute distance to all training points
    • Find k nearest neighbors
    • Take majority vote of their labels

Distance Metrics:

Euclidean (L2): \(d(x, x') = \sqrt{\sum_{j=1}^{D}(x_j - x'_j)^2}\)

Straight-line distance. Good for continuous features with similar scales. Most common choice.

Manhattan (L1): \(d(x, x') = \sum_{j=1}^{D}|x_j - x'_j|\)

City-block distance (only horizontal/vertical moves). Less sensitive to outliers. Good for high-dimensional data.

📝 Example: 2D Classification

Points: A(1,2), B(2,3), C(1,5), D(4,1). Classes: A,B→Red, C,D→Blue. Classify (2,2) with k=3:

  1. Distances: d(A)=√1=1.0, d(B)=√2=1.41, d(C)=√10=3.16, d(D)=√5=2.24
  2. 3 nearest: A(1.0), B(1.41), D(2.24)
  3. Vote: Red, Red, Blue → Red wins (2 vs 1)
Choosing k:
  • k=1: Overfits! Every training point creates island. Sensitive to noise.
  • Small k (3-5): Flexible boundaries, sensitive to local structure
  • Large k: Smoother boundaries, more stable but may underfit
  • k=N: Always predicts majority class (useless!)
  • Best practice: Use odd k to avoid ties. Choose via cross-validation.
Feature Scaling is CRITICAL! If feature 1 is in [0,1] and feature 2 is in [0,1000], distance is dominated by feature 2. Always standardize: x' = (x - μ)/σ before k-NN!

🔹 Logistic Regression

🎯 Why We Use Logistic Regression:

  • Probabilistic output: Gives P(class|x) not just hard decision - useful for confidence/ranking
  • Interpretable: Weights show feature importance (like linear regression)
  • Efficient: Fast to train and predict, scales well
  • Well-calibrated: Probabilities are meaningful (unlike SVM scores)
  • Smooth decision boundary: Not as jagged as decision trees
  • Regularization: Easily add L1/L2 to prevent overfitting

📅 When to Use Logistic Regression:

  • When you need probability estimates (not just class labels)
  • For binary or multi-class classification
  • When decision boundary is approximately linear
  • When you need interpretability (which features predict class?)
  • For online learning (can update weights incrementally)
  • Examples: Medical diagnosis, spam detection, click prediction

Sigmoid Function:

\[\sigma(z) = \frac{1}{1 + e^{-z}}\]

What this does: Squashes any real number z into range (0,1) to interpret as probability.
  • z → ∞: σ(z) → 1 (confident positive)
  • z = 0: σ(z) = 0.5 (neutral, decision boundary)
  • z → -∞: σ(z) → 0 (confident negative)
S-shaped curve. Smooth, differentiable everywhere.

Model:

\[P(y=1|x) = \sigma(w^T x) = \frac{1}{1 + \exp(-w^T x)}\]

What this means: First compute linear combination z = w^T x (like linear regression). Then apply sigmoid to get probability. Decision boundary is where w^T x = 0 (a hyperplane). Points on one side have P>0.5 (predict 1), other side P<0.5 (predict 0).

Loss Function (Binary Cross-Entropy):

\[L(w) = -\sum_{i=1}^{N}[y_i\log(\hat{y}_i) + (1-y_i)\log(1-\hat{y}_i)]\]

What this means: Penalizes wrong predictions heavily. If y=1, wants ŷ→1 (log(ŷ)→0). If y=0, wants ŷ→0 (log(1-ŷ)→0). Convex! Can optimize with gradient descent. Also called log-loss or negative log-likelihood.

Gradient for Optimization:

\[\frac{\partial L}{\partial w} = \sum_{i=1}^{N}(\hat{y}_i - y_i)x_i\]

Beautiful result! Gradient has same form as linear regression. Tells us how to adjust each weight. If ŷ > y (over-predicted), reduce weights for positive features. Update: w ← w - η∇L (gradient descent).

🧠 Geometric Intuition:

Decision boundary is a hyperplane w^T x = 0. Points far from boundary have high confidence (P near 0 or 1). Points close to boundary have low confidence (P near 0.5). Unlike hard SVM, transition is smooth via sigmoid.

Multi-class Strategies:
  • One-vs-Rest (OvR): Train K binary classifiers, one per class. Pick class with highest score. Simple but may have ambiguous regions.
  • One-vs-One (OvO): Train K(K-1)/2 classifiers for each pair. Majority vote. More classifiers but simpler problems.
  • Softmax (Multinomial): Generalization to K classes. \(P(y=k|x) = \frac{\exp(w_k^T x)}{\sum_j \exp(w_j^T x)}\). Probabilities sum to 1. Most principled approach.

🔹 Support Vector Machines (SVM)

🎯 Why We Use SVM:

  • Maximum margin: Finds most robust decision boundary (farthest from both classes)
  • Kernel trick: Efficiently works in infinite-dimensional spaces!
  • Works with high dimensions: Effective when features >> samples
  • Memory efficient: Only stores support vectors (small subset of data)
  • Robust to outliers: Soft margin tolerates noisy points
  • Strong theoretical foundation: Based on statistical learning theory

📅 When to Use SVM:

  • When you have high-dimensional data (text, genomics)
  • When clear margin exists between classes
  • For binary classification (natively designed for 2 classes)
  • When data is not too large (O(N²) to O(N³) training time)
  • Need non-linear boundaries via kernel trick
  • Examples: Text classification, face recognition, bioinformatics

DON'T use when:

  • Very large datasets (>100k samples) - too slow
  • Need probability estimates (SVM gives scores, not calibrated probabilities)
  • Data is very noisy (logistic regression may be better)

Hard Margin SVM (Linearly Separable):

\[\min_{w,b} \frac{1}{2}\|w\|^2 \quad \text{s.t.} \quad y_i(w^T x_i + b) \geq 1 \quad \forall i\]

What this means:
  • Objective ||w||²/2: Minimize weight norm = maximize margin (margin = 2/||w||)
  • Constraint y_i(w^T x_i + b) ≥ 1: All points correctly classified with at least distance 1/||w|| from boundary
  • Points with y_i(w^T x_i + b) = 1 are support vectors - they define the margin
  • Only works if data is perfectly separable!

Soft Margin SVM (Allow Errors):

\[\min_{w,b,\xi} \frac{1}{2}\|w\|^2 + C\sum_{i=1}^{N}\xi_i \quad \text{s.t.} \quad y_i(w^T x_i + b) \geq 1-\xi_i, \quad \xi_i \geq 0\]

What this means:
  • Slack variables ξ_i: Allow violation of margin. ξ_i=0 means correctly classified, ξ_i>0 means violation
  • C parameter: Trade-off between margin and violations. Large C = hard margin (few violations), small C = soft margin (wide margin, more violations)
  • Handles non-separable data and is more robust to outliers

Kernel Trick:

\[K(x, x') = \phi(x)^T \phi(x')\]

RBF Kernel: \(K(x, x') = \exp\left(-\gamma\|x-x'\|^2\right)\)

Magic of kernel trick: Instead of explicitly computing high/infinite-dimensional features φ(x), directly compute inner product via kernel K. This means:
  • Can work in infinite dimensions without computing them!
  • RBF kernel measures similarity: K=1 if x=x' (identical), K→0 if x far from x'
  • γ controls width: small γ = wide influence (smooth), large γ = narrow (complex boundary)
  • Creates non-linear decision boundaries in original space

🧠 What are Support Vectors?

Support vectors are the training points closest to the decision boundary (exactly on the margin). They "support" the hyperplane - if you remove other points, decision boundary stays same! Only need to store these (~5-20% of data typically).

📝 Example: Identifying Support Vectors

Given w=(2,1), b=-5, margin = 2/||w|| = 2/√5 ≈ 0.894

Check points:

  • A(1,2): y_A(2·1+1·2-5) = (+1)(-1) = -1 < 1 → VIOLATES (inside margin)
  • B(3,2): y_B(2·3+1·2-5) = (+1)(3) = 3 > 1 → Not SV (too far)
  • C(2,1): y_C(2·2+1·1-5) = (+1)(0) = 0 < 1 → SUPPORT VECTOR (on margin)
Hyperparameter Tuning:
  • C (regularization): C↑ → fewer margin violations but may overfit. C↓ → wider margin but more errors. Use CV to tune.
  • γ (RBF width): γ↑ → complex boundary (may overfit). γ↓ → simple boundary (may underfit). Typical range: [0.001, 10].
  • Trade-off: Often C and γ correlated - tune together via grid search + cross-validation

🔹 Decision Trees

🎯 Why We Use Decision Trees:

  • Highly interpretable: Can visualize and explain to anyone (if-then rules)
  • No feature scaling needed: Works with raw features (unlike k-NN, SVM, logistic)
  • Handles mixed data: Both categorical and numerical features
  • Non-parametric: No assumptions about data distribution
  • Captures interactions: Automatically finds feature combinations
  • Fast prediction: O(log N) for balanced tree

📅 When to Use Decision Trees:

  • When interpretability is critical (medical, legal, finance)
  • Data has categorical features and/or missing values
  • Need to explain decisions to non-technical users
  • As base learner for ensembles (Random Forest, Gradient Boosting)
  • Examples: Customer churn prediction, loan approval, diagnosis

DON'T use alone when:

  • Overfitting is concern (use ensemble or prune)
  • Linear relationships (logistic regression better)
  • Need best accuracy (use Random Forest instead)

Information Gain (IG):

\[IG = H(\text{parent}) - \sum_{\text{children}} \frac{N_{\text{child}}}{N_{\text{parent}}} H(\text{child})\]

What this means: Information gain measures how much a split reduces uncertainty (entropy). Choose split that maximizes IG = makes children most pure. Higher IG = better split.

Entropy:

\[H = -\sum_{k=1}^{K} p_k \log_2(p_k)\]

What this means: Measures impurity/uncertainty in a node.
  • H = 0: Pure node (all same class) - very certain!
  • H = 1 (for binary): Maximum impurity (50-50 split) - very uncertain
  • p_k = fraction of class k in node
  • Lower entropy = better (more pure)

📝 Example: Computing Information Gain

Parent node: 10 samples (6 Yes, 4 No)

H_parent = -(6/10)log₂(6/10) - (4/10)log₂(4/10) = 0.971

Split on feature X:

  • Left child: 6 samples (5 Yes, 1 No) → H_left = 0.650
  • Right child: 4 samples (1 Yes, 3 No) → H_right = 0.811

IG = 0.971 - (6/10)×0.650 - (4/10)×0.811 = 0.256

This split reduces uncertainty by 0.256 bits!

Preventing Overfitting:
  • Max depth: Limit tree depth (typical: 5-20)
  • Min samples split: Don't split nodes with
  • Min samples leaf: Each leaf must have ≥N samples (typical: 5-20)
  • Pruning: Grow full tree then remove branches that don't improve validation error
  • Use ensemble: Random Forest averages many trees → better generalization
Common Pitfalls: (1) Unpruned trees overfit badly. (2) Biased toward features with many values. (3) Unstable - small data changes cause different trees. (4) Poor at extrapolation. Solution: Use Random Forest!

3. UNSUPERVISED LEARNING

🔹 K-means Clustering

🎯 Why We Use K-means:

  • Simplicity: Easy to understand and implement
  • Efficiency: Very fast - O(nkd) per iteration (n=samples, k=clusters, d=dimensions)
  • Scalability: Works well with large datasets
  • Guaranteed convergence: Objective always decreases (though to local minimum)
  • Widely applicable: Customer segmentation, image compression, data preprocessing
  • Clear interpretation: Each cluster has a center (prototype)

📅 When to Use K-means:

  • When you know (or can estimate) number of clusters k
  • Clusters are spherical/globular in shape
  • Clusters have similar sizes
  • Need fast clustering on large data
  • Examples: Customer segmentation, image quantization, feature learning

DON'T use when:

  • Clusters have complex shapes (use DBSCAN or spectral clustering)
  • Clusters have very different densities/sizes (use GMM)
  • Don't know k (try hierarchical or use elbow method)
  • Data has outliers (k-means sensitive to them)

Objective Function:

\[J = \sum_{i=1}^{N}\sum_{k=1}^{K} r_{ik}\|x_i - \mu_k\|^2\]

What this means: Minimize total within-cluster variance (sum of squared distances from points to their cluster centers).
  • r_ik: Binary indicator (1 if point i in cluster k, 0 otherwise)
  • μ_k: Center (mean) of cluster k
  • ||x_i - μ_k||²: Squared distance from point to center
  • Goal: Make clusters compact (points close to their centers)

K-means Algorithm:

  1. Initialize: Randomly select k points as initial cluster centers μ₁,...,μ_k
    Or use k-means++ for smarter initialization (spread out centers)
  2. Assignment Step: For each point x_i, assign to nearest cluster
    \(r_{ik} = 1\) if k = argmin_j ||x_i - μ_j||², else \(r_{ik} = 0\)
  3. Update Step: Recompute cluster centers as mean of assigned points
    \(\mu_k = \frac{\sum_i r_{ik} x_i}{\sum_i r_{ik}}\) (average of points in cluster k)
  4. Repeat: Steps 2-3 until convergence (centers don't change or max iterations)
    Typically converges in 10-100 iterations

📝 Example: One K-means Iteration

Points: A(1,1), B(2,1), C(1,2), D(8,8), E(9,8), F(8,9). k=2

Initial centers: μ₁=(1,1), μ₂=(8,8)

Assignment:

  • A,B,C: Closer to μ₁ → Cluster 1
  • D,E,F: Closer to μ₂ → Cluster 2

Update:

  • New μ₁ = mean(A,B,C) = ((1+2+1)/3, (1+1+2)/3) = (1.33, 1.33)
  • New μ₂ = mean(D,E,F) = ((8+9+8)/3, (8+8+9)/3) = (8.33, 8.33)
Choosing k (Number of Clusters):
  • Elbow method: Plot J vs k. Choose k at "elbow" where improvement slows
  • Silhouette score: Measures how well points fit their clusters. Score in [-1,1], higher better
  • Domain knowledge: Sometimes k is known (e.g., k=3 customer tiers)
  • Gap statistic: Compare within-cluster variance to random data
K-means Limitations: (1) Sensitive to initialization (run multiple times with different seeds). (2) Assumes spherical clusters. (3) Need to choose k. (4) Sensitive to outliers and scale (normalize features!). (5) Converges to local minimum (not global).

🔹 Gaussian Mixture Models (GMM)

🎯 Why We Use GMM:

  • Soft clustering: Points can belong to multiple clusters with probabilities
  • Flexible shapes: Each cluster can have different shape (elliptical) and size
  • Probabilistic: Provides uncertainty estimates (unlike hard k-means)
  • Statistical foundation: Based on mixture of Gaussians (well-understood theory)
  • Generative model: Can generate new samples from learned distribution
  • Handles overlapping clusters: When boundaries are fuzzy

📅 When to Use GMM:

  • Clusters have different shapes/sizes/orientations
  • Need probabilistic assignments (not hard labels)
  • Clusters overlap (ambiguous boundaries)
  • Want to model data distribution and generate samples
  • Examples: Speaker recognition, image segmentation, anomaly detection

Comparison with K-means:

  • GMM: Soft clusters, elliptical shapes, slower but more flexible
  • K-means: Hard clusters, spherical shapes, faster but more restrictive (special case of GMM!)

Mixture Model:

\[p(x) = \sum_{k=1}^{K}\pi_k \mathcal{N}(x|\mu_k, \Sigma_k)\]

What this means: Data is generated from K Gaussian distributions mixed together.
  • π_k: Mixing coefficient (probability of cluster k). Must sum to 1: Σπ_k = 1
  • μ_k: Mean vector of cluster k (center)
  • Σ_k: Covariance matrix of cluster k (shape and orientation)
  • N(x|μ,Σ): Multivariate Gaussian distribution

Gaussian Distribution:

\[\mathcal{N}(x|\mu, \Sigma) = \frac{1}{(2\pi)^{D/2}|\Sigma|^{1/2}}\exp\left(-\frac{1}{2}(x-\mu)^T\Sigma^{-1}(x-\mu)\right)\]

What this means: Probability density of observing x in a Gaussian centered at μ with spread Σ.
  • (2π)^(D/2)|Σ|^(1/2): Normalization constant (ensures probabilities integrate to 1)
  • (x-μ)^T Σ^(-1) (x-μ): Mahalanobis distance (accounts for covariance)
  • If Σ is diagonal: axes-aligned ellipse. If Σ is identity: sphere (same as k-means!)

Responsibility (Soft Assignment):

\[\gamma_{ik} = P(\text{cluster } k|x_i) = \frac{\pi_k \mathcal{N}(x_i|\mu_k,\Sigma_k)}{\sum_{j=1}^{K}\pi_j \mathcal{N}(x_i|\mu_j,\Sigma_j)}\]

What this means: Probability that point i belongs to cluster k (Bayes theorem!).
  • γ_ik ∈ [0,1] and Σ_k γ_ik = 1 for each point i
  • Unlike k-means (r_ik ∈ {0,1}), GMM gives soft probabilities
  • If point near center: one γ_ik ≈ 1, others ≈ 0 (confident assignment)
  • If point between clusters: multiple γ_ik > 0 (uncertain assignment)

EM Algorithm for GMM:

  1. Initialize: Set initial parameters (π_k, μ_k, Σ_k)
    Can use k-means to get initial μ_k
  2. E-step (Expectation): Compute responsibilities γ_ik for all points
    Given current parameters, calculate soft cluster assignments
  3. M-step (Maximization): Update parameters using responsibilities
    • \(\pi_k = \frac{1}{N}\sum_i \gamma_{ik}\) (cluster weight)
    • \(\mu_k = \frac{\sum_i \gamma_{ik}x_i}{\sum_i \gamma_{ik}}\) (weighted mean)
    • \(\Sigma_k = \frac{\sum_i \gamma_{ik}(x_i-\mu_k)(x_i-\mu_k)^T}{\sum_i \gamma_{ik}}\) (weighted covariance)
    Given current assignments, find parameters that maximize likelihood
  4. Repeat: E and M steps until convergence (log-likelihood increases)
    Converges to local maximum (not global!)

🧠 K-means vs GMM Connection:

K-means is a special case of GMM where: (1) All covariances are identity: Σ_k = σ²I (spherical clusters), (2) σ² → 0 (hard assignments), (3) All mixing coefficients equal: π_k = 1/K. GMM generalizes this to soft, elliptical clusters!

Advantages of GMM over K-means:
  • ✅ Clusters can have different sizes and shapes (via Σ_k)
  • ✅ Soft assignments provide uncertainty (useful for decision-making)
  • ✅ Probabilistic framework (can use model selection criteria like BIC)
  • ✅ Can handle overlapping clusters
  • ⚠️ But: More parameters, slower, harder to interpret, needs more data

🔹 Principal Component Analysis (PCA)

🎯 Why We Use PCA:

  • Dimensionality reduction: Reduce D features to d<
  • Visualization: Project high-D data to 2D/3D for plotting
  • Remove correlation: Transform correlated features to uncorrelated components
  • Noise reduction: Keep signal (high variance), discard noise (low variance)
  • Speed up learning: Fewer features → faster training
  • Interpretability: Principal components show main patterns of variation

📅 When to Use PCA:

  • Have many correlated features (can combine into fewer components)
  • Need to visualize high-dimensional data
  • Want to compress data (fewer dimensions, less storage)
  • Need to remove multicollinearity before regression
  • As preprocessing before clustering/classification
  • Examples: Image compression, face recognition (eigenfaces), gene expression analysis

DON'T use when:

  • Features are already uncorrelated (PCA won't help)
  • Need interpretability of original features (PCA creates linear combinations)
  • Non-linear structure (use kernel PCA or t-SNE)

PCA Objective:

\[\max_{w_1} \text{Var}(w_1^T X) = \max_{w_1} w_1^T \Sigma w_1 \quad \text{s.t.} \quad \|w_1\|=1\]

What this means: Find direction w₁ that maximizes variance of projected data.
  • w₁^T X: Project data X onto direction w₁ (scalar for each point)
  • Var(w₁^T X): Variance along this direction (how spread out points are)
  • ||w₁||=1: Constraint that w₁ is unit vector (prevents unbounded solution)
  • First principal component = direction of maximum variance
  • Subsequent PCs: orthogonal directions of decreasing variance

Solution (Eigen-decomposition):

\[\Sigma w_j = \lambda_j w_j\]

What this means: Principal components are eigenvectors of covariance matrix Σ!
  • Σ = (1/N) X^T X: Covariance matrix (D×D, measures how features vary together)
  • w_j: j-th principal component (eigenvector)
  • λ_j: Variance explained by j-th PC (eigenvalue)
  • Order by eigenvalues: λ₁ ≥ λ₂ ≥ ... ≥ λ_D
  • Keep top d eigenvectors for dimensionality reduction

Projection & Reconstruction:

Projection: \(z = W^T x\) where W = [w₁ w₂ ... w_d] (D×d matrix)

Reconstruction: \(\tilde{x} = W z = W W^T x\)

What this means:
  • z: Low-dimensional representation (d numbers instead of D)
  • W^T x: Project D-dim point to d-dim space (compress)
  • W W^T x: Reconstruct back to D-dim (decompress, with some loss)
  • Reconstruction error: ||x - x̃||² (information lost by compression)

PCA Algorithm:

  1. Center data: X ← X - mean(X)
    Subtract mean of each feature. CRITICAL: PCA requires centered data!
  2. Compute covariance: Σ = (1/N) X^T X
    D×D matrix showing how features covary
  3. Eigen-decomposition: Find eigenvectors w_j and eigenvalues λ_j of Σ
    Use numerical methods (SVD is more stable than direct eigen-decomposition)
  4. Sort: Order eigenvectors by decreasing eigenvalues
    λ₁ ≥ λ₂ ≥ ... ≥ λ_D (first PC explains most variance)
  5. Select: Keep top d eigenvectors as W
    Choose d to retain desired % of variance (e.g., 95%)
  6. Transform: Z = X W (project all data to low-dim space)
    N×D matrix → N×d matrix

📝 Example: Choosing Number of Components

Eigenvalues: [5.2, 3.1, 0.8, 0.5, 0.3, 0.1] (total = 10)

Variance explained:

  • PC1: 5.2/10 = 52%
  • PC1+PC2: (5.2+3.1)/10 = 83%
  • PC1+PC2+PC3: (5.2+3.1+0.8)/10 = 91%

Decision: Keep d=2 for 83% variance (good compression: 6→2 dims)

Choosing Number of Components d:
  • Variance explained: Keep d PCs that explain ≥90-95% total variance
  • Scree plot: Plot eigenvalues, look for "elbow" where drop-off slows
  • Cross-validation: Try different d, see which gives best downstream task performance
  • Kaiser rule: Keep PCs with λ > 1 (for standardized data)
Important: (1) Always center (and often standardize) data before PCA. (2) PCA finds linear combinations - won't capture non-linear structure. (3) PCs are not interpretable in original feature space. (4) Sensitive to scale (standardize if features have different units!).

🔹 Kernel K-means

🎯 Why We Use Kernel K-means:

  • Non-linear clusters: Can find clusters that aren't spherical in original space
  • Kernel trick: Work in high/infinite-dim space without computing it explicitly
  • Flexible: Different kernels for different data types (RBF, polynomial, etc.)
  • Same complexity: Often similar computational cost to standard k-means

📅 When to Use Kernel K-means:

  • Clusters have non-linear shapes (concentric circles, interleaving spirals)
  • Standard k-means fails but you want k-means-like algorithm
  • Example: Concentric circles - impossible for linear k-means, easy for kernel k-means with RBF

Key Idea:

\[\|x_i - \mu_k\|^2 = \|\phi(x_i) - \mu_k\|^2 = K(x_i,x_i) - \frac{2}{|C_k|}\sum_{j \in C_k} K(x_i,x_j) + \frac{1}{|C_k|^2}\sum_{j,l \in C_k} K(x_j,x_l)\]

Beautiful trick: Can compute distances in feature space using only kernel evaluations K(x,x')! Never need to explicitly compute φ(x). This means:
  • Assign x_i to cluster k that minimizes kernel distance
  • All computations use kernel matrix K (N×N)
  • Can work in infinite dimensions (e.g., RBF kernel)

📝 Example from Past Papers: Concentric Circles

Problem: Inner circle (class 1) and outer ring (class 2)

Standard k-means: Fails! Draws vertical line, can't separate circles

Kernel k-means with RBF: Maps to high-dim space where circles become separable. Finds correct clustering!

Why it works: RBF kernel measures similarity. Points in same circle are similar (nearby), points in different circles are dissimilar (far).
