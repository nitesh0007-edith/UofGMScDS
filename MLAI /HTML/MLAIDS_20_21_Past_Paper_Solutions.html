  • Numerical overflow in polynomial computations
  • Ill-conditioned matrices (Φ^TΦ becomes nearly singular)

(c) RBF with Data Points as Centers [5 marks]

The radial basis function (RBF):

\[h_{n,k} = \exp\left(-\frac{\sum_{d=1}^{D}(x_{n,d} - \mu_{d,k})^2}{2s^2}\right), \quad n=1,\ldots,N; \quad k=1,\ldots,K\]

is a popular basis function. The parameter \(\mu_{d,k}\) is often set to be a data point \(x_{i,d}\), \(i=1,\ldots,N\). Outline the strength and risk of this setup for \(\mu_{d,k}\), and how would you mitigate the risk.

Solution:

Answer:

Strength: Flexibility [1 mark]

Risk: Numerical stability and overfitting [2 marks]

Mitigation: Use fewer centers or add small value to the diagonal of X^TX, using regularization [2 marks]

Detailed Explanation:

Strength (Flexibility):

  • Using data points as centers means K=N (one RBF per data point)
  • This makes the model extremely flexible - can fit any pattern
  • Each training point gets its own basis function
  • Can achieve perfect interpolation of training data

Risks:

  • Overfitting: With K=N basis functions, model can fit noise perfectly. Training error → 0 but test error very high
  • Numerical Instability: Design matrix Φ (N×N) can become ill-conditioned or singular. Matrix Φ^TΦ may not be invertible
  • Computational Cost: N² parameters to estimate, expensive for large datasets

Mitigation Strategies:

  • Use fewer centers (K < N): Select subset of data points (e.g., K-means clustering to find representative centers)
  • Regularization: Add Ridge (L2) or Lasso (L1) penalty to prevent large weights
  • Add to diagonal: Φ^TΦ + εI where ε is small (e.g., 0.001) ensures invertibility
  • Cross-validation: Choose optimal number of centers K and width parameter s

(d) Fourier Basis Functions [5 marks]

In addition to the polynomial function and RBF, linear regression can be generalized using other basis functions. One of the most widely used example is the Fourier analysis. Let's consider the following linear regression model:

\[t_n = \sum_{j} A_j \cos(jx_n + \theta_j)\]

What is the basis function of choice here? How would you deal with the unknown parameters \(A_j\) and \(\theta_j\)? (Hint: you might find the following trigonometry identity useful, \(\cos(a + b) = \cos(a)\cos(b) + \sin(a)\sin(b)\)).

Solution:

Solution 1 [3 marks total]:

Basis function: \(\cos(jx_n + \theta_j)\) [1 mark]

\(A_j\) is the regression parameter, use cross-validation for \(\theta_j\) [2 marks]

Solution 2 [Full marks - 5 marks]:

Apply the trigonometric identity \(\cos(a + b) = \cos(a)\cos(b) + \sin(a)\sin(b)\):

\[\cos(jx_n + \theta_j) = \cos(jx_n)\cos(\theta_j) + \sin(jx_n)\sin(\theta_j)\]

This gives us two basis functions: \(\cos(jx_n)\) and \(\sin(jx_n)\)

\(\theta_j\) becomes part of the linear regression parameters (absorbed into weights), the same as \(A_j\).

Detailed Explanation of Solution 2:

Rewriting the model:

\[t_n = \sum_j A_j[\cos(jx_n)\cos(\theta_j) + \sin(jx_n)\sin(\theta_j)]\]

\[= \sum_j [A_j\cos(\theta_j)]\cos(jx_n) + [A_j\sin(\theta_j)]\sin(jx_n)\]

\[= \sum_j w_j^{(c)}\cos(jx_n) + w_j^{(s)}\sin(jx_n)\]

where \(w_j^{(c)} = A_j\cos(\theta_j)\) and \(w_j^{(s)} = A_j\sin(\theta_j)\)

Key Insight: Now both \(w_j^{(c)}\) and \(w_j^{(s)}\) are linear regression parameters! We can learn them directly using ordinary least squares. No need to separately optimize \(A_j\) and \(\theta_j\).

Fourier Basis Functions:

  • Basis functions: \(\{\cos(jx), \sin(jx)\}\) for \(j=1,2,3,\ldots\)
  • Can represent any periodic function (Fourier series)
  • Useful for seasonal data, time series, signal processing
  • Each frequency j captures different wavelength patterns

Question 2: Classification (20 marks)

(a) Logistic Regression Likelihood [4 marks]

The likelihood of logistic regression is:

\[p(t_n|\mathbf{w}, \mathbf{x}_n) = g(\mathbf{w}^T\mathbf{x}_n)^{t_n}(1 - g(\mathbf{w}^T\mathbf{x}_n))^{1-t_n}\]

where \(g(a) = \frac{1}{1+\exp(-a)}\). Use an example of a few data points to explain how the likelihood function tells how well the parameter \(\mathbf{w}\) fits the data.

Solution:

Marking Scheme:

The example needs to have more than one pair of \(t_n\) and \(x_n\) [1 mark], demonstrating how to construct joint data likelihood [1 mark]. It should also include a parameter estimate representing a good fit and a parameter estimate representing a bad fit [2 marks].

Example Solution:

Dataset (3 points):

Point Feature x_n Label t_n
1 2.0 1
2 1.5 1
3 -1.0 0

Joint Likelihood (product of individual likelihoods):

\[L(\mathbf{w}) = p(t_1,t_2,t_3|\mathbf{w}) = \prod_{n=1}^{3} p(t_n|\mathbf{w},x_n)\]

Good Parameter (w=2):

  • Point 1: \(g(2 \times 2) = g(4) = 0.98\) → \(p(t_1=1) = 0.98\) ✓ High probability
  • Point 2: \(g(2 \times 1.5) = g(3) = 0.95\) → \(p(t_2=1) = 0.95\) ✓ High probability
  • Point 3: \(g(2 \times (-1)) = g(-2) = 0.12\) → \(p(t_3=0) = 1-0.12 = 0.88\) ✓ High probability
  • Joint Likelihood: \(L = 0.98 \times 0.95 \times 0.88 = 0.82\) (HIGH)

Bad Parameter (w=-1):

  • Point 1: \(g(-1 \times 2) = g(-2) = 0.12\) → \(p(t_1=1) = 0.12\) ✗ Low probability
  • Point 2: \(g(-1 \times 1.5) = g(-1.5) = 0.18\) → \(p(t_2=1) = 0.18\) ✗ Low probability
  • Point 3: \(g(-1 \times (-1)) = g(1) = 0.73\) → \(p(t_3=0) = 1-0.73 = 0.27\) ✗ Low probability
  • Joint Likelihood: \(L = 0.12 \times 0.18 \times 0.27 = 0.0058\) (VERY LOW)

Conclusion: Higher likelihood = better fit. Maximum likelihood estimation (MLE) finds w that maximizes L(w), which corresponds to parameters that make the observed labels most probable given the features.

(d) Falsely Labeled Data AUC [6 marks total]

Let's consider a binary classifier trained on a falsely labeled dataset. The issue is all positive (1) and negative (0) labels are swapped during training. The classifier outputs in the table below:

Correct label 0 0 0 1 1 1
False label during training 1 1 1 0 0 0
Probability of being the positive class 0.9 0.8 0.7 ? 0.2 0.1

(i) What would be the AUC (computed with the correct labels) when the classifier is perfectly trained on the false data? And why? [2 marks]

(ii) Provide the range of possible values for the missing output (labeled '?') that would be produced by the classifier in (i). Explain why. [2 marks]

(iii) What would be the AUC (computed with the correct labels) of a random classifier trained on the falsely labeled data? Why? [2 marks]

Solution:

(i) AUC = 0: [2 marks]

Answer: 0

Explanation: Perfect AUC on false labels is 1. Correct label is 1-1=0. Or arguing using the definition of AUC: the probability of a positive example having classification score higher than a negative one. Since labels are swapped, all true positives get low scores and all true negatives get high scores, so AUC with correct labels is 0 (worst possible).

Detailed Reasoning:

If the classifier is perfectly trained on FALSE labels:

  • False label 1 (true label 0) → High scores (0.9, 0.8, 0.7)
  • False label 0 (true label 1) → Low scores (?, 0.2, 0.1)

With CORRECT labels:

  • True label 0 → High scores (0.9, 0.8, 0.7) ✗ Should be low!
  • True label 1 → Low scores (?, 0.2, 0.1) ✗ Should be high!

Result: Classifier ranks ALL negatives above ALL positives → AUC = 0

(ii) Range: [0, 0.7): [2 marks]

Answer: [0, 0.7)

Explanation: All numbers ensure that all positive data (based on false label, i.e., true negatives) have higher score than any negative data (based on false label, i.e., true positives). For perfect training on false labels, the missing value must be less than 0.7 (the lowest score of false positives).

Why [0, 0.7)?

For perfect classification on FALSE labels:

  • All false label 1 (correct label 0) must have scores ≥ any false label 0 (correct label 1)
  • Lowest score for false label 1: 0.7
  • Highest score for false label 0: 0.2
  • Missing value (false label 0) must be < 0.7 to maintain perfect separation
  • Can be anywhere in [0, 0.7) including 0.2 (same as next point)

(iii) AUC = 0.5: [2 marks]

Answer: 0.5

Explanation: Random classifier will have 0.5 AUC on false labels, 1-0.5 is still 0.5.

Why 0.5?

A random classifier produces random scores regardless of labels:

  • AUC on false labels = 0.5 (random performance)
  • Swapping labels doesn't change random scores
  • AUC on correct labels = 0.5 (still random)
  • Key insight: Label swapping inverts perfect classifiers (1→0, 0→1) but doesn't affect random classifiers (0.5→0.5)

Suppose we want to avoid any data point from the inner ring being assigned to the same cluster with any data point from the outer ring. Outline two approaches to achieve this goal with the Gaussian mixture model? Hint: You don't have to use just 2 clusters.

Answer:

Approach 1:

  • Project the data onto a different space [1 mark]
  • where data points in the inner ring are well separated from the data points in the outer ring [1 mark]

Approach 2:

  • Restrict variance of each component to be small [1 mark]
  • and use more clusters [1 mark]

Detailed Explanation:

Approach 1: Feature Space Transformation (Kernel Method)

  • Use RBF kernel to map to higher dimensional space
  • Add radial coordinate: \(r = \sqrt{x^2 + y^2}\) as a feature
  • In this new space, inner and outer rings have different r values
  • GMM can easily separate them based on r
  • Example: inner ring (r ≈ 0.3), outer ring (r ≈ 0.8)

Approach 2: Many Small Clusters

  • Instead of K=2, use K=10 or K=20 small Gaussian components
  • Constrain each component to have small, similar covariance (almost spherical)
  • Each small cluster captures a local region
  • Post-process: merge clusters - all in inner ring = cluster A, all in outer ring = cluster B
  • This is similar to over-clustering then hierarchical grouping

Other Sensible Approaches:

  • Hierarchical GMM: First fit 2 components for rough split, then fit sub-GMMs within each
  • Constrained GMM: Add constraints that force components to have non-overlapping support

(d) Achieving GMM Results with K-means [4 marks]

Suppose Figure 3 (B) represents the results we want. Outline one approach to achieve this goal with K-means. Hint: Sufficient details of the approach are required to get full marks.

Solution:

Answer:

Kernel K-means [2 marks]

Details on how to kernelize K-means: using the kernel trick on the distance [1 mark] and only update the assignments [1 mark].

Detailed Explanation of Kernel K-means:

Key Idea:

Use kernel trick to implicitly map data to higher-dimensional space where diagonal clusters become more separable.

Distance Computation in Kernel K-means:

Standard K-means distance: \(\|x_i - \mu_k\|^2\)

In feature space: \(\|\phi(x_i) - \mu_k\|^2\) where \(\mu_k = \frac{1}{|C_k|}\sum_{j \in C_k}\phi(x_j)\)

Using kernel trick:

\[\|\phi(x_i) - \mu_k\|^2 = K(x_i,x_i) - \frac{2}{|C_k|}\sum_{j \in C_k}K(x_i,x_j) + \frac{1}{|C_k|^2}\sum_{j,l \in C_k}K(x_j,x_l)\]

Algorithm:

  1. Choose kernel (e.g., RBF: \(K(x,x') = \exp(-\gamma\|x-x'\|^2)\))
  2. Compute kernel matrix K for all pairs of points
  3. Initialize cluster assignments randomly
  4. Repeat:
    • For each point i, compute kernel distance to each cluster k
    • Assign point i to cluster with minimum kernel distance
    • Update cluster assignments (no explicit centroid update!)
  5. Stop when assignments converge

Why This Works for Diagonal Clusters:

  • RBF kernel can capture local similarity patterns
  • Points within same diagonal cluster have high similarity (high K value)
  • Points in different clusters have low similarity (low K value)
  • Kernel space makes clusters more "spherical" and separable
