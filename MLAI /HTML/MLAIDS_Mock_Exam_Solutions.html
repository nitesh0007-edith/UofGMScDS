Question 1: Linear Regression (20 marks)

Linear regression with models of the form \(t_n = \mathbf{w}^T\mathbf{x}_n\) is a common technique for learning real-valued functions from data.

(a) Squared vs Absolute Loss [6 marks]

Squared and absolute loss are defined as follows:

\[L_{squared} = (t_n - \mathbf{w}^T\mathbf{x}_n)^2, \quad L_{absolute} = |t_n - \mathbf{w}^T\mathbf{x}_n|\]

Describe, with a diagram if you like, why, when optimizing the parameters with the squared loss outliers have a larger effect than with the absolute loss.

Solution:

Analysis of Loss Function Behavior:

Defining \(e_n = t_n - \mathbf{w}^T\mathbf{x}_n\) as the error for data point n:

[2 marks] For small errors where \(|e_n| < 1\), the absolute loss is actually larger than the squared loss:

Example: If \(|e_n| = 0.5\):

\[L_{squared} = (0.5)^2 = 0.25 < 0.5 = L_{absolute}\]

[2 marks] However, as \(e_n\) increases, the squared loss increases much faster than the absolute loss due to the quadratic growth:

Example: If \(|e_n| = 5\) (outlier):

\[L_{squared} = 5^2 = 25 \quad \text{vs} \quad L_{absolute} = 5\]

The squared loss is 5× larger!

[2 marks] Outliers have a high \(e_n\) value and therefore contribute a much larger influence to the total squared loss than to the absolute loss. When minimizing the total loss, the model will adjust parameters more aggressively to reduce squared loss for outliers, potentially at the expense of fitting the bulk of the data well.

Graphical Intuition: If you plot both loss functions:
  • \(L_{squared} = e^2\) is a parabola that grows rapidly for large |e|
  • \(L_{absolute} = |e|\) is V-shaped with constant slope for large |e|
  • For outliers (large |e|), squared loss dominates the optimization

(b) Properties of Loss Functions [2 marks]

Which of the following statements is true:

  • A) Parameter estimation with the squared loss is not analytically tractable.
  • B) The squared loss is equivalent to assuming normally distributed noise.
  • C) The absolute loss is a popular choice for regularization.
  • D) The squared loss is a popular choice for regularization.

Solution:

Correct Answer: B [2 marks]

Explanation:

The squared loss corresponds to the assumption that the noise (error) in the data follows a Gaussian (normal) distribution. This comes from maximum likelihood estimation:

If \(t_n = \mathbf{w}^T\mathbf{x}_n + \epsilon_n\) where \(\epsilon_n \sim \mathcal{N}(0, \sigma^2)\), then:

\[p(t_n|\mathbf{x}_n, \mathbf{w}) = \frac{1}{\sqrt{2\pi\sigma^2}} \exp\left(-\frac{(t_n - \mathbf{w}^T\mathbf{x}_n)^2}{2\sigma^2}\right)\]

Maximizing log-likelihood is equivalent to minimizing squared loss.

Why other answers are wrong:

  • A is false: Squared loss has a closed-form solution: \(\mathbf{w} = (\mathbf{X}^T\mathbf{X})^{-1}\mathbf{X}^T\mathbf{t}\)
  • C and D are false: Neither loss function is used for regularization. L1 (absolute value of weights) and L2 (squared weights) are used for regularization, not loss functions on predictions.

(c) Model Complexity Selection [3 marks]

Discuss why the value of the squared loss on the training data cannot be used to choose the model complexity.

Solution:

[1 mark] When training the model, we are minimizing the loss on the training data.

[2 marks] As we make models more complex (e.g., adding more polynomial terms, more basis functions), the squared loss value at the minima will always decrease or stay the same - it never increases. This is because a more complex model can always achieve at least the performance of a simpler model (by setting extra parameters to zero) and usually does better.

Consequence: Hence, training loss will always favor more complex models, leading to overfitting. The model will memorize the training data rather than learning generalizable patterns.

What to use instead:
  • Validation set or cross-validation error
  • Information criteria (AIC, BIC) that penalize complexity
  • Regularization techniques (Ridge, Lasso)
These methods balance fit quality with model complexity to achieve good generalization.

(d) Model Prediction [2 marks]

For the particular model \(t_n = w_1x_n + w_2x_n^3\), I optimize the parameters and end up with \(\mathbf{w} = [2,1]^T\). What does the model predict for a test point at \(x_{new} = 3\)?

Solution:

Prediction Calculation: [2 marks]

\[t_{new} = w_1 \cdot x_{new} + w_2 \cdot x_{new}^3\]

\[= 2 \times 3 + 1 \times 3^3\]

\[= 6 + 27\]

\[= 33\]

Answer: The model predicts 33

(e) Radial Basis Functions (RBF) [4 marks]

The radial basis function (RBF):

\[h_{n,k} = \exp\left(-\frac{\sum_{d=1}^{D}(x_{n,d} - \mu_{d,k})^2}{2s^2}\right), \quad n = 1, \ldots, N; \quad k = 1, \ldots, K\]

is a popular choice for converting the original features, \(x_{n,d}\), into a new set of K features prior to training. Assume the value of s is given. Describe a procedure for determining the center parameter \(\mu_{d,k}\) and K.

Solution:

Procedure for Determining \(\mu_{d,k}\) and K:

[2 marks] Any reasonable approach for choosing K centers. The answer should clearly state how to choose K.

Common Approach 1: Use Data Points as Centers

  • Let \(\mu_{d,k}\) be a data point \(x_{i,d}\) where \(i \neq n\) (different from the nth data point)
  • In this case, K = N-1 (or K = N if we allow self-centers)
  • This ensures each basis function is centered at an actual data location

[2 marks] What is \(\mu_{d,k}\): The centers are the D-dimensional coordinates in feature space where each RBF is centered.

Common Approach 2: K-means Clustering

  • Choose K (e.g., via cross-validation, domain knowledge, or as a fraction of N)
  • Run K-means clustering on the training data points
  • Set \(\mu_{d,k}\) to be the K cluster centroids
  • This gives a more compact representation than using all data points

Common Approach 3: Uniform Grid

  • Place centers uniformly across the feature space range
  • K is chosen based on desired coverage/resolution
  • Works well for low-dimensional data
Key Point: The centers \(\mu_{d,k}\) define where each RBF "bump" is located in the D-dimensional feature space. Each basis function creates a localized response around its center, with width controlled by s.

(f) RBF vs Linear Model [3 marks]

With respect to the functions they can fit, describe the difference between RBF and the basic linear model \(\mathbf{w}^T\mathbf{x}_n\) with a graph.

Solution:

[1 mark] The RBF can produce a nonlinear model, i.e., a wiggly curve that can capture complex patterns and local variations in the data.

[1 mark] \(\mathbf{w}^T\mathbf{x}_n\) produces only a linear model, i.e., straight lines (in 2D) or hyperplanes (in higher dimensions).

[1 mark] Readable graph showing the difference.

Graphical Description:
  • Linear Model: A straight line that best fits the data in a least-squares sense. Cannot capture curvature or local patterns.
  • RBF Model: A smooth, curved function formed by weighted sum of Gaussian "bumps". Can capture:
    • Non-linear relationships
    • Local features (e.g., peaks and valleys)
    • Complex patterns by combining multiple RBF basis functions

Mathematical Comparison:

Linear: \(f(x) = w_0 + w_1x\) (straight line)

RBF: \(f(x) = \sum_{k=1}^{K} w_k \exp\left(-\frac{(x-\mu_k)^2}{2s^2}\right)\) (nonlinear combination of bumps)

Question 2: Classification (20 marks)

(a) Fundamental Concepts [4 marks total]

Use a classification algorithm, describe what is meant by:

(i) Generalisation [2 marks]

(ii) Over-fitting [2 marks]

Solution:

(i) Generalisation: [2 marks]

Generalisation refers to a model's ability to make accurate predictions on previously unseen data (test data or new examples not in the training set). A model that generalises well has learned the underlying patterns in the data rather than memorizing specific training examples, allowing it to perform well on new, similar data.

(ii) Over-fitting: [2 marks]

Over-fitting is the problem of memorizing the training data, including its noise and outliers, rather than learning the general patterns. An overfitted model performs very well on training data but poorly on test data because it has captured random fluctuations specific to the training set rather than the true underlying relationship.

Relationship: Over-fitting leads to poor generalisation. The model becomes too complex and fits the training data too closely, losing the ability to generalize to new examples.

(b) Confusion Matrix Analysis [6 marks total]

A classification algorithm has been used to make predictions on a test set, resulting in the following confusion matrix:

Truth
Positive Negative Total
Prediction Positive 23 5 28
Negative 10 12 22
Total 33 17 50

Compute the following quantities (expressing them as fractions is fine):

(i) Accuracy [2 marks]

(ii) Sensitivity [2 marks]

(iii) Specificity [2 marks]

Solution:

Given values from confusion matrix:

  • True Positives (TP) = 23
  • False Positives (FP) = 5
  • False Negatives (FN) = 10
  • True Negatives (TN) = 12
  • Total = 50

(i) Accuracy: [2 marks]

\[\text{Accuracy} = \frac{TP + TN}{\text{Total}} = \frac{23 + 12}{50} = \frac{35}{50} = 0.70 = 70\%\]

(ii) Sensitivity (Recall/True Positive Rate): [2 marks]

\[\text{Sensitivity} = \frac{TP}{TP + FN} = \frac{23}{23 + 10} = \frac{23}{33} \approx 0.697 = 69.7\%\]

This measures the proportion of actual positives correctly identified.

(iii) Specificity (True Negative Rate): [2 marks]

\[\text{Specificity} = \frac{TN}{TN + FP} = \frac{12}{12 + 5} = \frac{12}{17} \approx 0.706 = 70.6\%\]

This measures the proportion of actual negatives correctly identified.

(c) AUC and Confusion Matrix [4 marks]

Explain why it is not possible to compute the AUC from a confusion matrix.

Solution:

[1 mark] AUC (Area Under the Curve) is the area under the ROC (Receiver Operating Characteristic) curve.

[1 mark] The ROC curve is created by varying the threshold at which the algorithm calls something as belonging to the positive class. For each threshold, we plot the True Positive Rate (TPR) against the False Positive Rate (FPR).

[1 mark] A confusion matrix only gives us the performance at one single threshold - the specific threshold that was used to make the binary predictions shown in the matrix.

[1 mark] To compute AUC, we need to evaluate the classifier's performance across all possible thresholds (or a range of thresholds), which requires access to the continuous prediction scores, not just the final binary predictions.

What's needed for AUC:
  • The predicted probabilities or scores for each test example
  • The true labels
  • Compute TPR and FPR at multiple thresholds by varying the decision boundary
  • Plot ROC curve and calculate area under it

(d) Computing AUC from Predictions [4 marks]

Two binary classifiers are used to make predictions for the same set of six test points. These predictions are given below, along with the true labels. Compute the area under the curve (AUC) in each case.

Classifier 1 Classifier 2
Predicted probability of class 1 True class Predicted probability of class 1 True class
1.0 1 0.8 1
0.8 1 0.8 0
0.6 1 0.6 1
0.4 0 0.6 0
0.2 0 0.2 1
0.0 0 0.2 0

Solution:

Classifier 1 Analysis: [2 marks]

Actual positives (class 1): 3 examples (scores: 1.0, 0.8, 0.6)
Actual negatives (class 0): 3 examples (scores: 0.4, 0.2, 0.0)

The predictions are perfectly separated - all positive examples have higher scores than all negative examples. This means:

  • At any threshold between 0.4 and 0.6, we classify perfectly
  • TPR = 1.0 (all positives correctly classified)
  • FPR = 0.0 (no negatives incorrectly classified)

AUC for Classifier 1 = 1.0 (perfect classifier)

Classifier 2 Analysis: [2 marks]

Actual positives: scores 0.8, 0.6, 0.2
Actual negatives: scores 0.8, 0.6, 0.2

The predictions are completely mixed - there's no threshold that separates the classes better than random guessing. For every positive at a given score, there's a negative at the same or similar score:

  • The ranking of examples is no better than random
  • No matter what threshold we choose, we cannot separate the classes

AUC for Classifier 2 = 0.5 (random classifier, no better than coin flip)

AUC Interpretation:
  • AUC = 1.0: Perfect classifier
  • AUC = 0.5: Random classifier (no discriminative power)
  • AUC < 0.5: Worse than random (predictions inverted)
  • AUC > 0.5: Better than random

(e) SVM and Kernel Trick [2 marks]

Explain how the SVM can be extended via the kernel trick to perform non-linear classification.

Solution:

[1 mark] In the SVM objective function and decision rule, the data only appear in the form of inner products (dot products) between data points: \(\mathbf{x}_i \cdot \mathbf{x}_j\).

[1 mark] Inner products in the original space can be replaced by inner products in another (higher-dimensional) feature space via kernel functions: \(k(\mathbf{x}_i, \mathbf{x}_j) = \phi(\mathbf{x}_i) \cdot \phi(\mathbf{x}_j)\), where \(\phi\) maps data to a higher-dimensional space. Critically, we can compute \(k(\mathbf{x}_i, \mathbf{x}_j)\) directly without explicitly computing \(\phi\).

Common Kernels:

Linear: \(k(\mathbf{x}_i, \mathbf{x}_j) = \mathbf{x}_i \cdot \mathbf{x}_j\)

Polynomial: \(k(\mathbf{x}_i, \mathbf{x}_j) = (\mathbf{x}_i \cdot \mathbf{x}_j + c)^d\)

RBF/Gaussian: \(k(\mathbf{x}_i, \mathbf{x}_j) = \exp\left(-\frac{\|\mathbf{x}_i - \mathbf{x}_j\|^2}{2\sigma^2}\right)\)

Why this works: By using a kernel, we implicitly map data to a space where it becomes linearly separable, without the computational cost of explicitly computing the high-dimensional feature representation. The SVM finds a linear decision boundary in this high-dimensional space, which corresponds to a non-linear boundary in the original space.

Question 3: Unsupervised Learning (20 marks)

(a) K-means Algorithm [5 marks]

Provide pseudo code for K-means (assume that the number of clusters K is provided).

Solution:

K-means Pseudo Code:

K-MEANS ALGORITHM:

Given: Number of clusters, K; Data points x_1, ..., x_N

Initialization:
    Randomly initialize K cluster centers μ_1, ..., μ_K

Repeat until convergence:

    Assignment Step:
    For each object n = 1...N:                              [1 mark]
        Compute the distance between object n and
        each cluster k = 1...K                              [1 mark]

        Assign object n to the cluster corresponding
        to the smallest distance:                           [1 mark]
            c_n = argmin_k ||x_n - μ_k||²

    Update Step:
    For each cluster k = 1...K:
        Update the mean of cluster k:                       [1 mark]
            μ_k = (1/N_k) Σ x_n  (sum over n where c_n = k)

    Check Convergence:
    If assignments have changed, return to Assignment Step
    Else stop                                               [1 mark]

Output: Cluster assignments c_1, ..., c_N and centers μ_1, ..., μ_K
Key Points:
  • The algorithm alternates between assigning points to nearest clusters and updating cluster centers
  • Guaranteed to converge (assignments stop changing)
  • May converge to local optima depending on initialization
  • Computational complexity: O(N·K·D·iterations) where D is dimensionality

(b) Selecting Number of Clusters [2 marks]

Is the total Euclidean distance between data points and their cluster centers a good criterion to select number of clusters in K-means? Why?

Solution:

[1 mark] No, the total Euclidean distance is not a good criterion for selecting K.

[1 mark] As we increase the number of clusters K, the total Euclidean distance will always decrease (or stay the same). With more clusters, each point can be closer to its assigned cluster center, leading to smaller distances.

Extreme case: If K = N (number of clusters equals number of data points), every point is its own cluster center and the total distance becomes zero. This is clearly overfitting and not useful.

Better methods for choosing K:
  • Elbow method: Plot total distance vs K and look for an "elbow" where the decrease slows
  • Silhouette score: Measures how well points fit their clusters vs neighboring clusters
  • Gap statistic: Compares within-cluster variation to null reference distribution
  • Cross-validation: If clusters will be used for downstream tasks
  • Domain knowledge: Sometimes the number of clusters is known from the problem context

(c) Expectation Maximization (EM) for GMM [4 marks]

Gaussian mixture models can be fitted to data using the expectation maximization (EM) algorithm. The EM algorithm has two steps: E-step and M-step. Describe what parameters are being estimated in each step in Gaussian mixture models.

Solution:

E-step (Expectation Step): [2 marks]

In the E-step, the estimated parameters are the responsibilities (also called posterior probabilities) \(r_{nk}\), which represent the expected assignment probabilities for each data point n to each Gaussian component k.

\[r_{nk} = \frac{\pi_k \mathcal{N}(x_n|\mu_k, \Sigma_k)}{\sum_{j=1}^{K} \pi_j \mathcal{N}(x_n|\mu_j, \Sigma_j)}\]

This is the probability that point n belongs to cluster k, given current parameters.

M-step (Maximization Step): [2 marks]

In the M-step, we update the model parameters using the responsibilities computed in E-step:

  • Mean \(\mu_k\) of each Gaussian component
  • Covariance matrix \(\Sigma_k\) of each Gaussian component
  • Mixing coefficients \(\pi_k\) (the prior probability of each cluster)

\[\mu_k = \frac{\sum_{n=1}^{N} r_{nk} x_n}{\sum_{n=1}^{N} r_{nk}}, \quad \pi_k = \frac{1}{N}\sum_{n=1}^{N} r_{nk}\]

Iteration: These steps alternate until convergence. Each complete E-M iteration is guaranteed to increase (or maintain) the log-likelihood of the data under the model.

(d) K-means vs Gaussian Mixture Models [4 marks]

Describe three key differences between K-means and Gaussian mixture models.

Solution:

Difference 1: Assignment Type [2 marks total]

  • [1 mark] K-means uses hard assignment: Each point is assigned to exactly one cluster (binary membership)
  • [1 mark] Gaussian mixture models use soft assignment: Each point has a probability of belonging to each cluster (GMM estimates the probability of assignment)

Difference 2: Cluster Parameters [2 marks total]

  • [1 mark] K-means only estimates cluster centers (means)
  • [1 mark] GMM estimates both cluster centers (means) and covariance matrices, which define the shape and orientation of clusters

Difference 3: Cluster Shape

  • K-means assumes spherical clusters of similar size (uses Euclidean distance)
  • GMM can model elliptical clusters of different sizes and orientations (via covariance matrices)
Additional Differences:
  • Probabilistic foundation: GMM has a proper probabilistic model (can compute likelihoods), K-means is a heuristic algorithm
  • Mixing coefficients: GMM estimates \(\pi_k\) (prior probability of each cluster), K-means implicitly assumes equal cluster sizes
  • Computational cost: K-means is generally faster; GMM is more expensive due to covariance computations

(e) Overcoming Local Optima in K-means [3 marks]

K-means often converges to a local optimal solution. Describe a simple process for overcoming the local optimality of K-means.

Solution:

Multiple Random Restarts: [3 marks]

For a fixed number of clusters K, perform multiple re-starts of the K-means algorithm with different random initializations, and evaluate the total distance of points from their cluster mean (the objective function value) for each run. Keep the solution with the smallest distance.

Algorithm:

best_objective = ∞
best_solution = None

For restart = 1 to R (e.g., R = 10 or 20):
    Randomly initialize cluster centers
    Run K-means to convergence
    Compute objective: total_distance = Σ ||x_n - μ_{c_n}||²

    If total_distance < best_objective:
        best_objective = total_distance
        best_solution = current clusters and centers

Return best_solution
Why this works:
  • Different random initializations explore different parts of the solution space
  • Each run converges to a local optimum
  • By trying multiple initializations, we increase chances of finding the global optimum (or a good local optimum)
  • Common practice: Use 10-50 random restarts
Smarter initialization: K-means++ is an improved initialization method that spreads initial centers far apart, often reducing the need for many restarts.

(f) When to Choose Mixture Models over K-means [2 marks]

Describe two situations (with justification) where you might choose a mixture model over K-means.

Solution:

Situation 1: Non-isotropic Clusters [1 mark]

When we have data for which we don't want to be limited to isotropic (spherical) clusters. Mixture models (particularly GMM) can model elliptical clusters with different shapes and orientations using covariance matrices, while K-means assumes spherical clusters of similar size.

Example: Image segmentation where objects have elongated or irregular shapes, or gene expression data where clusters have different variances in different dimensions.

Situation 2: Non-Euclidean Data Types [1 mark]

When using a data type for which it is not obvious how to compute a distance metric (but for which we can compute a likelihood). Mixture models are based on likelihood functions and can be adapted to various data types by choosing appropriate component distributions.

Examples:

  • Text data: Use mixture of multinomials
  • Binary data: Use mixture of Bernoullis
  • Count data: Use mixture of Poissons

Additional Situations:

  • Soft assignments needed: When you need probability estimates for cluster membership (e.g., for downstream uncertainty quantification)
  • Probabilistic framework: When you need to compute likelihoods, perform model selection via BIC/AIC, or integrate clustering into a larger probabilistic model
  • Overlapping clusters: When clusters naturally overlap and points can partially belong to multiple clusters
