Question 1: Regression - World Population Growth Rate (20 marks)

Context: Consider using regression to predict the world population growth rate using data from 1950 to 2021.

(a) Rescaling Strategy for Year Variable [4 marks]

Propose a rescaling strategy (with enough details of the procedure) for the variable Year. Explain why the proposed strategy is appropriate.

Solution:

Proposed Rescaling Strategies: [2 marks for reasonable strategy]

Any of the following strategies would be appropriate:

1. Whitening (Standardization):

\[\text{Year}_{scaled} = \frac{\text{Year} - \mu}{\sigma}\]

where \(\mu\) is the mean year (≈1985.5) and \(\sigma\) is the standard deviation of years.

2. Min-Max Scaling:

\[\text{Year}_{scaled} = \frac{\text{Year} - 1950}{2021 - 1950} = \frac{\text{Year} - 1950}{71}\]

This scales the data to the range [0, 1].

3. Logarithmic Transformation:

\[\text{Year}_{scaled} = \log(\text{Year})\]

This compresses the scale of the year values.

Reasoning for Why This is Appropriate: [2 marks total]

  • [1 mark] The key is to reduce the absolute value of "year", such that high order polynomial will still produce well-behaved values (small). Without rescaling, raising large year values (e.g., 2021) to high powers would result in extremely large numbers (2021³ ≈ 8.2 billion) that cause numerical instability.
  • [1 mark] The matrix inversion in the least square solution is still stable. Large numbers in the design matrix \(\Phi\) lead to very large values in \(\Phi^T\Phi\), making the matrix ill-conditioned and its inversion numerically unstable.
Example: Without rescaling: 1950³ = 7.4×10⁹, 2021³ = 8.2×10⁹. With min-max scaling to [0,1]: values raised to any power remain in [0,1], which is numerically stable.

(c) Sigmoid Basis Functions vs Polynomials [4 marks]

Consider fitting the data in figure 1 with a regression using sigmoid basis functions:

\[h_{n,k} = \text{sigmoid}\left(\frac{(x_n - \mu_k)^2}{s}\right), \quad n = 1, \ldots, N; \quad k = 1, \ldots, K\]

where \(x_n\) represents each year and \(\text{sigmoid}(a) = \frac{1}{1 + \exp(-a)}\)

Outline one advantage and one disadvantage of using this sigmoid basis function over polynomials.

Solution:

Advantage: [2 marks total]

  • [1 mark] The variance is not equally distributed across x values - it is larger in small values (early years with more fluctuation) and smaller in large x values (recent years with smoother decline).
  • [1 mark] Using location-specific basis functions, sigmoid can model this localized effect better than polynomial functions which model global effect across all x values, resulting in better fitting performance.

Disadvantage: [2 marks total]

  • [1 mark] Need to choose hyperparameter \(\mu_k\) (centers).
  • [1 mark] Need to choose hyperparameter \(s\) (scale/bandwidth).
Key Insight: Sigmoid basis functions are local - they activate based on distance from centers \(\mu_k\). This allows adaptive fitting to regions with different variability. Polynomials affect the entire range uniformly and cannot adapt to local patterns.

Question 2: Classification - Tweet Hate Speech Detection (20 marks)

(a) Hate Speech Classifier Design [13 marks total]

You have been asked to design a classifier to automatically identify Tweets that are considered as 'hate speech' on Twitter. You collected a training dataset with 800 'regular' tweets and 100 'hate' tweets.

(i) Feature Engineering [3 marks]

Describe 2 features you might use for this task including their type (scalar/vector, real-valued or not).

Solution:

Marking: 1 mark for each feature and 1 mark for mentioning the feature type.

Example Features:

Feature 1: Presence/Absence of Disrespectful Words

  • Type: Binary feature (scalar)
  • Description: A binary indicator (0 or 1) showing whether the tweet contains words from a predefined list of disrespectful or offensive terms.

Feature 2: Count of Strongly Negative Words

  • Type: Real-valued scalar
  • Description: The number or proportion of words with strong negative sentiment (e.g., using a sentiment lexicon).

Feature 3: Word Embeddings (Alternative)

  • Type: Real-valued vector (e.g., 300-dimensional)
  • Description: Dense vector representation of the tweet using pre-trained word embeddings (e.g., Word2Vec, GloVe) averaged over all words.
Other Valid Features: Character n-grams, tweet length, presence of all-caps words, number of profanity words, toxicity scores from APIs, user mention count, hashtag count, etc. Any reasonable feature with correct type specification is acceptable.

(ii) Weighted Classification Accuracy [2 marks]

You learn a faulty classifier which always classifies a tweet as 'regular'. What would be the weighted classification accuracy of this classifier?

Solution:

Analysis:

  • Regular class accuracy: 100% (all 800 regular tweets correctly classified as regular)
  • Hate class accuracy: 0% (all 100 hate tweets incorrectly classified as regular)

\[\text{Weighted Accuracy} = \frac{\text{Regular Accuracy} + \text{Hate Accuracy}}{2} = \frac{100\% + 0\%}{2} = 50\%\]

Answer: 50% (average across classes)

Important Distinction: Note that the overall accuracy would be 800/900 = 88.9% (proportion of correct predictions), but the weighted classification accuracy (balanced accuracy) averages the per-class accuracies, giving 50%. This metric is more appropriate for imbalanced datasets.

(iii) Logistic Regression Calculation [4 marks]

Assume that we use Logistic Regression for the task considering only 2 scalar features. We get the following set of parameters w = [-1.8, 2.1, -0.3]ᵀ after training. For feature vector x = [1, 1]ᵀ, calculate the output of the logistic function (probability score).

Solution:

Step 1: Calculate the linear combination [2 marks]

\[z = w_0 + w_1 x_1 + w_2 x_2\]

\[z = -1.8 + 2.1 \times 1 + (-0.3) \times 1\]

\[z = -1.8 + 2.1 - 0.3 = 0\]

Step 2: Calculate the output probability score [2 marks]

\[\text{sigmoid}(z) = \frac{1}{1 + \exp(-z)} = \frac{1}{1 + \exp(0)} = \frac{1}{1 + 1} = \frac{1}{2} = 0.5\]

Answer: 0.5

Interpretation: A probability of 0.5 means the classifier is completely uncertain - it considers the tweet equally likely to be hate speech or regular. This makes sense since z=0 is exactly at the decision boundary.

(iv) Limitation of Linear Relationship Assumption [2 marks]

Logistic Regression assumes a linear relationship between the dependent and the independent variables (x). Why is that considered a limitation of the model?

Solution:

Limitation Explanation: [2 marks total - 1 mark for each observation]

[1 mark] This assumption lets Logistic Regression learn only linear decision boundaries.

[1 mark] Not suitable for data that requires a non-linear decision boundary - which is the case in many real problems.

Example: If hate speech is characterized by a complex combination of features (e.g., "high negative sentiment AND short length" or "contains profanity but NOT in quotes"), a linear boundary cannot capture these interactions. Real-world text classification often has such non-linear patterns.

Potential Solutions:

  • Add polynomial or interaction features (e.g., x₁×x₂, x₁²)
  • Use kernel methods
  • Use neural networks with non-linear activations

Decision Boundary and Margin: [3 marks]

Key Elements:

Hard Margin SVM Principle: The algorithm finds the hyperplane (line in 2D) that maximizes the margin (distance) between the two classes. The margin is defined by the closest points from each class, called support vectors.

(iii) Training Error [1 mark]

What is the training error?

Solution:

Training Error: 0 [1 mark for correct answer]

Since this is a hard margin SVM and the data is linearly separable, all training points are correctly classified. The decision boundary perfectly separates all red circles from all blue triangles with no misclassifications.

\[\text{Training Error} = \frac{\text{Number of misclassified points}}{\text{Total number of points}} = \frac{0}{8} = 0\]

Hard vs Soft Margin: Hard margin SVM only works when data is perfectly linearly separable (training error = 0 is possible). For overlapping classes, soft margin SVM is needed, which allows some training errors controlled by parameter C.

Question 3: Unsupervised Learning (20 marks)

Context: Clustering Two-Moon Manifolds

Consider using K-means to perform clustering on data arranged in two crescent moon shapes (scenario A1), where we expect to form two clusters as shown in A2.

(a) K-means with Euclidean Distance on Moon Shapes [2 marks]

Outline what would happen if we directly apply K-means with Euclidean distance to this data. Can it achieve the clustering objective? How will it split/group the data and why?

Solution:

Answer: [1 mark] K-means cannot split the data into two clusters correctly.

Explanation: [1 mark] Due to the Euclidean distance, points that are close together, although they belong to another manifold/cluster, will be clustered together.

Detailed Analysis:

The two moon shapes interleave with each other. Points from the inner moon are spatially close to points from the outer moon in certain regions. K-means creates clusters based on Euclidean distance to centroids, which results in:

  • Vertical or diagonal splits that cut across both moons
  • Each resulting cluster contains parts of both moons
  • The algorithm cannot capture the curved, non-convex structure of the moons
Fundamental Issue: K-means assumes convex, spherical clusters. The two-moon problem requires capturing non-convex, curved manifolds - something K-means with Euclidean distance cannot do. This is a classic example demonstrating K-means' limitations.

(b) Kernel K-means for Moon Shapes [3 marks]

An alternative approach is to use Kernel K-means. Would kernel K-means help in this dataset and why?

Solution:

Answer: [1 mark] Yes, a kernel would help in this case.

Explanation:

[1 mark] A kernel would project the data onto a different space where data can be easily separated.

[1 mark] A kernel also relaxes the dependency of K-means on Euclidean distance and allows more appropriate distance measures to be used for the problem at hand.

How It Works:

  • The kernel trick implicitly maps data to a higher-dimensional space
  • In this higher-dimensional space, the two moons may become linearly (or spherically) separable
  • K-means in the kernel space can then successfully separate the two clusters

Kernel K-means distance:

\[\|\phi(x_i) - \mu_k\|^2 = K(x_i, x_i) - \frac{2}{|C_k|}\sum_{x_j \in C_k} K(x_i, x_j) + \text{const}\]

where K is the kernel function and φ is the implicit feature map

Example Kernel: An RBF (Gaussian) kernel with appropriate bandwidth could map the two moons to a space where they are separable. The key is choosing a kernel that captures the manifold structure.

(c) Mixture Models for Moon Shapes [3 marks]

An alternative approach is to use mixture models. Would mixture models help to better classify this dataset than K-means and why?

Solution:

Answer: [1 mark] Mixture models might not be able to classify those data.

Explanation:

[1 mark] Mixture models assume a Gaussian distribution of the underlying clusters and they capture different variations based on the parameters of the distribution.

[1 mark] In this case, it is not possible to approximate the complex boundary shape with a Gaussian and the algorithm won't perform well.

Detailed Analysis:

  • Gaussian Mixture Models (GMMs) model each cluster as a Gaussian (elliptical) distribution
  • Even with anisotropic (non-spherical) Gaussians, each component is still convex
  • The crescent moon shapes are highly non-convex - no single Gaussian can fit them
  • Using multiple Gaussians per moon might help but requires knowing the true structure a priori
Key Limitation: While GMM is more flexible than K-means (soft assignments, different cluster shapes), it still assumes each cluster follows a Gaussian distribution. Non-convex, crescent-shaped manifolds violate this assumption. Kernel K-means or spectral clustering would be more appropriate.

(d) PCA - First Principal Component [2 marks]

The plot in D1 shows some 2D data. PCA is applied to this data. Sketch this plot, and indicate on your sketch what the first principal component would look like. Explain your reasoning.

Solution:

First Principal Component: [1 mark for plot, 1 mark for explanation]

Sketch: A line passing through the center of the data cloud along the direction of maximum spread (approximately diagonal from lower-left to upper-right).

Explanation: The first principal component will be across the direction of highest variance. Looking at the data distribution, the points spread most along a diagonal direction, so the first PC aligns with this direction.

PCA Principle:

PC1 = direction of eigenvector with largest eigenvalue of covariance matrix

This direction maximizes: \(\text{Var}(\mathbf{w}^T\mathbf{X})\) subject to \(\|\mathbf{w}\| = 1\)

Visual Guidance: Imagine fitting the longest possible line through the data cloud such that the projections of all points onto this line have maximum variance. This captures the "main axis" of variation in the data.

(e) PCA - Second Principal Component [2 marks]

Similar to previous question, sketch what the second principal component would look like and explain why.

Solution:

Second Principal Component: [1 mark for plot, 1 mark for explanation]

Sketch: A line perpendicular (orthogonal) to the first principal component, passing through the center of the data.

Explanation: PCA decomposes the data into orthogonal components. Therefore, the second principal component must be perpendicular to the first PC, along the direction of second-highest variance.

Orthogonality Constraint:

PC2 maximizes variance subject to: \(\mathbf{w}_2^T\mathbf{w}_1 = 0\) (orthogonal to PC1)

In 2D, there's only one direction perpendicular to PC1

2D Special Case: In 2D data, there are exactly 2 principal components. Once PC1 is determined, PC2 is uniquely determined as the perpendicular direction. Together, they form a rotated coordinate system aligned with the data's natural axes of variation.

(f) PCA Applications and Component Selection [2 marks]

Explain why PCA is used (provide at least two applications) and how to decide for the optimum number of principal components.

Solution:

Why PCA is Used (Applications): [1 mark]

  • Dimensionality Reduction: Reduce high-dimensional data to lower dimensions while preserving most variance
  • Intuitive Visualization: Project high-dimensional data to 2D or 3D for visualization
  • Feature Selection: Identify the most important directions of variation in the data
  • Noise Reduction: Remove components with low variance (likely noise)
  • Computational Efficiency: Speed up machine learning algorithms by reducing features

How to Decide Optimum Number of Components: [1 mark]

We can use cross-validation by leaving points at random and estimating the mean square error of the matrix factorization approach as we include more components.

Alternative Methods:

  • Explained Variance Ratio: Select k components that explain a threshold of total variance (e.g., 95%)
  • Scree Plot: Plot eigenvalues and look for an "elbow" where additional components add little variance
  • Kaiser Criterion: Keep components with eigenvalues > 1 (for normalized data)
  • Cross-Validation: Test reconstruction error on held-out data for different k values

Explained Variance Ratio:

\[\text{EVR}(k) = \frac{\sum_{i=1}^k \lambda_i}{\sum_{i=1}^d \lambda_i}\]

where λᵢ are eigenvalues and d is total dimensions

Practical Guidance: The optimal number depends on the application. For visualization, use 2-3 components. For preprocessing before classification, use cross-validation on the downstream task to select k.

(g) Projection vs Discriminative Feature Selection [6 marks]

Explain the advantages and disadvantages of feature selection based on projection compared to feature selection based on how well they can discriminate between two classes.

Solution:

Projection Methods (e.g., PCA):

How They Work: [1 mark]

Projection methods will map data into different dimensions/coordinate system, and they will select the components based on the maximum variance. These methods are based on unsupervised learning, and they don't require class labels.

Key Characteristics: [1 mark]

They project all features in a different space and therefore the new features are a combination of all the old ones.

Advantages: [2 marks]

  • Since they don't exclude completely specific features, they model better the intrinsic properties of the datasets in lower dimensions
  • They depend on the intrinsic properties of the data and their interrelationships
  • No need for labeled data (unsupervised)
  • If variables are highly correlated, projection methods can effectively combine them

Disadvantages: [2 marks]

  • The number of optimum components might be unstable and less meaningful
  • New features are linear combinations of old features, losing interpretability
  • May not focus on directions that are actually useful for classification
  • High variance doesn't always mean high discriminative power

Discriminative Methods (e.g., Filter/Wrapper Methods):

How They Work:

Select original features based on their ability to separate classes (e.g., using mutual information, chi-square test, or classifier performance).

Advantages:

  • Features are selected specifically to maximize class separability
  • Retains original features, maintaining interpretability
  • Directly optimizes for the classification objective

Disadvantages:

  • Requires labeled data (supervised)
  • May discard features that are individually weak but collectively strong
  • Risk of overfitting to specific class labels
  • Computationally expensive for wrapper methods

Comparison Summary:

Aspect Projection (PCA) Discriminative (e.g., LDA, Filter)
Supervision Unsupervised Supervised
Goal Max variance Max class separation
Features New combinations Original subset
Interpretability Low High
When to Use Each:
  • Projection (PCA): Exploratory analysis, visualization, when you don't have labels, reducing correlated features
  • Discriminative: When classification performance is the primary goal, when interpretability matters, when you have good labels

Hybrid Approach: Use PCA for initial dimensionality reduction, then apply discriminative methods on the reduced space.
