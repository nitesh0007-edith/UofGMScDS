01

The Curse of High Dimensions

Consider this real scenario: You want to build a classifier to diagnose leukaemia using gene expression data. You have 38 samples (27 healthy, 11 with leukaemia), but each sample has 3,751 features (gene expression levels).

The data matrix: 38 samples × 3,751 features — a recipe for overfitting

⚠️ Why is this a problem? With 3,751 features, a logistic regression model would need to estimate 3,751+ parameters from only 38 samples. This leads to severe overfitting — the model memorizes noise rather than learning patterns.

The solution? Reduce the number of features before training your model. There are two main approaches:

🎯

Feature Selection

Keep a subset of the original features. Choose the most informative ones and discard the rest.

🔄

Feature Projection

Create new features by combining the originals. Transform into a lower-dimensional space.

02

Feature Selection

The idea is simple: not all features are equally useful. Some genes might perfectly separate healthy from leukaemia samples, while others are just noise. Let's find the good ones!

The Separability Score

For classification problems, a good feature should have different values for different classes. We can measure this with a simple score:

Feature Separability Score
$$s = \frac{|\mu_1 - \mu_0|}{\sigma_0^2 + \sigma_1^2}$$
What this measures:
• Numerator: How far apart are the class means? (bigger = better)
• Denominator: How spread out is each class? (smaller = better)
• High s: Classes are well-separated on this feature
• Low s: Classes overlap — feature is not useful

The Feature Selection Process

1

Compute scores for all features

Calculate $s$ for each of the D features using training data only.

2

Rank features by score

Sort features from highest to lowest separability score.

3

Select top S features

Keep only the S features with highest scores. Use cross-validation to find optimal S.

4

Train your model

Build classifier using only selected features.

⚠️ Critical Warning: You must compute feature scores on training data only! If you use test labels to select features, you're implicitly using test information for training — this is data leakage and will give overly optimistic results.

03

Feature Projection

Instead of selecting existing features, we can create new features by combining the originals. Think of it like casting a shadow: a 3D hand creates a 2D shadow on the wall.

Projection: Like casting a shadow from high-dimensional space to low-dimensional space

The Math of Projection

Linear Projection
$$\mathbf{Z} = \mathbf{X}\mathbf{W}$$
• $\mathbf{X}$ is your data matrix: $N \times D$ (N samples, D original features)
• $\mathbf{W}$ is the projection matrix: $D \times M$ (maps D dims to M dims)
• $\mathbf{Z}$ is the projected data: $N \times M$ (N samples, M new features)
• Each column of $\mathbf{W}$ defines one new dimension

W D × M
=
Z N × M

💡 Key Insight: Different choices of $\mathbf{W}$ give different projections — like moving the light source changes the shadow. The question is: which $\mathbf{W}$ best preserves the important structure in our data?

04

Principal Component Analysis (PCA)

PCA answers the question: Which projection preserves the most variance? The idea is that directions with high variance contain the most "information" about the data structure.

The PCA Objective

For each new dimension $m$, PCA finds the direction $\mathbf{w}_m$ that maximizes the variance of projected data:

PCA Objective — Maximize Projected Variance
$$\text{Var}(\mathbf{z}_m) = \frac{1}{N}\sum_{n=1}^{N}(z_{mn} - \mu_m)^2$$
Where $\mathbf{z}_m = \mathbf{X}\mathbf{w}_m$ is the projection onto direction $\mathbf{w}_m$.

Constraint: Each new direction must be orthogonal (perpendicular) to all previous ones. This ensures the new dimensions capture different information.

Visualizing PCA

Consider projecting 2D data onto a 1D line. Different lines capture different amounts of variance:

σ² = 0.39 (Low variance)
σ² = 1.9 (High variance) ✓

The first principal component is the direction of maximum variance. The second principal component is the direction of maximum remaining variance, orthogonal to the first. And so on.

The Analytic Solution

The beautiful thing about PCA is that there's a closed-form solution. The principal components are the eigenvectors of the covariance matrix:

Covariance Matrix & Eigendecomposition
$$\mathbf{C} = \frac{\mathbf{X}^T\mathbf{X}}{n-1} = \mathbf{V}\mathbf{L}\mathbf{V}^T$$
• $\mathbf{C}$ = covariance matrix of the data ($D \times D$)
• $\mathbf{V}$ = matrix of eigenvectors (these are our $\mathbf{w}$ vectors!)
• $\mathbf{L}$ = diagonal matrix of eigenvalues (these tell us the variance in each direction)

Python: sklearn.decomposition.PCA
R: prcomp()

Connection to SVD

In practice, PCA is computed using Singular Value Decomposition (SVD), which is more numerically stable:

X = U Σ VT
(N×D) (N×N) (N×D) (D×D)

The columns of V are the principal components!

05

PCA in Practice

Applying to New Data

Once you've learned $\mathbf{W}$ from training data, you can project any new data into the same space:

Projecting New Data
$$\mathbf{Z}_{\text{new}} = \mathbf{X}_{\text{new}}\mathbf{W}$$
Use the same $\mathbf{W}$ learned from training. This ensures training and test data live in the same coordinate system.

How Many Components?

You can keep anywhere from 1 to D components. More components = more information retained, but also more complexity. Use cross-validation to find the sweet spot:

Cross-validation helps find the optimal number of principal components

06

Selection vs Projection

Aspect Feature Selection PCA (Projection)
What it does Picks subset of original features Creates new features from combinations
Interpretability High — features are still original variables Lower — new features are abstract combinations
Uses labels? Yes (supervised) No (unsupervised)
Captures interactions No — features evaluated independently Yes — combinations capture correlations
When to use Need to know which original features matter Care about prediction, not interpretation
07

Summary

1

The Problem

Too many features relative to samples leads to overfitting. Need to reduce dimensionality.

2

Feature Selection

Score features by class separability: $s = \frac{|\mu_1 - \mu_0|}{\sigma_0^2 + \sigma_1^2}$. Keep top S features.

3

Feature Projection

Create new features via linear transformation: $\mathbf{Z} = \mathbf{X}\mathbf{W}$

4

PCA

Choose $\mathbf{W}$ to maximize variance of projected data. Solution: eigenvectors of covariance matrix.

5

In Practice

Use cross-validation to choose number of features/components. Always compute on training data only!

🎯 Key Takeaway: Dimensionality reduction is essential when D >> N. Whether you select or project, you're trading some information for the ability to actually learn from your data without overfitting.
