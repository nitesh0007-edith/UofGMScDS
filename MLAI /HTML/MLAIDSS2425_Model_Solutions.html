Question 1: Regression (20 marks)

Context: Consider using regression to predict the birth rate in the US from 1909 to 2008.

(a) Polynomial Regression Order 10 - Numerical Issues [4 marks]

Consider fitting the data with a polynomial regression of order 10. Identify the numerical issue with model fitting and propose a solution with sufficient details.

Solution:

Numerical Issue Identification: [2 marks]

  • The value of the year could cause numerical issues when raised to order 10 [1 mark]
  • The numerical issue is matrix inversion in the least square solution being unstable when year^10 is too large [1 mark]
Example: Year 2000 raised to power 10 = 2000^10 ≈ 1.024 × 10^33, which is extremely large and causes numerical overflow and instability in matrix computations.

Solution: [2 marks]

Propose a reasonable rescaling method as a solution. Valid approaches include:

  • Whitening (Standardization): Transform features to have mean 0 and standard deviation 1

    \[x_{scaled} = \frac{x - \mu}{\sigma}\]

  • Min-Max Normalization: Scale to range [0,1] or [-1,1]

    \[x_{scaled} = \frac{x - x_{min}}{x_{max} - x_{min}}\]

  • Logarithmic Transformation: Take the logarithm to reduce magnitude

    \[x_{scaled} = \log(x)\]

(b) Polynomial Regression Order 2 - Poor Fit Regions [6 marks]

Consider fitting the data with a polynomial regression of order 2. Identify the two regions of most likely poorly fitted data points and explain why.

Solution:

Identifying Poorly Fitted Regions: [2 marks]

Several options for poorly fitted data points:

  • Region 1: x = 1945-1960, y ≈ 24 (post-war baby boom)
  • Region 2: x ≈ 1938-1940, y ≈ 17 (pre-war drop)
  • Region 3: x = 1975-1980 (data points showing decline)

Reasoning: [4 marks]

  1. A polynomial regression model with an order of 2 is a quadratic (or convex) curve [1 mark]
  2. The global minima of the curve would be in years after 2008 [1 mark]
  3. The left tail end would be following the downward trend from 1909 to 1920 [1 mark]
  4. The curve would cut through the drop from 1920-1940 and the upward trend after the 1940s [1 mark]

Optional mark [1]: The fitted curve will struggle to capture the drop in 1970-1980.

Key Insight: A quadratic model can only capture one "bend" in the data. The birth rate data has multiple inflection points (1920s drop, 1940s rise, 1970s drop), which a simple parabola cannot represent accurately.

(c) Sigmoid Basis Function Hyperparameters [4 marks]

Consider fitting the data with a linear regression model using the sigmoid basis function:

\[h_{n,k} = sigmoid\left(\frac{(x_n - \mu_k)^2}{s}\right), \quad n = 1,...,N; \quad k = 1,...,K\]

Explain the choice of hyperparameters μ_k (mu_k) and s that could lead to a well-fitted model.

Solution:

Choice of μ_k (centers): [2 marks]

  • μ_k can be set as x_n (i.e., place centers at actual data points) [1 mark]
  • Setting the center parameter to be x_n gives the model maximum flexibility to fit every data point [1 mark]

Choice of s (width/scale): [2 marks]

  • s is set to relatively small in relation to x_n, for example, s = 1 [1 mark]
  • Setting s to be small allows sharp turns between data points to be fitted [1 mark]
Trade-off: While this approach gives maximum flexibility, it can lead to overfitting. In practice, you would use regularization (Ridge or Lasso) to control model complexity while maintaining flexibility.

(d) Ridge vs Lasso - Identifying Regularization Method [6 marks]

Two fitting strategies (ridge regression and lasso) were used and obtained different fitting models. Identify which fitting strategy is used in each figure and explain why and how the chosen fitting method could have generated the result.

Figure 1.3 A: Shows a model with three flat regions

Figure 1.3 B: Shows a model poorly following the trend

Solution:

Figure 1.3 A - Lasso: [3 marks]

Figure 1.3 A is fitted with Lasso [1 mark]

Reasoning:

  • The fitted model has three flat regions. This happens when basis functions covering these regions have zero coefficients [1 mark]
  • The fitted model also suggests that the hyperparameter controlling the strength of L1 regularization is set to be too strong [1 mark, identifying L1 regularization is needed]
  • Lasso's L1 penalty (λΣ|w_j|) forces many coefficients to exactly zero, creating sparse solutions
  • When λ is too large, too many basis functions are eliminated, leaving flat regions where no basis functions are active

Figure 1.3 B - Ridge: [3 marks]

Figure 1.3 B is fitted with Ridge regression [1 mark]

Reasoning:

  • The fitted model is poorly following the trend. This happens when all basis functions have very small coefficients [1 mark]
  • The fitted model suggests that the hyperparameter controlling the strength of L2 regularization is set to be too strong [1 mark, identifying L2 regularization is needed]
  • Ridge's L2 penalty (λΣw_j²) shrinks all coefficients toward zero but doesn't eliminate them
  • When λ is too large, all coefficients become very small, resulting in a nearly flat line that barely responds to the data
Key Difference:
  • Lasso with strong λ: Creates sparse solutions (many zeros) → flat regions with no response
  • Ridge with strong λ: Shrinks all coefficients uniformly → smooth but overly dampened response

Question 2: Classification (20 marks)

Context: Training data in 2D plane with points and classes:

  • (2,0) → Class +1 (Red)
  • (0,2) → Class -1 (Blue)
  • (0,-2) → Class +1 (Red)
  • (-2,0) → Class +1 (Red)

(a)(i) k-NN with k=1 Classification [4 marks]

Design a k-NN classifier with k=1 and determine the class variables C1 through C4 for the following test points:

  • (0,1) → C1
  • (1.5,1) → C2
  • (-0.5,1) → C3
  • (0,0) → C4

Solution:

Calculate distances and classify:

Test point (0,1) - C1:

  • Distance to (2,0): √[(0-2)² + (1-0)²] = √5 ≈ 2.24
  • Distance to (0,2): √[(0-0)² + (1-2)²] = 1.0 ← Nearest
  • Distance to (0,-2): √[(0-0)² + (1-(-2))²] = 3.0
  • Distance to (-2,0): √[(0-(-2))² + (1-0)²] = √5 ≈ 2.24

C1 = -1 (Class -1/Blue)

Test point (1.5,1) - C2:

  • Distance to (2,0): √[(1.5-2)² + (1-0)²] = √1.25 ≈ 1.12 ← Nearest
  • Distance to (0,2): √[(1.5-0)² + (1-2)²] = √3.25 ≈ 1.80

C2 = 1 (Class +1/Red)

Test point (-0.5,1) - C3:

  • Distance to (-2,0): √[(-0.5-(-2))² + (1-0)²] = √3.25 ≈ 1.80
  • Distance to (0,2): √[(-0.5-0)² + (1-2)²] = √1.25 ≈ 1.12 ← Nearest

C3 = -1, but accept C3 = 1 if student argues for (-2,0) being closest

Test point (0,0) - C4:

  • Distance to (2,0): 2.0
  • Distance to (0,2): 2.0
  • Distance to (0,-2): 2.0
  • Distance to (-2,0): 2.0

C4 = unknown (all training points are equidistant - tie situation)

Results Summary: C1 = -1, C2 = 1, C3 = 1 (or -1 depending on calculation), C4 = unknown

(a)(ii) k-NN with k=3 for Point (0,0) [2 marks]

What would be the class variable C4 if we had used k=3?

Solution:

With k=3, we consider the 3 nearest neighbors to (0,0).

Since all 4 training points are equidistant (distance = 2.0), we take any 3 of them:

  • Possible neighbors: (2,0), (0,2), (0,-2), (-2,0)
  • Class distribution: 3 points are Class +1, 1 point is Class -1
  • Any selection of 3 points will include at least 2 from Class +1

C4 = 1 (Class +1) by majority vote

(a)(iii) Decision Boundary Equations [4 marks]

Write down the equations that specify the decision boundary between the two classes.

Solution:

Decision Boundary 1: [2 marks]

\[X_1 - X_2 = 0 \quad \text{in} \quad X_1 > 0 \text{ and } X_2 > 0\]

This represents the line x₁ = x₂ in the first quadrant (where both coordinates are positive).

Decision Boundary 2: [2 marks]

\[X_1 + X_2 = 0 \quad \text{in} \quad X_1 > 0 \text{ and } X_2 < 0\]

This represents the line x₁ = -x₂ in the fourth quadrant (where x₁ is positive and x₂ is negative).

Geometric Interpretation: The decision boundaries form a "plus sign" pattern through the origin, dividing the space based on which class point is closest in each region.

(b)(i) SVM Support Vectors and Predictor [6 marks]

Apply a linear SVM model with predictor y(X₁, X₂). Which data points are the support vectors? Write down the equation for y(X₁, X₂).

Solution:

Support Vectors: [3 marks]

The support vectors are the points closest to the decision boundary:

  • (2, 0) - Class +1
  • (0, 2) - Class -1
  • (-2, 0) - Class +1

Point (0, -2) is NOT a support vector as it lies further from the optimal decision boundary.

SVM Predictor Equation: [3 marks]

\[y(X_1, X_2) = -2X_2 + 1\]

Explanation:

  • Decision boundary: -2X₂ + 1 = 0, or X₂ = 0.5
  • For X₂ < 0.5: y > 0 → Class +1
  • For X₂ > 0.5: y < 0 → Class -1
  • This horizontal decision boundary optimally separates the classes

(b)(ii) Lagrange Multipliers [4 marks]

Specify the Lagrange multipliers α₁, α₂, α₃, α₄ for each of the data points (2,0), (0,2), (-2,0), and (0,-2), respectively.

Solution:

\[\alpha_1 = 0.5 \quad \text{(support vector)}\]

\[\alpha_2 = 1.0 \quad \text{(support vector)}\]

\[\alpha_3 = 0.5 \quad \text{(support vector)}\]

\[\alpha_4 = 0 \quad \text{(not a support vector)}\]

Key Property: Lagrange multipliers are non-zero (α > 0) only for support vectors. Non-support vectors have α = 0 because they don't influence the decision boundary.

Question 3: Unsupervised Learning (20 marks)

Context: Consider using K-means algorithm to cluster concentric circles data (two ring-shaped clusters).

(a) K-means with Euclidean Distance on Concentric Circles [3 marks]

Outline what would happen if we directly apply K-means with Euclidean distance to this data. Can it achieve the clustering objective? How will it split/group the data and why?

Solution:

K-means Behavior: [1 mark]

K-means clustering aims to partition the data into K clusters by minimizing the variance within each cluster. It does this by assigning each data point to the nearest cluster center (centroid) based on Euclidean distance.

Problem with Concentric Circles: [1 mark]

  • K-means assumes that clusters are spherical and equally sized, which is not the case with concentric circles
  • Concentric circles are non-spherical and have different radii
  • Euclidean distance measures straight-line distance, which doesn't work well for circular or ring-shaped clusters
  • Points on the inner circle are closer to the centroid of the outer circle than to the centroid of their own circle

Expected Result: [1 mark]

K-means will likely split the data into arbitrary segments rather than correctly identifying the two concentric circles. For example, it might divide the data into pie-like slices or other non-intuitive shapes based on which centroid is closer in Euclidean distance.

Key Limitation: K-means with Euclidean distance cannot separate clusters that are not linearly separable or spherical. Concentric circles require non-linear separation.

(b) Kernel K-means for Concentric Circles [2 marks]

Would kernel K-means help in this dataset and why?

Solution:

Yes, Kernel K-means would help.

Reason 1: [1 mark]

Kernel K-means extends the traditional K-means algorithm by using a kernel function to transform the data into a higher-dimensional space where the clusters become more separable.

Reason 2: [1 mark]

A Radial Basis Function (RBF) kernel would allow handling the non-linear cluster boundaries. In the transformed space, the concentric circles could become linearly separable.

RBF Kernel:

\[K(x_i, x_j) = \exp\left(-\gamma \|x_i - x_j\|^2\right)\]

(c) Gaussian Mixture Models for Concentric Circles [3 marks]

Would mixture models help to better classify this dataset than K-means and why?

Solution:

GMMs have limitations for this data:

Point 1: [1 mark]

GMMs assume that the data is generated from a mixture of several Gaussian distributions with unknown parameters.

Point 2: [1 mark]

GMMs assume that each cluster follows a Gaussian distribution, which is not ideal for ring-shaped data. Gaussian distributions are unimodal and radially symmetric around a center, but ring shapes are hollow with no data at the center.

Point 3: [1 mark]

With points only on the boundaries (rings), the model might place the means of the Gaussians in between the circles, leading to incorrect clustering.

Conclusion: While GMMs are more flexible than K-means for some non-spherical clusters, they still struggle with hollow ring-shaped clusters. Kernel K-means or spectral clustering would be more appropriate.

(d) PCA - First Principal Component [2 marks]

A 2D plot shows two separate clusters. Explain how the first principal component would look if overlaid on the plot.

Solution:

Direction of First PC: [1 mark]

The first principal component will be across the direction of the highest variance.

Application to Two Clusters: [1 mark]

In this case, we can note that there is a large variation in the axis that connects the two clusters. The first PC will be a line running through the data along the direction that separates the two clusters (the axis connecting their centers).

Visualization: If two clusters are horizontally separated, PC1 would be a horizontal line. If diagonally separated, PC1 would be a diagonal line in that direction.

(e) PCA - Second Principal Component [2 marks]

Explain what the second principal component would look like and why.

Solution:

Orthogonality Property: [1 mark]

PCA decomposes the data into orthogonal components.

Direction of Second PC: [1 mark]

Therefore, the second principal component will be orthogonal (perpendicular) to the first. If PC1 captures variance along the axis connecting the two clusters, PC2 captures variance within each cluster perpendicular to that axis.

Example: If PC1 is horizontal (separating clusters left-right), then PC2 is vertical (capturing variation up-down within each cluster).

(f) Cross-Validation for Kernel K-Means [4 marks]

Describe the four-step process to determine the number of clusters in Kernel K-Means. (Hint: Each step gets a mark.)

Solution:

Step 1: Split the Data [1 mark]

Divide the data into k folds (e.g., k=5 or k=10 for cross-validation).

Step 2: Kernel Computation [1 mark]

Compute the kernel matrix for the entire dataset. This matrix contains pairwise kernel similarities K(x_i, x_j) for all data points.

Step 3: Cross-Validation Loop [1 mark]

For each fold, train Kernel K-means on (k-1) folds and validate on the remaining fold. Repeat this process for different numbers of clusters (e.g., K=2, 3, 4, ...).

Step 4: Evaluate Performance [1 mark]

Use a performance metric (e.g., silhouette score, Davies-Bouldin index, or within-cluster sum of squares) to evaluate the clustering quality and average the results across all folds. Select the number of clusters K that gives the best average performance.

Common Metrics:
  • Silhouette Score: Measures how similar a point is to its own cluster compared to other clusters (range: -1 to 1, higher is better)
  • Elbow Method: Plot within-cluster sum of squares vs K, look for "elbow" point
  • Gap Statistic: Compares within-cluster dispersion to expected dispersion under null reference distribution

(g) Managing Curse of Dimensionality [4 marks]

Describe two approaches you could take to manage the curse of dimensionality in genetic data. For example, how would you overcome this if you had a high-dimensional dataset with thousands of genetic features but only hundreds of subjects?

Solution:

General Solution: [2 marks]

Feature selection can help overcome the problem of the curse of dimensionality.

Approach 1: Feature Selection [1 mark]

Use a feature selection method that picks the most relevant features. Options include:

  • Filter methods: Statistical tests (correlation, mutual information, ANOVA)
  • Wrapper methods: Recursive Feature Elimination (RFE), forward/backward selection
  • Embedded methods: Lasso regression (L1 penalty automatically selects features)
  • Domain knowledge: Select features known to be biologically relevant

Approach 2: Dimensionality Reduction [1 mark]

Use a dimensionality reduction technique like PCA that creates new features (principal components) as linear combinations of original features. Other options include:

  • PCA: Projects data onto directions of maximum variance
  • t-SNE or UMAP: Non-linear dimensionality reduction for visualization
  • Autoencoders: Neural network-based dimensionality reduction
  • Factor Analysis: Identifies latent variables explaining correlations
Specific to Genetic Data:
  • With thousands of genetic features and hundreds of samples, the ratio is very poor (features >> samples)
  • Feature selection can reduce from thousands to tens/hundreds of most relevant genes
  • PCA can reduce to first ~50-100 components capturing 80-90% of variance
  • This makes subsequent analysis (clustering, classification) more reliable and computationally feasible
