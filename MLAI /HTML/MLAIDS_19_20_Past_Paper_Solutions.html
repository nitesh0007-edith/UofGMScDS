Question 1: Polynomial Regression (20 marks)

A polynomial regression model is defined as:

\[t_n = \sum_{d=0}^{D} w_d x_n^d, \quad n = 1, \ldots, N\]

(a) Rescaling in Olympic Data [4 marks]

When applying this model to the Olympic data, where \(x_n \in \{1896, \ldots, 2008\}\), we always rescale x, e.g. \(x_n = \frac{x_n - 1896}{40}\). Explain why is this rescaling necessary.

Solution:

Answer:

In the Olympic data, high polynomials could make the value of \(x^d\) very big. [2 marks]

As a result, the computation of parameters becomes unstable. [2 marks]

Explanation: When dealing with years like 1896-2008, raising these large values to high powers (e.g., 2000^10) results in astronomically large numbers that cause:
  • Numerical overflow (numbers too large to store in computer memory)
  • Ill-conditioned matrices (Φ^TΦ becomes nearly singular)
  • Loss of numerical precision in floating-point arithmetic
Rescaling to a smaller range (e.g., [0,2.8]) keeps polynomial values manageable.

(b) RBF Basis Function Model [3 marks]

Write down the regression model if the following Radial basis function (RBF) with a range of different position parameter μ is applied to \(x_n\).

\[RBF(y; \mu, l) = \exp\left(-\frac{(y-\mu)^2}{l^2}\right)\]

Solution:

Answer:

\[t_n = \sum_{d=1}^{D} w_d \exp\left(-\frac{(x_n - \mu_d)^2}{l^2}\right), \quad n = 1, \ldots, N\]

[3 marks]

Explanation: The RBF model replaces polynomial terms \(x^d\) with Gaussian basis functions centered at different positions μ_d. Each basis function creates a "bump" centered at μ_d, and the final prediction is a weighted sum of these bumps. The width parameter l controls how wide each bump is.

(c) Nonlinear Regression Example [2 marks]

Give an example of nonlinear regression model. Also explain why it is nonlinear.

Solution:

Answer:

Any nonlinear function of the w and x, e.g., \(f(x, w) = \sin(w \cdot x)\) [1 mark]

State clearly which relationship is nonlinear (e.g., the model is nonlinear in the parameters w, or nonlinear in x) [1 mark]

Other Examples:
  • \(f(x, w) = w_1 e^{w_2 x}\) - exponential model, nonlinear in w_2
  • \(f(x, w) = \frac{w_1}{1 + w_2 x}\) - rational function, nonlinear in w_2
  • \(f(x, w) = w_1 \log(w_2 x)\) - logarithmic model, nonlinear in w_2
Key Point: "Nonlinear" typically means nonlinear in the parameters w, not just in x. Polynomial regression is linear in w (even though polynomial in x).

(d) Polynomial Regression and Outliers [3 marks]

Explain why polynomial regression could suffer from outliers.

Solution:

Answer:

Outliers favour higher polynomial orders [1 mark], as they can fit the outliers better [1 mark]. However, the high order models are more likely to overfit the data [1 mark].

Detailed Explanation: When outliers are present:
  • Low-degree polynomials (d=1,2) cannot fit outliers well, leading to high error on those points
  • High-degree polynomials (d≥5) can bend dramatically to pass through outliers
  • This causes the polynomial to have extreme oscillations between data points
  • The model overfits, capturing noise rather than the underlying trend
  • Test error becomes very high even though training error is low
Solution: Use regularization (Ridge/Lasso) or robust regression methods that are less sensitive to outliers.

(e) L2 Regularization with Contour Plot [8 marks]

L2 Regularised regression can be used to deal with this problem. Use a contour plot (Assuming the dimension of the parameter is 2) of parameters and the loss function to explain why.

Solution:

Marking Scheme:

  • 2 marks for the correct contour of the mean squared error (ellipses centered away from origin)
  • 2 marks for the correct contour for the L2 regularization (circles centered at origin)
  • 2 marks for highlighting the correct intersection between the two
  • 2 marks for stating the fact that optimal parameter shifted closer to (0, 0)
Visual Description:

Without Regularization (MSE only):

  • Contours are ellipses of constant MSE: \(\sum (y_i - w^T x_i)^2 = c\)
  • Minimum is at the center of the ellipses (ordinary least squares solution)
  • This minimum may have large weight values if overfitting

L2 Regularization penalty:

  • Contours are circles: \(w_1^2 + w_2^2 = c\) (or \(\|w\|^2 = c\))
  • Centered at origin (0,0)
  • Penalty increases as we move away from origin

Combined objective (MSE + λ\|w\|²):

  • Optimal solution is where MSE contour touches smallest L2 penalty circle
  • This point is closer to (0,0) than the unregularized solution
  • Larger λ → solution closer to origin → smaller weights → less overfitting

Why this helps with outliers: Smaller weights mean the model is less flexible and cannot bend as dramatically to fit outliers, leading to better generalization.

Question 2: Classification (20 marks)

(a) Regression vs Classification [3 marks]

Classification and regression are both supervised learning problems. Describe a way to turn a regression problem into a classification problem? (Please state the differences between the two.)

Solution:

Answer:

In regression, the target variable is a continuous/real-valued variable [1 mark].

In classification, the target variable is a discrete/binary/categorical variable [1 mark].

To obtain a classification problem from regression, one can cluster/group the continuous/real-valued variable into groups [1 mark].

Examples:
  • Regression: Predict house price (continuous: $100K, $250K, $500K, etc.)
  • Classification: Predict price category (discrete: "Low", "Medium", "High")
  • Conversion: Set thresholds: Low (<$200K), Medium ($200K-$400K), High (>$400K)
  • Regression: Predict student exam score (0-100)
  • Classification: Predict pass/fail (Pass if score ≥50, Fail otherwise)

(b) ROC Curve [3 marks]

The receiver operating characteristic (ROC) curve is a standard way to visualize the performance of classifiers. Outline how to draw a ROC curve.

Solution:

Answer:

The ROC curve is created by varying the threshold at which the classifier calls something as belonging to the positive class. [1 mark]

Each point is consisted of false positive rate or 1-specificity (normally on the x-axis) [1 mark]

and true positive rate or sensitivity (normally on the y-axis) [1 mark].

Step-by-Step Process:
  1. Sort instances by predicted probability/score (highest to lowest)
  2. Start with threshold = +∞ (all predicted negative): TPR=0, FPR=0 → point (0,0)
  3. For each threshold:
    • Classify instances above threshold as positive, below as negative
    • Calculate: TPR = TP/(TP+FN), FPR = FP/(FP+TN)
    • Plot point (FPR, TPR)
  4. End with threshold = -∞ (all predicted positive): TPR=1, FPR=1 → point (1,1)
  5. Connect points to form curve
Perfect Classifier: ROC goes from (0,0) → (0,1) → (1,1) (AUC = 1)
Random Classifier: Diagonal line from (0,0) to (1,1) (AUC = 0.5)

(c) AUC Calculation [7 marks total]

For the classifier outputs in the table below, provide a value for the missing output (labeled '?') that would:

Class Label 0 0 0 1 1 1
Output 0.1 0.25 0.4 ? 0.6 0.9

(i) Give an AUC equal to 1 [2 marks]

(ii) Give an AUC less than 1 [2 marks]

(iii) Now assuming you can change any output of the six data points, give an example of the outputs, such that the AUC is 0.5 [3 marks]

Solution:

(i) AUC = 1: [2 marks]

Answer: Anything between 0.4 and 1.0

Explanation: For perfect classification (AUC=1), all positive class outputs must be higher than all negative class outputs. The highest negative output is 0.4, and the lowest known positive output is 0.6. So the missing value must be in the range (0.4, 1.0) to maintain perfect separation.

(ii) AUC < 1: [2 marks]

Answer: Anything less than 0.4

Explanation: If the missing positive class output is less than 0.4, there will be at least one negative class instance (with output 0.4) that has a higher score than a positive class instance. This creates misclassification at some thresholds, reducing AUC below 1.

(iii) AUC = 0.5: [3 marks]

Answer: As long as the output for 0s and 1s are the same.

Example: All outputs = 0.5, or outputs [0.3, 0.3, 0.3, 0.3, 0.3, 0.3]

Explanation: AUC = 0.5 represents random guessing. This occurs when the classifier cannot distinguish between classes at all. If all outputs are identical, the classifier provides no information, resulting in a diagonal ROC curve (random performance).

AUC Interpretation:
  • AUC = 1.0: Perfect classifier (100% separation)
  • AUC = 0.5: Random classifier (no discriminative power)
  • AUC < 0.5: Worse than random (predictions are inverted!)
  • AUC represents probability that a random positive instance is ranked higher than a random negative instance

(d) Linear SVM Diagram [4 marks]

Use a diagram (some data in 2D) to describe how linear Support Vector Machines (SVMs) operate (how they make classification decisions, what and how parameters have to be set, what data needs to be stored etc).

Solution:

Marking Scheme:

  • 1 mark for drawing that demonstrates the margin
  • 1 mark for highlighting the decision boundary
  • 1 mark for highlighting support vectors
  • 1 mark for pointing out only support vectors need to be stored
Diagram Should Include:

Key Components:

  • Decision boundary: The hyperplane that separates the two classes (solid line in 2D)
  • Margin: The region between the two parallel hyperplanes on either side of the decision boundary (shaded area or indicated by dashed lines)
  • Support vectors: The data points that lie exactly on the margin boundaries (circled or highlighted). These are the critical points that define the decision boundary.
  • Other points: Data points that are farther from the boundary (can be shown but less emphasized)

Mathematical Details:

  • Decision boundary: \(w^T x + b = 0\)
  • Margin boundaries: \(w^T x + b = +1\) (positive class) and \(w^T x + b = -1\) (negative class)
  • Margin width: \(\frac{2}{\|w\|}\)
  • Classification: Sign of \(w^T x + b\) determines predicted class

Storage Requirement:

  • Only support vectors and their coefficients need to be stored (typically 5-20% of training data)
  • All other training points can be discarded after training
  • This makes SVM memory-efficient for prediction

(e) Logistic Regression Likelihood [3 marks]

Logistic regression uses the sigmoid function to make classification decision. Now, we have defined the following probability with a sigmoid function:

\[p(t_n = 0|\mathbf{w}, \mathbf{x}_n) = \frac{1}{1 + \exp(-\mathbf{w}^T \mathbf{x}_n)}\]

Write down the corresponding likelihood function for nth data point, \(p(t_n|\mathbf{w}, \mathbf{x}_n)\).

Solution:

Answer:

\[p(t_n|\mathbf{w}, \mathbf{x}_n) = \left(1 - \frac{1}{1 + \exp(-\mathbf{w}^T \mathbf{x}_n)}\right)^{t_n} \left(\frac{1}{1 + \exp(-\mathbf{w}^T \mathbf{x}_n)}\right)^{1-t_n}\]

[3 marks]

Derivation:

Given:

  • \(p(t_n = 0|w, x_n) = \sigma(w^T x_n) = \frac{1}{1 + \exp(-w^T x_n)}\)
  • Then: \(p(t_n = 1|w, x_n) = 1 - \sigma(w^T x_n) = 1 - \frac{1}{1 + \exp(-w^T x_n)} = \frac{\exp(-w^T x_n)}{1 + \exp(-w^T x_n)}\)

The likelihood for a single point can be written compactly as:

  • If \(t_n = 1\): use \(p(t_n=1|w,x_n)\) raised to power 1, and \(p(t_n=0|w,x_n)\) raised to power 0
  • If \(t_n = 0\): use \(p(t_n=1|w,x_n)\) raised to power 0, and \(p(t_n=0|w,x_n)\) raised to power 1

This gives the formula: \(p(t_n|w,x_n) = [p(t_n=1)]^{t_n} [p(t_n=0)]^{1-t_n}\)

Note: This can also be simplified to:

\[p(t_n|\mathbf{w}, \mathbf{x}_n) = \sigma(\mathbf{w}^T \mathbf{x}_n)^{1-t_n} (1-\sigma(\mathbf{w}^T \mathbf{x}_n))^{t_n}\]

where \(\sigma(z) = \frac{1}{1+\exp(-z)}\)

Question 3: Unsupervised Learning (20 marks)

Consider using the K-means algorithm to perform clustering on the following data.

[Diagram shows concentric circles - inner circle and outer ring pattern]

We want to cluster data in the outer ring in one cluster and the data in the inner circle as a different cluster.

(a) K-means with Euclidean Distance [2 marks]

Outline what would happen if we directly apply K-means with Euclidean distance to this data. Can it achieve the clustering objective? How will it split/group the data?

Solution:

Answer:

K-means cannot split the data into outer ring and inner circle clusters [1 mark].

It will group parts of the outer ring or inner circle [1 mark].

Detailed Explanation:

K-means with Euclidean distance assumes clusters are spherical/globular. The algorithm will:

  • Draw a straight line (linear decision boundary) between two cluster centers
  • This creates a vertical or horizontal split through the data
  • One cluster might contain: left half of both inner circle and outer ring
  • Other cluster: right half of both inner circle and outer ring
  • Result: Each cluster is a mixture of both true classes - fails to separate concentric patterns

Why it fails: The true cluster shapes are non-convex (ring shape cannot be captured by spherical clusters). K-means can only create convex, spherical clusters separated by linear boundaries.

(b) Kernel K-means [3 marks]

An alternative approach is to use Kernel K-means. Explain how the kernel could help in this dataset.

Solution:

Answer:

A kernel that projects the data onto a different space [1 mark]

where data can be easily separated [1 mark].

The space could have higher or lower number of dimensions compared to the original data [1 mark].

Detailed Explanation:

How kernel helps with concentric circles:

  • Original 2D space: Inner circle and outer ring are not linearly separable
  • RBF kernel transformation: Maps data to infinite-dimensional space
  • In transformed space: The two groups become separable by a hyperplane
  • Result: K-means in the transformed space correctly separates inner/outer

Intuition for RBF kernel:

  • RBF kernel: \(K(x_i, x_j) = \exp(-\gamma \|x_i - x_j\|^2)\)
  • Measures similarity: Points close together → K ≈ 1, far apart → K ≈ 0
  • Inner circle points are all close to each other (high similarity)
  • Outer ring points are all close to each other (high similarity)
  • Inner and outer points are far apart (low similarity)
  • This similarity structure helps kernel K-means separate the groups

(c) Kernel Statements [2 marks]

Which one of the following statements about kernel is NOT correct?

A. One could use the RBF kernel, \(K(x_n, x_j) = \exp(-\gamma(x_n - x_j)^T(x_n - x_j))\), to achieve the clustering target.

B. The RBF kernel projects the data onto an infinite dimensional space. The free parameter γ can be estimated with cross-validation.

C. One could use the linear kernel, \(K(x_n, x_j) = x_n^T x_j\), to achieve the clustering target.

D. The linear kernel projects the data onto itself, and there is no free parameter to tune.

Solution:

Answer: C [2 marks]

Explanation of Each Statement:
  • A - CORRECT: RBF kernel can achieve the clustering target. It maps data to infinite dimensions where concentric circles become separable.
  • B - CORRECT: RBF kernel does project to infinite-dimensional space (can be shown using Taylor series expansion). Parameter γ controls the width and can be tuned via cross-validation.
  • C - INCORRECT: Linear kernel \(K(x_n, x_j) = x_n^T x_j\) projects data onto itself (identity mapping). This is equivalent to standard K-means with Euclidean distance, which we established in part (a) cannot separate concentric circles. Therefore, linear kernel CANNOT achieve the clustering target.
  • D - CORRECT: Linear kernel indeed projects data onto itself (no transformation), and has no hyperparameters to tune.

(d) K-means Pseudocode [5 marks]

Write some pseudo-code to perform K-means.

Solution:

Answer:

Given: Number of clusters, K

2. For each cluster k = 1...K:                                      [Initialization step implied]

3. For each object n = 1...N:                                       [1 mark]

4. Compute the distance between object n and cluster k              [1 mark]

5. Assign object n to the cluster corresponding to smallest distance [1 mark]

6. Update the mean of each cluster                                  [1 mark]

7. If assignments have changed, return to 2. Else stop.             [1 mark]
More Detailed Algorithm: 
Input: Data points {x₁, ..., x_N}, number of clusters K

1. Initialize:
   Randomly select K data points as initial cluster centers μ₁, ..., μ_K

2. Repeat until convergence:

   2a. Assignment Step: [1 mark]
       For each data point n = 1 to N:
           Compute distance to each cluster center k:        [1 mark]
               d_nk = ||x_n - μ_k||²
           Assign x_n to nearest cluster:                    [1 mark]
               c_n = argmin_k d_nk

   2b. Update Step: [1 mark]
       For each cluster k = 1 to K:
           Recompute cluster center as mean of assigned points:
               μ_k = (1/|C_k|) Σ_{n ∈ C_k} x_n

   2c. Check Convergence: [1 mark]
       If cluster assignments haven't changed (or max iterations reached):
           Stop
       Else:
           Continue to next iteration

Output: Cluster assignments {c₁, ..., c_N} and centers {μ₁, ..., μ_K}

(e) Cross-validation for Selecting K [8 marks]

Outline how and why cross-validation can be used to select the number of clusters K.

Solution:

Answer:

Cross-validation with total or average Euclidean distance between data points and their cluster centers [2 marks].

At each CV cycle, use the training data to determine the mean of the clusters [2 marks].

Test these means by computing total or average Euclidean distance between testing data points and their nearest cluster centers [2 marks].

CV allows the trained number of clusters and means to be tested on a different dataset in which the trained K-means may or may not be a good fit [2 marks].

Detailed Cross-Validation Procedure:

Why use Cross-Validation for K selection:

  • Training error always decreases as K increases (K=N gives zero training error)
  • Need to evaluate generalization to unseen data
  • CV provides unbiased estimate of how well clustering generalizes
  • Prevents overfitting (too many clusters)

How to use Cross-Validation:

  1. For each candidate K value (e.g., K = 2, 3, 4, ..., 10):
  2. Split data into folds (e.g., 5-fold CV):
  3. For each fold f:
    • Training set: All folds except f (e.g., 80% of data)
    • Validation set: Fold f (e.g., 20% of data)
  4. Train K-means on training set:
    • Run K-means algorithm to convergence
    • Obtain K cluster centers: μ₁, ..., μ_K
  5. Evaluate on validation set:
    • For each validation point x_n, find nearest cluster center
    • Compute distance: d_n = min_k ||x_n - μ_k||²
    • Average over validation set: Error_f = (1/N_val) Σ d_n
  6. Average across all folds:
    • CV_Error(K) = (1/F) Σ_f Error_f
  7. Select K with minimum CV error

Alternative Metrics:

  • Silhouette score: Measures how similar points are to their own cluster vs other clusters
  • Gap statistic: Compares within-cluster dispersion to that expected under null reference distribution
  • Elbow method: Plot within-cluster sum of squares vs K, look for "elbow"

Important Note: CV for unsupervised learning is tricky because:

  • No true labels to evaluate against
  • Must use reconstruction error or similar proxy metric
  • Different random initializations can give different results (run multiple times)
