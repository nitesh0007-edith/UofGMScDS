Unsupervised Learning

Gaussian Mixture Models
& The EM Algorithm

A beginner-friendly journey into probabilistic clustering — where data points belong to clusters with probabilities, not hard assignments.

01

Thinking Generatively

Imagine you're looking at a scatter plot of data points. You notice they seem to form distinct groups or clusters. But here's a deeper question: What process could have generated this data?

Data points naturally grouping into three distinct clusters

The Generative Idea: What if each data point was generated by first picking one of several "source" distributions, then sampling from that distribution? If we can figure out what those distributions are, we've essentially discovered the clusters!

This is the core idea behind Mixture Models. Instead of just finding cluster centers (like K-means), we model each cluster as a full probability distribution. The most common choice? The Gaussian (Normal) distribution.

02

The Gaussian Distribution

Before diving into mixtures, let's understand the building block: the Gaussian (or Normal) distribution. It's the famous "bell curve" you've probably seen before.

1D Gaussian

Univariate Gaussian PDF
$$f(X) = \frac{1}{\sigma\sqrt{2\pi}} \exp\left(-\frac{1}{2}\left(\frac{X - \mu}{\sigma}\right)^2\right)$$
What each part means:
• $\mu$ (mu) = mean — the center of the bell curve
• $\sigma$ (sigma) = standard deviation — how spread out the data is
• $\sigma^2$ = variance — the square of standard deviation
• The exponential term creates the bell shape
• The fraction in front ensures the area under the curve equals 1

2D (Multivariate) Gaussian

In higher dimensions, we need a covariance matrix Σ instead of just variance. This matrix captures not only how spread out each dimension is, but also how dimensions are correlated with each other.

Multivariate Gaussian PDF
$$p(\mathbf{X}|\boldsymbol{\mu}, \boldsymbol{\Sigma}) = \frac{1}{(2\pi)^{n/2}|\boldsymbol{\Sigma}|^{1/2}} \exp\left(-\frac{1}{2}(\mathbf{X}-\boldsymbol{\mu})^T\boldsymbol{\Sigma}^{-1}(\mathbf{X}-\boldsymbol{\mu})\right)$$
New elements:
• $\boldsymbol{\mu}$ = mean vector (center point in n-dimensional space)
• $\boldsymbol{\Sigma}$ = covariance matrix (n × n matrix)
• $|\boldsymbol{\Sigma}|$ = determinant of covariance matrix
• $\boldsymbol{\Sigma}^{-1}$ = inverse of covariance matrix
• The exponent now involves a quadratic form (matrix multiplication)

How Covariance Shapes the Distribution

Full Covariance

Σ = [Σ₁₁ Σ₁₂]
     [Σ₂₁ Σ₂₂]

Tilted ellipse — dimensions are correlated

Diagonal Covariance

Σ = [Σ₁₁  0 ]
     [ 0  Σ₂₂]

Axis-aligned — no correlation

Spherical (Isotropic)

Σ = σ²I = σ²[1 0]
            [0 1]

Perfect circle — equal variance

03

Gaussian Mixture Model (GMM)

Now we combine multiple Gaussians! A Gaussian Mixture Model assumes our data comes from a mixture of K different Gaussian distributions.

The Generative Story

For each data point $\mathbf{x}_n$:

  1. Pick a component k from {1, 2, ..., K} with probability $\pi_k$
  2. Sample $\mathbf{x}_n$ from the Gaussian $\mathcal{N}(\boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)$

Key Parameters

A GMM with K components has the following parameters to learn:

GMM Parameters (Δ)
$$\Delta = \{\pi_1, ..., \pi_K, \boldsymbol{\mu}_1, ..., \boldsymbol{\mu}_K, \boldsymbol{\Sigma}_1, ..., \boldsymbol{\Sigma}_K\}$$
For each component k:
• $\pi_k$ = mixing coefficient (probability of choosing this component, $\sum_k \pi_k = 1$)
• $\boldsymbol{\mu}_k$ = mean of component k
• $\boldsymbol{\Sigma}_k$ = covariance matrix of component k

The Latent Variable

We introduce a latent (hidden) variable $z_{nk}$ that tells us which component generated point $\mathbf{x}_n$:

Latent Variable Indicator
$$z_{nk} = \begin{cases} 1 & \text{if } \mathbf{x}_n \text{ came from component } k \\ 0 & \text{otherwise} \end{cases}$$
This is what we want to know but don't observe. The whole point of the algorithm is to estimate these assignments!

Posterior Probability (Responsibility)

Given a data point, how likely is it to belong to each cluster? This is computed using Bayes' theorem:

Responsibility — The Posterior Probability
$$q_{nk} = \frac{\pi_k \cdot p(\mathbf{x}_n | z_{nk}=1, \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)}{\sum_{j=1}^{K} \pi_j \cdot p(\mathbf{x}_n | z_{nj}=1, \boldsymbol{\mu}_j, \boldsymbol{\Sigma}_j)}$$
Reading this formula:
• Numerator: (prior probability of cluster k) × (likelihood of data given cluster k)
• Denominator: Sum over all clusters — this normalizes so probabilities sum to 1
• $q_{nk}$ is the "responsibility" that cluster k takes for point n

Soft vs Hard Assignment: Unlike K-means which assigns each point to exactly one cluster, GMM gives each point a probability distribution over all clusters. A point might be 70% cluster 1 and 30% cluster 2!

04

The EM Algorithm

We can't directly maximize the likelihood because we don't know the cluster assignments. The Expectation-Maximization (EM) algorithm solves this by alternating between two steps.

EM Algorithm for GMM

1

Initialize Parameters

Make initial guesses for $\boldsymbol{\mu}_k$, $\boldsymbol{\Sigma}_k$, and $\pi_k$ for all K components. Random initialization or K-means initialization are common choices.

E

Expectation Step (E-step)

Compute the responsibilities $q_{nk}$ — the probability that each point belongs to each cluster, given current parameters:

$q_{nk} = \frac{\pi_k \mathcal{N}(\mathbf{x}_n | \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)}{\sum_j \pi_j \mathcal{N}(\mathbf{x}_n | \boldsymbol{\mu}_j, \boldsymbol{\Sigma}_j)}$

M

Maximization Step (M-step)

Update parameters to maximize likelihood given current responsibilities:

4

Repeat Until Convergence

Go back to E-step. Continue until parameters stop changing significantly or log-likelihood converges.

M-step Update Formulas

Effective Number of Points in Cluster k
$$\hat{N}_k = \sum_{i=1}^{N} q_{ik}$$
Sum of all responsibilities for cluster k. This is the "soft count" of how many points belong to cluster k.
Updated Mean
$$\hat{\boldsymbol{\mu}}_k = \frac{1}{\hat{N}_k} \sum_{i=1}^{N} q_{ik} \mathbf{x}_i$$
Weighted average of all points, where weights are the responsibilities. Points more likely to belong to cluster k have more influence on its mean.
Updated Covariance
$$\hat{\boldsymbol{\Sigma}}_k = \frac{1}{\hat{N}_k} \sum_{i=1}^{N} q_{ik} (\mathbf{x}_i - \hat{\boldsymbol{\mu}}_k)(\mathbf{x}_i - \hat{\boldsymbol{\mu}}_k)^T$$
Weighted covariance matrix. Each point's contribution is weighted by how likely it belongs to cluster k.
Updated Mixing Coefficient
$$\hat{\pi}_k = \frac{\hat{N}_k}{N}$$
Fraction of points belonging to cluster k. If cluster k "claims" 30% of the total responsibility, its mixing coefficient becomes 0.3.

Convergence Guarantee

EM is guaranteed to converge to a local maximum of the likelihood. However, it may not find the global maximum! Running EM multiple times with different initializations helps find better solutions.

06

GMM vs K-Means

You might notice GMM and K-means have similar iterative structures. Here's how they compare:

K-Means

Assignment: Hard (each point belongs to exactly one cluster)

Clusters: Spherical only

Output: Cluster labels

Objective: Minimize within-cluster variance

Speed: Faster

GMM

Assignment: Soft (probabilistic membership)

Clusters: Elliptical (any shape via covariance)

Output: Probabilities per cluster

Objective: Maximize likelihood

Speed: Slower but more flexible

Fun fact: K-means is actually a special case of GMM! If you use spherical covariances ($\Sigma_k = \sigma^2 I$) with equal, fixed variances, and take the limit as $\sigma \to 0$, EM for GMM becomes equivalent to K-means.

07

Choosing the Number of Components K

A critical question: How many Gaussian components should we use?

The Overfitting Problem

Adding more components always increases the training likelihood! With K = N (one Gaussian per point), you could achieve perfect fit — but this is meaningless overfitting.

Solution: Cross-Validation

Use held-out log-likelihood on validation data:

Cross-validation helps find the optimal number of components

Other approaches include BIC (Bayesian Information Criterion) and AIC (Akaike Information Criterion), which penalize model complexity.

08

Beyond Gaussian Components

The mixture model framework is very flexible! You can use any distribution for the components, as long as you can define $p(\mathbf{x}_n | z_{nk}=1, \Delta_k)$.

Example: Binary/Bernoulli Mixture

For binary data (0s and 1s), use Bernoulli distributions:

Bernoulli Mixture Component
$$p(\mathbf{x}_n | z_{nk}=1, \Delta_k) = \prod_{d=1}^{D} p_{kd}^{x_{nd}} (1 - p_{kd})^{1-x_{nd}}$$
• $x_{nd}$ is the d-th dimension of point n (either 0 or 1)
• $p_{kd}$ is the probability of dimension d being 1 in cluster k
• The M-step update: $p_{kd} = \frac{\sum_n q_{nk} x_{nd}}{\sum_n q_{nk}}$ (weighted average)

This is useful for document clustering, survey data, or any binary feature vectors!

09

Summary

Concept Key Points
Generative Model Data is generated by first picking a component, then sampling from its distribution
GMM Parameters Mixing coefficients $\pi_k$, means $\mu_k$, covariances $\Sigma_k$ for each component
E-step Compute responsibilities $q_{nk}$ (soft assignments) using current parameters
M-step Update $\mu_k$, $\Sigma_k$, $\pi_k$ using weighted statistics from responsibilities
Convergence Guaranteed to reach local maximum; use multiple restarts for better solutions
Model Selection Use cross-validation, BIC, or AIC to choose K
Flexibility Can use any distribution — Gaussian, Bernoulli, Poisson, etc.

The Big Picture: GMM gives you a probabilistic view of clustering. Instead of asking "which cluster does this point belong to?", you ask "what's the probability this point came from each cluster?" This uncertainty quantification is valuable in many real-world applications!
