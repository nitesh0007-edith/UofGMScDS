📈 REGRESSION

Linear Regression

Model: \(y = w^Tx + b\)

MSE Loss: \(L = \frac{1}{N}\sum(y_i - w^Tx_i)^2\)

Solution: \(w = (\Phi^T\Phi)^{-1}\Phi^Ty\)

Squared vs Absolute Loss: Squared loss \(e^2\) grows faster than absolute \(|e|\) → outliers have much larger influence with squared loss

Basis Functions

Polynomial: \(\phi(x) = [1, x, x^2, ..., x^d]\)

RBF/Gaussian: \(\phi_k(x) = \exp\left(-\frac{(x-\mu_k)^2}{2s^2}\right)\)

Sigmoid: \(\phi_k(x) = \sigma\left(\frac{x-\mu_k}{s}\right)\)

RBF advantages: Local features, handles gaps in data. RBF disadvantages: Need to choose centers \(\mu_k\) and width \(s\)

Regularization

Ridge (L2) Lasso (L1)
Penalty \(\lambda\sum w_j^2\) \(\lambda\sum|w_j|\)
Effect Shrinks → 0 Sets = 0
Use All features relevant Feature selection

Ridge solution: \(w = (\Phi^T\Phi + \lambda I)^{-1}\Phi^Ty\)

Rescaling/Normalization

Whitening/Z-score: \(x' = \frac{x - \mu}{\sigma}\) → mean=0, std=1

Min-Max: \(x' = \frac{x - \min}{\max - \min}\) → range [0,1]

Log transform: \(x' = \log(x)\) → reduces skewness

When to rescale: High-degree polynomials, features with different scales, distance-based algorithms

Model Complexity & Validation

Training loss cannot choose complexity → always decreases with complexity → overfitting. Use validation/cross-validation instead.

K-fold CV: Split into K folds, train on K-1, validate on 1, average K results

Common K: 5 or 10. K=N is LOOCV

Bias-Variance Trade-off

\(\text{Error} = \text{Bias}^2 + \text{Variance} + \text{Noise}\)

High Bias: Underfitting, too simple
High Variance: Overfitting, too complex
🎯 CLASSIFICATION

k-Nearest Neighbors (k-NN)

Algorithm:

  1. Compute distances to all training points
  2. Find k nearest neighbors
  3. Majority vote for classification

Distance: \(d = \sqrt{\sum(x_i - x_j)^2}\) (Euclidean)

Decision boundary: Perpendicular bisectors between nearest neighbors of different classes (for k=1)
Small k: Complex boundaries, noise-sensitive
Large k: Smooth boundaries, may underfit

Logistic Regression

Sigmoid: \(\sigma(z) = \frac{1}{1+e^{-z}}\)

Model: \(P(y=1|x) = \sigma(w^Tx + b)\)

Loss: \(L = -\frac{1}{N}\sum[y_i\log(\hat{y}_i) + (1-y_i)\log(1-\hat{y}_i)]\)

Why sigmoid? Maps to (0,1), smooth/differentiable, probabilistic interpretation

Likelihood for dataset {(x₁,y₁), (x₂,y₂)}:

\(L = P(y_1|x_1) \times P(y_2|x_2)\)

If y₁=1, y₂=0: \(L = \sigma(w^Tx_1) \times (1-\sigma(w^Tx_2))\)

Support Vector Machines (SVM)

Decision boundary: \(w^Tx + b = 0\)

Margin: \(\frac{2}{\|w\|}\) → SVM maximizes this

Prediction: \(y = \text{sign}(w^Tx + b)\)

Hard margin: \(\min \frac{1}{2}\|w\|^2\) s.t. \(y_i(w^Tx_i+b) \geq 1\)

Soft margin: \(\min \frac{1}{2}\|w\|^2 + C\sum\xi_i\) s.t. \(y_i(w^Tx_i+b) \geq 1-\xi_i\)

Support vectors: Points on/within margin (\(\alpha_i > 0\))
Large C: Small margin, fewer violations
Small C: Large margin, more violations

Lagrange Multipliers (for SVM)

Weight vector: \(w = \sum_{i=1}^{N}\alpha_i y_i x_i\)

Only support vectors have \(\alpha_i > 0\)

Example: If points (2,0)→class +1, (0,2)→class -1, (0,-2)→class -1 are support vectors and \(\alpha_1=0.5, \alpha_2=\alpha_3=0.25\):

\(w = 0.5(+1)[2,0] + 0.25(-1)[0,2] + 0.25(-1)[0,-2] = [1,0]\)

Kernel Trick

Linear: \(K(x_i,x_j) = x_i^Tx_j\)

Polynomial: \(K(x_i,x_j) = (x_i^Tx_j + c)^d\)

RBF: \(K(x_i,x_j) = \exp(-\gamma\|x_i-x_j\|^2)\)

Key idea: Data appears only as inner products → replace \(x_i^Tx_j\) with \(K(x_i,x_j)\) to get non-linear boundaries without computing \(\phi(x)\) explicitly

Decision Trees

Entropy: \(H = -\sum p_i\log_2(p_i)\)

Info Gain: \(IG(S,A) = H(S) - \sum\frac{|S_v|}{|S|}H(S_v)\)

Choose attribute with highest information gain for splitting

Generalization & Overfitting

Generalization: Ability to perform well on unseen data
Overfitting: Memorizing training data (including noise) → poor test performance
📊 EVALUATION METRICS

Confusion Matrix

Pred + Pred -
Actual + TP FN
Actual - FP TN

Classification Metrics

Accuracy: \(\frac{TP+TN}{TP+TN+FP+FN}\)

Precision: \(\frac{TP}{TP+FP}\) (of predicted +, how many correct?)

Recall/Sensitivity: \(\frac{TP}{TP+FN}\) (of actual +, how many found?)

Specificity: \(\frac{TN}{TN+FP}\) (of actual -, how many correct?)

F1-Score: \(\frac{2 \cdot Precision \cdot Recall}{Precision + Recall}\)

Accuracy misleading with imbalanced classes! Use precision/recall/F1 instead

ROC Curve & AUC

ROC: Plot TPR vs FPR at different thresholds

AUC = 1.0: Perfect classifier

AUC = 0.5: Random guessing

AUC < 0.5: Worse than random

Cannot compute AUC from single confusion matrix → need predictions at multiple thresholds

Regression Metrics

MSE: \(\frac{1}{N}\sum(y_i-\hat{y}_i)^2\)

MAE: \(\frac{1}{N}\sum|y_i-\hat{y}_i|\) (robust to outliers)

R²: \(1 - \frac{\sum(y_i-\hat{y}_i)^2}{\sum(y_i-\bar{y})^2}\) (0 to 1)

🔍 UNSUPERVISED LEARNING

K-means Clustering

Objective: \(\min \sum_{k=1}^{K}\sum_{x \in C_k}\|x-\mu_k\|^2\)

Algorithm:

  1. Initialize K centroids randomly
  2. Assign each point to nearest centroid
  3. Update centroids: \(\mu_k = \frac{1}{|C_k|}\sum_{x \in C_k}x\)
  4. Repeat 2-3 until convergence
Total distance always decreases with more K → cannot use for selecting K. Use elbow method, silhouette score, or cross-validation instead
Overcoming local optima: Run multiple times with different random initializations, keep best (lowest objective)

Kernel K-means

For anisotropic/elongated Gaussian clusters: Use Gaussian kernel to encode anisotropy. Gives better fit than standard K-means for non-spherical clusters

Gaussian Mixture Models (GMM)

Model: \(p(x) = \sum_{k=1}^{K}\pi_k\mathcal{N}(x|\mu_k,\Sigma_k)\)

EM Algorithm:

E-step: Calculate responsibilities

\(r_{ik} = \frac{\pi_k\mathcal{N}(x_i|\mu_k,\Sigma_k)}{\sum_j\pi_j\mathcal{N}(x_i|\mu_j,\Sigma_j)}\)

M-step: Update \(\mu_k, \Sigma_k, \pi_k\)

\(\mu_k = \frac{\sum r_{ik}x_i}{\sum r_{ik}}\), \(\pi_k = \frac{1}{N}\sum r_{ik}\)

K-means vs GMM

Aspect K-means GMM
Assignment Hard (binary) Soft (probabilistic)
Parameters Centers only \(\mu_k, \Sigma_k, \pi_k\)
Shape Spherical Elliptical (any orientation)
When to use GMM over K-means:
  • Non-spherical/anisotropic clusters
  • Need probability estimates
  • Data type where distance unclear but likelihood computable

GMM Model Selection

Choosing K:

  1. Use k-fold cross-validation
  2. For each K, compute average log-likelihood on validation folds
  3. Select K with maximum average log-likelihood

GMM Outlier Detection

Algorithm:

  1. Fit GMM using EM algorithm
  2. Compute responsibilities \(r_{nk}\) for each point
  3. If \(\max_k(r_{nk}) < \text{threshold}\) → outlier

Intuition: If a point doesn't fit any cluster well (low responsibility for all clusters), it's an outlier

Principal Component Analysis (PCA)

Steps:

  1. Center data: \(X' = X - \bar{X}\)
  2. Compute covariance: \(C = \frac{1}{N}X'^TX'\)
  3. Find eigenvalues \(\lambda_i\) and eigenvectors \(v_i\)
  4. Sort by eigenvalue (largest first)
  5. Select top k eigenvectors

Variance explained: \(\frac{\sum_{i=1}^{k}\lambda_i}{\sum_{j=1}^{D}\lambda_j}\)

PC1: Direction of maximum variance
PC2: Orthogonal to PC1, next highest variance
PCs are orthogonal (uncorrelated)
💡 EXAM TIPS

Common Question Types

1. Calculate metrics from confusion matrix
  • Write formula first
  • Identify TP, FP, TN, FN carefully
  • Substitute values
  • Show all working
2. k-NN classification
  • Calculate ALL distances
  • Show distance formula
  • Identify k nearest
  • Count class votes
3. Compare methods (Ridge/Lasso, K-means/GMM)
  • Use table format
  • Cover: objective, assumptions, pros/cons
  • Give specific examples
4. Explain concepts
  • Clear definition
  • Mathematical formulation
  • Intuition/purpose
  • Concrete example

Common Mistakes

❌ Confusing precision and recall
Precision = of predicted +, how many correct?
Recall = of actual +, how many found?
❌ Not showing working
Always: Formula → Substitute → Calculate → Answer
❌ Forgetting assumptions
State: linearly separable (hard SVM), normally distributed noise (linear regression with MSE), spherical clusters (K-means)
