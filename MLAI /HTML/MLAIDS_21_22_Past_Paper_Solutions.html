Question 1: Regression - CO2 Emissions and Temperature Anomaly (20 marks)

Context: Consider using regression to predict global temperature anomaly from cumulative CO2 emissions data.

(a) Rescaling Strategy for High Order Polynomial Regression [4 marks]

Propose a rescaling strategy (with enough details of the procedure) for the cumulative CO2 emissions when using high order polynomial regression. Explain why the proposed strategy is appropriate.

Solution:

Proposed Rescaling Strategies: [2 marks for reasonable strategy]

Any of the following strategies would be appropriate:

1. Whitening (Standardization):

\[x_{scaled} = \frac{x - \mu}{\sigma}\]

where \(\mu\) is the mean and \(\sigma\) is the standard deviation of cumulative CO2 emissions.

2. Min-Max Scaling:

\[x_{scaled} = \frac{x - x_{min}}{x_{max} - x_{min}}\]

This scales the data to the range [0, 1].

3. Logarithmic Transformation:

\[x_{scaled} = \log(x)\]

This compresses the scale of large values.

Reasoning for Why This is Appropriate: [2 marks total]

  • [1 mark] The key is to reduce the absolute value of CO2 emissions, such that high order polynomial will still produce well-behaved values (small numbers). Without rescaling, raising large CO2 emission values (e.g., 1.5×10¹²) to high powers would result in extremely large numbers that cause numerical instability.
  • [1 mark] The matrix inversion in the least squares solution is still stable. Large numbers in the design matrix \(\Phi\) lead to very large values in \(\Phi^T\Phi\), making the matrix ill-conditioned and its inversion numerically unstable.
Example: If CO2 = 1.5×10¹², then CO2² = 2.25×10²⁴ and CO2³ = 3.375×10³⁶, which are computationally problematic. After rescaling to [0,1], these values become manageable.

(c) RBF vs Polynomial Basis Functions [4 marks]

Consider fitting the data in figure 1 with a regression using radial basis functions (RBF):

\[h_{n,k} = \exp\left(-\frac{(x_n - \mu_k)^2}{2s^2}\right), \quad n = 1, \ldots, N; \quad k = 1, \ldots, K\]

where \(x_n\) represents each cumulative CO2 emission. Outline one advantage and one disadvantage of using RBF over polynomials for the data in figure 1.

Solution:

Advantage: [2 marks total]

  • [1 mark] The data is not equally distributed across x values - it is denser in small values and relatively sparser in large x values.
  • [1 mark] Using location-specific basis functions, RBF can model this localized effect better than polynomial functions which model global effect across all x values, resulting in better fitting performance.

Disadvantage: [2 marks total]

  • [1 mark] RBF has more hyper-parameters (centers \(\mu_k\), bandwidth \(s^2\), number of basis functions K).
  • [1 mark] Poorly chosen hyper-parameters could lead to overfitting.
Key Insight: RBF basis functions are local - each basis function only "activates" for data points near its center \(\mu_k\). This allows adaptive fitting to regions with different data densities. Polynomials are global - each term affects the entire range equally.

Question 2: Classification (20 marks)

(a) Logistic Regression Likelihood Comparison [6 marks]

The likelihood of logistic regression is:

\[p(t_n|\mathbf{w}, \mathbf{x}_n) = g(\mathbf{w}^T\mathbf{x}_n)^{t_n}(1 - g(\mathbf{w}^T\mathbf{x}_n))^{1-t_n}\]

where \(g(a) = \frac{1}{1 + \exp(-a)}\)

Consider fitting this model to a dataset with 2 classes, 2 binary features and 2 examples per class:

  • Class 0: Example 1 = [1,1], Example 2 = [1,0]
  • Class 1: Example 1 = [1,1], Example 2 = [0,1]

Use the likelihood function to demonstrate which of the following two parameter hypotheses: [0.6, 0.1] and [0.6, 0.8] fits this dataset better.

Solution:

Marking: Correct computation of each data likelihood [4 marks total]. Correct comparison of total likelihood [2 marks].

Step 1: Calculate likelihood for each data point

For each data point, we compute \(\mathbf{w}^T\mathbf{x}_n\) and then \(g(\mathbf{w}^T\mathbf{x}_n)\):

Step 2: Calculate log-likelihood for each data point

Step 3: Calculate joint likelihood

Joint log-likelihood = -1.1032 - 1.0375 - 0.4032 - 0.6444 = -3.1883

Conclusion:

Parameter candidate [0.6, 0.1] fits the data better because it has higher joint likelihood (0.0412 > 0.0388) and higher log-likelihood (-3.1883 > -3.2494).

Interpretation: The first parameter set [0.6, 0.1] gives less weight to the second feature, which better matches this contradictory dataset where [1,1] appears in both classes. The second parameter [0.6, 0.8] gives too much weight to the second feature, leading to overconfident (and wrong) predictions.

Consider an SVM trained on a dataset where two data points are mislabeled by a non-expert annotator. The classifier outputs are in the table below:

(i) What would be the AUC (computed with the correct labels) if the missing value is 0.6? (Detailed calculation required) [2 marks]

(ii) What would be the maximum achievable AUC (computed with the corrupted labels) and corresponding range of possible values for the missing value? Explain why. [2 marks]

(iii) If you could correct one of the two corrupted labels to get better AUC (computed with the labels with one remaining wrongly labeled data), assuming the missing value is 0.6 and rest of the scores do not change, which will you correct? Explain why. [2 marks]

Solution:

(i) AUC with missing value = 0.6: [2 marks]

First, let's organize the data with correct labels and scores:

2 0 (negative) 8.8
3 0 (negative) 0.7
4 1 (positive) 0.6

AUC is calculated as the probability that a randomly chosen positive sample has a higher score than a randomly chosen negative sample.

Positive samples: 4 (score 0.6), 5 (score 2.2), 6 (score 0.3)

Negative samples: 1 (score -9.6), 2 (score 8.8), 3 (score 0.7)

Count correctly ranked pairs (positive score > negative score):

  • Positive 4 (0.6) vs Negative 1 (-9.6): 0.6 > -9.6 ✓
  • Positive 4 (0.6) vs Negative 2 (8.8): 0.6 < 8.8 ✗
  • Positive 4 (0.6) vs Negative 3 (0.7): 0.6 < 0.7 ✗
  • Positive 5 (2.2) vs Negative 1 (-9.6): 2.2 > -9.6 ✓

  • Positive 6 (0.3) vs Negative 3 (0.7): 0.3 < 0.7 ✗

Correctly ranked pairs: 4 out of 9 total pairs

This ensures that positive data (based on corrupted labels) have higher scores than any negative data (based on corrupted labels).

(c) SVM Configuration for Noisy Labels [4 marks]

Noisy labels may produce outliers in the training set. How will you configure the SVM in terms of margin and kernel to deal with outliers? Explain why.

Solution:

Margin Configuration: [2 marks]

Use soft margin SVM to allow outliers to go across the decision boundary without heavily influencing the boundary position. The soft margin allows some misclassification through the slack variables, making the model robust to noisy labels.

Soft margin objective: \(\min \frac{1}{2}\|\mathbf{w}\|^2 + C\sum_{i}\xi_i\)

Use smaller C value to tolerate more violations (outliers)

Kernel Configuration: [2 marks]

Choose a less powerful kernel to avoid overfitting to the noisy labels. For example:

  • Use linear kernel instead of RBF if data is roughly linearly separable
  • If using RBF kernel, use larger bandwidth parameter \(\sigma\) to create smoother decision boundaries
Summary: Soft margin (small C) + simple kernel = robust to outliers from noisy labels. This prevents the model from contorting the decision boundary to fit mislabeled points perfectly.

(d) K-NN Scoring for AUC Calculation [4 marks]

Calculating AUC requires a classifier to give a score for each data point. A K-nearest neighbor classifier does not normally provide a score, but directly predicts the class for a data point. Outline two approaches to produce scores for computing AUC for a K-nearest neighbor classifier.

Solution:

Approach 1: Vote Proportions [2 marks]

Convert vote counts to vote proportions. For a test point:

  • Find K nearest neighbors
  • Count how many belong to the positive class
  • Score = (number of positive neighbors) / K

Example: If K=5 and 3 neighbors are positive, score = 3/5 = 0.6

Approach 2: Majority Margin [2 marks]

Use the margin of the majority vote as the confidence score:

  • Find K nearest neighbors
  • Count votes for each class: n₊ for positive, n₋ for negative
  • Score = (n₊ - n₋) / K (or simply n₊ - n₋)

Example: If K=5, with 4 positive and 1 negative, score = (4-1)/5 = 0.6

Alternative Approaches: Distance-weighted voting (inverse distance weighting), using the distance to the K-th nearest neighbor, or averaging distances to positive vs negative neighbors.

Question 3: Unsupervised Learning (20 marks)

Context: Clustering Ellipsoid-Shaped Data

Consider using K-means to perform clustering on data arranged in three ellipsoid shapes (Figure 3A), where we expect to form three clusters as shown in Figure 3B.

(a) K-means with Euclidean Distance on Ellipsoids [2 marks]

Outline what would happen if we directly apply K-means with Euclidean distance to this data. Can it achieve the clustering objective? How will it split/group the data and why?

Solution:

Answer: [1 mark] K-means cannot split the data into three clusters along the respective ellipsoids.

Explanation: [1 mark] Due to the Euclidean distance, points that are close together but in neighboring ellipsoids will be clustered together. K-means creates circular (spherical in higher dimensions) cluster boundaries, so it cannot capture the elongated ellipsoid shapes. Points at the ends of different ellipsoids that are spatially close will be grouped together, rather than respecting the ellipsoid structure.

Key Issue: K-means assumes clusters are isotropic (same variance in all directions), but ellipsoids are anisotropic (different variances along different axes).

(b) Kernel K-means for Ellipsoids [3 marks]

An alternative approach is to use Kernel K-means. Would kernel K-means help in this dataset and why?

Solution:

Answer: [1 mark] Yes, kernel K-means could help separate these ellipsoid clusters.

Explanation: [2 marks] A kernel projects the data onto a different space where data can be more easily separated. The space could have a higher or lower number of dimensions compared to the original data. By choosing an appropriate kernel (e.g., polynomial or RBF), the ellipsoids that overlap in the original 2D space might become linearly separable in the kernel space, allowing K-means to successfully cluster them.

Example: An RBF kernel can map the data to an infinite-dimensional space where the ellipsoids might be separated with spherical boundaries in that space.

(c) Mixture Models for Ellipsoids [3 marks]

An alternative approach is to use mixture models. Would mixture models help to better classify the dataset in Figure 3A than K-means and why?

Solution:

Answer: Yes, mixture models should be able to better classify the data than K-means.

Explanation: Mixture models (specifically Gaussian Mixture Models) are able to model clusters as a mixture of Gaussian distributions with anisotropic Gaussian distributions - where diagonal elements of the covariance matrix are not equal. This means each cluster can have:

  • Different variances along different dimensions (capturing elongated shapes)
  • Rotated orientations (through full covariance matrices)
  • Different cluster sizes and densities

This naturally matches the ellipsoid structure where clusters are elongated in certain directions.

GMM allows: \(\Sigma_k = \begin{bmatrix} \sigma^2_{k,1} & \rho \\ \rho & \sigma^2_{k,2} \end{bmatrix}\)

where \(\sigma^2_{k,1} \neq \sigma^2_{k,2}\) (anisotropic)

Key Advantage: GMM explicitly models the covariance structure of each cluster, allowing it to fit ellipsoid shapes naturally, whereas K-means only uses distances to centroids.

Context: Clustering Data with Different Variances

Now consider clustering data in Figure 4A into three clusters as shown in Figure 4B, where the middle cluster has considerably different variance than the outer two clusters.

(d) K-means on Clusters with Different Variances [2 marks]

Outline what would happen if we directly apply K-means with Euclidean distance to this data. Can it achieve the clustering objective? How will it split/group the data?

Solution:

Answer: K-means cannot split the data well into three clusters.

Explanation: The variance in the middle cluster is considerably different than the variance in the other clusters. K-means only considers distance to centroids and assumes all clusters have similar variance (isotropic, spherical clusters). Therefore, considering only distance won't be sufficient - K-means will struggle to correctly assign points in the diffuse middle cluster, potentially splitting it or merging parts of it with the dense outer clusters.

Problem: Points in the sparse middle cluster might be closer to centroids of the dense outer clusters than to their own cluster's centroid, leading to incorrect assignments.

(e) Kernel K-means for Different Variances [3 marks]

An alternative approach is to use Kernel K-means. Would kernel K-means help in this dataset and why?

Solution:

Answer: Kernel K-means would not significantly help in this dataset.

Explanation: Kernel K-means does not explicitly model variance and since it is still based on distance (just in a transformed space), it won't be robust in classifying data with anisotropic variance. The fundamental issue is that different clusters have different spreads/variances, and kernel K-means still uses the same distance-to-centroid logic, just in a different space. This doesn't address the variance mismatch problem.

Limitation: Kernels help with non-linear separability, not with handling different cluster densities or variances.

(f) Mixture Models for Different Variances [3 marks]

An alternative approach is to use mixture models. Explain whether mixture models could help to better classify this dataset and why?

Solution:

Answer: Yes, mixture models with anisotropic variance would work well to model this data.

Explanation: Variance in the data is a parameter for each cluster in Gaussian Mixture Models. Each cluster k has its own covariance matrix \(\Sigma_k\), which can model:

  • Different variances for different clusters
  • The middle cluster can have large variance (\(\sigma^2_{middle}\) large)
  • The outer clusters can have small variance (\(\sigma^2_{outer}\) small)

GMM uses probabilistic assignment based on the likelihood of each point belonging to each Gaussian component, which naturally accounts for the different spreads.

For cluster k: \(p(x|\theta_k) = \mathcal{N}(x|\mu_k, \Sigma_k)\)

Each cluster can have different \(\Sigma_k\)

Success: GMM will assign high probability to the middle cluster for points within its large variance region, and high probability to outer clusters for points in their small variance regions.

(g) Feature Selection Methods [4 marks]

Explain why there is a need for feature selection and list two methods and their main characteristics.

Solution:

Why Feature Selection is Needed: [2 marks]

  • Due to the curse of dimensionality, which states that the number of required samples increases exponentially with the number of features, it is desirable to reduce dimensionality.
  • Feature selection is also important for visualizing data and identifying anomalies.
  • Reduces computational cost and helps avoid overfitting.

In high dimensions, data becomes sparse: volume of hypercube = \(L^d\) where d is dimensions

Most of the volume is in the corners, far from the center

Method 1: Feature Subset Selection [1 mark]

Characteristics: Use a subset of the original features. Choose those features that maximize the difference between the classes (e.g., using filter methods like correlation, chi-square test, or wrapper methods like forward/backward selection).

Method 2: Feature Combination/Transformation [1 mark]

Characteristics: Combine the original features and find new dimensions. For example, use Principal Component Analysis (PCA) to find dimensions that maximize the variance. This creates new features that are linear combinations of the original features.

PCA: Find principal components: \(\mathbf{z} = \mathbf{W}^T\mathbf{x}\)

where \(\mathbf{W}\) contains eigenvectors of the covariance matrix

Comparison: Subset selection keeps interpretability (original features), while transformation methods like PCA can achieve better dimensionality reduction but lose direct interpretability.
