1. Introduction to Language Modeling

Language modeling is the task of building a predictive model of language. While previous lectures dealt with unordered (bags of) word representations, today we'll start modeling sequences of words.

🎯 What is a Language Model?

A language model is something that can tell us:

  1. Probability of a whole sentence: $P(w_1, w_2, \ldots, w_n)$
  2. Probability of the next word: $P(w_{k+1} | w_1, \ldots, w_k)$
Language Modeling Exercise

Which words make sense to complete: "Denied the _____"

  • ✅ "allegations" — makes sense!
  • ❌ "sock" — doesn't fit
  • ✅ "claim" — makes sense!

Neuroscience suggests humans predict the next word when listening. We'll use word statistics to do the same!

Why Do We Need Language Models?

Application How LMs Help
Speech Recognition "butter" vs "but her" — choose what's more likely
Spelling Correction Pick the most probable correction in context
Machine Translation Choose the most natural translation
Auto-completion Predict what the user is typing
Text Generation Generate coherent, contextual text (ChatGPT!)

⚠️ LLMs Don't "Know" What They're Saying

Language models may appear to "know" things and "reason" — but their skills can often be an illusion. They're predicting statistically likely continuations, not understanding meaning. Just like the emoji exercise — you could predict patterns without understanding the "language"!

2. Probability Fundamentals

Before diving into language models, let's review essential probability concepts using a deck of cards (4 suits, 13 cards each, 52 total).

Basic Probability

Probability Rules

A probability must satisfy:

$$0 \leq P(x) \leq 1 \quad \text{and} \quad \sum_x P(x) = 1$$

Examples:

  • $P(\text{Card is 8}) = \frac{4}{52}$
  • $P(\text{Card is red}) = \frac{26}{52}$

Joint Probability — $P(A, B)$

The chance of a set of events happening together.

Joint Probability
$$P(\text{Card is 8 AND Card is red}) = P(\text{Card is 8}, \text{Card is red}) = \frac{2}{52}$$

Conditional Probability — $P(A|B)$

The probability of event A given that event B has occurred.

Conditional Probability
$$P(A|B) = \frac{P(A, B)}{P(B)}$$

Examples:

  • $P(\text{Card is 8} | \text{Card is red}) = \frac{2}{26}$
  • $P(\text{Card is red} | \text{Card is 8}) = \frac{2}{4}$

Chain Rule

Decomposing joint probabilities into products of conditional probabilities:

Chain Rule
$$P(A, B, C, D) = P(A|B,C,D) \cdot P(B|C,D) \cdot P(C|D) \cdot P(D)$$

For language, this means:

$$P(w_1, w_2, \ldots, w_n) = P(w_1) \cdot P(w_2|w_1) \cdot P(w_3|w_1,w_2) \cdots P(w_n|w_1,\ldots,w_{n-1})$$

Independence

Events A and B are independent if knowing one tells you nothing about the other:

Independence
$$P(A|B) = P(A) \quad \text{and} \quad P(B|A) = P(B)$$ $$\Rightarrow P(A,B) = P(A) \cdot P(B)$$

Bayes' Theorem

Bayes' Theorem
$$P(A|B) = \frac{P(B|A) \cdot P(A)}{P(B)}$$

3. Unigram Language Model

The simplest language model treats each word independently and ignores all context — this is the unigram model.

Unigram Probability
$$P(w) = \frac{\text{count}(w)}{N}$$

where $N$ is the total number of tokens in the corpus.

Sequence Probability (Unigram):

$$P(w_1, w_2, \ldots, w_n) = \prod_{i=1}^{n} P(w_i) = \prod_{i=1}^{n} \frac{\text{count}(w_i)}{N}$$
Example Calculation

Corpus has 1000 tokens, "the" appears 50 times:

$$P(\text{"the"}) = \frac{50}{1000} = 0.05 = 5\%$$

Calculate Token Probability

Python Step 2: Unigram Token Probability 
def unigram_token_prob(next_token, unigram_counts, unigram_N):
    """
    Calculate the probability of a single token in a unigram model.
    
    Args:
        next_token: The token to calculate probability for
        unigram_counts: Counter object with token counts
        unigram_N: Total number of tokens in corpus
    
    Returns:
        Probability of the token (float between 0 and 1)
    """
    count = unigram_counts[next_token]
    probability = count / unigram_N
    return probability

# Example usage
prob = unigram_token_prob('irn', posts_unigram_counts, posts_unigram_N)
print(f"P('irn') = {prob:.6f}")  # e.g., 0.000234

# Unseen token has probability 0
prob_unseen = unigram_token_prob('totallynotatoken', posts_unigram_counts, posts_unigram_N)
print(f"P('totallynotatoken') = {prob_unseen}")  # 0

Calculate Sequence Probability

Python Step 3: Sequence Probability 
def unigram_sequence_prob(sequence, unigram_counts, unigram_N):
    """
    Calculate the probability of a sequence using unigram model.
    
    Args:
        sequence: List of tokens
        unigram_counts: Counter object with token counts
        unigram_N: Total number of tokens
    
    Returns:
        Probability of the entire sequence
    """
    probability = 1.0
    
    for token in sequence:
        token_prob = unigram_token_prob(token, unigram_counts, unigram_N)
        probability *= token_prob
        
        # If any token has probability 0, whole sequence is 0
        if probability == 0:
            break
    
    return probability

# Example
seq = ['i', 'like', 'irn', 'bru']
prob = unigram_sequence_prob(seq, posts_unigram_counts, posts_unigram_N)
print(f"P({seq}) = {prob:.10e}")  # Very small number!

Log Probabilities — Avoiding Underflow

⚠️ The Underflow Problem

When multiplying many small probabilities, computers quickly hit numerical underflow — numbers too small to represent. The solution: work with logarithms!

Log Probability of Sequence
$$\log P(w_1, \ldots, w_n) = \sum_{i=1}^{n} \log P(w_i)$$

Multiplication becomes addition — numerically stable!

Python Log Probability Implementation 
import math

def unigram_sequence_logprob(sequence, unigram_counts, unigram_N):
    """
    Calculate log2 probability of a sequence.
    More numerically stable than regular probabilities.
    
    Returns:
        Log2 probability (negative number, or -inf if any token unseen)
    """
    log_probability = 0.0
    
    for token in sequence:
        token_prob = unigram_token_prob(token, unigram_counts, unigram_N)
        
        if token_prob == 0:
            return float('-inf')  # Log of 0 is -infinity
        
        log_probability += math.log2(token_prob)
    
    return log_probability

# Example
seq = ['i', 'like', 'irn', 'bru']
log_prob = unigram_sequence_logprob(seq, posts_unigram_counts, posts_unigram_N)
print(f"log2 P({seq}) = {log_prob:.2f}")  # e.g., -45.23

# Verify: log2 of regular prob should equal log prob
regular_prob = unigram_sequence_prob(seq, posts_unigram_counts, posts_unigram_N)
print(f"Verification: {math.log2(regular_prob):.2f}")  # Same!

💡 Why Log Base 2?

Using log base 2 has an information-theoretic interpretation: the negative log probability tells us how many bits of information we gain when observing that sequence.

4. N-Gram Language Models

Unigrams ignore context. N-gram models improve by considering the previous N-1 words when predicting the next word. This uses the Markov assumption.

🔗 The Markov Assumption

The probability of a word depends only on a fixed number of previous words, not the entire history:

$$P(w_i | w_1, \ldots, w_{i-1}) \approx P(w_i | w_{i-n+1}, \ldots, w_{i-1})$$

N-Gram Model Types

Model Context Size Formula
Unigram 0 (no context) $P(w_i)$
Bigram 1 previous word $P(w_i | w_{i-1})$
Trigram 2 previous words $P(w_i | w_{i-2}, w_{i-1})$
4-gram 3 previous words $P(w_i | w_{i-3}, w_{i-2}, w_{i-1})$

Bigram Model

Bigram Probability
$$P(w_i | w_{i-1}) = \frac{\text{count}(w_{i-1}, w_i)}{\text{count}(w_{i-1})}$$

Sequence Probability:

$$P(w_1, \ldots, w_n) \approx \prod_{i=1}^{n} P(w_i | w_{i-1})$$
Bigram Example

Suppose "irn" appears 100 times, and "irn bru" appears 80 times:

$$P(\text{"bru"} | \text{"irn"}) = \frac{\text{count}(\text{"irn bru"})}{\text{count}(\text{"irn"})} = \frac{80}{100} = 0.8$$

80% of the time "irn" appears, it's followed by "bru"!

Python Counting Bigrams 
def count_bigrams(flattened):
    """
    Count all bigrams (consecutive token pairs) in the corpus.
    
    Args:
        flattened: List of all tokens in corpus
    
    Returns:
        Counter object mapping bigram tuples to counts
    """
    bigrams = []
    
    # Create bigrams by pairing consecutive tokens
    for i in range(len(flattened) - 1):
        bigram = (flattened[i], flattened[i+1])
        bigrams.append(bigram)
    
    return Counter(bigrams)

# Example
posts_bigram_counts = count_bigrams(posts_flattened_tokens)

# How often does "irn bru" appear?
print(posts_bigram_counts[('irn', 'bru')])  # e.g., 35
Python Bigram Token Probability 
def bigram_token_prob(prev_token, next_token, unigram_counts, bigram_counts):
    """
    Calculate P(next_token | prev_token) using bigram model.
    
    Args:
        prev_token: The previous token (context)
        next_token: The token we're calculating probability for
        unigram_counts: Token counts
        bigram_counts: Bigram counts
    
    Returns:
        Conditional probability
    """
    # Count of the bigram
    bigram_count = bigram_counts[(prev_token, next_token)]
    
    # Count of the previous token
    prev_token_count = unigram_counts[prev_token]
    
    # Avoid division by zero
    if prev_token_count == 0:
        return 0.0
    
    probability = bigram_count / prev_token_count
    return probability

# Example
prob = bigram_token_prob('irn', 'bru', posts_unigram_counts, posts_bigram_counts)
print(f"P('bru' | 'irn') = {prob:.4f}")

Trigram Model

Trigram Probability
$$P(w_i | w_{i-2}, w_{i-1}) = \frac{\text{count}(w_{i-2}, w_{i-1}, w_i)}{\text{count}(w_{i-2}, w_{i-1})}$$
Python Counting and Calculating Trigrams 
def count_trigrams(flattened):
    """
    Count all trigrams (consecutive token triples) in the corpus.
    """
    trigrams = []
    
    for i in range(len(flattened) - 2):
        trigram = (flattened[i], flattened[i+1], flattened[i+2])
        trigrams.append(trigram)
    
    return Counter(trigrams)

posts_trigram_counts = count_trigrams(posts_flattened_tokens)


def trigram_token_prob(prev_prev_token, prev_token, next_token,
                       bigram_counts, trigram_counts):
    """
    Calculate P(next_token | prev_prev_token, prev_token)
    """
    # Count of the full trigram
    trigram_count = trigram_counts[(prev_prev_token, prev_token, next_token)]
    
    # Count of the bigram context
    bigram_count = bigram_counts[(prev_prev_token, prev_token)]
    
    if bigram_count == 0:
        return 0.0
    
    probability = trigram_count / bigram_count
    return probability

# Example
prob = trigram_token_prob('i', 'like', 'irn', posts_bigram_counts, posts_trigram_counts)
print(f"P('irn' | 'i', 'like') = {prob:.4f}")

🎯 Trade-offs in N-Gram Models

  • Higher N: More context → better predictions, but sparser data
  • Lower N: Less context → simpler predictions, more data available
  • Sweet spot: Trigrams often work well for many applications

5. Application: Spelling Correction

Language models can fix spelling mistakes by finding the most likely correction. This builds on ideas from Peter Norvig's spelling corrector.

The Approach

  1. Generate Candidates: Find words 1-2 edits away from the misspelled word
  2. Filter by Vocabulary: Keep only candidates that appear in our corpus
  3. Rank by Probability: Use language model to find the most likely word

Edit Distance Operations

A word can be "1 edit away" through:

  • Deletion: remove one letter ("hello" → "helo")
  • Transposition: swap adjacent letters ("hello" → "hlelo")
  • Replacement: change one letter ("hello" → "hallo")
  • Insertion: add one letter ("hello" → "heallo")
Python Generate Edit Distance Candidates 
def generate_edits1(token):
    """Generate all tokens that are 1 edit away from input."""
    letters = 'abcdefghijklmnopqrstuvwxyz'
    splits = [(token[:i], token[i:]) for i in range(len(token) + 1)]
    
    # Delete one letter
    deletes = [L + R[1:] for L, R in splits if R]
    
    # Transpose two adjacent letters
    transposes = [L + R[1] + R[0] + R[2:] for L, R in splits if len(R) > 1]
    
    # Replace one letter
    replaces = [L + c + R[1:] for L, R in splits if R for c in letters]
    
    # Insert one letter
    inserts = [L + c + R for L, R in splits for c in letters]
    
    return set(deletes + transposes + replaces + inserts)


def generate_edits2(token):
    """Generate all tokens that are 2 edits away."""
    return set(e2 for e1 in generate_edits1(token) 
                  for e2 in generate_edits1(e1))

# Example
print(generate_edits1('teh'))  # Includes 'the', 'tea', 'ten', etc.
Python Get Valid Candidate Tokens 
def get_candidate_tokens(token, unigram_counts):
    """
    Get candidate corrections that appear in our vocabulary.
    
    Priority:
    1. If token exists in corpus, return only that token
    2. Otherwise, return all 1-2 edit distance tokens in corpus
    3. If no candidates found, return original token
    """
    # If the token is already known, don't correct it
    if token in unigram_counts:
        return {token}
    
    # Try edits at distance 1
    candidates = generate_edits1(token) & set(unigram_counts.keys())
    
    if candidates:
        return candidates
    
    # Try edits at distance 2
    candidates = generate_edits2(token) & set(unigram_counts.keys())
    
    if candidates:
        return candidates
    
    # No candidates found, return original token
    return {token}

# Example
candidates = get_candidate_tokens('gmaig', posts_unigram_counts)
print(candidates)  # {'gaming', 'gmail', 'image', ...}

Unigram Spelling Corrector

Python Unigram Spelling Correction 
def unigram_spelling_correct(token, unigram_counts, unigram_N):
    """
    Correct spelling using unigram model.
    Choose the candidate with highest probability.
    """
    candidates = get_candidate_tokens(token, unigram_counts)
    
    # Find candidate with maximum probability
    best_candidate = max(candidates,
                        key=lambda c: unigram_token_prob(c, unigram_counts, unigram_N))
    
    return best_candidate

# Example
correction = unigram_spelling_correct('gmaig', posts_unigram_counts, posts_unigram_N)
print(f"'gmaig' corrected to '{correction}'")  # Likely 'gaming'

Bigram Spelling Corrector (Context-Aware)

Python Context-Aware Bigram Spelling Correction 
def bigram_spelling_correct(prev_token, token, unigram_counts, bigram_counts):
    """
    Correct spelling using bigram model (considers previous word).
    More accurate than unigram when context matters.
    """
    candidates = get_candidate_tokens(token, unigram_counts)
    
    # Find candidate with maximum conditional probability
    best_candidate = max(candidates,
                        key=lambda c: bigram_token_prob(prev_token, c,
                                                       unigram_counts,
                                                       bigram_counts))
    
    return best_candidate

# Example showing context matters!
print(bigram_spelling_correct('e', 'gmaig', 
                             posts_unigram_counts, 
                             posts_bigram_counts))  # 'gmail'

print(bigram_spelling_correct('the', 'gmaig', 
                             posts_unigram_counts, 
                             posts_bigram_counts))  # 'gaming'
Context Matters!

The bigram model gives different corrections based on context:

  • "e gmaig" → "e gmail" (email address likely)
  • "the gmaig" → "the gaming" (article before noun)

6. Application: Text Generation

Language models are generative — they can create new text that sounds natural! This is the foundation of systems like ChatGPT.

The Generation Process

  1. Start with a seed sequence (e.g., "I like")
  2. Calculate probability distribution for next word
  3. Sample from this distribution (higher probability = more likely)
  4. Add the chosen word to sequence
  5. Repeat until desired length reached

⚠️ Why Not Just Pick the Most Likely Word?

Always picking the most probable next token creates boring, repetitive text! Instead, we sample from the distribution — high probability words are picked more often, but there's still variety.

Python Get Probability Distribution for Next Token 
def get_trigram_distribution(prev_prev_token, prev_token,
                              unigram_counts, bigram_counts, trigram_counts):
    """
    Get probability distribution for all possible next tokens
    given two previous tokens.
    
    Returns:
        Dictionary mapping token -> probability
    """
    prob_distribution = {}
    
    for token in unigram_counts:
        p = trigram_token_prob(prev_prev_token, prev_token, token,
                              bigram_counts, trigram_counts)
        prob_distribution[token] = p
    
    return prob_distribution

# Example: What comes after "i like"?
dist = get_trigram_distribution('i', 'like',
                               posts_unigram_counts,
                               posts_bigram_counts,
                               posts_trigram_counts)
# dist = {'to': 0.15, 'the': 0.12, 'it': 0.08, ...}
Python Sample from Distribution 
import random

def sample_from_distribution(distribution):
    """
    Randomly choose a token weighted by probabilities.
    Higher probability = higher chance of selection.
    """
    population = list(distribution.keys())
    probabilities = list(distribution.values())
    
    choice, = random.choices(population, weights=probabilities, k=1)
    return choice

# Each time you run this, you might get a different result!
sample = sample_from_distribution(dist)
print(sample)  # Could be 'to', 'the', 'it', etc.
Python Generate Complete Sequence (Hallucinate Text) 
def hallucinate_text(sequence, max_length,
                     unigram_counts, bigram_counts, trigram_counts):
    """
    Generate text using trigram model.
    
    Args:
        sequence: Starting tokens (list of at least 2 tokens)
        max_length: Desired final length
        unigram_counts, bigram_counts, trigram_counts: Model data
    
    Returns:
        Generated sequence of tokens
    """
    # Make a copy to avoid modifying original
    sequence = list(sequence)
    
    # Keep generating until we reach max_length
    while len(sequence) < max_length:
        # Get last two tokens for context
        prev_prev_token = sequence[-2]
        prev_token = sequence[-1]
        
        # Get probability distribution for next token
        trigram_dist = get_trigram_distribution(
            prev_prev_token, prev_token,
            unigram_counts, bigram_counts, trigram_counts
        )
        
        # Sample next token from distribution
        next_token = sample_from_distribution(trigram_dist)
        
        # Add to sequence
        sequence.append(next_token)
    
    return sequence

# Generate text!
start = ["i", "like"]
generated = hallucinate_text(start, 20,
                            posts_unigram_counts,
                            posts_bigram_counts,
                            posts_trigram_counts)
print(" ".join(generated))

Advanced Generation Techniques

For better text generation, consider:

  • Top-K sampling: Filter to only the top K most probable words before sampling
  • Top-P (nucleus) sampling: Filter to words covering the top P% of probability
  • Beam search: Grow several candidate sequences and keep the best ones
  • Repetition blocking: Prevent repeated n-grams

7. Smoothing Techniques

Real-world text will always contain words we haven't seen during training. Without smoothing, unseen words get probability 0, making entire sentences probability 0!

⚠️ The Zero Probability Problem

Example: A speech recognition system choosing between:

  • "ginchy toast with but her" — probability 0 (rare word)
  • "ginchy toast with butter" — probability 0 (same rare word)

One is obviously more plausible, but both have zero probability because "ginchy" (1950s slang for "excellent") wasn't in training data!

Add-K (Laplace) Smoothing

The idea: pretend we saw every possible word K extra times. This ensures no word has zero probability.

Add-K Smoothed Unigram Probability
$$P_{\text{smooth}}(w) = \frac{\text{count}(w) + k}{N + k|V|}$$

where:

  • $k$ = smoothing parameter (often 0.1 or 1)
  • $N$ = total number of tokens
  • $|V|$ = vocabulary size (number of unique tokens)
Original Distribution           Smoothed Distribution
                                                    
    ┌─────┐                         ┌─────┐        
    │█████│                         │████ │        
    │█████│ ┌───┐                   │████ │ ┌───┐  
    │█████│ │███│                   │████ │ │██ │  
    │█████│ │███│ ┌─┐               │████ │ │██ │ ┌─┐ ┌─┐
    │█████│ │███│ │█│               │████ │ │██ │ │█│ │░│
────┴─────┴─┴───┴─┴─┴───────    ────┴─────┴─┴───┴─┴─┴─┴─┴───
  "the"   "cat" "xyz" (unseen)    "the"   "cat" "xyz" (now>0!)
                                                    
Take some probability from common words, give to rare/unseen words
Python Add-K Smoothing Implementation 
def unigram_token_prob_ksmooth(next_token, unigram_counts, unigram_N, k):
    """
    Calculate smoothed unigram probability.
    Ensures unseen tokens still have non-zero probability.
    
    Args:
        next_token: Token to calculate probability for
        unigram_counts: Token counts from training data
        unigram_N: Total tokens in training data
        k: Smoothing parameter (e.g., 0.1 or 1.0)
    
    Returns:
        Smoothed probability
    """
    # Get count (0 for unseen tokens)
    count = unigram_counts[next_token]
    
    # Vocabulary size
    vocab_size = len(unigram_counts)
    
    # Smoothed probability
    probability = (count + k) / (unigram_N + k * vocab_size)
    
    return probability

# Example: Even unknown words have probability!
prob_known = unigram_token_prob_ksmooth('the',
                                       posts_unigram_counts,
                                       posts_unigram_N, 0.1)
prob_unknown = unigram_token_prob_ksmooth('abracadabra',
                                         posts_unigram_counts,
                                         posts_unigram_N, 0.1)

print(f"P('the') = {prob_known:.6f}")         # e.g., 0.045123
print(f"P('abracadabra') = {prob_unknown:.10f}")  # Small but > 0!

Other Smoothing Methods

Method Description
Add-1 (Laplace) K=1, simple but often too aggressive
Add-K K can be a fraction (0.1), tuned experimentally
Interpolation Combine trigram + bigram + unigram with weights
Kneser-Ney State-of-the-art, considers word versatility
Stupid Backoff Used at Google: if trigram=0, use 0.4×bigram
Interpolated Smoothing

Combines higher-order (more sparse) with lower-order (less sparse) models:

$$P(w_3|w_1,w_2) = \lambda_1 P_{\text{tri}}(w_3|w_1,w_2) + \lambda_2 P_{\text{bi}}(w_3|w_2) + \lambda_3 P_{\text{uni}}(w_3)$$

where $\lambda_1 + \lambda_2 + \lambda_3 = 1$ (weights determined experimentally)

8. Evaluation: Perplexity & Cross-Entropy

How do we know if our language model is good? We need metrics to evaluate performance.

Train/Test Split

🔑 Key Principle

Never evaluate on the same data you trained on!

  • Training Set (80%): Build the model (count tokens, n-grams)
  • Test Set (20%): Evaluate how well it predicts new text
Python Split Data into Train/Test

Cross-Entropy

Cross-entropy measures the average "surprise" of the model when seeing new text. It's the negative average log probability:

Cross-Entropy
$$H = -\frac{1}{N} \sum_{i=1}^{N} \log_2 P(w_i)$$

where $N$ is the total number of tokens in the test set.

Interpretation: Cross-entropy tells us the average number of bits needed to encode each word given our model.

Perplexity

Perplexity is just $2^{\text{cross-entropy}}$ — it's easier to interpret!

Perplexity
$$\text{Perplexity} = 2^H = 2^{-\frac{1}{N}\sum_{i=1}^{N} \log_2 P(w_i)}$$

Also written as:

$$\text{Perplexity} = \sqrt[N]{\frac{1}{\prod_{i=1}^{N} P(w_i)}}$$
Understanding Perplexity

Perplexity of X means the model is as confused as if it had to choose uniformly from X possible words at each step.

  • Perplexity = 1: Perfect model (always predicts correctly)
  • Perplexity = 100: As confused as choosing from 100 words
  • Perplexity = 10000: Very confused model (random over vocabulary)

Lower is better!

    for token in sequence:
        token_prob = unigram_token_prob_ksmooth(token, unigram_counts,

        log_prob += math.log2(token_prob)

# Evaluate on TRAINING set

test_logprob = unigram_sequence_logprob_ksmooth(

print(f"\nTest Cross-Entropy: {test_cross_entropy:.4f}")
print(f"Test Perplexity: {test_perplexity:.2f}")

📊 Key Points for Evaluation

  1. Always use smoothing when evaluating on test data
  2. Test perplexity > Training perplexity is normal and expected
  3. Compare models: Use perplexity to choose between different models
  4. Context helps: Bigram/trigram models usually have lower perplexity than unigram

9. Information Theory Connection

Perplexity and cross-entropy come from Information Theory, pioneered by Claude Shannon. Let's build intuition with "20 Questions"!

Playing 20 Questions

If someone can only think of 2 things (elephant or giraffe) with equal probability, how many yes/no questions do you need?

        ┌─────────────────┐
        │ Elephant (50%)  │
        │ or              │
        │ Giraffe (50%)   │
        └─────────────────┘
                │
                ▼
          1 question!
      "Is it an elephant?"

If they can think of 8 things with equal probability?

   ┌────┬────┬────┬────┬────┬────┬────┬────┐
   │ ✈️  │ 🚁  │ 🦉  │ 🪰  │ 🦢  │ 🚜  │ 🐕  │ 🐋  │
   └────┴────┴────┴────┴────┴────┴────┴────┘
                      │
                      ▼
               3 questions!
     "Does it fly?" "Is it man-made?" "Is it X?"

Entropy

Entropy measures the average uncertainty (in bits) of a probability distribution:

Entropy Formula
$$H(P) = -\sum_x P(x) \log_2 P(x)$$

Intuition: It's the expected number of yes/no questions (bits) needed to identify an outcome.

Entropy Examples

Uniform over 2 items: $H = -2 \times 0.5 \times \log_2(0.5) = 1$ bit

Uniform over 8 items: $H = -8 \times 0.125 \times \log_2(0.125) = 3$ bits

Non-uniform (peaked): Lower entropy! Easier to predict.

Cross-Entropy: Mismatched Models

What if you use a wrong model to encode data from the true distribution?

Cross-Entropy
$$H(P, Q) = -\sum_x P(x) \log_2 Q(x)$$

where $P$ is the true distribution and $Q$ is your model's distribution.

Interpretation: The average bits needed if you use model $Q$ to encode data from $P$. Always ≥ entropy of $P$.

🎯 The Connection

When we evaluate our language model on test data:

  • $P$ = the "true" distribution of language (approximated by test data)
  • $Q$ = our language model's probability distribution
  • Cross-entropy = how well our model "compresses" real language
  • Perplexity = $2^{\text{cross-entropy}}$ = interpretable measure

Formula Quick Reference

All Key Formulas

Unigram Probability:

$$P(w) = \frac{\text{count}(w)}{N}$$

Bigram Probability:

$$P(w_i | w_{i-1}) = \frac{\text{count}(w_{i-1}, w_i)}{\text{count}(w_{i-1})}$$

Trigram Probability:

$$P(w_i | w_{i-2}, w_{i-1}) = \frac{\text{count}(w_{i-2}, w_{i-1}, w_i)}{\text{count}(w_{i-2}, w_{i-1})}$$

Add-K Smoothing:

$$P_{\text{smooth}}(w) = \frac{\text{count}(w) + k}{N + k|V|}$$

Cross-Entropy:

$$H = -\frac{1}{N}\sum_{i=1}^{N} \log_2 P(w_i)$$

Perplexity:

$$\text{Perplexity} = 2^H$$
