Tokenization: Breaking Text into Pieces

Tokenization is the process of splitting text into smaller units called tokens. These tokens can be words, characters, or even sentences.

Why Tokenize?

Before we can analyze text computationally, we need to break it down into manageable pieces. Think of it like breaking a sentence into individual LEGO blocks that we can count, compare, and analyze.

Try It Yourself: Tokenization Demo

Example: Processing "IRN BRU is made in Scotland from girders"

1 Original Text: "IRN BRU is made in Scotland from girders"
2 Whitespace Split: ["IRN", "BRU", "is", "made", "in", "Scotland", "from", "girders"]
3 Lowercase: ["irn", "bru", "is", "made", "in", "scotland", "from", "girders"]
4 Remove Stopwords: ["irn", "bru", "made", "scotland", "girders"]
Common Tokenization Methods:
  • Whitespace: Split on spaces, tabs, newlines
  • Punctuation-based: Remove or split on punctuation marks
  • Rule-based: Use complex rules (e.g., spaCy tokenizer)
  • Subword: Break words into smaller units (e.g., WordPiece, BPE)

Regular Expressions (Regex) for Text Processing

Regular Expressions (regex) are powerful patterns used to match, search, and manipulate text. They are essential tools for tokenization and text processing in NLP.

Why Use Regex?

  • Flexible and powerful pattern matching
  • Find and extract specific patterns (emails, URLs, phone numbers)
  • Clean and normalize text data
  • Advanced tokenization beyond simple whitespace splitting

Basic Regex Syntax

Pattern Description Example Matches
. Any single character c.t cat, cot, cut
* 0 or more repetitions ca*t ct, cat, caat
+ 1 or more repetitions ca+t cat, caat (not ct)
? 0 or 1 occurrence colou?r color, colour
^ Start of string ^The The (at start)
$ End of string end$ end (at end)
[abc] Character class [aeiou] Any vowel
[^abc] Negated class [^0-9] Non-digit
\d Digit [0-9] \d{3} 123, 456
\w Word character [a-zA-Z0-9_] \w+ word123
\s Whitespace \s+ spaces, tabs
\b Word boundary \bcat\b cat (whole word)
(abc) Capturing group (the|a) the or a

Regex-Based Tokenizer

Regex in Python: 
import re

# Find all words
text = "Hello, world! 123"
words = re.findall(r'\w+', text)
# Result: ['Hello', 'world', '123']

# Find all email addresses
emails = re.findall(r'\b[\w.-]+@[\w.-]+\.\w+\b', text)

# Split on whitespace
tokens = re.split(r'\s+', text)

# Replace patterns
cleaned = re.sub(r'[^\w\s]', '', text)  # Remove punctuation

Regex for Advanced Tokenization

Unlike simple split(), regex allows you to:

  • Keep punctuation attached: Match "can't" as one token
  • Handle contractions: Recognize "don't", "I'm", "you'll"
  • Extract entities: Find all dates, numbers, or special patterns
  • Context-aware splitting: Split on spaces but not within quotes
Example: Contractions
Pattern: \w+'\w+|\w+
Text: "I can't believe it's working!"
Tokens: ["I", "can't", "believe", "it's", "working"]
Practice Tips:
  • Start simple and build complexity gradually
  • Use online regex testers (regex101.com, regexr.com) for testing
  • Test your patterns on diverse examples
  • Remember: regex can be slow on very large texts
  • For production NLP, consider using spaCy or NLTK tokenizers

Text Preprocessing

Stopwords Removal

Stopwords are common words (like "the", "is", "in") that often don't carry significant meaning for analysis. Removing them can help focus on important content.

Example Stopwords: a, an, the, is, are, in, on, at, to, from, with, by, for, of, etc.

Stemming

Stemming reduces words to their root form by removing suffixes. It's fast but can be imprecise.

fishing → fish
fished → fish
fisher → fish
Porter Stemmer Rules (Examples):
  • SSES → SS (caresses → caress)
  • IES → I (ponies → poni)
  • SS → SS (caress → caress)
  • S → (cats → cat)

Lemmatization

Lemmatization uses vocabulary and morphological analysis to return the base or dictionary form (lemma) of a word. It's more accurate but slower than stemming.

better → good
running → run
was → be

N-grams: Capturing Context

An N-gram is a contiguous sequence of N items (characters or words) from a text.

Types of N-grams:

  • Unigram (1-gram): Single tokens
  • Bigram (2-gram): Pairs of consecutive tokens
  • Trigram (3-gram): Triples of consecutive tokens

Example: "The quick brown fox"

1 Word Unigrams: [the, quick, brown, fox]
2 Word Bigrams: [the quick, quick brown, brown fox]
3 Word Trigrams: [the quick brown, quick brown fox]

N-gram Generator

Why use N-grams? N-grams capture context and word relationships. They're useful for:
  • Language modeling and text generation
  • Spelling correction
  • Text classification
  • Finding similar documents

Set Operations for Text Analysis

Once we have tokens, we can treat documents as sets of unique tokens and apply set theory operations.

Basic Set Operations:

Operation Symbol Description Example
Union X ∪ Y All elements in either set {1,2} ∪ {2,3} = {1,2,3}
Intersection X ∩ Y Elements in both sets {1,2} ∩ {2,3} = {2}
Difference X \ Y Elements in X but not in Y {1,2} \ {2,3} = {1}
Cardinality |X| Number of elements |{1,2,3}| = 3

Document Similarity Measures

How do we measure how similar two documents are? We can use set-based similarity measures!

1. Overlap Coefficient

Measures the overlap between two sets relative to the smaller set.

Overlap(X, Y) = |X ∩ Y| / min(|X|, |Y|)
Interpretation:
  • Range: 0 to 1
  • 1 = Perfect overlap (smaller set is subset of larger)
  • 0 = No common elements
  • Good for comparing documents of different lengths

Example Calculation:

Set A: {irn, bru, made, scotland, girders} → |A| = 5

Set B: {irn, bru, scottish, soft, drink} → |B| = 5

A ∩ B: {irn, bru} → |A ∩ B| = 2

Overlap: 2 / min(5, 5) = 2/5 = 0.4 or 40%

2. Jaccard Similarity

Measures the size of intersection relative to the size of union.

Jaccard(X, Y) = |X ∩ Y| / |X ∪ Y|
Interpretation:
  • Range: 0 to 1
  • 1 = Identical sets
  • 0 = No common elements
  • Most commonly used similarity measure

Example Calculation:

Set A: {irn, bru, made, scotland, girders} → |A| = 5

Set B: {irn, bru, scottish, soft, drink} → |B| = 5

A ∩ B: {irn, bru} → |A ∩ B| = 2

A ∪ B: {irn, bru, made, scotland, girders, scottish, soft, drink} → |A ∪ B| = 8

Jaccard: 2 / 8 = 0.25 or 25%
