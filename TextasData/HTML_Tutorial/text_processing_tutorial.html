2. Python Basics Review

2.1 Lists and Iteration

Lists are ordered collections of items. We'll use them extensively to store tokens (words).

# Creating a list
words = ["hello", "world", "python"]

# Iterating through a list
for word in words:
    print(word)

# List comprehension (compact way to create lists)
uppercase_words = [word.upper() for word in words]
# Result: ['HELLO', 'WORLD', 'PYTHON']

2.2 Sets - Finding Unique Items

Sets store unique items and are perfect for finding overlaps between documents.

# Creating sets
doc1_words = {"cat", "dog", "bird"}
doc2_words = {"dog", "fish", "bird"}

# Intersection (common words)
common = doc1_words & doc2_words
# Result: {'dog', 'bird'}

# Union (all unique words)
all_words = doc1_words | doc2_words
# Result: {'cat', 'dog', 'bird', 'fish'}

# Difference (words only in doc1)
unique_to_doc1 = doc1_words - doc2_words
# Result: {'cat'}

2.3 Counter - Counting Items

The Counter class helps us count how many times each item appears.

from collections import Counter

words = ["apple", "banana", "apple", "cherry", "apple"]
word_counts = Counter(words)

print(word_counts)
# Output: Counter({'apple': 3, 'banana': 1, 'cherry': 1})

# Find most common word
most_common = word_counts.most_common(1)
# Result: [('apple', 3)]

3. Regular Expressions: Pattern Matching

Regular expressions (regex) are powerful tools for finding patterns in text. Think of them as "super find" - you can search for complex patterns, not just exact matches.

3.1 Basic Patterns

Pattern Meaning Example
. Any single character c.t matches "cat", "cot", "c9t"
? Previous character is optional colou?r matches "color" and "colour"
* Zero or more of previous ab*c matches "ac", "abc", "abbc"
+ One or more of previous ab+c matches "abc", "abbc" (not "ac")
\d Any digit (0-9) \d\d\d\d matches "2024", "1999"
\w Any word character (letter, digit, _) \w+ matches whole words
\s Whitespace (space, tab, newline) \s+ matches any amount of whitespace

3.2 Practical Examples

Example 1: Finding Years

Pattern: 20\d\d finds years from 2000-2099

import re

text = "The years 2020 and 2021 were difficult."
pattern = re.compile(r'20\d\d')
years = pattern.findall(text)
print(years)
# Output: ['2020', '2021']

Example 2: Flexible Text Matching

Pattern: irn.?bru finds "Irn-Bru", "Irn Bru", "IrnBru"

import re

text = "I love Irn-Bru, IRN BRU, and Irn Bru!"
pattern = re.compile(r'irn.?bru', re.I)  # re.I = case insensitive
matches = pattern.findall(text)
print(matches)
# Output: ['Irn-Bru', 'IRN BRU', 'Irn Bru']
Common Pitfall: Remember to use raw strings (r'pattern') for regex patterns in Python. This prevents backslashes from being interpreted as escape sequences.

4. Tokenization: Breaking Text into Pieces

Tokenization is the process of splitting text into smaller units called tokens (usually words). This is the first step in almost all text processing tasks.

Tokenization Visual Example

Input: "Hello, World! How are you?"

Output: ["Hello", ",", "World", "!", "How", "are", "you", "?"]

4.1 Simple Tokenization: Split by Spaces

def tokenize_simple(text):
    return text.split()  # Split on whitespace

text = "Hello world from Python"
tokens = tokenize_simple(text)
print(tokens)
# Output: ['Hello', 'world', 'from', 'Python']
Problem: This doesn't handle punctuation well!
"Hello, world!" becomes ["Hello,", "world!"] - punctuation is stuck to words.

4.2 Advanced Tokenization: Handling Punctuation

We need rules to separate punctuation from words:

import string

def tokenize_advanced(text):
    tokens = []
    current_token = ""
    prev_type = None

    for char in text:
        if char in string.whitespace:
            if current_token:
                tokens.append(current_token)
                current_token = ""
            prev_type = 'space'
        else:
            # Determine character type
            if char in string.ascii_letters:
                char_type = 'letter'
            elif char in string.digits:
                char_type = 'digit'
            else:
                char_type = 'punctuation'

            # Start new token if type changed
            if prev_type and prev_type != char_type:
                tokens.append(current_token)
                current_token = char
            else:
                current_token += char

            prev_type = char_type

    if current_token:
        tokens.append(current_token)

    return tokens

text = "Hello, world! It's 2024."
tokens = tokenize_advanced(text)
print(tokens)
# Output: ['Hello', ',', 'world', '!', 'It', "'", 's', '2024', '.']

4.3 Professional Tokenization with spaCy

In practice, we use libraries like spaCy that have sophisticated tokenization rules built-in: 

import spacy

# Load English language model
nlp = spacy.load("en_core_web_sm")

text = "Hello, world! It's 2024."
doc = nlp(text)

tokens = [token.text for token in doc]
print(tokens)
# Output: ['Hello', ',', 'world', '!', 'It', "'s", '2024', '.']
Why use spaCy? It handles complex cases like contractions ("don't" → "do", "n't"), hyphenated words, URLs, emails, and more. It's battle-tested on millions of documents!

5. Stemming & Lemmatization: Standardizing Words

Different forms of the same word should often be treated as the same thing. For example: "running", "runs", "ran" all refer to the concept of "run".

5.1 What is Stemming?

Stemming removes word endings to get the "stem" (base form).

Examples:
  • "running" → "run"
  • "happily" → "happi"
  • "studies" → "studi"
Note: The stem doesn't have to be a real word!

Stemming Example

running
→
Remove "ing"
→
run

5.2 Simple Stemming Rules

def simple_stem(word):
    # Rule 1: Remove 'ing'
    if word.endswith('ing'):
        return word[:-3]

    # Rule 2: Remove 'ed'
    elif word.endswith('ed'):
        return word[:-2]

    # Rule 3: Remove 's' (but not 'is')
    elif word.endswith('s') and word != 'is':
        return word[:-1]

    return word

words = ["running", "jumped", "cats", "is"]
stemmed = [simple_stem(w) for w in words]
print(stemmed)
# Output: ['runn', 'jump', 'cat', 'is']

5.3 Lemmatization (Better than Stemming!)

Lemmatization is smarter - it considers the context and returns actual dictionary words (lemmas).

Examples:
  • "better" → "good" (not "bett")
  • "running" → "run"
  • "mice" → "mouse"
import spacy

nlp = spacy.load("en_core_web_sm")

text = "The mice were running faster than better athletes"
doc = nlp(text)

for token in doc:
    print(f"{token.text:15} → {token.lemma_}")

# Output:
# The             → the
# mice            → mouse
# were            → be
# running         → run
# faster          → fast
# than            → than
# better          → good
# athletes        → athlete

6.1 What are Stopwords?

Stopwords are common words that don't carry much meaning: "the", "is", "and", "a", etc.

We usually remove them because:
  • They appear in almost every document
  • They don't help distinguish between documents
  • Removing them speeds up processing

6.2 Building the Complete Pipeline

import spacy

nlp = spacy.load("en_core_web_sm")

def process_text(text):
    """
    Complete text processing pipeline
    Returns: list of clean, processed tokens
    """
    # Step 1: Parse text with spaCy
    doc = nlp(text)

    # Step 2: Process each token
    clean_tokens = []
    for token in doc:
        # Skip if:
        if token.is_stop:           # It's a stopword
            continue
        if token.is_punct:          # It's punctuation
            continue
        if token.is_space:          # It's whitespace
            continue

        # Get lemma and convert to lowercase
        clean_token = token.lemma_.lower()
        clean_tokens.append(clean_token)

    return clean_tokens

# Example
text = "The quick brown foxes are jumping over the lazy dogs!"
processed = process_text(text)
print(processed)
# Output: ['quick', 'brown', 'fox', 'jump', 'lazy', 'dog']

Let's See Each Step:

Stage Result
Original "The quick brown foxes are jumping over the lazy dogs!"
Tokenized ["The", "quick", "brown", "foxes", "are", "jumping", "over", "the", "lazy", "dogs", "!"]
Lowercased ["the", "quick", "brown", "foxes", "are", "jumping", "over", "the", "lazy", "dogs", "!"]
Lemmatized ["the", "quick", "brown", "fox", "be", "jump", "over", "the", "lazy", "dog", "!"]
Final (no stopwords/punct) ["quick", "brown", "fox", "jump", "lazy", "dog"]

7. Document Similarity: Finding Similar Texts

Now that we can process text, how do we measure if two documents are similar? The key idea: similar documents share many words.

7.1 Token Overlap

The simplest measure: count how many words two documents have in common.

Example:

Document 1:
"I love coffee and tea"
Tokens: {love, coffee, tea}
Document 2:
"I prefer tea and hot coffee"
Tokens: {prefer, tea, coffee, hot}

Overlap: {coffee, tea} = 2 words in common 

def count_overlap(tokens1, tokens2):
    # Convert to sets to find intersection
    set1 = set(tokens1)
    set2 = set(tokens2)
    overlap = set1 & set2  # Intersection
    return len(overlap)

doc1 = ["love", "coffee", "tea"]
doc2 = ["prefer", "tea", "coffee", "hot"]

overlap = count_overlap(doc1, doc2)
print(f"Overlap: {overlap} words")
# Output: Overlap: 2 words

7.2 Jaccard Similarity (Better!)

Problem with simple overlap: Longer documents naturally have more overlaps!

Solution: Jaccard Similarity - normalize by the total number of unique words.

Jaccard Similarity Formula

Jaccard(A, B) = |A ∩ B| ÷ |A ∪ B|

|A ∩ B| = Number of words in both documents
|A ∪ B| = Number of unique words across both documents

Calculating Jaccard Similarity:

Doc 1 tokens: {love, coffee, tea}
Doc 2 tokens: {prefer, tea, coffee, hot}

Intersection (∩): {coffee, tea} → 2 words
Union (∪): {love, coffee, tea, prefer, hot} → 5 words

Jaccard = 2 ÷ 5 = 0.4 (40% similar) 

def jaccard_similarity(tokens1, tokens2):
    """
    Calculate Jaccard similarity between two token lists
    Returns: float between 0 (no similarity) and 1 (identical)
    """
    set1 = set(tokens1)
    set2 = set(tokens2)

    # Intersection: words in both
    intersection = set1 & set2

    # Union: all unique words
    union = set1 | set2

    # Avoid division by zero
    if len(union) == 0:
        return 0.0

    return len(intersection) / len(union)

# Example
doc1 = ["love", "coffee", "tea"]
doc2 = ["prefer", "tea", "coffee", "hot"]

similarity = jaccard_similarity(doc1, doc2)
print(f"Similarity: {similarity:.2f}")
# Output: Similarity: 0.40

7.3 Complete Example: Finding Similar Reddit Posts

import spacy

nlp = spacy.load("en_core_web_sm")

# Sample Reddit posts
posts = [
    "I love drinking Irn Bru. Best Scottish soft drink!",
    "Playing Nintendo Switch all weekend. Love the new Zelda game!",
    "Irn Bru tastes amazing. Great orange soda from Scotland.",
    "Coffee is my favorite morning beverage."
]

# Process all posts
processed_posts = [process_text(post) for post in posts]

# Find posts similar to the first one
target = processed_posts[0]
print(f"Target post: {posts[0]}")
print("\nSimilarity scores:")

for i, post in enumerate(processed_posts):
    similarity = jaccard_similarity(target, post)
    print(f"Post {i}: {similarity:.2f} - {posts[i][:50]}...")

# Expected Output:
# Target post: I love drinking Irn Bru. Best Scottish soft drink!
#
# Similarity scores:
# Post 0: 1.00 - I love drinking Irn Bru. Best Scottish soft drink!
# Post 1: 0.07 - Playing Nintendo Switch all weekend. Love the new Z...
# Post 2: 0.45 - Irn Bru tastes amazing. Great orange soda from Scot...
# Post 3: 0.00 - Coffee is my favorite morning beverage.
Analysis: Post 2 (0.45 similarity) is correctly identified as most similar because it shares key terms: "irn", "bru", "scottish", "drink". The system works!
