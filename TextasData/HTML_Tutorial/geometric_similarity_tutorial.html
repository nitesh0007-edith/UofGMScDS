Review: Set-based Similarity

Previously, we used sets to represent documents. Let's review the limitations:

Jaccard Similarity
Jaccard(X, Y) = |X ∩ Y| / |X ∪ Y|
Example: X={"bru","i","irn","like"}, Y={"bad","bru","irn","is"} → Jaccard = 2/6 = 1/3
⚠️ Limitations of Set-based Methods
  • No importance weighting: All words treated equally
  • No frequency info: Word repetition ignored
  • Common words dominate: "the", "is" appear everywhere

From Sets to Vectors

We represent documents as vectors in a high-dimensional space where each dimension corresponds to a term in our vocabulary.

Building the Vocabulary

Collect all unique tokens and assign each an integer ID:

IDToken
0a
1bad
2bru
3i
4irn
5like
Python
def make_vocabulary(corpus):
    """Create vocabulary mapping from corpus."""
    vocab = {}
    for doc in corpus:
        for token in doc:
            if token not in vocab:
                vocab[token] = len(vocab)
    return vocab

# Example:
corpus = [['irn', 'bru', 'good'], ['irn', 'bru', 'bad']]
vocab = make_vocabulary(corpus)
# {'irn': 0, 'bru': 1, 'good': 2, 'bad': 3}

One-Hot Encoding

Create a vector where tokens present = 1, others = 0:

Document: {"i", "like", "irn", "bru"}

Token: a bad bru i irn like
Value: 0 0 1 1 1 1

Sparse vs Dense Representations

With vocabulary of 150,000+ tokens, most values are 0. Storing zeros wastes memory!

❌ Dense

[0, 0, 1, 1, 1, 0, 1, 0, ...]

~600KB per document!

✅ Sparse

{2: 1, 3: 1, 4: 1, 6: 1}

Only ~8 bytes per token!

Python - Sparse Functions
def make_onehot_sparse(tokens, vocab):
    """Create sparse one-hot vector (dict format)."""
    vector = {}
    for token in tokens:
        if token in vocab:
            vector[vocab[token]] = 1
    return vector

def sparse_to_dense(sv, length):
    """Convert sparse to dense."""
    dense = [0] * length
    for idx, val in sv.items():
        dense[idx] = val
    return dense
03

Term Frequency (TF)

One-hot encoding loses frequency info. Count how many times each term appears:

Document: "i like irn bru, irn bru taste like"

Token: bru i irn like taste
TF: 2 1 2 2 1
Python
from collections import Counter

def make_tf_sparse(tokens, vocab):
    """Create term frequency sparse vector."""
    counts = Counter(tokens)
    vector = {}
    for token, count in counts.items():
        if token in vocab:
            vector[vocab[token]] = count
    return vector
⚠️ Problem with Raw TF

Is a doc with 10 mentions of "Djokovic" really 10× more about him than one with 1 mention? Aboutness doesn't scale linearly!

Sublinear TF Scaling
TF_log = 1 + log₁₀(TF) if TF > 0, else 0
Example: TF=1→1, TF=10→2, TF=1000→4

Inverse Document Frequency (IDF)

Some words are common everywhere ("the", "is"). We want rare terms to have higher weight!

IDF Formula
IDF(t) = log₁₀(N / DF_t)
N = total docs, DF_t = docs containing term t
TermDF (in 1M docs)IDFMeaning
calpurnia16.0Very informative
animal1005.0Informative
fly10,0003.0Common
the1,000,0001.0Useless
Python
def doc_frequency(corpus):
    """Calculate document frequency for each token."""
    df = Counter()
    for doc in corpus:
        for token in set(doc):  # Count once per doc
            df[token] += 1
    return df

TF-IDF: Putting It Together

TF-IDF Formula
TF-IDF(t, d) = TF(t, d) × IDF(t)
With sublinear scaling: (1 + log₁₀(tf)) × log₁₀(N/df)

✅ High TF-IDF

  • Term frequent in THIS doc
  • Term rare in corpus

❌ Low TF-IDF

  • Term rare in this doc
  • Term common everywhere
Python - TF-IDF
import math

def make_tfidf_sparse(tokens, vocab, doc_freq, N):
    """Create TF-IDF sparse vector."""
    counts = Counter(tokens)
    vector = {}
    for token, count in counts.items():
        if token in vocab:
            tf = 1 + math.log10(count)  # Sublinear TF
            df = doc_freq.get(token, 1)
            idf = math.log10(N / df)
            vector[vocab[token]] = tf * idf
    return vector
04

Zipf's Law

Zipf's Law: Word frequency is inversely proportional to rank. A few words are very common; most are rare.

Zipf's Law: Linear relationship on log-log scale
📊 Why Zipf's Law Matters for TF-IDF

Using logarithms normalizes extreme differences between common and rare words. Without log scaling, a few words would dominate all calculations!

05

Geometric Similarity

With documents as vectors, we measure similarity by comparing vectors in high-dimensional space.

Documents as vectors - direction matters more than length!
⚠️ Problem with Euclidean Distance

d₂ and d₃ point in the same direction (both negative reviews) but have different Euclidean distances due to length. Longer docs shouldn't be "more different"!

Cosine Similarity

Measure the angle between vectors, not distance. Same direction = similar!

Cosine Similarity
cos(θ) = (v · w) / (||v|| × ||w||)
v·w = dot product, ||v|| = magnitude

θ = 0°

cos = 1
Same direction

θ = 45°

cos ≈ 0.7
Somewhat similar

θ = 90°

cos = 0
Perpendicular

Python - Cosine Similarity
import math

def sparse_dot_prod(sv1, sv2):
    """Dot product of sparse vectors."""
    result = 0
    for idx, val in sv1.items():
        if idx in sv2:
            result += val * sv2[idx]
    return result

def sparse_cosine_similarity(sv1, sv2):
    """Cosine similarity between sparse vectors."""
    dot = sparse_dot_prod(sv1, sv2)
    mag1 = math.sqrt(sum(v**2 for v in sv1.values()))
    mag2 = math.sqrt(sum(v**2 for v in sv2.values()))
    if mag1 == 0 or mag2 == 0:
        return 0
    return dot / (mag1 * mag2)

def normalize_sparse_vector(sv):
    """Normalize to unit length."""
    mag = math.sqrt(sum(v**2 for v in sv.values()))
    if mag == 0: return {}
    return {idx: val/mag for idx, val in sv.items()}
💡 Performance Tip

Pre-normalize all vectors! Then cosine similarity = just a dot product (much faster).

K-Means Clustering

Clustering groups similar documents without predefined categories. K-Means is the most popular algorithm.

📰 News Grouping

Group articles about same topic

👥 Customer Segments

Group by purchasing patterns

🔍 Search Results

Organize into categories

K-Means Algorithm

Initialize

Pick K random centroids

Assign

Each point to nearest centroid

Update

Recalculate centroids as mean of assigned points

Repeat

Until convergence (centroids stop moving)

✅ Advantages

  • Fast: O(N×K) complexity
  • Simple to implement
  • Scales to large datasets

❌ Limitations

  • Must choose K upfront
  • Sensitive to initialization
  • May find local minimum

Choosing K: Elbow Method

Plot distortion vs K, look for the "elbow" where adding clusters provides diminishing returns.

Silhouette Coefficient

s(i) = (b(i) - a(i)) / max(a(i), b(i))
a = avg dist to same cluster, b = avg dist to nearest other cluster
Range: -1 (wrong cluster) to +1 (perfect fit)

Python - Complete Workflow
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score
import spacy

# Load spaCy
nlp = spacy.load("en_core_web_sm")

def text_pipeline_spacy(text):
    """Tokenize with spaCy, remove stopwords/punct."""
    tokens = []
    doc = nlp(text)
    for t in doc:
        if not t.is_stop and not t.is_punct and not t.is_space:
            tokens.append(t.lemma_.lower())
    return tokens

# Vectorize documents
documents = [
    "My favourite soft drink is Irn Bru.",
    "Irn Bru is a Scottish soft drink.",
    "I love playing football on weekends.",
    "The weather in Glasgow is often rainy.",
]

vectorizer = TfidfVectorizer(
    tokenizer=text_pipeline_spacy,
    token_pattern=None
)
tfidf_matrix = vectorizer.fit_transform(documents)

# Find similar documents
query_vec = tfidf_matrix[0]
similarities = tfidf_matrix.dot(query_vec.T).toarray().flatten()
print("Similarities to doc 0:", similarities)

# K-Means clustering
kmeans = KMeans(n_clusters=2, random_state=42, n_init=10)
labels = kmeans.fit_predict(tfidf_matrix)
score = silhouette_score(tfidf_matrix, labels)
print(f"Cluster labels: {labels}")
print(f"Silhouette score: {score:.3f}")

import math
from collections import Counter

# === VOCABULARY ===
def make_vocabulary(corpus):
    vocab = {}
    for doc in corpus:
        for token in doc:
            if token not in vocab:
                vocab[token] = len(vocab)
    return vocab

# === VECTORS ===
def make_onehot_sparse(tokens, vocab):
    return {vocab[t]: 1 for t in tokens if t in vocab}

def make_tf_sparse(tokens, vocab):
    counts = Counter(tokens)
    return {vocab[t]: c for t, c in counts.items() if t in vocab}

# === TF-IDF ===
def doc_frequency(corpus):
    df = Counter()
    for doc in corpus:
        for token in set(doc):
            df[token] += 1
    return df

def make_tfidf_sparse(tokens, vocab, doc_freq, N):
    counts = Counter(tokens)
    vector = {}
    for token, count in counts.items():
        if token in vocab:
            tf = 1 + math.log10(count)
            idf = math.log10(N / doc_freq.get(token, 1))
            vector[vocab[token]] = tf * idf
    return vector

# === SIMILARITY ===
def sparse_euclidean_distance(sv1, sv2):
    indices = set(sv1.keys()) | set(sv2.keys())
    return math.sqrt(sum((sv1.get(i,0)-sv2.get(i,0))**2 for i in indices))

def sparse_dot_prod(sv1, sv2):
    return sum(v * sv2.get(i, 0) for i, v in sv1.items())

def normalize_sparse_vector(sv):
    mag = math.sqrt(sum(v**2 for v in sv.values()))
    return {i: v/mag for i, v in sv.items()} if mag > 0 else {}

def sparse_cosine_similarity(sv1, sv2):
    dot = sparse_dot_prod(sv1, sv2)
    mag1 = math.sqrt(sum(v**2 for v in sv1.values()))
    mag2 = math.sqrt(sum(v**2 for v in sv2.values()))
    return dot / (mag1 * mag2) if mag1 > 0 and mag2 > 0 else 0

📋 Summary

🎯 Vector Representations

  • Documents → high-dimensional vectors
  • Use sparse representation for efficiency
  • Bag of Words loses order, keeps counts

⚖️ TF-IDF Weighting

  • TF: term importance in document
  • IDF: term rarity in corpus
  • Use logarithms to dampen extremes

📐 Cosine Similarity

  • Measures angle, not distance
  • Pre-normalize for speed
  • Range: 0 (orthogonal) to 1 (identical)

🎨 K-Means Clustering

  • Unsupervised grouping
  • Use elbow/silhouette to choose K
  • Fast but requires K upfront
