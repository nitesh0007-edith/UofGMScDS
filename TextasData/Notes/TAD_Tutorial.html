Introduction, Tokens & Document Similarity

How computers read text: cleaning, splitting into tokens, and measuring how similar two documents are

1.1 The NLP Pipeline

Before a computer can understand text, it needs to process it step-by-step. Think of it like preparing ingredients before cooking:

Real-Life Analogy
Imagine you receive a messy handwritten recipe card. You would: (1) wipe off stains, (2) separate each word, (3) standardize abbreviations like "tbsp" → "tablespoon", (4) ignore common filler words like "the". That's exactly what the NLP pipeline does with text!
1
Data Cleaning: Remove non-relevant content — HTML tags, URLs, tables, boilerplate headers/footers.
Example: <p>Hello <b>world</b></p> → Hello world
2
Tokenization: Split text into individual units (tokens). Tokens can be words, subwords, or characters.
Example: "I can't wait!" → ["I", "ca", "n't", "wait", "!"]
3
Normalization: Reduce variation — convert everything to lowercase.
Example: "Apple" → "apple", so "Apple" and "apple" are treated the same
4
Stemming: Chop off word endings using rules (fast but crude).
Example: "running" → "run", "textiles" → "textil"
5
Lemmatization: Use a dictionary to find the base form (slower but more accurate).
Example: "better" → "good", "mice" → "mouse"
6
Stopword Removal: Remove high-frequency, low-information words.
Example: Remove "the", "is", "a", "of" — they appear everywhere and don't help distinguish documents
Exam Tip
When asked to justify preprocessing steps, always give a domain-specific example. E.g., for patent classification: case folding ensures "Textile" (start of sentence) and "textile" (mid-sentence) are treated identically; stemming ensures "textiles" and "textile" map to the same token.

1.2 Character Encodings

Computers store text as numbers. An encoding tells the computer which number represents which character.

EncodingBytes per CharacterCharacters SupportedUse Case
ASCII1 byte128 (English letters, digits, symbols)Simple English text
Unicode (UTF-8)1-4 bytes150,000+ (all world scripts)Multilingual text, emojis
Why This Matters
If your system uses ASCII and encounters Chinese text (你好) or emojis (😊), it will break! Unicode (UTF-8) is the modern standard that handles everything — English, Arabic, Japanese, and even emojis.

1.3 Tokenization Approaches

Tokenization means splitting text into meaningful pieces. There are three main approaches:

1.4 Set-Based Similarity Measures

Given two documents as sets of unique tokens X and Y, how similar are they?

Critical Point
These measures work on sets (unique elements only). If a word appears 5 times, it only counts once! Don't confuse types (unique tokens) with tokens (total occurrences).
MeasureFormulaRangeIntuition
Overlap Count$|X \cap Y|$$[0, \min(|X|,|Y|)]$How many words in common?
Overlap Coefficient$\frac{|X \cap Y|}{\min(|X|, |Y|)}$$[0, 1]$Overlap relative to smaller set
Sorensen-Dice$\frac{2|X \cap Y|}{|X| + |Y|}$$[0, 1]$Overlap relative to average size
Jaccard$\frac{|X \cap Y|}{|X \cup Y|}$$[0, 1]$Overlap relative to total unique words
Key Relationship
similarity(X, Y) is proportional to -distance(X, Y). More similar = less distance.
Real-Life Example: Comparing Shopping Lists
Alice's list: {milk, eggs, bread, butter, cheese}
Bob's list: {milk, eggs, bread, apples, oranges, grapes}

Intersection: {milk, eggs, bread} → $|X \cap Y| = 3$
Union: {milk, eggs, bread, butter, cheese, apples, oranges, grapes} → $|X \cup Y| = 8$
// Overlap Coefficient $|X \cap Y| / \min(|X|,|Y|) = 3 / \min(5,6) = 3/5 = $ 0.60 // Sorensen-Dice $2|X \cap Y| / (|X|+|Y|) = 2 \times 3 / (5+6) = 6/11 = $ 0.545 // Jaccard $|X \cap Y| / |X \cup Y| = 3 / 8 = $ 0.375
Exam Practice: Set Similarity (2025 Q1c style) 4 marks
Given DocX with tokens {A, C, G, H} and DocY with tokens {A, B, D, E, G, H, I, J}, compute the Overlap Coefficient and Jaccard Similarity.
Solution
Set(DocX) = {A, C, G, H} → |DocX| = 4 Set(DocY) = {A, B, D, E, G, H, I, J} → |DocY| = 8 Intersection = {A, G, H} → |Intersection| = 3 Union = {A, B, C, D, E, G, H, I, J} → |Union| = 9
Overlap Coefficient $\text{Overlap} = \frac{|X \cap Y|}{\min(|X|, |Y|)} = \frac{3}{\min(4, 8)} = \frac{3}{4} = \mathbf{0.75}$
Jaccard Similarity $\text{Jaccard} = \frac{|X \cap Y|}{|X \cup Y|} = \frac{3}{9} = \frac{1}{3} \approx \mathbf{0.333}$
Why the difference?
Overlap divides by the smaller set (generous), Jaccard divides by the union (strict). Overlap is high when one set is a subset of the other.

1.5 Properties of a Metric (Distance Function)

A distance function $d$ is a metric if it satisfies ALL four properties:

The Four Metric Properties 1. Identity: $d(X, X) = 0$  —  distance to yourself is zero
2. Positivity: $d(X, Y) > 0$ when $X \neq Y$  —  different things have positive distance
3. Symmetry: $d(X, Y) = d(Y, X)$  —  A-to-B same as B-to-A
4. Triangle Inequality: $d(X, Z) \leq d(X, Y) + d(Y, Z)$  —  shortcut can't be longer than detour
Real-Life Analogy: City Distances
Think of distances between cities:
• London → London = 0 km (identity) ✔
• London → Paris > 0 km (positivity) ✔
• London → Paris = Paris → London (symmetry) ✔
• London → Tokyo ≤ London → Paris + Paris → Tokyo (triangle inequality) ✔
Important Definitions
• A semi-metric violates triangle inequality.
• A pseudo-metric violates positivity (different objects can have distance 0).
Exam Tip (2022 Q2d)
Sorensen-Dice is a semi-metric — it breaks triangle inequality. To show this: find three sets A, B, C where $d(A,C) > d(A,B) + d(B,C)$. Remember to convert similarity to distance first: $d = 1 - \text{sim}$.

1.7 N-grams

An n-gram is a contiguous sequence of n items from text. Think of it as a sliding window.

Example: "the cat sat on the mat"
Unigrams (n=1): [the, cat, sat, on, the, mat]
Bigrams (n=2): [the cat, cat sat, sat on, on the, the mat]
Trigrams (n=3): [the cat sat, cat sat on, sat on the, on the mat]
Maximum Possible N-grams For alphabet size $A$ and text length $L$:
If $A^n \leq L - n + 1$: max = $A^n$ (all combinations possible)
If $A^n > L - n + 1$: max = $L - n + 1$ (limited by text length)

Geometric Similarity, Text Distributions & Clustering

Converting text to numbers (vectors), measuring similarity with angles/distances, and grouping similar documents

2.1 Term Frequency (TF) Variants

We need to represent documents as numbers. The simplest approach: count how often each word appears.

MethodFormulaWhen to Use
Raw TF$\text{tf}(t,d) = \text{count of } t \text{ in } d$Simplest, linear scaling
Log-transformed TF$1 + \log_{10}(\text{tf}(t,d))$ if tf > 0, else 0Dampens effect of very frequent words
Real-Life Example: News Articles
In an article about football, "goal" might appear 20 times. Raw TF = 20. But does appearing 20 times vs 10 times make it twice as relevant? Probably not. Log TF dampens this: $1 + \log_{10}(20) = 2.3$ vs $1 + \log_{10}(10) = 2.0$.

2.2 IDF and TF-IDF

Some words are common across ALL documents (like "the"). We need to down-weight them. That's what IDF (Inverse Document Frequency) does.

IDF Formula $$\text{IDF}(t) = \log\left(\frac{N}{\text{df}(t)}\right)$$ Where $N$ = total number of documents, $\text{df}(t)$ = number of documents containing term $t$
TF-IDF Formula $$\text{TF-IDF}(t, d) = (1 + \log(\text{tf}(t,d))) \times \log\left(\frac{N}{\text{df}(t)}\right)$$ Or with linear TF: $\text{TF-IDF}(t,d) = \text{tf}(t,d) \times \text{IDF}(t)$
Real-Life Example: Google Search
Imagine 1,000 web pages. The word "the" appears in all 1,000 → IDF = log(1000/1000) = 0. It's useless for search!
The word "quantum" appears in only 5 → IDF = log(1000/5) = 2.3. Very distinctive!

TF-IDF rewards words that are frequent in a document BUT rare overall.
Zipf's Law
A few words are extremely frequent; most words are very rare. The frequency of a word is inversely proportional to its rank. Word #1 appears about twice as often as word #2, three times as often as word #3, etc.

To reconstruct text from TF-IDF: divide by IDF to get TF. But word order is lost — bag-of-words only tells you WHAT words appear and HOW MANY times, not the order.

2.3 Vector Similarity Measures

Once documents are vectors, we can measure distance/similarity geometrically.

MeasureFormulaKey Property
Euclidean Distance$d(\vec{x}, \vec{y}) = \sqrt{\sum_i (x_i - y_i)^2}$Sensitive to vector magnitude (length)
Cosine Similarity$\cos(\theta) = \frac{\vec{d_1} \cdot \vec{d_2}}{||\vec{d_1}|| \times ||\vec{d_2}||}$Magnitude-independent (measures angle)
Why Cosine is Better for Text
Imagine two documents about "machine learning". Document A is 100 words, Document B is 10,000 words (a textbook chapter). Their TF vectors have very different magnitudes. Euclidean distance would say they're far apart. But cosine similarity measures the angle between them — if they use the same words in the same proportions, the angle is small (high similarity), regardless of length!
Cosine Similarity Worked Example $\vec{A} = [1, 2, 3], \quad \vec{B} = [2, 4, 6]$

$\vec{A} \cdot \vec{B} = (1)(2) + (2)(4) + (3)(6) = 2 + 8 + 18 = 28$
$||\vec{A}|| = \sqrt{1+4+9} = \sqrt{14}$,   $||\vec{B}|| = \sqrt{4+16+36} = \sqrt{56}$
$\cos(\theta) = \frac{28}{\sqrt{14} \times \sqrt{56}} = \frac{28}{28} = \mathbf{1.0}$

Perfect similarity! B is just A scaled by 2 — same direction, different magnitude.

2.4 K-Means Clustering

Clustering groups similar documents together without labels. K-Means is the most common algorithm.

Real-Life Example: Email Organization
Imagine you have 10,000 emails and want to automatically organize them into groups. K-Means might discover clusters like: "work emails", "online shopping", "social media notifications", "newsletters" — all without you telling it what groups to look for!
0
Initialize: Pick k random points as initial centroids (cluster centers)
1
Assign: Each data point goes to its nearest centroid
2
Update: Recalculate each centroid as the mean of its assigned points
R
Repeat: Steps 1-2 until assignments stop changing (convergence)
Initialization MethodHow It Works
ForgyRandomly select k data points as initial centroids
Random PartitionRandomly assign each point to a cluster, then compute centroids
Choosing k: The Elbow Method
Plot total within-cluster distance vs. k. The "elbow" point (where improvement diminishes sharply) is a good choice for k. Too few clusters = things that are different get grouped. Too many = things that are similar get split.

2.5 Measuring Clustering Quality

Two Types of Evaluation
Intrinsic (no labels needed): How well-separated are clusters?
Extrinsic (needs labels): Do clusters match known categories?

Silhouette Score combines two ideas: cohesion (how close points are within a cluster) and separation (how far apart clusters are).

Silhouette Score $$s(i) = \begin{cases} 1 - \frac{\text{cohesion}(i)}{\text{separation}(i)} & \text{if cohesion} < \text{separation} \\ 0 & \text{if cohesion} = \text{separation} \\ \frac{\text{separation}(i)}{\text{cohesion}(i)} - 1 & \text{if cohesion} > \text{separation} \end{cases}$$
Range: $[-1, 1]$. Higher = better. Negative = point might be in wrong cluster.
Definitions
Cohesion(i): Average distance from point i to all OTHER points in the SAME cluster (lower = better)
Separation(i): Minimum average distance from point i to points in ANY OTHER cluster (higher = better)
Purity: (extrinsic) Fraction of each cluster that belongs to the majority class
Exam Practice: Silhouette Score 3 marks
Point X has cohesion = 2.0 and separation = 5.0. Calculate its silhouette score. Is this a good clustering for point X?
Solution
cohesion = 2.0, separation = 5.0 Since cohesion < separation: s(X) = 1 - cohesion/separation = 1 - 2.0/5.0 = 1 - 0.4 = 0.6 This is a good score! Point X is closer to its own cluster than to the nearest other cluster.

Language Modelling

Predicting the next word: probability foundations, n-gram models, smoothing, and text generation

3.1 Probability Foundations

Language modelling is about predicting what word comes next. We need some probability basics first.

Essential Probability Rules $0 \leq P(x) \leq 1$   and   $\sum P(x) = 1$

Bayes' Rule: $P(A|B) = \frac{P(B|A) \times P(A)}{P(B)}$

Chain Rule: $P(A,B,C) = P(A|B,C) \times P(B|C) \times P(C)$

Independence: A and B are independent iff $P(A|B) = P(A)$
Real-Life Example: Phone Autocomplete
When you type "I want to" on your phone and it suggests "eat", "go", "know" — that's a language model! It estimates $P(\text{next word} | \text{previous words})$ and shows the highest-probability options.

3.2 N-gram Language Models

The Markov assumption: the probability of a word depends only on the previous n-1 words.

ModelAssumptionFormulaExample
UnigramWords are independent$P(w_i) = \frac{\text{Count}(w_i)}{N}$P("the") = 200/1000 = 0.2
BigramDepends on previous word$P(w_i|w_{i-1}) = \frac{\text{Count}(w_{i-1}, w_i)}{\text{Count}(w_{i-1})}$P("cat"|"the") = Count("the cat")/Count("the")
TrigramDepends on previous 2 words$P(w_i|w_{i-2},w_{i-1}) = \frac{\text{Count}(w_{i-2},w_{i-1},w_i)}{\text{Count}(w_{i-2},w_{i-1})}$P("mat"|"the","cat") = ...
Real-Life Example: Text Prediction
Training text: "the cat sat on the mat the cat ate the fish"

Bigram counts after "the": cat×2, mat×1, fish×1 (total: 4)
$P(\text{cat}|\text{the}) = 2/4 = 0.5$
$P(\text{mat}|\text{the}) = 1/4 = 0.25$
$P(\text{fish}|\text{the}) = 1/4 = 0.25$

So after typing "the", the model predicts "cat" is most likely!

3.3 Smoothing

Problem: What if we encounter an n-gram not seen in training? Its count = 0 → probability = 0 → the ENTIRE sequence probability becomes 0!

The Zero-Probability Problem
If your model sees "the dog" 100 times but never "the elephant", it thinks $P(\text{elephant}|\text{the}) = 0$. That's too harsh — "the elephant" is perfectly valid English!

Add-k Smoothing

Add-k Smoothing Formula $$P_{\text{add-k}}(w | \text{context}) = \frac{\text{Count}(w, \text{context}) + k}{\sum_{w'} \text{Count}(w', \text{context}) + k \times |V|}$$ When $k=1$, this is called Laplace smoothing. $|V|$ = vocabulary size.

Interpolation

Interpolation Formula $$P_{\text{interp}}(w_i | w_{i-1}, w_{i-2}) = \lambda_1 \cdot P_{\text{tri}} + \lambda_2 \cdot P_{\text{bi}} + \lambda_3 \cdot P_{\text{uni}}$$ Where $\lambda_1 + \lambda_2 + \lambda_3 = 1$. Mix higher-order and lower-order models.

Jelinek-Mercer Smoothing

Jelinek-Mercer Formula $$P_{\text{JM}}(t | \text{doc}) = \lambda \cdot P(t | \text{doc}) + (1 - \lambda) \cdot P(t | \text{collection})$$ Mix document-specific probability with corpus-wide probability.

3.4 Language Model Evaluation

MetricFormulaInterpretation
Entropy$H = -\sum P(x) \cdot \log_2 P(x)$Uncertainty in distribution. Higher = more uncertain
Cross-Entropy$H = -\frac{1}{n} \sum \log_2 P(w_i)$How well model predicts test data. Lower = better
Perplexity$PP = 2^{\text{Cross-Entropy}}$Effective number of choices. Lower = better model
Intuition for Perplexity
Perplexity of 10 means the model is as confused as if it had to choose randomly between 10 equally likely words at each position. A good English language model might have perplexity around 50-100 (out of 50,000+ vocabulary words).
Evaluation Types
Intrinsic: Evaluate LM directly (perplexity)
Extrinsic: Evaluate LM as part of another task (e.g., machine translation quality)
Syntactic: Does generated text follow grammar rules?
Semantic: Does generated text convey actual meaning?

3.5 Text Generation Strategies

StrategyHow It WorksOutput Quality
GreedyAlways pick highest-probability next tokenDeterministic, boring, repetitive

Exam Practice: N-gram Generation (2024 Q2 style) 8 marks
Text: BCBBCBBABCCCBBCCBCCB (20 chars, alphabet: {A, B, C}). Generate 3 characters using: (a) greedy unigram, (b) trigram greedy starting from context "CB".
Solution

(a) Greedy Unigram:

// Count each character: B = 10, C = 8, A = 2. Total = 20 // Greedy always picks highest probability: P(B) = 10/20 = 0.5 ← highest P(C) = 8/20 = 0.4 P(A) = 2/20 = 0.1 // Generate: always pick B Result: BBB

(b) Trigram Greedy from "CB":

// Look at last 2 chars: CB. Find all trigrams starting with CB: CBB: 3 times, CBC: 1 time, CBA: 0 times Total: 4 // Pick highest: B (prob = 3/4) Generated: B. Context is now: BB // Last 2 chars: BB. Find all trigrams starting with BB: BBA: 1, BBB: 0, BBC: 1 → Tie A and C! // Break tie alphabetically: pick A Generated: BA. Context is now: BA // Last 2 chars: BA. Find trigrams starting with BA: BAB: 1 → pick B Result: BAB
Common Mistake
Don't forget: newly generated characters become part of the context for subsequent predictions! After generating "B" from "CB", the context becomes "BB" (not still "CB").

3.6 The Zero-Probability Problem

A bigram model cannot generate a sequence if any constituent bigram has zero count in the training data.

Solution
Apply add-k smoothing to give all unseen n-grams a small probability, then use sampling (not greedy) to allow selection of low-probability tokens.

Text Classification

Assigning categories to text: types, metrics, evaluation, and common pitfalls

4.1 Types of Classification

TypeDescriptionReal-Life Example
Binary2 classesEmail: Spam or Not Spam
Multi-class1 label from n mutually exclusive classesMovie review: Positive / Negative / Neutral
Multi-labelMultiple labels per instanceNews article: tagged as "Politics" AND "Economy" AND "UK"
Exam Tip (2025 Q2a)
If categories overlap (e.g., a recipe is both "Italian" AND "pasta"), it's multi-label, not multi-class. This distinction is frequently tested! Multi-class means exactly ONE label per item.
Real-Life Example: Netflix
Multi-class: "Is this movie rated G, PG, PG-13, or R?" (exactly one rating)
Multi-label: "What genres? Action, Comedy, Romance..." (a movie can be multiple genres)

4.2 Classification Metrics

Binary Confusion Matrix — the foundation of all classification metrics:

Predicted PositivePredicted Negative
Actual PositiveTP (True Positive)FN (False Negative)
Actual NegativeFP (False Positive)TN (True Negative)
Core Classification Metrics $$\text{Accuracy} = \frac{TP + TN}{TP + FP + TN + FN}$$ $$\text{Precision} = \frac{TP}{TP + FP} \quad \text{"Of predicted positives, how many correct?"}$$ $$\text{Recall} = \frac{TP}{TP + FN} \quad \text{"Of actual positives, how many found?"}$$ $$F_1 = \frac{2 \times P \times R}{P + R} = \frac{2TP}{2TP + FP + FN}$$ $$F_\beta = \frac{(1 + \beta^2) \times P \times R}{\beta^2 \times P + R}$$
Real-Life Example: Medical Testing
A COVID test:
• High Precision = When the test says "positive", it's almost always right (few false alarms)
• High Recall = The test catches almost all infected people (few missed cases)
• For a deadly disease, recall is more important — missing a case (FN) is worse than a false alarm (FP)
Key Rule
$F_1$ is the harmonic mean of Precision and Recall. It MUST always be between P and R. If $F_1 > \max(P, R)$, something is wrong! (2025 Q2c tested this)

Multi-class Averaging

MethodHow It WorksWhen to Use
MacroUnweighted average of per-class metricWhen all classes are equally important
WeightedWeighted by class sample countWhen class sizes vary
MicroSum TP/FP/TN/FN across all classes firstWhen you care about overall performance
Exam Practice: Confusion Matrix Metrics 6 marks
Given this confusion matrix for a spam classifier, compute Precision, Recall, and F1.
Pred: SpamPred: Not Spam
Actual: Spam40 (TP)10 (FN)
Actual: Not Spam5 (FP)145 (TN)
Solution
Precision = TP / (TP + FP) = 40 / (40 + 5) = 40/45 = 0.889 Recall = TP / (TP + FN) = 40 / (40 + 10) = 40/50 = 0.800 F1 = 2 * P * R / (P + R) = 2 * 0.889 * 0.800 / (0.889 + 0.800) = 1.422 / 1.689 = 0.842 // Sanity check: F1 (0.842) is between P (0.889) and R (0.800) ✔

4.3 ROC Curve and AUROC

The ROC curve plots True Positive Rate (recall) vs False Positive Rate at different thresholds.

Key Concepts
How to build ROC: Start bottom-left. Order predictions by confidence. For each: actual positive → move UP, actual negative → move RIGHT.

AUROC = Area Under the ROC Curve.
• 0.5 = random guessing (diagonal line)
• 1.0 = perfect classifier
• < 0.5 = worse than random (model might have labels flipped)

4.4 Data Splitting & Cross-Validation

ApproachHow It WorksWhen to Use
Train / Val / TestSplit into 3 parts. Train on train, tune on val, final eval on testLarge datasets (1000+ samples)
K-Fold CVSplit into k folds, rotate test fold k times, average resultsSmall datasets (< 1000 samples)
Never peek at test data!
The test set must ONLY be used for final evaluation. If you tune hyperparameters on it, your results are biased (test set leakage).

Exam Practice: Overfitting vs Underfitting (2022 Q3d style) 4 marks
System A: Train accuracy slightly better than baseline*, Test accuracy better than baseline*. System B: Train accuracy much better than baseline*, Test accuracy worse than baseline. (* = statistically significant) Diagnose each.
Solution
System A: Good generalization - Both train and test improve over baseline - Test improvement is statistically significant - Model has learned genuine patterns System B: Overfitting - Train performance is much better (memorized training data) - Test performance is worse (can't generalize) - Gap between train and test = classic overfitting sign

4.6 Anomaly Detection in Reported Metrics (2025 Q2c)

In exams, you may be asked to spot errors in a reported experimental setup. Common anomalies:

  • Perplexity reported for classification — wrong metric (it's for language models)
  • Different classes in train vs test confusion matrices
  • Accuracy doesn't match confusion matrix values
  • Class distribution mismatch between train and test
  • Validation set larger than training set
  • Perfect test with poor train — suspicious data leakage
  • $F_1$ not between Precision and Recall — mathematically impossible
  • Macro P/R/F1 for binary — just use regular P/R/F1

Contextual Word Embeddings

From static word vectors to transformers: Word2Vec, BERT, GPT, attention, BPE, and transfer learning

5.1 Static Word Embeddings

The idea: represent each word as a dense vector of numbers (e.g., 300 dimensions) based on the words it appears near.

Distributional Hypothesis
"You shall know a word by the company it keeps" — words that appear in similar contexts have similar meanings. "King" and "queen" appear near similar words, so they get similar vectors.

Word2Vec

  • Learns dense vectors via noise contrastive estimation
  • Negative samples help learning by contrasting true context vs random words
  • Famous example: $\vec{\text{king}} - \vec{\text{man}} + \vec{\text{woman}} \approx \vec{\text{queen}}$
The Polysemy Problem
Word2Vec produces ONE vector per word. But "bank" means a financial institution AND a river bank. "Python" is a snake AND a programming language. The single vector mixes both meanings, hurting downstream tasks. Static embeddings handle synonymy (similar words → similar vectors) but NOT polysemy (one word, multiple meanings).
Window Size Trade-off
Too large k: Contexts too broad and noisy, vectors inconsistent
Too small k: Not enough information to represent word meaning
Function word problem: Words like "the", "of" appear in ALL contexts, so $P(\text{context}|\text{word})$ is near-uniform — their vectors are unreliable.

5.2 Contextual Word Embeddings

The breakthrough: give each word a different vector depending on its context!

ModelArchitectureKey FeatureYear
ELMoBi-LSTMDifferent vector per context2018
BERTTransformer (Encoder)Bidirectional, masked LM2018
GPTTransformer (Decoder)Unidirectional, causal LM2018+
How Contextual Embeddings Solve Polysemy
Sentence 1: "I deposited money in the bank"
Sentence 2: "I sat by the river bank"

In Word2Vec, "bank" has ONE vector mixing both meanings.
In BERT, "bank" gets different vectors in each sentence because the surrounding words are different! The financial context pushes it toward money-related meanings; the river context pushes it toward nature-related meanings.

5.3 Self-Attention Mechanism

The core of transformers. Each word "attends" to every other word to decide what's important.

Self-Attention Formula $$Q = \text{Input} \times W_Q \quad \text{(Queries: "what am I looking for?")}$$ $$K = \text{Input} \times W_K \quad \text{(Keys: "what do I contain?")}$$ $$V = \text{Input} \times W_V \quad \text{(Values: "what information do I provide?")}$$ $$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{Q \times K^T}{\sqrt{d_k}}\right) \times V$$
Real-Life Analogy: Library Research
Imagine you're in a library:
• Query (Q): Your research question — "Tell me about climate change"
• Key (K): Each book's title/summary — helps you find relevant books
• Value (V): The actual content of each book — what you actually read

You compare your query with each book's key (dot product), focus on the most relevant ones (softmax), and read their content (multiply by V).
Why divide by $\sqrt{d_k}$?
Without scaling, large dot products push softmax to extreme values (nearly 0 or 1), making gradients vanish. Dividing by $\sqrt{d_k}$ keeps values in a manageable range.

Multi-head attention: Run multiple attention heads in parallel (each with different $W_Q, W_K, W_V$), then concatenate results. This lets the model attend to different types of relationships simultaneously.

5.4 Subword Tokenization: BPE Algorithm

Byte Pair Encoding (BPE) is how modern models tokenize text — between word-level and character-level.

Training Phase (Learning Merge Rules)

1
Pre-tokenize text into words
2
Initialize vocabulary with all individual characters
3
Find the most common adjacent symbol pair
4
Merge all instances into a new symbol, add to vocabulary
R
Repeat 3-4 until target vocabulary size reached

Encoding Phase (Applying Learned Rules)

Split word into characters, apply merge rules in the order they were learned.

Exam Practice: BPE Encoding (2024 Q1c style) 6 marks
Encode "mississippi pie" using learned rules: is, pi, ie, pie, ppi, sippi (in order)
Solution
// Encode "mississippi" Start: m i s s i s s i p p i Rule 1 (i,s → is): m is s is s i p p i Rule 2 (p,i → pi): m is s is s i p pi (no more p,i pairs adjacent) Rule 3 (i,e → ie): no i,e pairs → same Rule 4 (p,ie → pie): no p,ie pairs → same Rule 5 (p,pi → ppi): m is s is s i ppi Rule 6 (si,ppi → sippi): m is s is sippi Result: m is s is sippi // Encode "pie" Start: p i e Rule 1 (i,s → is): no i,s → same Rule 2 (p,i → pi): pi e Rule 3 (i,e → ie): no separate i,e → same Rule 4 (p,ie → pie): no p,ie → already merged to pi Actually: pi + e, no merge applicable Result: pi e
BPE Vocabulary Size Trade-offs (2024 Q1d)
Too low (e.g., 100): Most tokens are single characters, losing morphological info; sequences become very long
Too high (e.g., 100M): Approaches word-level; rare words get poor embeddings; wastes memory

5.6 Encoder (BERT) vs Decoder (GPT)

FeatureEncoder (BERT)Decoder (GPT)
AttentionBidirectional (sees all tokens)Causal/masked (sees only left context)
Best forClassification, NER, understandingGeneration, completion
TrainingMasked Language Model (predict [MASK])Causal LM (predict next word)
Intuition
BERT (Encoder) is like reading an entire paragraph and answering questions about it. It sees everything at once.
GPT (Decoder) is like writing a story word by word. It can only look at what's been written so far, not what comes next.

5.7 Transfer Learning: Pretrain + Fine-tune

1
Pretrain on massive unlabelled corpus (expensive, done once). The model learns general language understanding.
2
Fine-tune on small labelled task-specific data (cheap, done per task). The model specializes.
Analogy
Pretraining = Going to university and getting a broad education
Fine-tuning = Getting specialized on-the-job training for a specific role
Exam Tip (2022 Q3e)
To add POS information to BERT: create a new embedding layer (one embedding per POS tag), add it to the input alongside token and positional embeddings. This differs from bag-of-features models where each word-POS pair is a separate sparse feature.

5.8 Stopwords: BoW vs Transformers (2024 Q1a)

Key Distinction
Removed in BoW: Stopwords add noise since BoW ignores word order. "The cat sat on the mat" and "mat the the cat on sat" are identical in BoW.

Kept in Transformers: Transformers use positional encoding and attention to model word relationships. Stopwords carry syntactic meaning. "Not good" vs "good" — removing "not" completely changes sentiment!

POS Tagging & Parsing

Labelling words with grammatical roles and understanding sentence structure

6.1 Hidden Markov Model (HMM) for POS Tagging

POS tagging assigns grammatical labels (noun, verb, adjective, etc.) to each word. HMMs use two probability components:

HMM Components Transition probabilities: $P(y_i | y_{i-1})$ — probability of tag given previous tag
"After a determiner (DET), a noun (NN) is likely"

Emission probabilities: $P(x_i | y_i)$ — probability of word given its tag
"Given tag VERB, the word 'run' has probability 0.05"
Real-Life Example
Sentence: "The dog runs fast"

Tags: DET → NN → VBZ → RB
• $P(\text{NN}|\text{DET})$ = high (after "the", a noun usually follows)
• $P(\text{dog}|\text{NN})$ = the probability of seeing "dog" when the tag is noun
• The best tag sequence maximizes the product of all transitions × emissions

6.2 Decoding Strategies

StrategyHow It WorksQualitySpeed
GreedyPick best transition × emission at each token independentlySuboptimal (local decisions)Fast
ViterbiDynamic programming: find globally optimal path through all possible tag sequencesOptimalSlower but polynomial
Analogy: GPS Navigation
Greedy: At each intersection, take the road that looks shortest right now. You might end up on a longer route overall.
Viterbi: Calculate ALL possible routes first, then pick the shortest total path. Guaranteed optimal.

6.3 Domain Mismatch Problem (2022 Q3c)

A POS tagger trained on news text may fail on patents, medical text, or social media because:

  • Specialized vocabulary not in training data
  • Different sentence structures (legal text is very different from news)
Solutions
1. Fine-tune the tagger on domain-specific annotated data
2. Pre/post-processing with domain-specific dictionaries for known domain words

6.4 Constituency & Dependency Parsing

Constituency Parsing

Breaks sentences into nested phrases (tree structure):

// "The big cat sat on the mat" S → NP VP NP → DET ADJ NN → "The big cat" VP → VBD PP → "sat on the mat" PP → IN NP → "on the mat"

Dependency Parsing

Creates a directed graph of word relationships (subject, object, modifier).

Properties of Dependency Trees
• Weakly connected, acyclic
• Single root
• Each word has exactly one head (except root)

Transition-based Dependency Parsing

ActionEffect
ShiftMove top of buffer to stack
Left-ArcCreate arc from stack top to second element; remove second
Right-ArcCreate arc from second element to stack top; remove stack top

Information Extraction (Advanced)

Extracting structured knowledge from unstructured text: NER, entity linking, relation extraction, coreference

8.1 Named Entity Recognition (NER)

NER identifies and classifies named entities (people, places, organizations) in text.

IOB Tagging Scheme
B-TYPE: Beginning of an entity
I-TYPE: Inside (continuation of) an entity
O: Outside any entity
Example
[Barack]B-PER [Obama]I-PER [visited]O [Paris]B-LOC

"Barack Obama" is a PERSON entity (2 tokens). "visited" is not an entity. "Paris" is a LOCATION entity.
Real-Life Use: Medical Records
"Patient took Aspirin 500mg for headache."
NER identifies: Aspirin = B-DRUG, headache = B-SYMPTOM
This helps automatically extract medical information from doctor's notes!

Approaches: CRF (Conditional Random Fields), Transformer-based sequence labeling

8.2 Entity Linking

Goal: Map entity mentions (strings) to knowledge base entries (things).

"From Strings to Things"
"Paris" in text → Is it Paris, France? Paris, Texas? Paris Hilton?
Entity linking uses context to resolve ambiguity.
1
Candidate Generation: Find possible matches (dictionary lookup, dense vector retrieval)
2
Candidate Reranking: Use context to score each candidate (Transformer model)
Example
"I visited Paris and saw the Eiffel Tower."
Context: "Eiffel Tower" makes it clear → Paris, France (not Texas!)

8.3 Relation Extraction

Goal: Extract triples (subject, relation, object) from text.

ApproachHowPros/Cons
Co-occurrenceTwo entities in same sentenceNo actual relation type extracted
Rule-basedPatterns like "X, who starred in Y"High precision, low recall
BERT-basedEntity markers + pooling for classificationBest performance, needs training data
BERT-based Approach
Input: [E1] Barack Obama [/E1] visited [E2] Paris [/E2]
The model uses [E1] and [E2] representations to classify the relation as "visited" or "traveled_to".

8.4 Coreference Resolution

Determining which mentions in text refer to the same entity.

TypeDefinitionExample
AnaphoraRefers to something mentioned earlier"Barack Obama gave a speech. He was eloquent."
CataphoraRefers to something mentioned later"Before he left, Obama thanked everyone."
ExophoraRefers to something outside the text"Look at that!" (pointing at something)
1
Mention detection: Find all pronouns, named entities, NPs
2
Mention clustering: Determine which mentions refer to the same entity

Hobbs' Algorithm: A syntactic tree-based method to find the noun phrase a pronoun refers to.

Large Language Models (LLMs)

From GPT to ChatGPT: training pipeline, prompt engineering, issues, and practical considerations

9.1 LLM Training Pipeline

1
Pretraining: Learn language on massive text corpora (books, web pages, code). Uses causal LM objective (predict next word). Costs millions of dollars.
2
Instruction Tuning: Fine-tune on instruction-following examples. "Summarize this article" → [summary]. Makes the model helpful and responsive.
3
RLHF (Reinforcement Learning from Human Feedback): Humans rank model outputs. A reward model is trained on these rankings, then used to fine-tune the LLM to align with human preferences.
Real-Life Analogy
Pretraining: A child reads millions of books and learns how language works
Instruction Tuning: The child goes to school and learns to follow instructions
RLHF: A teacher gives feedback: "This answer is good, but that one could be harmful" — aligning behavior with societal values

9.2 Prompt Engineering

StrategyDescriptionExample Prompt
Zero-shotNo examples provided"Classify this review as positive or negative: ..."
Few-shotProvide examples in prompt"Example 1: 'Great!' → positive. Example 2: 'Terrible' → negative. Now classify: ..."
Chain-of-ThoughtAsk model to reason step by step"Let's think step by step..."
Explicit InstructionsDetailed task description"You are a sentiment classifier. Output only 'positive' or 'negative'."
Practical Tip
Few-shot usually beats zero-shot. Chain-of-Thought is especially effective for reasoning tasks (math, logic). Be specific in instructions — vague prompts get vague answers.

9.3 LLM Issues

IssueDescriptionWhy It Matters
HallucinationGenerates plausible but false informationCan spread misinformation, dangerous in medical/legal contexts
Dangerous GenerationsHarmful, biased, or toxic outputsCan perpetuate stereotypes or provide harmful instructions
Test Set LeakageTraining data may contain benchmark test setsInflates reported performance; model doesn't truly generalize
ExpenseInference costs are significantGPT-4 costs ~$30-60 per million tokens; not feasible for all applications
Zero-Shot Evaluation Caveat (2024 Q3a)
Before claiming "zero-shot" performance, verify that:
• The LLM's pretraining data did NOT include the test dataset
• The prompt itself doesn't contain examples from the dataset
• The model was not instruction-tuned on similar tasks

9.4 Computational Cost Ranking (2025 Q3a)

Cost Ranking (Least → Most Expensive) Logistic Regression (TF-IDF) < BERT-base (~110M params) < LLaMA-70B (~70B params)
Scale Comparison
• Logistic Regression: Runs on any laptop CPU in milliseconds
• BERT-base: Needs a GPU, ~110 million parameters
• LLaMA-70B: Needs multiple high-end GPUs, 70 billion parameters (600x larger than BERT!)

9.5 Multilingual Considerations (2025 Q3c)

Problems with New Languages
Logistic Regression: Vocabulary is language-specific; all tokens in a new language are unknown features
BERT: Subword tokenizer trained on specific language(s); other languages split into meaningless character sequences

Solution: Use multilingual models (mBERT, XLM-R) or collect/train on new language data

9.6 Subword Tokenization Pitfalls (2025 Q3d)

Real-World Problem
Brand name "BeStep" might tokenize as ["Best", "##ep"]. The sub-token "Best" overlaps with the positive word "best", causing the sentiment classifier to ALWAYS predict positive for any mention of BeStep, regardless of context!
Solution
Replace brand names with neutral placeholders like "[BRAND]" in the training data, or add the brand name as a special token in the tokenizer vocabulary.

10.5 Common Mistakes to Avoid

Top 7 Mistakes Students Make
  1. Confusing similarity and distance (they are inversely related)
  2. Forgetting that Sorensen-Dice uses set cardinality, not token count (duplicates don't count)
  3. Confusing types (unique tokens) vs tokens (total occurrences)
  4. Not converting similarity to distance before checking triangle inequality
  5. Using wrong context in n-gram generation (newly generated tokens ARE part of context)
  6. Reporting Macro metrics for binary classification (just use P/R/F1)
  7. $F_1$ MUST always be between Precision and Recall — if not, something is wrong

Crib Sheet Reference (All Formulas)

Every formula you need in one place — your exam cheat sheet

Set Similarity Measures

$\text{Overlap Count} = |X \cap Y|$

$\text{Overlap Coefficient} = \frac{|X \cap Y|}{\min(|X|, |Y|)}$

$\text{Sorensen-Dice} = \frac{2|X \cap Y|}{|X| + |Y|}$

$\text{Jaccard} = \frac{|X \cap Y|}{|X \cup Y|}$

Distance & Vector Measures

$\text{Euclidean: } d(\vec{x},\vec{y}) = \sqrt{\sum_i (x_i - y_i)^2}$

$\text{Cosine Similarity: } \cos(\theta) = \frac{\vec{d_1} \cdot \vec{d_2}}{||\vec{d_1}|| \times ||\vec{d_2}||}$

TF-IDF

$\text{Log TF} = 1 + \log_{10}(\text{tf})$   if tf > 0, else 0

$\text{IDF} = \log\left(\frac{N}{\text{df}_t}\right)$

$\text{TF-IDF} = (1 + \log(\text{tf})) \times \log\left(\frac{N}{\text{df}_t}\right)$

Language Model Formulas

$\text{Add-k: } P = \frac{\text{count} + k}{N + k \times |V|}$

$\text{Interpolation: } \lambda_1 P_{\text{tri}} + \lambda_2 P_{\text{bi}} + \lambda_3 P_{\text{uni}}$   where $\sum \lambda = 1$

$\text{Jelinek-Mercer: } \lambda \cdot P(t|\text{doc}) + (1-\lambda) \cdot P(t|\text{collection})$

$\text{Cross-Entropy} = -\frac{1}{n} \sum \log_2 P(w_i)$

$\text{Perplexity} = 2^{\text{Cross-Entropy}}$

$\text{Entropy} = -\sum P(x) \cdot \log_2 P(x)$

Classification Metrics

$\text{Accuracy} = \frac{TP + TN}{TP + FP + TN + FN}$

$\text{Precision} = \frac{TP}{TP + FP}$

$\text{Recall} = \frac{TP}{TP + FN}$

$F_1 = \frac{2 \times P \times R}{P + R} = \frac{2TP}{2TP + FP + FN}$

$F_\beta = \frac{(1 + \beta^2) \times P \times R}{\beta^2 \times P + R}$

Self-Attention

$Q = \text{Input} \times W_Q, \quad K = \text{Input} \times W_K, \quad V = \text{Input} \times W_V$

$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{Q \times K^T}{\sqrt{d_k}}\right) \times V$

HMM

$\text{Transitions: } P(y_i | y_{i-1})$

$\text{Emissions: } P(x_i | y_i)$

Silhouette Score

$$s(i) = \begin{cases} 1 - \frac{\text{cohesion}}{\text{separation}} & \text{if cohesion} < \text{separation} \\ 0 & \text{if equal} \\ \frac{\text{separation}}{\text{cohesion}} - 1 & \text{if cohesion} > \text{separation} \end{cases}$$ Range: $[-1, 1]$. Higher = better clustering.

Metric Properties

1. Identity: $d(X, X) = 0$
2. Positivity: $d(X, Y) > 0$ for $X \neq Y$
3. Symmetry: $d(X, Y) = d(Y, X)$
4. Triangle Inequality: $d(X, Z) \leq d(X, Y) + d(Y, Z)$
