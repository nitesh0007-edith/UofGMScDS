SQL & NoSQL

Relational Databases (RDBMS)

  • Tabular data organization
  • Normalized for reduced redundancy
  • Primary & Foreign Keys
  • SQL for queries & transactions
Oracle PostgreSQL MySQL SQL Server

ACID Properties

  • Atomicity: All or nothing
  • Consistency: Valid state transitions
  • Isolation: Sequential-like execution
  • Durability: Permanent commits

NoSQL Databases

  • Key-Value stores (Redis)
  • Wide Column (Cassandra)
  • Document stores (MongoDB)
  • Graph & Vector databases
BASE Transactions Eventual Consistency

Use Cases

  • In-memory caching
  • Time series data
  • Geospatial queries
  • Full-text search (Elasticsearch)
🕸️

Graph Databases

Graph Structure

  • G = (V, E)
  • V: Set of nodes/vertices
  • E: Set of edges/relations
  • Directed or undirected edges
  • Node & edge features/attributes

Graph Types

  • Homogeneous: Single node/edge type
  • Heterogeneous: Multiple types
  • Used in recommendation systems

Representations

  • Adjacency Matrix: O(|V|²) space
  • Adjacency List: O(|E|) space
  • Latent/Embeddings: O(|V| × D)

Downstream Tasks

  • Graph-level: Property prediction
  • Node-level: Classification
  • Edge-level: Link prediction
  • Community detection, Ranking

Query Languages (GQL)

  • GraphQL: APIs & microservices
  • Cypher: Neo4j pattern matching
  • Gremlin: Graph traversals
  • SPARQL: RDF/Semantic web
  • AQL: ArangoDB multi-model
Neo4j ArangoDB TigerGraph

Vector Databases

AI & Embeddings

  • High-dimensional vectors
  • Search engines & RecSys
  • Personalized recommendations
  • Similarity-based retrieval

RAG Architecture

  • Retrieval Augmented Generation
  • External knowledge sources

  • Reduce hallucinations
LLM Integration

AKNN Search

  • Approximate K Nearest Neighbors
  • Euclidean distance, Cosine similarity
  • Recall = |V ∩ V*| / |V*|

AKNN Algorithms

  • Tree-based: KD-tree, R-tree
  • Quantization: IVF_FLAT, PQ
  • Graph-based: HNSW, NSG
  • Hashing: LSH

Libraries & Tools

  • Facebook Faiss (GPU support)
  • Microsoft SPTAG
  • Spotify Annoy
  • SIMD optimizations

Vector DB Pipeline

  • 1. Indexing: Build search structure
  • 2. Querying: Find nearest neighbors
  • 3. Post-processing: Re-ranking

📋 Key Comparisons & Trade-offs

SQL vs NoSQL

ACID vs BASE • Strong vs Eventual Consistency • Vertical vs Horizontal Scaling

Graph DB Advantage

Relationship-first queries • No JOIN overhead • Flexible schema evolution

Vector DB Purpose

Similarity search • ML/AI embeddings • Sub-linear query time
