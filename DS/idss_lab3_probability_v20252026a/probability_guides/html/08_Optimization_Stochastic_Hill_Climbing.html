🎯 Introduction: The Optimization Problem

Many problems in probability and statistics boil down to optimization:

  • Find the MLE (maximize likelihood)
  • Minimize prediction error
  • Choose the best search strategy
  • Find optimal parameter values
The Core Problem:

Given a function f(x), find x* such that:
f(x*) is maximum (or minimum)

Example: Finding the peak of a mountain!

📊 Optimization Basics

Optimization Problem

Find the value that maximizes (or minimizes) an objective function:

\[x^* = \arg\max_{x} f(x)\]

Read as: "The value of x that maximizes f(x)"

Equivalent for minimization:

\[\arg\min_{x} f(x) = \arg\max_{x} -f(x)\]

Local vs Global Optima

⚠️ Critical Distinction

Local Maximum: Best in the neighborhood

  • \(f(x^*) \geq f(x)\) for all x near \(x^*\)
  • Like a hilltop

Global Maximum: Best everywhere

  • \(f(x^*) \geq f(x)\) for ALL x
  • The highest mountain peak
Landscape Analogy:

Gradient-Based Methods

Gradient Descent/Ascent

Use the derivative to find the direction of steepest change:

\[x_{t+1} = x_t + \alpha \nabla f(x_t)\]

Where:

  • \(\alpha\) = learning rate (step size)
  • \(\nabla f\) = gradient (derivative)

Problem: Can get stuck in local optima!

⛰️ Hill Climbing Algorithm

Stochastic Hill Climbing

Iteratively move to better neighboring solutions with randomness:

Algorithm:

1. Start at random position x
2. Repeat:
a. Generate random neighbor x'
b. If f(x') > f(x):
Move to x' (accept improvement)
c. Otherwise:
Stay at x (reject worse solution)
3. Until convergence or max iterations

🏔️ Example: Finding Maximum

Function: \(f(x) = -(x-3)^2 + 10\) (parabola, max at x=3)

Iteration 0: x = 0.5, f(x) = 3.75
Try neighbor: x' = 1.2
f(x') = 6.76 > 3.75 ✓ Accept!

Iteration 1: x = 1.2, f(x) = 6.76
Try neighbor: x' = 2.0
f(x') = 9.0 > 6.76 ✓ Accept!

Iteration 2: x = 2.0, f(x) = 9.0
Try neighbor: x' = 2.8
f(x') = 9.96 > 9.0 ✓ Accept!

Iteration 3: x = 2.8, f(x) = 9.96
Try neighbor: x' = 3.1
f(x') = 9.99 > 9.96 ✓ Accept!

...converges to x ≈ 3

📊 Strengths and Weaknesses

Strengths:

  • Simple to implement
  • No gradient needed
  • Works for discrete spaces

Weaknesses:

  • Gets stuck in local optima
  • Sensitive to starting point
  • Can't escape plateaus

🔥 Simulated Annealing

Simulated Annealing

Enhanced hill climbing that can escape local optima by occasionally accepting worse solutions!

Key Idea: Accept worse solutions with probability that decreases over time (like cooling metal)

Algorithm

Simulated Annealing:

1. Start at random x, set temperature T = T₀
2. Repeat:
a. Generate neighbor x'
b. Calculate Δf = f(x') - f(x)
c. If Δf > 0:
Accept (move to x')
d. Else:
Accept with probability exp(Δf/T)
e. Decrease temperature: T = α·T
3. Until T ≈ 0 or convergence

Acceptance Probability:

\[P(\text{accept worse}) = \exp\left(\frac{f(x') - f(x)}{T}\right)\]

Where T is the "temperature" parameter

🌡️ Temperature Schedule

High Temperature (early):

  • Accept worse solutions frequently
  • Explore broadly
  • Can escape local optima

Low Temperature (late):

  • Rarely accept worse solutions
  • Exploit current region
  • Fine-tune solution

Cooling Schedule:

\[T_t = T_0 \cdot \alpha^t \quad \text{where } 0 < \alpha < 1\]

Common: \(\alpha = 0.95\) (5% cooling per step)

❄️ Example: Escaping Local Optimum

Function: Multiple peaks with local maxima

Current: x = 2, f(x) = 8 (local max)
Neighbor: x' = 2.5, f(x') = 6 (worse!)
Δf = 6 - 8 = -2

Temperature T = 5:
P(accept) = exp(-2/5) = exp(-0.4) ≈ 0.67
→ 67% chance of accepting!

Temperature T = 1:
P(accept) = exp(-2/1) = exp(-2) ≈ 0.14
→ Only 14% chance

Temperature T = 0.1:
P(accept) = exp(-2/0.1) = exp(-20) ≈ 0.000002
→ Almost never accept

Early on, we might accept this worse move and discover the global optimum elsewhere!

🌍 Applications

🔍 Application 1: Submarine Search Strategy

Optimize: Search order to maximize probability of finding submarine

Approach:

  1. Represent strategy as sequence of squares
  2. Objective: Expected search time
  3. Use simulated annealing to find good ordering
  4. Accept worse strategies early to explore

Handles complex, discrete optimization problem!

🗺️ Application 2: Traveling Salesman Problem

Problem: Visit n cities with minimum total distance

Classic simulated annealing application:

  • State = tour (permutation of cities)
  • Neighbor = swap two cities
  • Objective = total distance
  • Can escape local optima to find better tours

📊 Application 3: Model Fitting

When MLE has no closed form:

  • Use optimization to maximize likelihood
  • Hill climbing for simple landscapes
  • Simulated annealing for multimodal likelihoods

Example: Fitting mixture models with multiple components

🧬 Application 4: Protein Folding

Find 3D structure that minimizes energy:

  • Energy landscape extremely complex
  • Millions of local minima
  • Simulated annealing mimics physical process
  • High temp = unfolded, low temp = folded

🎮 Application 5: Game AI

Optimize game-playing strategies:

  • State = current position
  • Objective = win probability
  • Neighbors = possible moves
  • Hill climbing for move selection

Convergence Criteria

🛑 When to Stop

Common stopping conditions:

  1. Max iterations: Run for N steps
  2. No improvement: f(x) unchanged for k iterations
  3. Small changes: |f(x_{t+1}) - f(x_t)| < ε
  4. Temperature: T < threshold (simulated annealing)
  5. Time limit: Stop after T seconds

Often use combination of multiple criteria!

🎓 Summary: Key Takeaways

Optimization:

\[x^* = \arg\max_{x} f(x)\]

Hill Climbing:

  • Always accept better solutions
  • Simple but gets stuck in local optima
  • No gradient needed

Simulated Annealing:

\[P(\text{accept worse}) = \exp\left(\frac{\Delta f}{T}\right)\]
  • Occasionally accept worse solutions
  • Can escape local optima
  • Temperature controls exploration/exploitation

Key Trade-off:

  • Exploration: Search broadly for global optimum
  • Exploitation: Refine current solution
