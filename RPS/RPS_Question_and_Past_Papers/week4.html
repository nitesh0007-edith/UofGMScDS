🎯 Variables and Factors

Why Learn These Terms?

These terms are the language of experimental design. You need to know them to design, describe, and critique experiments properly!

The Key Variables You Must Know

Independent Variable (IV)

What You Deliberately Change

The variable that the experimenter controls and deliberately manipulates.

  • Has different "levels" (e.g., configuration 1, 2, 3)
  • If multiple IVs, a "condition" is a combination of IV levels
Example: If "Input Device" is the IV, levels might be: Mouse, Trackpad, Eye-gaze
Dependent Variable (DV)

The Data You Collect

A variable whose value(s) depend on the independent variable.

Any observable, measurable behaviour can be a DV:

  • Task execution time
  • Typing speed
  • Accuracy / Error rate
  • Questionnaire responses
  • Comfort / Emotional response
Control Variable

Factors That Don't Change

Factors that do not change to ensure a "fair" comparison.

  • Not under investigation (not IVs)
  • Might influence participant behaviour
  • Controlled by setting up study in controlled environment
Example: Same room, same computer, same chair for all participants
Random Variable

Factors You Allow to Vary

Factors you knowingly allow to randomly change/vary.

  • Instead of controlling everything, allow some to vary randomly
  • Helps generalize results
  • Controlling everything is probably impossible anyway!
Examples: Height, finger size, hand size, prior experience
Confounding Factor

⚠️ The Dangerous Variable

Factors that change and impact results (often only identified AFTER data collection).

  • Could have influenced the relationship between IV and DV
  • Prevent cause-and-effect relationships from being inferred
  • Need to be controlled for - this is a central skill!
Example: Practice effects, prior experience with an interface, different measurement types for IV levels

🎯 What We Want an Experiment to Show

A change in the IV → A change in the DV

We want to be sure that any change in the DV is a result of the change in the IV, NOT a result of change in other factors!

Human Characteristics in User Studies

For user studies, differences in human characteristics are important factors:

  • Age, gender, height, weight
  • Left/right-handed
  • Personality traits, political viewpoints
  • Ethnicity, cultural background
  • First language

How to handle:

  • Analyse data separately for different characteristics
  • Specify as random variables with acknowledged limitations
  • Adjust questions for participants (e.g., age-appropriate for children)

⚖️ Within vs Between Subject Designs

Why Do We Need Different Designs?

We need to structurally control how we conduct experiments to ensure a fair comparison. This helps mitigate things like ordering effects and learning!

🔄 Within-Subjects Design

Same participants experience ALL conditions

P1
→ Condition A → Condition B
P2
→ Condition A → Condition B
  • ✅ Fewer participants needed
  • ✅ Controls individual differences
  • ⚠️ Risk of order effects
  • ⚠️ Learning/fatigue effects

👥 Between-Subjects Design

Different participants for EACH condition

P1
P2
→ Condition A
P3
P4
→ Condition B
  • ✅ No order effects
  • ✅ No learning effects
  • ⚠️ More participants needed
  • ⚠️ Individual differences affect results

⚠️ Order Effects in Within-Subjects

The Problem

In within-subjects designs, the order participants experience conditions can affect results:

  • Learning effect: Performance improves with practice
  • Fatigue effect: Performance decreases due to tiredness
  • Carryover effect: One condition affects the next

Solutions: Counterbalancing

🎲 Randomisation

Randomly assign the order of conditions for each participant.

Helps "average out" order effects across all participants.

🔢 Latin Squares

A systematic way to counterbalance conditions so each condition appears in each position equally.

Participant Order 1 Order 2 Order 3 Order 4
P1 A B C D
P2 B C D A
P3 C D A B
P4 D A B C

Each condition (A, B, C, D) appears exactly once in each position!

🎹 Worked Example: Keyboard Comparison

Problem Statement

"Propose an experiment to determine which of two new keyboards is better"

1

Define "Better"

"Better" could mean many things:

  • Latency of input
  • Speed of typing
  • Accuracy while typing
  • User experience

Choice: "User experience using the device"

2

Research Questions

RQ1: Are there any significant differences in user comfort whilst using either keyboard over a long period of time (e.g., 1 week)?

RQ2: Are there any significant differences between the user workloads whilst using either keyboard?

3

Define Variables

Variable Type Definition
Dependent Variables • Comfort (5-point Likert scale / qualitative feedback)
• Workload (NASA TLX questionnaire)
Independent Variable The two keyboards
Conditions Keyboard 1, Keyboard 2
Task Recreation of office work (50 min): emails, Slack, web searching, report writing
Experimental Objects Human participants
Control Variables Same task, same room, same monitor/chair for all participants
Random Variables Human characteristics (e.g., prior tech experience)
Confounding Factors Prior typing experience (touch typists vs single-finger typers)

📝 Don't Forget!

  • How you intend to analyse your data
  • The ethics of running the experiment (especially with human participants)

📊 Types of Data

Why Data Types Matter

Different types of data require different analysis methods. Knowing what type of data you're collecting helps you choose the right statistical tests!

Quantitative vs Qualitative

🔢

Quantitative Data

Data that expresses a numerical quantity

  • Compilation time
  • Classification accuracy (%)
  • Number of errors
  • Reaction time
  • Task success (Y/N)
📝

Qualitative Data

Data that is non-numerical

  • Interview transcripts
  • Observation notes
  • Ethnographic diaries
  • Documents
  • Videos

Continuous vs Discrete

〰️

Continuous Data

Values can be anywhere in a range, including fractional values

Examples: Time taken (2.5 seconds), height (175.3 cm), temperature
🔵

Discrete Data

Values are at specific points rather than anywhere along a range

Examples: Number of errors (12 or 13, never 12.5), number of clicks

The Data Hierarchy (NOIR)

🏷️

Nominal

Categories with no order

Only members vs non-members - nothing more!

Examples:
Gender, Hair colour, Eye colour
📊

Ordinal

Categories WITH natural order/rank

Has ranking but intervals not equal

Example:
"Very unsatisfied → Unsatisfied → Neutral → Satisfied → Very satisfied"
🌡️

Interval

Equal intervals, NO true zero

Shows order AND difference between values

Example:
Temperature in Celsius (0°C is still a temperature, not "no heat")
📏

Ratio

Equal intervals WITH true zero

Zero means absence of the thing

Examples:
Height, Weight, Time (0 = no time)

📥 Data Collection & Management

Plan Your Data Collection!

We don't want data collection to be random. We don't want to finish an experiment only to discover problems!

⚠️ Common Mistakes to Avoid

  • Collected the wrong data
  • Didn't collect data to sufficient accuracy (e.g., seconds vs milliseconds)
  • Collected data format is unusable
  • Broke data collection protocols at your employment
  • Didn't collect data ethically
  • Designed experiments that don't yield useful data
  • Too many IVs making it difficult to isolate effects

Best Practices

1

Prepare Analysis Scripts Early

Before running an experiment, set up your data analysis scripts!

  • Helps you know what data you need to capture
  • Select data analysis methods in advance
  • Generate fake data to test your scripts
Benefits: Saves time post-experiment and highlights shortcomings of your approach!
2

Run a Pilot Test

Test the experiment setup and data collection method:

  • With target participants: Early insight into actual data you might collect
  • With knowledgeable colleagues: Can critique experimental design, setup, and data collection

Data Visualisation & Exploration

🔍

During Analysis

Visualise and explore your data as you analyse it

  • Spot patterns and anomalies
  • Check data quality
  • Guide hypothesis formation
📊

When Presenting

Visualise data in reports and publications

  • Communicate findings clearly
  • Support your conclusions
  • Make data accessible

📖 "A Hypothesis is a Liability" — Yanai & Lercher

Interesting research showing how focusing too much on a specific hypothesis can prevent discoveries!

☀️ Day Science

Structured testing of predefined hypotheses

We falsify hypotheses and observe which are left standing

🌙 Night Science

Exploring unstructured realm of possible hypotheses

We create hypotheses through exploration

Key insight: Both are complementary — the shortcomings of one are corrected by the other!

🌐 Generalisability of Results

What is Generalisability?

The extent to which your results can be applied beyond your specific experiment. Can you claim your findings apply to other contexts?

⚠️ Don't Overclaim!

Overclaiming is when you claim you prove/show more than your results actually do.

A large part of what you do as a researcher is convince peers your results are correct/valid. Overclaiming damages credibility!

The Control-Generalisability Trade-off

More Control = Less Generalisability

When we tightly control variables, our results only apply to those fixed control values!

Example: Graph Layout Algorithms

Imagine an experiment testing graph layout algorithms with:

  • One specific graph
  • One specific task (find shortest path)
  • Fixed node pairs
  • Fixed graph orientation

Result: "Algorithm A is better than Algorithm B for finding shortest paths between nodes X and Y in this specific graph with this orientation..."

This isn't very generalisable — too many "if X and if Y and if Z..."!

How to Improve Generalisability

1

Increase Variability in Control Variables

  • Use more than one graph
  • Use more than one task
  • Test across different contexts
2

Post-Experiment Analysis

Don't control, but analyse data with respect to other factors:

  • If graph density might be confounding → analyse by density (high/medium/low)
  • If gender might have an effect → analyse separately
3

Randomisation

Mitigate possible confounding factors through randomisation:

  • Randomly choose different node pairs for each task
  • Choose random orientation for each graph
  • "Average out" the effects of other potential causes

💡 Three Ways to Ensure IV Caused DV Changes

  1. Control Variables: Explicitly set all other potential variables
  2. Post-Experiment Analysis: Analyse data with respect to other factors
  3. Randomisation: Average out effects through random assignment
