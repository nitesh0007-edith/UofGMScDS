🎯 VARIABLES - QUICK REFERENCE

🔵 Independent Variable (IV)

What: What experimenter deliberately changes

Example: Type of keyboard (A vs B)

Levels: Different values of the IV

🟢 Dependent Variable (DV)

What: What you measure/collect

Example: Typing speed, error rate

Depends on: The IV

🟡 Control Variables

What: Kept constant for fair comparison

Example: Same computer, same room

Purpose: Eliminate alternative explanations

🟣 Random Variables

What: Allowed to vary randomly

Example: Participant age, experience

Purpose: Improve generalizability

🔴 Confounding Factors

What: Changes & impacts results

Example: Prior knowledge, practice

Problem: Prevents cause-effect inference

📊 Condition

What: Combination of IV levels

Example: Keyboard A + Task 1

Note: Participants assigned to conditions

⚖️ WITHIN vs BETWEEN SUBJECTS

Aspect Within-Subjects Between-Subjects
Definition Same participants in ALL conditions Different participants in EACH condition
Participants Needed ✅ Fewer (e.g., 20 total) ❌ More (e.g., 40 total for 2 conditions)
Individual Differences ✅ Controlled (same people) ❌ Problem (different abilities)
Order Effects ❌ Risk (practice, fatigue) ✅ No problem
When to Use Few participants, large individual differences Learning effects likely, enough participants
Counterbalancing ✅ Required (A→B, B→A) ❌ Not needed
Statistical Test Paired/Repeated-measures t-test Independent t-test

📊 DATA TYPES (NOIR)

Type Categories Ordered Equal Intervals True Zero Examples

Ordinal ✅ ✅ ❌ ❌ Satisfaction (1-5), rankings

Continuous: Can be any value (2.5 seconds, 175.3 cm)

🎯 HYPOTHESIS

A hypothesis is a SPECIFIC and TESTABLE explanation of a phenomenon
Good Hypothesis Structure:
"[New method] will result in [better outcome] compared to [old method]"
Example: "Students using LocoPro will complete navigation tasks faster and report higher immersion scores than those using LocoOriginal."
What it needs:
  • Specific prediction (not vague)
  • Testable (can design experiment)
  • Clear DV stated or implied
  • Directional (which is better)

⚠️ COMPUTER MISUSE ACT

Key Point: Creating viruses is NOT automatically illegal!
Legal: Creating/using viruses WITH owner's permission (e.g., security testing)
Illegal: Creating/using to access, modify, or impair computers WITHOUT authorization
Three Main Offenses:
  1. Unauthorized access to computer material
  2. Unauthorized access with intent to commit further offenses
  3. Unauthorized modification of computer material

🧪 RESEARCH ETHICS PRINCIPLES

Informed Consent
Participants must know what study involves and agree voluntarily
Right to Withdraw
Can leave study anytime without penalty
No Harm
Minimize physical/psychological harm
Confidentiality
Protect participant identity and data
Debriefing
Explain study purpose after participation
Ethics Approval
Get committee approval before starting
Common Ethical Issues:
  • Coercion - Power imbalance (teacher-student)
  • Deception - Misleading participants
  • Privacy - Collecting data without consent
  • Mandatory participation - Can't refuse (intertwined with essential activities)

📥 DATA COLLECTION METHODS

1. Questionnaires/Surveys

Closed: Fixed responses (Likert scale, Y/N, MCQ)

Open: Free-form text responses

Use: Large samples, standardized data

2. Interviews

Structured: Fixed questions

Semi-structured: Flexible follow-ups

Use: In-depth insights, qualitative data

3. Observation

Direct: Researcher present

Indirect: Video recording

Use: Actual behavior, contextual understanding

4. Think-Aloud Protocol

Participants verbalize thoughts while performing tasks

Use: Usability testing, understanding thought process

5. Automatic Data Collection

Logs, analytics, eye-tracking, biometrics

Use: Objective, precise measurements

6. Document/Artifact Analysis

Existing records, code repos, forum posts

Use: Historical data, authentic behavior

📈 STATISTICAL ANALYSIS BASICS

Null Hypothesis (H₀): No difference between conditions
Alternative Hypothesis (H₁): There IS a difference
Significance Level: p < 0.05 (5% chance results due to random chance)
Design Conditions Statistical Test
Within-subjects 2 conditions Paired t-test (Repeated-measures)
Between-subjects 2 conditions Independent t-test
Within-subjects 3+ conditions Repeated-measures ANOVA
Between-subjects 3+ conditions One-way ANOVA
In your answer, specify:
  • Test name (e.g., "repeated-measures t-test")
  • Null/alternative hypotheses
  • Significance level (α = 0.05)
  • What you're comparing (which DV)

©️ COPYRIGHT & DATABASE PROTECTION

Copyright Requirements:
  • Must be original work
  • Result of author's own intellectual creation
  • Fixed in tangible form
Database Copyright Needs:
  • Substantial investment in obtaining, verifying, presenting
  • Original selection/arrangement
  • Sufficient complexity
NOT Copyrightable:
  • Simple alphabetical ordering
  • Standard categorizations
  • Facts themselves (only arrangement)

🔬 SCIENTIFIC METHOD (8 STEPS)

  1. Define a question
  2. Gather information and resources (observe)
  3. Form explanatory hypothesis
  4. Test hypothesis by performing experiment
  5. Analyze the data
  6. Interpret data and draw conclusions
  7. Publish results
  8. Retest (often by other scientists)

❌ COMMON MISTAKES TO AVOID

Don't:
  • Mix up IV and DV
  • Say "collect data" (say HOW)
  • Say "statistical tests" (NAME the test)
  • Forget to justify design choices
  • Leave questions blank
  • Spend too long on low-mark questions
Do:
  • Use bullet points for clarity
  • Be specific with numbers (30 participants)
  • Define before explaining
  • Use correct terminology
  • Check command words
  • Show all aspects of design
