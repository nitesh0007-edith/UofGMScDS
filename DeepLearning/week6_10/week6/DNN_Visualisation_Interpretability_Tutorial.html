Chapter 01

Why Do We Need to Interpret Neural Networks?

Imagine you go to a doctor who says, "You have disease X." You ask, "How do you know?" and the doctor replies, "I just know. Trust me." Would you trust that diagnosis? Probably not. You'd want to know why the doctor thinks that.

Deep neural networks face the exact same trust problem. They can classify images, translate languages, and even drive cars — but they rarely tell us why they made a particular decision. This is the core motivation behind interpretability.

💡
Real-World Analogy

Think of a DNN as a brilliant employee who always gives you the right answer but can never explain their reasoning. In critical fields like healthcare, self-driving cars, or criminal justice, "just trust me" isn't good enough.

Key Reasons for Interpretability

🏥
Trust & Safety

In medical imaging, a doctor needs to verify why the AI flagged an X-ray as cancerous before acting on it.

⚖️
Fairness & Bias

If a hiring AI rejects candidates, we must ensure it's not secretly using race or gender as a factor.

🐛
Debugging

When a model fails, visualising what it "sees" helps engineers find and fix the problem quickly.

📜
Regulatory Compliance

Laws like the EU's GDPR give people the "right to an explanation" for automated decisions that affect them.

Chapter 02

The "Black Box" Problem

A deep neural network can have millions (or even billions) of parameters — tiny numerical weights adjusted during training. No human can look at millions of numbers and understand what's going on. That's why we call DNNs "black boxes".

The Black Box Problem: We can see the input and output, but what happens inside is a mystery.

Interpretability means peeking inside that box to understand which features, pixels, or patterns the network uses to make its decisions.

🎯 Key Distinction

Visualisation = "What does the network see?" (showing filters, activations, feature maps). Interpretability = "Why did the network make this specific decision?" (saliency maps, Grad-CAM, LIME).

Chapter 03

Filter (Kernel) Visualisation

The simplest technique is to directly look at the learned filters (also called kernels) of a CNN. Each filter is a small matrix responsible for detecting a specific pattern.

💡
Beginner Analogy

Think of filters like coloured sunglasses. If you put on "edge-detecting sunglasses," you'd only see the edges of objects. A CNN learns hundreds of these "glasses" automatically!

What Filters Learn at Different Layers

Different layers learn different levels of abstraction — from simple edges to entire objects:

▤
Layer 1
Edges & Colours
→
◫
Layer 2-3
Textures & Corners
→
◉
Layer 4-5
Parts (eyes, wheels)
→
🖼
Deep Layers
Full Objects & Faces
Feature hierarchy: Early layers detect simple patterns, deeper layers detect complex concepts.

How Convolution Works (Quick Recap)

A filter is a small matrix (e.g., 3×3) that slides across the input image. At each position, it performs element-wise multiplication and sums up the results to produce one number in the output (feature map).

Convolution Operation
$$\text{Output}(i, j) = \sum_{m}\sum_{n} \text{Input}(i+m,\; j+n) \cdot \text{Filter}(m, n) + b$$
Where Input is the image, Filter is the learned kernel, and b is a bias term. The filter slides (or "convolves") across the entire image.

Interactive Example: Edge Detection Filters

A vertical edge filter (Sobel) applied to a 3×3 image patch. High output = strong edge present.

PyTorch Code Example

python — visualise_filters.py
import torch
import torchvision.models as models
import matplotlib.pyplot as plt

# Load a pre-trained VGG16 model
model = models.vgg16(pretrained=True)

# Get the first convolutional layer's filters
filters = model.features[0].weight.data.clone()

# Normalise filters to [0, 1] for display
filters = filters - filters.min()
filters = filters / filters.max()

# Plot the first 16 filters
fig, axes = plt.subplots(4, 4, figsize=(8, 8))
for i, ax in enumerate(axes.flat):
    ax.imshow(filters[i].permute(1, 2, 0))
    ax.set_title(f"Filter {i}")
    ax.axis("off")
plt.suptitle("First Layer Filters of VGG16")
plt.show()
📝 What You'll See

The first-layer filters typically look like colourful edge detectors — horizontal lines, vertical lines, diagonal lines, and colour blobs. This matches how the human visual cortex starts by detecting edges!

Chapter 04

Activation Maximisation

Filter visualisation shows us what filters look like, but not what kind of input excites a neuron the most. Activation maximisation answers: "If this neuron could dream, what would it dream about?"

💡
Beginner Analogy

Imagine asking a music critic: "What is your absolute favourite type of music?" Instead of showing you their notes, they'd play you their ideal song. That ideal song is the activation-maximised input — the image that makes a neuron "the happiest."

How It Works

We freeze the weights and instead optimise the input image to maximise a specific neuron's activation. We start with random noise and iteratively adjust it using gradient ascent.

1
Start with Random Noise

Create a random image (like TV static) as the starting point.

2
Forward Pass

Feed this noise image through the network and record the activation of the target neuron.

3
Compute Gradient w.r.t. Input

Calculate how each pixel should change to increase the target neuron's activation.

4
Update the Image (Gradient Ascent)

Slightly modify the input in the direction that increases the activation. Repeat many times.

Gradient Ascent on Input
$$x^* = \arg\max_{x} \; a_k(x) - \lambda \|x\|^2$$
We find the input x* that maximises the activation a_k(x) of neuron k. The term λ||x||² is a regulariser that prevents the image from becoming too noisy.
Update Rule
$$x_{t+1} = x_t + \eta \cdot \frac{\partial \, a_k}{\partial \, x}\bigg|_{x=x_t}$$
At each step, update the image x by adding the gradient of the activation w.r.t. the input, scaled by learning rate η. This is gradient ascent (not descent!) because we want to maximise.

Visual: Random Noise → Meaningful Pattern

Activation maximisation gradually transforms random noise into a pattern that maximally excites a "face" neuron.

PyTorch Code Example

python — activation_maximisation.py
import torch

# Assume `model` is a pre-trained CNN
model.eval()

# Start with a random image (requires grad!)
input_img = torch.randn(1, 3, 224, 224, requires_grad=True)
optimizer = torch.optim.Adam([input_img], lr=0.05)

for step in range(200):
    optimizer.zero_grad()
    output = model(input_img)
    target_activation = output[0, target_class]

    # Negate because optimizer minimises
    loss = -target_activation
    loss.backward()
    optimizer.step()

# input_img now shows what the neuron "dreams" of
🎨 Deep Dream Connection

Google's famous DeepDream algorithm is based on activation maximisation! It amplifies the patterns a network detects in a real image, creating those psychedelic, fractal-like visuals of dog faces and eyes.

Chapter 05

Saliency Maps (Vanilla Gradients)

Saliency maps answer a powerful question: "Which pixels in the input image matter most for the network's decision?"

💡
Beginner Analogy

Imagine taking a test and the teacher highlights the words that are most important for finding the answer. Saliency maps highlight the pixels most important for the network's prediction.

The Core Idea

We compute the gradient of the output class score with respect to each input pixel. If changing a pixel slightly would greatly change the output, that pixel is "salient" (important).

Saliency Map Formula
$$S(x) = \left| \frac{\partial \, y_c}{\partial \, x} \right|$$
Where y_c is the score for class c (before softmax), and x is the input image. We take the absolute value because we care about the magnitude of influence, not the direction.

Step-by-Step Process

1
Feed the Image

Pass the image through the trained network and get class scores.

2
Backpropagate

Compute gradients of the target class score w.r.t. every input pixel.

3
Take Absolute Value

Both positive and negative gradients indicate importance, so we take absolute value.

4
Visualise as Heatmap

Display gradient magnitudes as a heatmap overlaid on the image. Bright areas = important pixels.

PyTorch Implementation

python — saliency_map.py
# Enable gradient tracking on the input
image.requires_grad_()

# Forward pass
output = model(image)
target_score = output[0, predicted_class]

# Backward pass
target_score.backward()

# Get saliency: max gradient across colour channels
saliency, _ = image.grad.data.abs().max(dim=1)

# Plot
plt.imshow(saliency[0], cmap='hot')
plt.title("Saliency Map")
plt.show()
⚠️ Limitation

Vanilla saliency maps tend to be noisy. SmoothGrad helps by averaging saliency maps over slightly noisy versions of the image:

SmoothGrad — Reducing Noise
$$\hat{S}(x) = \frac{1}{N} \sum_{i=1}^{N} S\!\left(x + \mathcal{N}(0, \sigma^2)\right)$$
Add random Gaussian noise N times, compute a saliency map each time, and average them. This smooths out noise while keeping truly important regions bright.
Chapter 06

Grad-CAM — Gradient-weighted Class Activation Mapping

Grad-CAM is one of the most popular interpretability techniques. Unlike noisy saliency maps, it produces smooth, class-specific heatmaps highlighting which regions the network focused on.

💡
Beginner Analogy

Imagine a thermal camera showing "hot spots" where the AI is looking. Ask "Is this a cat?" and hot spots appear over the cat. Ask "Is this a dog?" and they shift to the dog. That's Grad-CAM!

How Grad-CAM Works

The key insight: the last convolutional layer has the best mix of spatial information (where) and semantic information (what). Grad-CAM uses gradients flowing into this layer to find which feature maps matter for a given class.

1
Forward Pass

Feed the image through the CNN. Record feature maps \(A^k\) from the last convolutional layer.

2
Backward Pass

Compute gradients of the target class score \(y^c\) w.r.t. each feature map \(A^k\).

3
Compute Importance Weights

Global-average-pool the gradients to get one weight \(\alpha_k^c\) per feature map.

4
Weighted Combination + ReLU

Multiply each feature map by its weight, sum up, and apply ReLU to keep only positive influences.

Step 1 — Importance Weights
$$\alpha_k^c = \frac{1}{Z} \sum_{i}\sum_{j} \frac{\partial \, y^c}{\partial \, A^k_{ij}}$$
α_k^c tells us how important feature map k is for class c. Computed by global-average-pooling gradients over spatial dimensions (i, j). Z = total pixels in the feature map.
Step 2 — Generate Heatmap
$$L_{\text{Grad-CAM}}^c = \text{ReLU}\!\left(\sum_{k} \alpha_k^c \cdot A^k\right)$$
Weighted sum of feature maps A^k using weights α_k^c, then ReLU keeps only positive contributions. Result: a coarse heatmap showing where the network "looked."

PyTorch Implementation

python — grad_cam.py
import torch
import torch.nn.functional as F

# Hook to capture feature maps and gradients
feature_maps = {}
gradients = {}

def forward_hook(module, inp, output):
    feature_maps['val'] = output

def backward_hook(module, grad_in, grad_out):
    gradients['val'] = grad_out[0]

# Register hooks on the last conv layer
last_conv = model.features[-1]
last_conv.register_forward_hook(forward_hook)
last_conv.register_full_backward_hook(backward_hook)

# Forward + backward
output = model(image)
class_score = output[0, target_class]
model.zero_grad()
class_score.backward()

# Compute Grad-CAM
grads = gradients['val']           # [1, K, H, W]
fmaps = feature_maps['val']        # [1, K, H, W]
weights = grads.mean(dim=[2,3])   # Global avg pool

cam = (weights[:,:,None,None] * fmaps).sum(dim=1)
cam = F.relu(cam)                   # Only positive
cam = cam / cam.max()               # Normalise

Chapter 07

Deconvolution & Guided Backpropagation

These techniques produce sharper, cleaner visualisations by modifying how gradients flow backwards through the network.

Deconvolution (Zeiler & Fergus, 2014)

The idea is to reverse the CNN's operations — projecting a neuron's activation back to input pixel space to see what pattern activated it.

Key: How ReLU is Handled Differently

In a normal backward pass, gradients are zero wherever the forward input to ReLU was negative. Each method handles this differently:

Standard Backprop

Mask gradients based on forward pass activations. If input was < 0, gradient = 0.

Deconvolution

Mask based on backward signal. If reconstructed gradient < 0, set to 0.

Guided Backpropagation

Guided backprop combines both masks — zeroes out gradients when either the forward activation OR the backward signal is negative. This produces the sharpest visualisations.

Guided Backpropagation Rule
$$\frac{\partial\, \text{out}}{\partial\, x_i} = \frac{\partial\, \text{out}}{\partial\, f_i} \cdot \mathbb{1}(f_i > 0) \cdot \mathbb{1}\!\left(\frac{\partial\, \text{out}}{\partial\, f_i} > 0\right)$$
Gradient passes back only when both forward activation f_i > 0 and incoming gradient is positive. 𝟙 = indicator function (1 if true, 0 if false).

Visual Comparison

Comparison of gradient visualisation quality across the three methods.

Chapter 08

LIME & SHAP — Model-Agnostic Interpretability

All previous techniques are model-specific — they require peeking inside the network. But what if we want to explain any model, even one we can't open? That's where model-agnostic methods come in.

LIME — Local Interpretable Model-Agnostic Explanations

💡
Beginner Analogy

Imagine you can't open a locked machine, but you can poke it with different inputs and observe outputs. By systematically turning parts of the input on and off, you figure out which parts matter most. That's LIME!

1
Segment the Image

Break the image into "superpixels" — groups of nearby similar pixels.

2
Create Perturbed Versions

Randomly grey out some superpixels to create many modified versions of the image.

3
Query the Model

Feed all modified images to the black-box model and record predictions.

4
Fit a Simple Model

Train a simple linear model on perturbation results. Its weights reveal which superpixels matter most.

Visual: How LIME Works

LIME workflow: segment, perturb, query, and fit a simple interpretable model to explain the black box.
LIME Objective
$$\xi(x) = \arg\min_{g \in G} \; \mathcal{L}\!\left(f, g, \pi_x\right) + \Omega(g)$$
f = complex model, g = simple interpretable model, π_x = weighting function (nearby perturbations get higher weight), Ω(g) = complexity penalty to keep explanations simple.

SHAP — SHapley Additive exPlanations

SHAP uses Shapley values from game theory to fairly distribute the model's prediction among input features: "How much credit does each feature deserve?"

💡
Beginner Analogy

Imagine a group project graded 95%. SHAP figures out each member's contribution: Alice (+20%), Bob (+15%), Carol (-5% for errors). It's fair because it considers every possible combination of team members.

Shapley Value
$$\phi_i = \sum_{S \subseteq N \setminus \{i\}} \frac{|S|!\;(|N|-|S|-1)!}{|N|!} \left[f(S \cup \{i\}) - f(S)\right]$$
For each feature i, compute its marginal contribution [f(S ∪ {i}) - f(S)] over all possible subsets S. The combinatorial weighting ensures fairness. N = set of all features.

LIME vs SHAP Comparison

AspectLIMESHAP
ApproachFit a local linear modelGame-theoretic fair allocation
SpeedFaster (sampling-based)Slower (exact requires all subsets)
ConsistencyCan vary across runsTheoretically guaranteed
ScopeLocal explanations onlyLocal + can aggregate to global
Model-Agnostic?YesYes (with model-specific speedups)
python — lime_example.py
from lime import lime_image

explainer = lime_image.LimeImageExplainer()

explanation = explainer.explain_instance(
    image,                     # The image to explain
    model.predict,              # Model's prediction function
    top_labels=5,
    num_samples=1000,          # Number of perturbations
    hide_color=0               # Grey out hidden regions
)

temp, mask = explanation.get_image_and_mask(
    explanation.top_labels[0],
    positive_only=True,
    num_features=5

Chapter 09

t-SNE — Visualising High-Dimensional Representations

Previous techniques focus on individual predictions. Sometimes we want to understand the network's overall representation — how it organises and clusters inputs internally. t-SNE does this.

💡
Beginner Analogy

Imagine 10,000 students each described by 512 exam scores. You can't visualise 512 dimensions! t-SNE is a magical projector that squishes 512D down to 2D on a flat screen, keeping similar students near each other.

How t-SNE Works

1
Compute High-D Similarities

For each pair of points, compute similarity using a Gaussian kernel. Nearby points get high similarity.

2
Initialise Random 2D Positions

Place all points randomly on a 2D canvas.

3
Compute 2D Similarities

Compute similarities in 2D using a Student's t-distribution (heavier tails than Gaussian).

4
Minimise KL Divergence

Move 2D points until 2D similarities match high-D similarities as closely as possible.

High-D Similarity (Gaussian)
$$p_{j|i} = \frac{\exp\!\left(-\|x_i - x_j\|^2 / 2\sigma_i^2\right)}{\sum_{k \neq i} \exp\!\left(-\|x_i - x_k\|^2 / 2\sigma_i^2\right)}$$
Measures how likely x_j is a neighbour of x_i in high-D space. Closer points = higher probability. σ_i is chosen based on the "perplexity" parameter.
Low-D Similarity (Student-t)
$$q_{ij} = \frac{\left(1 + \|y_i - y_j\|^2\right)^{-1}}{\sum_{k \neq l} \left(1 + \|y_k - y_l\|^2\right)^{-1}}$$
In 2D, similarities use a Student's t-distribution with 1 degree of freedom. Heavier tails prevent the "crowding problem" where all points collapse into the centre.
Objective: KL Divergence
$$C = \text{KL}(P \| Q) = \sum_{i} \sum_{j} p_{ij} \log \frac{p_{ij}}{q_{ij}}$$
t-SNE minimises KL divergence between high-D distribution P and 2D distribution Q, ensuring the 2D map faithfully represents neighbourhood structure.

Visual: What t-SNE Output Looks Like

Simulated t-SNE plot: each colour = a different class. Similar items cluster together.

Code Example

python — tsne_visualisation.py
from sklearn.manifold import TSNE
import matplotlib.pyplot as plt

# Extract features from last hidden layer
features = []
labels = []

with torch.no_grad():
    for images, lbls in dataloader:
        feat = model.features(images)
        feat = feat.view(feat.size(0), -1)
        features.append(feat)
        labels.extend(lbls.tolist())

features = torch.cat(features).numpy()

# Run t-SNE
tsne = TSNE(n_components=2, perplexity=30, n_iter=1000)
embeddings = tsne.fit_transform(features)

# Plot
plt.scatter(embeddings[:,0], embeddings[:,1],
           c=labels, cmap='tab10', s=5, alpha=0.7)
plt.title("t-SNE of CNN Features")
plt.show()
⚠️ t-SNE Pitfalls

Cluster sizes don't mean anything — t-SNE can artificially shrink or expand clusters. Distances between clusters aren't meaningful either. Always run multiple times with different random seeds. The perplexity (typically 5–50) controls balance between local and global structure.

🧠 The Big Picture

Interpretability isn't just academic — it's a requirement for deploying AI responsibly. As models become more powerful, understanding and trusting them becomes critical.

Best practice: use multiple techniques together. Grad-CAM for quick visual overview, SHAP for rigorous feature attribution, and t-SNE for understanding internal representations.

🚀 Quick Decision Guide

Need a quick visual check? → Grad-CAM
Need pixel-level detail? → Guided Backprop or Guided Grad-CAM
Working with a non-CNN model? → LIME or SHAP
Want to see global clustering? → t-SNE
Need theoretical guarantees? → SHAP
