Titanic Simple Neural Network

Model: Linear(12,12) → Sigmoid → Linear(12,8) → Sigmoid → Linear(8,2) → Softmax(dim=1); CrossEntropyLoss, Adam, lr=0.1.

Part (a) · 1 mark
How many inputs and outputs does this network have?

Inputs: 12. First layer is Linear(12,12) → 12 input features.
Outputs: 2. Last layer is Linear(8,2) → 2 outputs (2-class probabilities: survived / didn't).

Part (b) · 3 marks
Calculate how many parameters are in this model.

Formula per Linear(m,n): \(m\cdot n + n\) (weights + biases).

  • linear1: \(12\cdot 12 + 12 = 144 + 12 = \mathbf{156}\)
  • linear2: \(12\cdot 8 + 8 = 96 + 8 = \mathbf{104}\)
  • linear3: \(8\cdot 2 + 2 = 16 + 2 = \mathbf{18}\)

Total: \(156 + 104 + 18 = 278\) parameters. Sigmoid and Softmax have no trainable parameters.

Part (c) · 3 marks
Name and describe, in physical terms, two key innovations Adam has over plain SGD.

1. Momentum (1st moment). Adam keeps a running average \(m_t\) of past gradients. Physically, this is like giving the optimiser inertia — a ball rolling downhill. Consistent gradient direction accelerates the update, while zig-zagging gradients cancel out. This smooths descent in narrow valleys and keeps moving across plateaus.

2. Adaptive per-parameter learning rate (2nd moment). Adam tracks a running average \(v_t\) of squared gradients and divides the step by \(\sqrt{v_t}\). Physically, parameters with consistently large gradients take small steps, while those with small gradients take large steps — each parameter effectively has its own learning rate that adapts to the local curvature.

Part (d) · 3 marks
Draw one example of a loss surface with two weights that demonstrates both innovations.

A classic example: an elongated narrow valley (eccentric bowl). Imagine a 2D surface where curvature along \(w_1\) is huge (steep walls) but curvature along \(w_2\) is tiny (gentle floor), with the minimum far along the \(w_2\) axis.

  w2 ─────────────►
  ╔═══════════════════════╗
  ║ ▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒ ║   steep in w1

  • Momentum benefit: Plain SGD zig-zags violently across the steep \(w_1\) direction and creeps along the gentle \(w_2\) direction. Momentum averages past gradients so the \(w_1\)-oscillations cancel while the \(w_2\)-direction accumulates speed — converges faster and smoother.
  • Adaptive-LR benefit: Because gradients in \(w_1\) are huge, \(\sqrt{v_t^{(1)}}\) is large and the effective LR in that direction shrinks; in \(w_2\) gradients are tiny so the effective LR grows. Both axes now converge at similar speeds.
Part (e) · 5 marks
Compare Saliency Analysis, SmoothGrad and Integrated Gradients.

Saliency. \(S(x)=|\partial F_c(x)/\partial x|\). Cheap (one backward pass), model-agnostic. Tells you the model's sensitivity to each input pixel. Weaknesses: noisy, sensitive to small input changes, and saturates — a neuron firing at its maximum has near-zero gradient even though it dominates the prediction.

SmoothGrad. Add small Gaussian noise to the input \(N\) times, compute saliency each time, average: \(\frac{1}{N}\sum_i S(x+\epsilon_i)\). Result is visually much cleaner, removing high-frequency noise. Costs \(N\times\) more compute. Still suffers from saturation because it averages the same flawed measure.

Integrated Gradients. Integrate gradients along a straight-line path from a baseline \(x'\) (e.g. a black image) to the input \(x\): \(\mathrm{IG}_i=(x_i-x'_i)\int_0^1 \partial F(x'+\alpha(x-x'))/\partial x_i\, d\alpha\). Measures contribution rather than sensitivity; satisfies completeness (attributions sum to \(F(x)-F(x')\)) and implementation invariance. Fixes saturation because it sums gradient along the path, visiting regions where the gradient is still informative. More expensive than saliency, and requires a meaningful baseline choice.

Part (f) · 5 marks
Comment on a coding mistake and three ways to fix it.

Mistake: The model ends with nn.Softmax(dim=1), but the loss is nn.CrossEntropyLoss(). PyTorch's CrossEntropyLoss applies LogSoftmax internally — so feeding softmax-probabilities through it is equivalent to applying softmax twice, producing numerically unstable, miscalibrated gradients.

Three fixes

  1. Remove the final Softmax layer and let the model output raw logits; keep CrossEntropyLoss. This is idiomatic PyTorch.
  2. Keep the Softmax but change the loss to nn.NLLLoss combined with log(probs) — or replace Softmax with nn.LogSoftmax(dim=1) and use nn.NLLLoss(). Mathematically equivalent to option 1.
  3. Keep the Softmax and switch to one-hot targets with BCE-style loss: use nn.BCELoss on the probabilities, with targets one-hot encoded. Produces a near-equivalent classifier, though slightly different gradient dynamics.

ConvNet on an 18×18 colour image

Architecture: Conv1 (8 filters, 3×3, stride 1, pad 1) → ReLU → MaxPool1 (2×2, stride 2) → FC1+ReLU → FC2.

Part (a) · 4 marks
How many neurons in FC1?

Input: 3×18×18.

  • Conv1: pad=1, kernel=3, stride=1, \(\lfloor(18+2-3)/1\rfloor+1 = 18\). With 8 filters → 8×18×18.
  • MaxPool1: kernel=2, stride=2 → \(\lfloor(18-2)/2\rfloor+1 = 9\) → 8×9×9.
  • Flatten for FC1: \(8\cdot 9\cdot 9 = \mathbf{648}\) neurons.
Part (b) · 3 marks
Receptive field of a central MaxPool1 output.

Work backwards from the pool output.

  • MaxPool 2×2 draws from a 2×2 window of Conv1's output.
  • Each Conv1 output pixel sees a 3×3 patch of the image.
  • Two adjacent 3×3 patches one pixel apart cover \(3+1 = 4\) pixels of input in each spatial direction.

Receptive field = 4×4 pixels of the input image.

Part (e) · 2 marks
For regression of annual profit, what activation and loss for FC2?

Activation: linear / identity — we need an unbounded real-valued output (profit can be any real number, positive or negative). Sigmoid/tanh/ReLU would all clip the range.
Loss: MSE (nn.MSELoss). L1 / Huber are acceptable alternatives if robustness to outliers is prioritised.

Part (f) · 5 marks
Ethical risks of using a ConvNet trained on management photos to predict and recruit senior management.
  • Correlation, not causation. The model learns statistical correlates of "existing profitable managers" — almost certainly demographics (age, gender, ethnicity, dress, attractiveness) — not genuine ability. Predicting profits from faces is pseudoscientific (physiognomy).
  • Historical bias baked in. Senior management is historically dominated by particular demographic groups. The network will memorise and reproduce that demographic profile, so women, ethnic minorities, and disabled candidates will be systematically under-scored. This is direct discrimination — illegal under UK Equality Act 2010 and EU AI Act.
  • Runaway feedback loop. The system retrains only on new recruits "that did increase profits" — which the system itself selected. Any bias in the original model is amplified each iteration: the training distribution narrows toward a self-selected clique. Diversity collapses, predictions become overconfident on that clique and unreliable on anyone else.
  • Spurious causal direction. A new recruit's contribution to profit depends on market conditions, team, strategy — not on their face. The system conflates "was recruited into a profitable era" with "caused profit".
  • No accountability / explainability. Rejected candidates have no meaningful reason given. Under GDPR Art. 22 and the EU AI Act (high-risk category: employment), such an automated hiring decision requires the right to human review and explanation.
  • Privacy & consent. Training and scoring on facial images collected from management may exceed the consent under which those photos were taken.
Question 3 · 20 marks

Vanilla RNN & GRU

\(h_t = \mathrm{ReLU}(w h_{t-1} + u x_t)\), \(y_t = c + v h_t\), \(L = \tfrac12(y-y_t)^2\). Parameters \(u=-2\), \(v=3\), \(w=0.5\), \(c=-1\), \(h_{-1}=0\); inputs \(x_0=-1\), \(x_1=+1\).

Part (a) · 5 marks
Forward pass over two timesteps.

t = 0

\(z_0 = w h_{-1} + u x_0 = 0.5\cdot 0 + (-2)\cdot(-1) = 2\).
\(h_0 = \mathrm{ReLU}(2) = 2\).
\(y_0 = c + v h_0 = -1 + 3\cdot 2 = \mathbf{5}\).

t = 1

\(z_1 = 0.5\cdot 2 + (-2)\cdot 1 = 1 - 2 = -1\).
\(h_1 = \mathrm{ReLU}(-1) = 0\).
\(y_1 = -1 + 3\cdot 0 = \mathbf{-1}\).

Part (b) · 3 marks
\(\partial y_0/\partial x_0\) for both ReLU regimes.

\(y_0 = c + v\cdot\mathrm{ReLU}(w h_{-1} + u x_0)\).

  • ReLU input ≤ 0 → \(h_0 = 0\), so \(\partial y_0/\partial x_0 = \mathbf{0}\).
  • ReLU input > 0 → \(h_0 = w h_{-1} + u x_0\), so \(\partial y_0/\partial x_0 = v\cdot u = \mathbf{uv} = (-2)(3) = -6\) for the given values.
Part (c) · 3 marks
\(\partial y_1/\partial x_0\) considering all ReLU states.

Chain: \(y_1\to h_1\to z_1 \to h_0 \to z_0 \to x_0\). Let \(r_t = 1\) if ReLU is active at step \(t\), else 0. Then:

\[ \frac{\partial y_1}{\partial x_0} = v\cdot r_1\cdot w\cdot r_0\cdot u = u\,v\,w\, r_0 r_1 \]

  • Both ReLUs active (\(r_0=r_1=1\)): \(uvw = (-2)(3)(0.5) = \mathbf{-3}\).
  • Either ReLU inactive: \(\mathbf{0}\).
Part (d) · 3 marks
Why compute \(\partial y_t/\partial x_0\) for different \(t\)?

Three reasons:

  1. Training via BPTT. To update weights that produced \(x_0\)'s effect on all later outputs, backprop-through-time must sum \(\partial L/\partial \theta\) across \(t\), which requires \(\partial y_t/\partial x_0\) for every \(t\).
  2. Diagnose vanishing / exploding gradients. The derivative has the form \((wr)^t u v\). If \(|w|r < 1\), gradients decay exponentially with \(t\) (vanishing — can't learn long-range dependencies); if >1 they explode.
  3. Interpretability / sensitivity analysis. Understand how far into the future input \(x_0\) actually influences predictions — important for deciding RNN capacity vs LSTM/Transformer alternatives.
Part (e) · 3 marks
Why sigmoid in some parts of a GRU and tanh in others?

Sigmoid (output 0–1) acts as a soft gate. In the GRU, the reset gate \(r_t\) and update gate \(z_t\) are sigmoids because their output multiplies another vector element-wise and controls "how much of this signal passes through" — 0 = block, 1 = pass. A bounded \([0,1]\) range is exactly the semantics of a switch.

Tanh (output -1 to +1) produces the candidate update. The candidate hidden state \(\tilde h_t\) must be able to take signed values so it can push the state up or down. Tanh is zero-centred, stabilising learning, and its bounded range prevents activations from exploding in a recurrent loop.

Part (f) · 3 marks
A major advantage of GRU over vanilla RNN, and which part enables it.

Advantage: GRU learns much longer-range dependencies without suffering vanishing gradients.

What enables it: the update gate \(z_t\) and the additive combination \(h_t = (1-z_t)\odot h_{t-1} + z_t\odot \tilde h_t\). When \(z_t\approx 0\) the previous state passes through almost unchanged, so during backprop the gradient flows through the linear identity path with derivative ≈ 1 — no repeated multiplication by \(w\phi'\) as in a vanilla RNN. This breaks the vanishing-gradient chain.

Two models processing torch.size([3,1,224,224])

model1: Flatten → Linear(224·224, 28·28) → Sigmoid.
model2: three blocks of Conv2d(in,out,3,stride=2,padding=1) → Sigmoid, with channel counts 1→2→4→1, then Flatten.

Part (a) · 2 marks
What does the tensor shape [3,1,224,224] represent?

A mini-batch of 3 grayscale images, each 224×224 pixels.

  • Dim 0 (size 3): batch size — three separate examples processed in parallel.
  • Dim 1 (size 1): channel count — 1 means grayscale (3 would be RGB colour).
  • Dim 2 (size 224): image height in pixels.
  • Dim 3 (size 224): image width in pixels.
Part (b) · 4 marks
Parameter count of model1.

Input features: \(224\cdot 224 = 50\,176\). Output features: \(28\cdot 28 = 784\).

Linear \((m,n)\) = \(m n + n\) params:

\[ 50\,176 \times 784 + 784 = 39\,337\,984 + 784 = \mathbf{39\,338\,768} \]

Sigmoid and Flatten are parameter-free. ~39.3 million parameters — a huge number for a single layer.

Part (c) · 4 marks
Parameter count of model2.

Conv2d\((c_\text{in}, c_\text{out}, k)\) = \(c_\text{out}(c_\text{in}\cdot k\cdot k + 1)\) params.

  • conv1: \(2(1\cdot 3\cdot 3 + 1) = 2\cdot 10 = \mathbf{20}\)
  • conv2: \(4(2\cdot 3\cdot 3 + 1) = 4\cdot 19 = \mathbf{76}\)
  • conv3: \(1(4\cdot 3\cdot 3 + 1) = 1\cdot 37 = \mathbf{37}\)

Total: \(20 + 76 + 37 = 133\) parameters. 300,000× fewer than model1 for an input of the same size.

Part (e) · 3 marks
Advantages and disadvantages.

model1 (fully connected)

  • ✓ Simple, one line of code.
  • ✗ Enormous parameter count (~39M) → slow training, heavy overfitting risk, huge memory.
  • ✗ Ignores spatial locality — pixel (0,0) and pixel (0,1) are treated as independent features.
  • ✗ Not translation-invariant — a shifted image needs different weights.
  • ✗ Fixed to 224×224; any other size crashes.

model2 (convolutional)

  • ✓ 300,000× fewer parameters → faster, less overfitting.
  • ✓ Exploits spatial locality and translation equivariance.
  • ✓ Works on images of any size (see part f).
  • ✗ Slightly more complex to reason about.
  • ✗ Uses sigmoid — saturates, vanishing gradients; modern CNNs use ReLU + batch-norm.

Convolution, max-pool, manual backprop of a 2-layer MLP

Part (a) · 3 marks
Apply a (kernel + image from Figure 1 — same horizontal-edge pattern as 2025 Q2). (Using typical image/kernel from this year's booklet.)

Method: slide the kernel over the image, at each position compute \(\sum K_{ij}\cdot I_{ij}\). With padding 0, stride 1 and a 3×3 kernel on a 5×5 image, output is 3×3. Exact values depend on the printed figure; apply dot-product at each location.

b General approach, since the figure isn't text-extractable: shown explicitly in the 2025 Q2(a)(ii) solution below for the same kernel type.

Part (b) · 1 mark
2×2 max-pool, stride 1 on the previous result.

For a 3×3 input with pool 2×2, stride 1: output is 2×2. Each cell = max of its 2×2 window. If the conv output is (say) \([[0,0,0],[3,3,3],[0,0,0]]\), max-pool yields \([[3,3],[3,3]]\).

Part (c) · 3 marks
Write \(y = y(\mathbf{h})\), \(\mathbf{h} = \mathbf{h}(\mathbf{z})\), \(\mathbf{z}=\mathbf{z}(\mathbf{x})\) in matrix form.

\(\mathbf{z} = M\mathbf{x}\) (no bias assumed in the figure).
\(\mathbf{h} = \sigma(\mathbf{z})\) — element-wise sigmoid.
\(y = N\mathbf{h}\) (scalar output from a 1×2 row matrix).

Part (d) · 4 marks
Write \(M\) and \(N\) from the figure.

From the standard figure used in this series: weights on edges \(x_1\to z_1, x_2\to z_1, x_1\to z_2, x_2\to z_2\) form \(M\); \(h_1\to y, h_2\to y\) form \(N\). Using typical labels \(m_{11}, m_{12}, m_{21}, m_{22}\) and \(n_1, n_2\):

\[ M = \begin{pmatrix} m_{11} & m_{12} \\ m_{21} & m_{22} \end{pmatrix}, \qquad N = \begin{pmatrix} n_1 & n_2 \end{pmatrix} \]

Part (e) · 2 marks
\(\nabla_\mathbf{h} y\).

\(y = n_1 h_1 + n_2 h_2\), so \(\nabla_\mathbf{h} y = (n_1, n_2)^\top = N^\top\).

Part (f) · 3 marks
\(\nabla_\mathbf{z} y\) by back-prop.

Apply chain rule through the element-wise sigmoid: \(\partial h_i/\partial z_i = \sigma(z_i)(1-\sigma(z_i)) = h_i(1-h_i)\).

\[ \nabla_\mathbf{z} y = \big(n_1\, h_1(1-h_1),\ n_2\, h_2(1-h_2)\big)^\top \]

\(\partial y/\partial x_1\).

\(\partial y/\partial x_1 = \sum_i (\partial y/\partial z_i)(\partial z_i/\partial x_1)\). With \(z_i = m_{i1}x_1 + m_{i2}x_2\), \(\partial z_i/\partial x_1 = m_{i1}\).

\[ \frac{\partial y}{\partial x_1} = n_1 h_1(1-h_1)\,m_{11} + n_2 h_2(1-h_2)\,m_{21} \]

Question 3 · 20 marks

City-centre surveillance ConvNet

Part (a) · 3 marks
Characteristics of this sequence learning problem.
  • Long, variable-length input — a video is a sequence of image frames (spatial + temporal).
  • Temporal dependencies matter — whether behaviour is criminal often depends on context across many seconds (e.g. approach → strike). A single frame isn't enough.
  • Multiple outputs per input — per-person segmentation (dense pixel prediction) plus per-person behaviour classification. Both spatial and classification tasks must be solved jointly.
  • Highly imbalanced classes — criminal events are rare compared to normal.
  • Noisy, partial, occluded observations — crowds, bad lighting, occlusions.
  • High stakes — false positives trigger police deployment; false negatives miss real crime.
Part (b) · 4 marks
How could neural attention improve the system?
  • Spatial attention focuses the classifier on the person / action under judgement rather than the background — higher accuracy in crowded scenes.
  • Temporal attention lets the model weight the most informative frames in a clip (e.g. the actual strike rather than idle walking). Improves performance on long videos where most frames are uninformative.
  • Self-attention across persons captures group context — a fight involves several people interacting; attention can model these relations explicitly.
  • Interpretability. Attention weights form a heatmap showing which frames / people / regions drove the decision — a human reviewer can verify the model's reasoning and prosecutors have a visual justification to present. Essential for accountability in a legal setting.
  • Handles variable sequence length gracefully without the truncation issues of plain RNNs.
Part (c) · 3 marks
One alternative visualisation technique.

Grad-CAM. Computes \(\alpha_k^c = \frac{1}{Z}\sum_{ij}\partial Y^c/\partial A^k_{ij}\) and overlays \(\mathrm{ReLU}(\sum_k \alpha_k^c A^k)\) on the input image to highlight class-discriminative regions.

Advantages

  • Works on any CNN — no architectural change needed; retrofits to an already-deployed model.
  • Fast — one forward + one backward pass per image.
  • Visually intuitive: a heatmap that non-technical reviewers (police, lawyers) can read.

Disadvantages

  • Coarse resolution — operates at the final conv feature-map resolution (e.g. 7×7), so highlights regions, not pixels.
  • Only highlights "where", not "why" — doesn't separate whether the model attended to a weapon, a gesture, or a clothing choice.
  • Sensitive to class choice: maps for "criminal" and "antisocial" may overlap.
Part (d) · 3 marks
Risks of the auto-annotation feedback loop.
  • Runaway bias amplification. Any systematic error (e.g. over-classifying particular demographics as criminal) is labelled into the next training set and gets reinforced. Within months the model's decisions become self-confirming regardless of ground truth.
  • Distribution drift. The system only auto-annotates confident cases, leaving hard / rare cases under-represented — so the model never improves on the cases where it's weakest.
  • Human-annotator bias transfer. Human annotators resolving the "uncertain" cases introduce their own biases (fatigue, prejudice, fashion trends). Those biases become the new training signal.
  • Label leakage. If policing action follows a classification, future frames capturing that action will be labelled consistently — training becomes self-fulfilling rather than causal.
  • No mechanism for correcting past mistakes — a person wrongly flagged contributes their frames to future training, poisoning the dataset permanently.
Part (e) · 7 marks
Other ethical and practical risks.

Ethical

  • Mass surveillance — continuous classification of citizens in public space is a prima-facie intrusion on privacy rights (ECHR Art. 8, UK Data Protection Act, EU AI Act's "real-time biometric identification" restrictions).
  • Disparate impact — training data likely under-represents minority groups or over-represents them in "criminal" classes, leading to disproportionate policing — a civil-rights violation.
  • Chilling effect on free expression and assembly — people change their behaviour knowing they're being classified.
  • Due-process erosion — an automated classification becomes evidence; defendants have no practical way to contest the algorithm.
  • Consent impossible — citizens walking in public did not consent to this processing.

Practical

  • High false-positive cost — innocent people detained. At even 99% specificity, across millions of frames per day, this is thousands of wrongful interventions.
  • Adversarial attacks — clothing patterns, accessories, or printable adversarial patches fool classifiers; criminals can deliberately evade.
  • Robustness — weather, lighting, camera angle, crowd density all degrade accuracy far below lab benchmarks.
  • Explainability gap — the operational officer gets a label with no robust justification; they cannot override responsibly.
  • Security / hack risk — centralised model + camera feeds is an attractive attack surface; compromised cameras poison training data.
  • Regulatory non-compliance — EU AI Act categorises this as "high-risk" or "prohibited" (biometric real-time categorisation in public); UK ICO has issued binding guidance against similar systems.
  • Recommendation: the system should not be deployed. If it must be, mandatory human-in-the-loop for every adverse decision, independent audit, rigorous per-demographic performance reports, and a mechanism for citizens to query and contest classifications.

Part (c) · 4 marks
Role of attention in linear RNNs vs RNNs without attention.

Without attention, an RNN decoder sees only the final hidden state \(h_T\) (a fixed-size summary). For long sequences, \(h_T\) becomes a bottleneck: information from early timesteps is compressed, diluted, or lost through the recurrence.

With attention, the decoder has direct access to every encoder hidden state \(h_1,\dots,h_T\). At each output step it computes a weighted combination based on similarity between its current query and each encoder key. Benefits:

  • No information bottleneck — long-range dependencies preserved.
  • Selective focus — different output positions can attend to different input positions.
  • Parallelism — in linear models, all attention scores can be computed in a single matmul.
  • Interpretability — attention weights visualise which inputs drive each output.

A linear RNN (no non-linearity) without attention is expressively equivalent to a single linear transformation of the input sum — very limited. Adding attention gives it data-dependent selection, hugely expanding its capacity.

Part (d) · 2 marks
Computation of attention scores and context vectors.
  1. Attention scores: \(s_t = q^\top k_t\) (optionally scaled by \(1/\sqrt{d_k}\) and passed through softmax to produce attention weights \(\alpha_t\)).
  2. Context vector: \(\mathbf{c} = \sum_t \alpha_t v_t\) — the weighted sum of value vectors, with weights given by the softmax of the scores.
Part (e) · 4 marks
Real-world example showing attention's advantage.

Machine translation (English → French). Consider translating "the red book on the wooden table is mine" into French. Without attention, the decoder sees one fixed vector summarising the whole English sentence. To emit "rouge" the decoder needs to recover information about the adjective "red" several words back; for long sentences this is lossy.

With attention, when generating "rouge" the decoder's query picks out the encoder position where "red" was processed — attention weight near 1 on that token, near 0 elsewhere. When generating "table", attention shifts to "table". This matches the intuition that translation is alignment between source and target words.

Why it's particularly advantageous here: (i) French and English have different word order, so positional shift is essential; (ii) sentence length varies enormously; (iii) attention weights visualise a word-alignment matrix that linguists already understood pre-neural MT — interpretability bonus; (iv) translation quality scales almost linearly with sentence length under attention, while without attention, BLEU degrades rapidly past ~20 tokens.

Part (d.i) · 2 marks
Effect of too-large / too-small learning rate.

Too large: updates overshoot the minimum. Loss oscillates, possibly divergently; may produce NaNs if gradients explode. Training never converges.

Too small: loss decreases painfully slowly. The optimiser may stall in saddle points or flat regions because each step barely moves. May not reach a good minimum within the training budget.

Part (d.ii) · 3 marks
Momentum.

Momentum maintains a velocity vector \(v_t = \beta v_{t-1} + \nabla L\) and updates \(\mathbf{w}\leftarrow \mathbf{w}-\alpha v_t\) — i.e. it accumulates an exponentially-decaying average of past gradients. Physically, a ball rolling downhill.

Loss-surface types it helps with:

  • Narrow elongated valleys — SGD zig-zags across the short axis. Momentum cancels the oscillations and accelerates along the long axis.
  • Plateaus / saddles — tiny gradients would stall SGD; accumulated velocity pushes through.
  • Noisy gradients (small batches) — momentum averages out the noise and tracks the true gradient direction.
Part (d.iii) · 3 marks
Weight decay.

Weight decay is L2 regularisation implemented as a step-by-step shrinkage. At each update, in addition to the gradient descent step, the weights are multiplied by \((1-\alpha\lambda)\), i.e. \(\mathbf{w}\leftarrow(1-\alpha\lambda)\mathbf{w}-\alpha\nabla L\). Equivalent to adding \(\tfrac{\lambda}{2}\|\mathbf{w}\|^2\) to the loss.

What it helps with:

  • Overfitting. Penalises large weights → smoother, simpler decision functions. Especially important for our 151-parameter network on a small 2D dataset where it could easily memorise training points.
  • Feature decorrelation — prevents weights from cooperating in extreme, redundant ways.
  • Improved generalisation (lower variance in bias-variance decomposition).
Part (e) · 5 marks
The loss curve shows overfitting (training loss decreasing, validation loss rising). Four ways to fix.

The gap between training and validation loss, widening over epochs, indicates the model is memorising training data. Four specific fixes:

  1. Increase weight_decay. The current value 0.001 may be too weak. Try 0.01 — stronger L2 penalty shrinks weights, reducing effective capacity.
  2. Add Dropout. Insert nn.Dropout(p=0.3) between fc1→fc2 and fc2→fc3. Randomly zeroing activations during training prevents co-adaptation and acts as an ensemble of thinned networks.
  3. Early stopping. Monitor validation loss; stop training at the epoch where it starts rising. Most direct cure for overfitting caused by over-training.
  4. Reduce model capacity. Shrink hidden layers from 10 → 5 units, or drop one hidden layer entirely. With only 2 input features, 151 parameters is very overcapacitated.

(Other acceptable answers: more training data, data augmentation, ensembling, label smoothing, transfer learning from a pretrained model.)

Part (b.i) · 4 marks
Describe the two architectural parts of a GAN.

Generator \(G\)

A neural network that takes a random noise vector \(z\sim p_z\) (typically a standard Gaussian or uniform distribution, dim ~100) and produces a fake image \(G(z)\). Usually a deconvolutional / transposed-convolutional network: dense layer → reshape → repeated upsample+conv blocks, outputting a tensor of image shape (e.g. 3×64×64). Purpose: learn a differentiable mapping from a simple latent distribution to the data distribution, so new samples can be generated by drawing \(z\) from noise and running forward.

Discriminator \(D\)

A binary classifier that receives either a real image \(x\sim p_\text{data}\) or a fake \(G(z)\), and outputs \(D(\cdot)\in (0,1)\) = the probability the input is real. Typically a convolutional network (strided convs → flatten → dense → sigmoid). Purpose: tell real from fake; its gradient signal trains the generator. At equilibrium \(D\) outputs 0.5 everywhere (can't distinguish).

Part (b.ii) · 6 marks
Meaning of the terms in the GAN objective.

Equation 1: \(\min_G \max_D\ \mathbb{E}_{x\sim p_\text{data}(x)}[\log D(x)] + \mathbb{E}_{z\sim p_z(z)}[\log(1-D(G(z)))]\).

  • \(G\): the generator network (parameters \(\theta_G\)). Minimises the objective — tries to reduce \(\log(1-D(G(z)))\), i.e. fool \(D\).
  • \(D\): the discriminator network (parameters \(\theta_D\)). Maximises the objective — tries to classify reals as real and fakes as fake.
  • \(x\sim p_\text{data}(x)\): a real sample drawn from the true (unknown) data distribution. In practice, a minibatch drawn from the training set.
  • \(D(x)\): discriminator's estimated probability that real image \(x\) is real. \(D\) wants this ≈ 1; the term \(\log D(x)\) is maximised at 1.
  • \(z\sim p_z(z)\): a random latent vector drawn from a simple known prior (typically \(\mathcal{N}(0, I)\)). Acts as the "seed" for generation.
  • \(D(G(z))\): discriminator's estimated probability that the generated (fake) image is real. \(D\) wants this ≈ 0 (so \(\log(1-D(G(z)))\) ≈ 0, its maximum); \(G\) wants this ≈ 1 (so \(\log(1-D(G(z)))\) is strongly negative, minimising the objective).

Global equilibrium when \(p_G = p_\text{data}\) and \(D(x)=D(G(z))=1/2\) everywhere.

Part (a.ii) · 5 marks
Expressions for \(\partial y_0/\partial x_0\) in both ReLU regimes.

\(y_0 = c + v\cdot \mathrm{ReLU}(w h_{-1} + u x_0)\).

  • ReLU input ≤ 0: \(h_0 = 0\), so \(y_0 = c\) (constant in \(x_0\)). \(\boxed{\partial y_0/\partial x_0 = 0}\).
  • ReLU input > 0: \(h_0 = w h_{-1} + u x_0\) (linear passthrough). Chain: \(\partial y_0/\partial x_0 = \partial y_0/\partial h_0\cdot\partial h_0/\partial x_0 = v\cdot u = \boxed{uv}\).

For the given values \(u=-3, v=3\), the non-zero case gives \(uv = -9\). At \(x_0=-1\), ReLU input is 3 (positive) ⇒ \(\partial y_0/\partial x_0 = -9\).

Part (b) · 4 marks
Advantages and limitations of Transformer vs RNN for text.

Advantages of Transformer

  • Parallelism. All tokens are processed simultaneously — training on modern GPUs/TPUs is orders of magnitude faster than an RNN's sequential unrolling.
  • Long-range dependencies. Self-attention gives direct O(1)-path connections between any two tokens; RNNs must propagate information through \(T\) recurrent steps, causing vanishing gradients for long contexts.
  • Interpretability. Attention matrices visualise which tokens the model relates to which, useful for debugging and analysis.
  • Scales beautifully with data and compute — produced GPT, BERT, T5, and the entire modern NLP stack.

Limitations of Transformer

  • Quadratic cost in sequence length \(O(T^2)\) for both compute and memory — prohibitive for long documents without sparse/efficient attention variants.
  • Position-agnostic by default; needs explicit positional encoding (sinusoidal or learned) — an extra design choice RNNs don't need.
  • Data-hungry — lacks the inductive bias of sequential processing; needs large corpora to match an RNN's performance on small datasets.
  • Fixed context window in practice (e.g. 2048, 8192, 128k tokens); a stateful RNN has in principle unbounded memory.
Part (c) · 6 marks
Self-attention parameter count for \(l=1024, d_e=100, H=2, d_k=d_q=10, d_v=20, d_o=100\).

Self-attention has four weight matrices: \(W_Q, W_K, W_V\) per head, and an output projection \(W_O\). Ignoring biases (as usually stated), each is a matrix with no free-standing parameters beyond \(\text{rows}\times\text{cols}\).

Per head

  • \(W_Q^{(h)}\): projects \(d_e\)-dim embedding into \(d_q\) query dim → shape \((d_e, d_q) = (100, 10)\) → \(100\cdot 10 = 1000\) params.
  • \(W_K^{(h)}\): \((d_e, d_k) = (100, 10)\) → 1000 params.
  • \(W_V^{(h)}\): \((d_e, d_v) = (100, 20)\) → 2000 params.
  • Per-head total: \(1000+1000+2000 = 4000\).

All heads

\(H=2\): \(2\times 4000 = 8000\) parameters across all \(W_Q, W_K, W_V\).

Output projection

Concatenated head output has dim \(H\cdot d_v = 2\cdot 20 = 40\); \(W_O\) projects \(40\to d_o=100\): \(40\cdot 100 = 4000\) params.

Total

\[ \boxed{\ 8000 + 4000 = 12{,}000\ \text{parameters}\ } \]

Note: sequence length \(l=1024\) does not appear in the parameter count — attention weights are position-independent. Compute and activation memory scale with \(l\) (via the \(l\times l\) attention matrix), but trainable parameters don't.

One-page revision checklist
  • Parameters: Linear(m,n) = mn+n; Conv(c_in, c_out, k) = c_out(c_in·k²+1); Attention per head = d_e(2d_k + d_v).
  • Output size: ⌊(I+2P-F)/S⌋ + 1 — applies to conv and max-pool.
  • Loss ↔ activation: binary → sigmoid+BCE (or BCEWithLogits, no final sigmoid); multi-class → raw logits + CrossEntropyLoss (CE applies LogSoftmax internally); regression → linear + MSE.
  • ReLU gradient: 1 if input > 0, else 0. Vanilla RNN gradient: products of (w·r)-factors → vanishing/exploding.
  • GRU/LSTM: sigmoid = gate, tanh = candidate. Update gate enables identity path = fixes vanishing gradient.
  • Optimisation: momentum = 1st moment (inertia), adaptive LR = 2nd moment (scale by √v). Adam = both + bias correction.
  • Regularisation: weight_decay = L2, dropout, early stopping, data aug. Fight overfitting.
  • Viz: Saliency (fast, noisy, saturates) → SmoothGrad (denoises) → Integrated Gradients (no saturation, completeness).
  • Ethics: bias, feedback loops, accountability, explainability, privacy, disparate impact.
