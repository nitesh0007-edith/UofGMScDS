Training Models

Master gradient descent, SGD, momentum, and Adam—the optimization algorithms that make deep learning possible.

← Previous Next →

Gradient Descent

The core optimization algorithm: move parameters in the direction that decreases loss.

φ ← φ - α · ∇L(φ)

Where α is the learning rate and ∇L is the gradient (direction of steepest ascent).

🔑 Learning Rate

Too large → overshoots, unstable. Too small → slow convergence. Start with 0.001 for Adam, 0.01 for SGD.

Stochastic Gradient Descent

Instead of computing gradients on all data, use random mini-batches:

  • Batch GD: All data per update (slow)
  • SGD: One sample per update (noisy)
  • Mini-batch: 32-256 samples (best tradeoff)

Momentum

Accumulate past gradients to smooth updates and accelerate through valleys:

v ← βv + ∇L(φ)
φ ← φ - αv

β ≈ 0.9 is typical. Like a ball rolling downhill, building speed.

Adam Optimizer

The most popular optimizer—combines momentum with adaptive per-parameter learning rates:

📖 Adam Defaults

α = 0.001, β₁ = 0.9, β₂ = 0.999, ε = 1e-8. Works well out-of-the-box for most problems.

Key Takeaways

  • Gradient descent moves parameters to minimize loss
  • Mini-batch SGD balances speed and accuracy
  • Momentum smooths and accelerates training
  • Adam adapts learning rates automatically—great default
  • Learning rate schedules can improve final performance
