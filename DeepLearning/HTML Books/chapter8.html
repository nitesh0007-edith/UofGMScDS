Chapter 8

Measuring Performance

Learn how to properly evaluate models, understand the bias-variance tradeoff, and avoid common pitfalls in model assessment.

← Previous Next →

Train-Test Split

Never evaluate on training data! Split your dataset:

  • Training set (60-80%): Learn parameters
  • Validation set (10-20%): Tune hyperparameters, early stopping
  • Test set (10-20%): Final evaluation only
⚠️ Data Leakage

Never let test data influence training decisions. This includes: normalization statistics, feature selection, hyperparameter tuning. Always compute these on training data only.

Bias-Variance Tradeoff

Model error decomposes into three components:

Error = Bias² + Variance + Noise
  • Bias: Error from oversimplified assumptions (underfitting)
  • Variance: Error from sensitivity to training data (overfitting)
  • Noise: Irreducible error in the data
💡 The Tradeoff

Simple model: High bias, low variance (underfits)
Complex model: Low bias, high variance (overfits)
Just right: Balance that minimizes total error

Cross-Validation

Get more reliable estimates with limited data using k-fold CV:

  1. Split data into k folds
  2. Train on k-1 folds, validate on remaining fold
  3. Repeat k times, each fold serving as validation once
  4. Average the k validation scores

Common choice: k = 5 or k = 10

Classification Metrics

Confusion Matrix

Breaks down predictions into true positives (TP), false positives (FP), true negatives (TN), false negatives (FN).

Key Metrics

  • Accuracy: (TP + TN) / Total — can be misleading for imbalanced data
  • Precision: TP / (TP + FP) — "Of predicted positives, how many are correct?"
  • Recall: TP / (TP + FN) — "Of actual positives, how many did we find?"
  • F1 Score: 2 × (Precision × Recall) / (Precision + Recall)
  • AUC-ROC: Area under receiver operating characteristic curve
🔑 When to Use What

Balanced data: Accuracy is fine
Imbalanced data: Use F1, precision/recall, or AUC
Cost-sensitive: Tune threshold based on precision/recall tradeoff

Regression Metrics

  • MSE: Mean squared error (sensitive to outliers)
  • RMSE: √MSE (same units as target)
  • MAE: Mean absolute error (robust to outliers)
  • R²: Coefficient of determination (explained variance)

Key Takeaways

  • Always split data into train/validation/test sets
  • Bias-variance tradeoff: simple models underfit, complex overfit
  • Cross-validation gives more reliable estimates
  • Choose metrics appropriate for your problem
  • Accuracy can be misleading for imbalanced classes
  • Never peek at test data during development
