Loss Functions

Learn how to measure the gap between predictions and reality. The loss function is the compass that guides network training toward better solutions.

← Previous: Deep Networks Next: Training Models →

What is a Loss Function?

A loss function (also called cost function or objective function) measures how wrong our model's predictions are. Training aims to minimize this loss.

📖 Definition

Loss Function: A function L(y, ŷ) that quantifies the difference between the true value y and the predicted value ŷ. Lower loss = better predictions.

Properties of Good Loss Functions

  • Non-negative: L ≥ 0, with L = 0 for perfect predictions
  • Differentiable: We can compute gradients for optimization
  • Aligned with goals: Minimizing the loss should improve performance on our actual task

Losses for Regression

For regression tasks where we predict continuous values, several loss functions are commonly used.

Mean Squared Error (MSE)

The most common regression loss:

MSE = (1/n) Σᵢ (yᵢ - ŷᵢ)²
🔑 Properties of MSE

• Heavily penalizes large errors (squared term)
• Differentiable everywhere
• Corresponds to Maximum Likelihood under Gaussian noise
• Sensitive to outliers

Mean Absolute Error (MAE)

MAE = (1/n) Σᵢ |yᵢ - ŷᵢ|

MAE is more robust to outliers than MSE because it doesn't square errors. However, it's not differentiable at zero.

Huber Loss

A hybrid that combines the best of both:

Huber(y, ŷ) = { 0.5(y-ŷ)²    if |y-ŷ| ≤ δ
                  δ(|y-ŷ| - 0.5δ)   otherwise }

Uses MSE for small errors (smooth optimization) and MAE for large errors (outlier robustness).

Losses for Classification

For classification, we predict probabilities over classes. The losses are designed for this probabilistic output.

Binary Cross-Entropy

For binary classification with sigmoid output:

BCE = -[y·log(ŷ) + (1-y)·log(1-ŷ)]

where y ∈ {0, 1} is the true label and ŷ ∈ (0, 1) is the predicted probability.

💡 Intuition

• If y = 1: Loss = -log(ŷ). High ŷ → low loss.
• If y = 0: Loss = -log(1-ŷ). Low ŷ → low loss.

The loss is very high when we're confident and wrong (e.g., predict 0.99 when y = 0).

Categorical Cross-Entropy

For multi-class classification with softmax output:

CCE = -Σₖ yₖ·log(ŷₖ)

where y is a one-hot encoded vector and ŷ is the softmax probability distribution. Since only one yₖ = 1, this simplifies to -log(ŷc) where c is the correct class.

⚠️ Numerical Stability

Computing log(ŷ) can cause issues when ŷ is very close to 0 (log(0) = -∞). In practice, we add a small epsilon: log(ŷ + 1e-7) or use numerically stable implementations that combine softmax and cross-entropy.

The Probabilistic View

Loss functions have a beautiful connection to probability theory through Maximum Likelihood Estimation.

Maximum Likelihood Principle

Given data, find parameters that maximize the probability of observing that data:

φ* = argmax P(data | φ) = argmax Πᵢ P(yᵢ | xᵢ, φ)

Taking the negative log (to convert product to sum and maximize to minimize):

φ* = argmin -Σᵢ log P(yᵢ | xᵢ, φ)

Connection to Common Losses

🔑 Key Connections

MSE = MLE assuming Gaussian noise
P(y | x, φ) = Normal(f(x, φ), σ²)

Cross-Entropy = MLE for categorical distribution
P(y = k | x, φ) = softmax(f(x, φ))ₖ

This connection means our loss functions are principled—they're the right thing to minimize if our probability assumptions are correct.

The Loss Landscape

The loss function defines a surface over the parameter space. Training is about navigating this landscape to find low points.

Gradient Descent on Loss Landscape

Landscape Features

  • Global minimum: The lowest point—what we ideally want to find
  • Local minima: Low points that aren't the lowest—can trap optimization
  • Saddle points: Points that go up in some directions, down in others
  • Plateaus: Flat regions where gradients are tiny
🔑 High-Dimensional Landscapes

In high dimensions (millions of parameters), true local minima are rare. Most "stuck" points are saddle points, which optimization can escape. This is one reason deep learning works despite non-convex landscapes.

Choosing the Right Loss

The choice of loss function should match your problem:

Quick Reference

Task Output Activation Loss Function
Regression None (linear) MSE or Huber
Binary Classification Sigmoid Binary Cross-Entropy
Multi-class Classification Softmax Categorical Cross-Entropy
Multi-label Classification Sigmoid (per class) Binary Cross-Entropy (summed)

Special Considerations

  • Imbalanced classes: Use weighted cross-entropy or focal loss
  • Outliers in regression: Use MAE or Huber instead of MSE
  • Probabilistic outputs needed: Ensure loss matches your output (sigmoid/softmax + cross-entropy)

Key Takeaways

Summary
  • Loss functions measure prediction error—lower is better
  • MSE for regression; penalizes large errors heavily
  • Cross-entropy for classification; measures probability distribution mismatch
  • Loss functions connect to Maximum Likelihood Estimation
  • The loss landscape has global minima, local minima, saddle points
  • Choose loss based on task type and output activation
  • Numerical stability matters—use proper implementations
