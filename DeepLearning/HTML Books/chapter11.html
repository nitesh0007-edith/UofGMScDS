Residual Networks

The breakthrough that enabled training networks with hundreds of layers—skip connections and the elegant residual learning framework.

← Previous Next →

The Degradation Problem

Surprisingly, deeper networks can perform worse than shallow ones—not from overfitting, but from optimization difficulty.

⚠️ The Paradox

A 56-layer network has higher training error than a 20-layer network. If deeper could at least copy the shallow solution, this shouldn't happen. Something makes optimization hard.

Skip Connections

The revolutionary idea: add the input directly to the output of a block.

y = F(x) + x

Instead of learning y = H(x), learn the residual F(x) = H(x) - x.

🔑 Why It Works

Easy identity: If F(x) = 0 is optimal, just push weights to zero
Gradient flow: Skip connection provides direct path for gradients
Ensemble effect: Network behaves like ensemble of shallower networks

ResNet Building Block

📖 Basic Block

x → Conv → BN → ReLU → Conv → BN → (+x) → ReLU

The input x is added before the final ReLU.

Bottleneck Block

For deeper networks (ResNet-50+), use 1×1 convolutions to reduce computation:

x → 1×1 Conv (reduce) → 3×3 Conv → 1×1 Conv (expand) → (+x)

ResNet Variants

  • ResNet-18/34: Basic blocks, good for smaller datasets
  • ResNet-50/101/152: Bottleneck blocks, ImageNet standard
  • ResNeXt: Grouped convolutions for efficiency
  • Wide ResNet: Fewer layers, more channels

Impact and Legacy

ResNets fundamentally changed deep learning:

  • Enabled training 100+ layer networks reliably
  • Won ImageNet 2015 with 3.57% error (human ~5%)
  • Skip connections now standard in most architectures
  • Inspired Transformers, U-Nets, and more

Key Takeaways

  • Deep networks suffer degradation, not just overfitting
  • Skip connections: y = F(x) + x (learn residuals)
  • Enables gradient flow through hundreds of layers
  • ResNet-50 is a common baseline for vision tasks
  • Skip connections now ubiquitous in modern architectures
