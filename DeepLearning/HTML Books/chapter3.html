Chapter 3 Part II: Supervised Learning

Shallow Neural Networks

Discover how adding just one hidden layer with nonlinear activation functions creates networks that can approximate any continuous function.

← Previous: Supervised Learning Next: Deep Neural Networks →

From Linear to Neural

Linear models are powerful but limited—they can only represent linear relationships. Real-world patterns are complex and nonlinear. Neural networks solve this by combining linear transformations with nonlinear activation functions.

📖 Definition

Shallow Neural Network: A neural network with one hidden layer between input and output. Despite being "shallow," these networks are universal function approximators—they can approximate any continuous function to arbitrary accuracy.

Network Architecture

A shallow neural network has three layers:

  1. Input Layer: Receives the raw input features x
  2. Hidden Layer: Transforms inputs using weights, biases, and activation functions
  3. Output Layer: Produces the final prediction
Shallow Neural Network Structure

Mathematical Formulation

For an input x, the shallow network computes:

h = a(W₁x + b₁)     [Hidden layer]

y = W₂h + b₂     [Output layer]

where:

  • W₁, b₁ are the weights and biases for the hidden layer
  • a(·) is the activation function (applied element-wise)
  • W₂, b₂ are the weights and biases for the output layer

Activation Functions

The activation function introduces nonlinearity. Without it, stacking multiple linear layers would just produce another linear function.

ReLU (Rectified Linear Unit)

The most popular activation function in modern deep learning:

ReLU(z) = max(0, z)
🔑 Why ReLU is Popular

• Simple: Easy to compute and implement
• Sparse activation: Many neurons output zero, creating efficient representations
• No vanishing gradient: Gradient is 1 for positive inputs (unlike sigmoid/tanh)
• Biological plausibility: Similar to how real neurons fire

Other Common Activations

Sigmoid:

σ(z) = 1 / (1 + e^(-z))

Squashes output to (0, 1). Used for binary classification outputs.

Tanh:

tanh(z) = (e^z - e^(-z)) / (e^z + e^(-z))

Squashes output to (-1, 1). Zero-centered, which can help training.

Leaky ReLU:

LeakyReLU(z) = max(αz, z), where α ≈ 0.01

Allows small gradient for negative inputs, addressing "dying ReLU" problem.

Universal Approximation Theorem

One of the most important theoretical results in neural network research:

📖 Universal Approximation Theorem

A feedforward neural network with a single hidden layer containing a finite number of neurons can approximate any continuous function on a compact subset of ℝⁿ, given appropriate activation functions and sufficient neurons.

This means shallow networks are theoretically capable of learning any reasonable function. However, "can approximate" doesn't mean "easy to train" or "will generalize well."

Limitations in Practice

Despite the universal approximation theorem:

  • The required number of hidden neurons may be exponentially large
  • Training may be difficult due to optimization challenges
  • The network may overfit without proper regularization
  • Deep networks often achieve the same results more efficiently
💡 Intuition: Why It Works

Think of each hidden neuron as creating a "bump" function—it's active for certain inputs and inactive for others. By combining many such bumps with different positions and heights (determined by the output layer weights), we can sculpt any shape we want.

Visualizing Network Behavior

Let's understand how a shallow network with ReLU creates complex functions:

Single Hidden Neuron

With one ReLU neuron, the network creates a "hinge" function—linear with a kink at some point. The position of the kink is determined by the bias.

Multiple Hidden Neurons

Each additional neuron adds another potential kink. With N neurons, we can create a piecewise linear function with up to N+1 linear pieces.

🔑 Key Insight

A shallow ReLU network computes a piecewise linear function. More hidden neurons = more linear pieces = ability to approximate more complex curves. In higher dimensions, these become piecewise linear surfaces.

Effect of Network Width

The number of hidden neurons (network width) controls the model's capacity:

  • Too few neurons: Underfitting—can't capture the pattern
  • Too many neurons: Overfitting risk—can memorize training data
  • Just right: Good generalization to new data

Counting Parameters

Understanding parameter count is crucial for assessing model complexity:

For a network with:

  • D input dimensions
  • H hidden neurons
  • K output dimensions

Parameter Count:

Hidden layer: D × H weights + H biases
Output layer: H × K weights + K biases

Total: D×H + H + H×K + K = H(D + K + 1) + K
💡 Example Calculation

For D=784 (MNIST images), H=256, K=10 (digit classes):

Hidden: 784 × 256 + 256 = 200,960
Output: 256 × 10 + 10 = 2,570

Total: 203,530 parameters

Training Preview

Training a neural network means finding the optimal values for all parameters (weights and biases). The process involves:

  1. Initialize parameters randomly
  2. Forward pass: Compute predictions for training inputs
  3. Loss calculation: Measure how wrong predictions are
  4. Backward pass: Compute gradients of loss w.r.t. parameters
  5. Update: Adjust parameters to reduce loss
  6. Repeat until convergence

Chapters 5-7 will cover loss functions, training algorithms, and backpropagation in detail.

Key Takeaways

Summary
  • Shallow neural networks have one hidden layer between input and output
  • Activation functions introduce essential nonlinearity
  • ReLU is the most popular activation: ReLU(z) = max(0, z)
  • Universal approximation: shallow networks can approximate any continuous function
  • In practice, more neurons = more capacity, but also more overfitting risk
  • With ReLU, networks compute piecewise linear functions
  • Parameter count scales with input dimension × hidden size × output dimension
