Deep Neural Networks

Explore why depth matters—how stacking multiple layers creates exponentially more powerful representations than wider shallow networks.

← Previous: Shallow Networks Next: Loss Functions →

Why Go Deep?

The universal approximation theorem tells us that shallow networks can represent any function. So why do we need deep networks?

🔑 Key Insight

While shallow networks can represent any function, they may need exponentially many neurons. Deep networks can represent the same functions much more efficiently—with far fewer total parameters.

The Efficiency Argument

Consider computing the function f(x) = x × x × x × ... (16 multiplications). A shallow network would need many neurons to approximate this. But a deep network can do it with just 4 layers, each squaring its input.

Layer 1: h₁ = x²
Layer 2: h₂ = h₁² = x⁴
Layer 3: h₃ = h₂² = x⁸
Layer 4: h₄ = h₃² = x¹⁶

This exponential compression is why deep learning is so powerful for complex real-world patterns.

Deep Network Architecture

A deep neural network (DNN) consists of multiple hidden layers stacked together:

h₁ = a(W₁x + b₁)
h₂ = a(W₂h₁ + b₂)
h₃ = a(W₃h₂ + b₃)
⋮
hₗ = a(Wₗhₗ₋₁ + bₗ)
y = Wₒhₗ + bₒ

Each layer transforms the representation from the previous layer, building increasingly abstract features.

Terminology

  • Depth: The number of layers (input → output)
  • Width: The number of neurons in a layer
  • Feedforward: Information flows in one direction (no loops)
  • Fully connected: Every neuron connects to all neurons in adjacent layers

Hierarchical Representations

The magic of deep learning lies in hierarchical feature learning. Each layer learns increasingly abstract representations.

💡 Image Recognition Example

Layer 1: Detects edges (horizontal, vertical, diagonal)
Layer 2: Combines edges into corners, curves, textures
Layer 3: Recognizes object parts (eyes, wheels, windows)
Layer 4: Identifies whole objects (faces, cars, houses)
Layer 5: Classifies scenes and contexts

This hierarchy emerges automatically during training—we don't explicitly program what each layer should learn. The network discovers useful representations on its own.

Compositionality

Deep networks exploit the compositional nature of the world. Complex things are built from simpler parts:

  • Words are composed of letters
  • Sentences are composed of words
  • Faces are composed of eyes, noses, mouths
  • Cars are composed of wheels, windows, doors

Each layer learns one level of this composition, allowing the network to reuse features across different concepts.

Depth vs. Width

Given a fixed parameter budget, should we make networks deeper or wider?

Wider Networks (More Neurons per Layer)

  • More expressive per layer
  • Better at memorizing training data
  • Easier to train (less depth means shorter gradient paths)
  • May require exponentially more parameters for complex functions

Deeper Networks (More Layers)

  • More efficient representation of hierarchical features
  • Better generalization (less memorization)
  • Can capture complex compositions with fewer parameters
  • Harder to train (vanishing/exploding gradients)
🔑 The Goldilocks Principle

Modern networks balance depth and width. Very deep narrow networks are hard to train, while very wide shallow networks are inefficient. The sweet spot depends on the problem and available data.

Challenges with Depth

Training deep networks isn't easy. Several challenges emerge as networks get deeper:

Vanishing Gradients

During backpropagation, gradients are multiplied through each layer. If these multipliers are small (< 1), gradients shrink exponentially:

gradient ∝ (factor)^depth

With sigmoid/tanh activations (which squash values), gradients become tiny in early layers. The network can't learn effective representations in those layers.

⚠️ Vanishing Gradient Problem

For a 10-layer network with factor 0.25 per layer:
Gradient at layer 1 ≈ 0.25¹⁰ ≈ 0.000001

The first layer barely learns anything!

Exploding Gradients

The opposite problem: if multipliers are large (> 1), gradients explode exponentially. This causes unstable training with NaN values.

Solutions

Modern deep learning addresses these with:

  1. ReLU activation: Gradient is 1 for positive inputs (no squashing)
  2. Careful initialization: Set initial weights to maintain variance
  3. Batch normalization: Normalize activations between layers
  4. Residual connections: Add skip connections (Chapter 11)
  5. Gradient clipping: Cap gradient magnitude to prevent explosion

Weight Initialization

Proper initialization is crucial for training deep networks. The goal: keep activation and gradient variances roughly constant across layers.

Xavier/Glorot Initialization

For layers with tanh/sigmoid activation:

W ~ Uniform(-√(6/(nᵢₙ + nₒᵤₜ)), √(6/(nᵢₙ + nₒᵤₜ)))

He Initialization

For layers with ReLU activation:

W ~ Normal(0, √(2/nᵢₙ))

where nᵢₙ is the number of input connections to each neuron.

🔑 Why It Matters

Bad initialization can cause activations to vanish (all zeros) or explode (all huge) in forward pass, making the network impossible to train. Proper initialization gives the network a fighting chance.

Practical Considerations

How Deep Should You Go?

  • Simple problems: 2-4 layers often suffice
  • Image classification: 20-150 layers (ResNets)
  • NLP/Transformers: 12-96 layers

Common Architectures

  • MLP: Fully connected layers, general purpose
  • CNN: Convolutional layers for images (Chapter 10)
  • ResNet: Deep networks with skip connections (Chapter 11)
  • Transformer: Attention-based architecture (Chapter 12)

Parameter Efficiency

Deep networks have a parameter advantage. A 10-layer network with 100 neurons per layer has ~100,000 parameters but can represent functions that a shallow network would need millions of parameters for.

Key Takeaways

Summary
  • Depth allows exponentially more efficient representations than width
  • Deep networks learn hierarchical features automatically
  • Each layer builds increasingly abstract representations
  • Vanishing/exploding gradients are key challenges with depth
  • ReLU activation, proper initialization, and normalization help
  • Balance depth and width based on your problem and data
  • Modern deep networks range from 10 to 100+ layers
