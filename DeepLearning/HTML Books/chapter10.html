Convolutional Networks

The architecture that revolutionized computer vision—learn how CNNs exploit spatial structure with local connectivity and weight sharing.

← Previous Next →

Why CNNs for Images?

Fully connected layers for images are problematic:

  • A 224×224 RGB image has 150,528 pixels
  • FC layer with 1000 neurons = 150 million parameters!
  • Ignores spatial structure of images
🔑 Key Insights

Local connectivity: Pixels nearby are more related
Translation invariance: A cat is a cat regardless of position
Hierarchical features: Edges → shapes → parts → objects

The Convolution Operation

A filter/kernel slides across the input, computing dot products:

output[i,j] = Σₘ Σₙ input[i+m, j+n] × kernel[m, n]

Key Properties

  • Local connectivity: Each output depends only on a small region
  • Weight sharing: Same filter applied everywhere
  • Translation equivariance: Shift input → shift output
💡 Convolution Example

A 3×3 edge detection filter:
[[-1, 0, 1], [-2, 0, 2], [-1, 0, 1]]

Sliding this across an image highlights vertical edges!

CNN Building Blocks

Convolutional Layer

Multiple filters produce multiple output channels (feature maps). Each filter detects a different pattern.

Pooling Layer

Downsamples spatial dimensions while keeping channels:

  • Max pooling: Take maximum in each window
  • Average pooling: Take average in each window

Provides translation invariance and reduces computation.

Stride and Padding

  • Stride: How far filter moves each step (stride=2 halves spatial size)
  • Padding: Add zeros around border to control output size

Classic Architectures

LeNet (1989)

First successful CNN. Conv → Pool → Conv → Pool → FC

AlexNet (2012)

Deeper, ReLU activation, dropout. Won ImageNet dramatically.

VGGNet (2014)

Very deep (16-19 layers), only 3×3 convolutions. Simple and elegant.

📖 Typical CNN Pattern

[Conv → ReLU → Pool] × N → Flatten → [FC → ReLU] × M → Output

Spatial size decreases, channel count increases as we go deeper.

Parameter Efficiency

CNNs are much more parameter-efficient than FC networks:

💡 Comparison

Input: 224×224×3 image

FC layer (1000 outputs): 150M parameters
Conv layer (64 filters, 3×3): 1,728 parameters

Over 80,000× fewer parameters!

Key Takeaways

  • CNNs exploit spatial structure with local connectivity
  • Weight sharing makes them parameter-efficient
  • Convolution: slide filter across input, compute dot products
  • Pooling: downsample spatially, provide invariance
  • Deep CNNs learn hierarchical features automatically
  • Classic pattern: Conv-ReLU-Pool repeated, then FC layers
