Chapter 2 Part II: Supervised Learning

Supervised Learning

Learn the fundamental paradigm of supervised learning: using labeled examples to train models that can make predictions on new, unseen data.

← Previous: Introduction Next: Shallow Neural Networks →

What is Supervised Learning?

Supervised learning is a type of machine learning where we have input-output pairs and want to learn a function that maps inputs to outputs. The "supervision" comes from the labeled examples we use for training.

📖 Definition

Supervised Learning: Given a dataset of input-output pairs {(x₁, y₁), (x₂, y₂), ..., (xₙ, yₙ)}, find a function f such that f(x) ≈ y for all training examples, and more importantly, generalizes well to new, unseen inputs.

The Two Main Tasks

Supervised learning problems fall into two categories based on the type of output:

  • Regression: The output y is continuous (a real number). Example: predicting house prices.
  • Classification: The output y is discrete (a category). Example: classifying emails as spam or not spam.

Regression

In regression, we predict a continuous value. The simplest form is linear regression, where we assume the relationship between input x and output y is linear.

Linear Regression Model

For a single input variable, the linear model is:

f(x) = wx + b

where:

  • w is the weight (slope) - how much y changes when x changes
  • b is the bias (intercept) - the value of y when x = 0
💡 Example: House Price Prediction

Suppose we want to predict house prices based on square footage:

Model: price = w × (square_feet) + b
If: w = 200 (dollars per sq ft), b = 50,000 (base price)
Then: A 1,500 sq ft house costs 200 × 1,500 + 50,000 = $350,000

Multiple Input Variables

With multiple inputs (features), the model becomes:

f(x) = w₁x₁ + w₂x₂ + ... + wₙxₙ + b = wᵀx + b

In vector notation, w is a vector of weights and x is a vector of input features. This is still called "linear" regression because the output is a linear combination of inputs.

Nonlinear Relationships

Real-world relationships are rarely perfectly linear. We can model nonlinear relationships in two ways:

  1. Feature Engineering: Add polynomial features (x², x³, etc.) or other transformations
  2. Neural Networks: Use multiple layers with nonlinear activations (covered in later chapters)

Classification

In classification, we assign inputs to discrete categories. The simplest case is binary classification with two classes.

Binary Classification

We can't directly use linear regression for classification because it outputs any real number, while we need probabilities between 0 and 1.

The solution is logistic regression, which applies the sigmoid function to squash outputs into [0, 1]:

P(y=1|x) = σ(wᵀx + b) = 1 / (1 + e^(-(wᵀx + b)))
📖 The Sigmoid Function

The sigmoid function σ(z) = 1/(1+e^(-z)) maps any real number to (0, 1):
• Large positive z → output near 1
• Large negative z → output near 0
• z = 0 → output = 0.5

Sigmoid Function Visualization

Multi-Class Classification

When we have more than two classes, we use the softmax function, which converts a vector of scores into probabilities that sum to 1:

P(y=k|x) = e^(zₖ) / Σⱼe^(zⱼ)

where zₖ is the score (logit) for class k. Softmax ensures all probabilities are positive and sum to 1.

💡 Example: Digit Recognition

For recognizing handwritten digits (0-9), the model outputs 10 scores. Softmax converts these to probabilities:

Scores: [2.0, 0.5, 0.1, 0.3, 0.2, 0.1, 0.4, 8.0, 0.3, 0.1]
Softmax: [0.002, 0.001, ..., 0.99, ...]
Prediction: Class 7 (highest probability)

The Model as a Function

In both regression and classification, we're learning a function f(x, φ) where:

  • x is the input data
  • φ (phi) represents all the learnable parameters (weights and biases)

The goal of training is to find the optimal parameters φ* that make our model's predictions as close as possible to the true outputs.

🔑 Key Insight

A model is just a parameterized function. Training is the process of finding good values for those parameters. The architecture (linear, neural network, etc.) determines what kinds of functions the model can represent.

Model Capacity

Different models can represent different sets of functions. A model's capacity refers to how complex the functions it can learn are:

  • Linear models: Can only represent linear functions (low capacity)
  • Polynomial models: Can represent polynomial functions
  • Neural networks: Can approximate any continuous function (high capacity)

We'll explore model capacity and its implications in Chapter 8 (Measuring Performance) and Chapter 9 (Regularization).

Training Overview

Training a supervised learning model involves three key components:

1. Loss Function

A loss function measures how wrong our predictions are. We want to minimize this. Common choices:

  • Mean Squared Error (MSE) for regression
  • Cross-Entropy Loss for classification

Chapter 5 covers loss functions in detail.

2. Optimization Algorithm

We need an algorithm to find parameters that minimize the loss. The most common approach is gradient descent:

  1. Start with random parameters
  2. Compute the gradient (direction of steepest increase)
  3. Move parameters in the opposite direction
  4. Repeat until convergence

Chapter 6 covers training algorithms in detail.

3. Evaluation

We need to measure how well our model performs on data it hasn't seen during training. This tells us if the model has truly learned useful patterns or just memorized the training data.

Chapter 8 covers performance measurement.

Generalization

The ultimate goal of supervised learning is generalization: performing well on new, unseen data, not just the training data.

⚠️ Overfitting vs. Underfitting

Overfitting: The model learns the training data too well, including noise and irrelevant patterns. It performs great on training data but poorly on new data.

Underfitting: The model is too simple to capture the underlying patterns. It performs poorly on both training and test data.

Finding the right balance between overfitting and underfitting is a central challenge in machine learning. We'll explore this in detail in later chapters.

Train-Test Split

To evaluate generalization, we split our data into:

  • Training set: Used to learn the model parameters
  • Validation set: Used to tune hyperparameters and prevent overfitting
  • Test set: Used only once, at the end, to report final performance

Key Takeaways

Summary
  • Supervised learning uses labeled input-output pairs to learn a mapping function
  • Regression predicts continuous values; classification predicts discrete categories
  • Linear regression models: f(x) = wᵀx + b
  • Logistic regression adds sigmoid for binary classification
  • Softmax extends to multi-class classification
  • Training minimizes a loss function using optimization algorithms
  • Generalization to unseen data is the ultimate goal
  • Overfitting and underfitting are key challenges to balance
