Chapter 9

Regularization

Fight overfitting with techniques like dropout, weight decay, and data augmentation—essential tools for training models that generalize.

← Previous Next →

The Overfitting Problem

Deep networks have enormous capacity—they can memorize training data instead of learning patterns. Regularization techniques constrain the model to prefer simpler solutions.

📖 Regularization

Any technique that helps a model generalize better to unseen data, typically by constraining model complexity or introducing noise during training.

Weight Decay (L2 Regularization)

Add a penalty on large weights to the loss:

L_reg = L + λ Σ w²

This encourages smaller weights, leading to smoother functions. λ controls regularization strength.

🔑 L1 vs L2

L2 (Ridge): Shrinks weights toward zero (smoother)
L1 (Lasso): Drives weights exactly to zero (sparse)
Typical λ values: 1e-4 to 1e-2

Dropout

Randomly set neurons to zero during training with probability p:

  • Prevents neurons from co-adapting
  • Acts like training an ensemble of sub-networks
  • At test time, use all neurons but scale by (1-p)
💡 Typical Usage

• p = 0.5 for hidden layers
• p = 0.1-0.2 for input layer
• Not used after convolutional layers (use batch norm instead)
• Disable during evaluation: model.eval()

Batch Normalization

Normalize activations within each mini-batch:

ĥ = (h - μ_batch) / σ_batch
output = γĥ + β

Benefits: stabilizes training, allows higher learning rates, provides regularization effect.

Data Augmentation

Artificially expand training data by applying transformations:

For Images

  • Random crops, flips, rotations
  • Color jittering, brightness changes
  • Cutout, MixUp, CutMix

For Text

  • Synonym replacement
  • Back-translation
  • Random insertion/deletion
🔑 Key Insight

Data augmentation is often the most effective regularization technique. It directly addresses the root cause of overfitting: insufficient training data diversity.

Early Stopping

Stop training when validation loss stops improving:

  1. Monitor validation loss each epoch
  2. Save model when validation loss improves
  3. Stop if no improvement for N epochs (patience)
  4. Restore best saved model

Key Takeaways

  • Regularization prevents overfitting by constraining model complexity
  • Weight decay (L2) penalizes large weights
  • Dropout randomly zeros neurons during training
  • Batch normalization stabilizes and regularizes
  • Data augmentation is highly effective—expand your data
  • Early stopping prevents training too long
  • Combine multiple techniques for best results
