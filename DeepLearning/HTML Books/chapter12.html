The Attention Mechanism

Attention allows each position to look at and gather information from all other positions.

📖 Self-Attention

For each token, compute a weighted sum of all token representations, where weights depend on relevance (similarity).

Query, Key, Value

Each input is projected into three vectors:

  • Query (Q): "What am I looking for?"
  • Key (K): "What do I contain?"
  • Value (V): "What information do I provide?"
Attention(Q, K, V) = softmax(QKᵀ / √d) V

Multi-Head Attention

Run attention multiple times in parallel with different projections:

🔑 Why Multiple Heads?

Different heads can attend to different types of relationships:
• Head 1: Syntactic relationships
• Head 2: Semantic similarity
• Head 3: Positional patterns
Typical: 8-16 heads

Transformer Architecture

Encoder Block

Multi-Head Attention → Add & Norm → FFN → Add & Norm

Decoder Block

Masked Self-Attention → Cross-Attention → FFN (each with residuals)

Positional Encoding

Since attention is permutation-invariant, we add position information:

PE(pos, 2i) = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))

Transformer Variants

  • BERT: Encoder-only, bidirectional, for understanding
  • GPT: Decoder-only, autoregressive, for generation
  • T5: Encoder-decoder, text-to-text framework
  • Vision Transformer (ViT): Transformers for images

Scaling Laws

Transformer performance scales predictably with:

  • Parameters: More layers, wider FFN
  • Data: More training tokens
  • Compute: More training FLOPs

This predictability enabled GPT-3, GPT-4, and modern LLMs.

Key Takeaways

  • Attention: weighted sum based on query-key similarity
  • Self-attention lets each position see all others
  • Multi-head attention captures different relationship types
  • Positional encoding adds sequence order information
  • Transformers scale predictably with size
  • Foundation for GPT, BERT, and modern AI
