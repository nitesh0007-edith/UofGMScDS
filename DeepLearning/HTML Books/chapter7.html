Backpropagation

The algorithm that makes deep learning possible—efficiently computing gradients through complex networks using the chain rule.

← Previous Next →

The Problem

To train a network, we need gradients of the loss with respect to every parameter. With millions of parameters and complex compositions, how do we compute this efficiently?

📖 Backpropagation

Backpropagation: An algorithm that efficiently computes gradients by propagating error signals backward through the network, reusing intermediate computations via the chain rule.

The Chain Rule

The mathematical foundation of backprop. For composed functions f(g(x)):

∂f/∂x = (∂f/∂g) · (∂g/∂x)

For a chain of functions f₁ → f₂ → ... → fₙ → L:

∂L/∂f₁ = (∂L/∂fₙ) · (∂fₙ/∂fₙ₋₁) · ... · (∂f₂/∂f₁)
💡 Simple Example

Let L = (wx + b)²

Forward: z = wx + b, L = z²
Backward: ∂L/∂z = 2z, ∂z/∂w = x
Result: ∂L/∂w = 2z · x = 2(wx + b) · x

Computational Graphs

Neural networks are represented as directed acyclic graphs (DAGs):

  • Nodes: Operations (add, multiply, activation)
  • Edges: Data flow between operations

Forward Pass

Compute outputs by traversing the graph from inputs to loss.

Backward Pass

Compute gradients by traversing from loss back to parameters, applying chain rule at each node.

The Backprop Algorithm

🔑 Step by Step

1. Forward pass: Compute and cache all intermediate values
2. Compute loss: Evaluate L at the output
3. Initialize: ∂L/∂L = 1
4. Backward pass: For each layer from output to input:
   • Compute local gradients using cached values
   • Multiply by incoming gradient (chain rule)
   • Pass gradient to previous layer
5. Update: Use computed ∂L/∂φ to update parameters

Gradients for Common Layers

Linear Layer: y = Wx + b

∂L/∂W = (∂L/∂y) · xᵀ
∂L/∂b = ∂L/∂y
∂L/∂x = Wᵀ · (∂L/∂y)

ReLU: y = max(0, x)

∂L/∂x = (∂L/∂y) · 𝟙(x > 0)

Gradient passes through where input was positive, blocked where negative.

Softmax + Cross-Entropy

Combined gradient has elegant form:

∂L/∂z = ŷ - y

Just the difference between prediction and target!

Automatic Differentiation

Modern frameworks (PyTorch, TensorFlow) implement backprop automatically:

Python (PyTorch)
# Define computation
x = torch.tensor([1.0], requires_grad=True)
y = x ** 2 + 2 * x + 1
    
# Compute gradients automatically
y.backward()
print(x.grad)  # dy/dx = 2x + 2 = 4
🔑 You Don't Need to Implement Backprop!

Frameworks handle gradient computation automatically. Understanding the concept helps with debugging and designing custom layers, but you won't write backprop from scratch.

Key Takeaways

  • Backpropagation efficiently computes gradients using the chain rule
  • Forward pass: compute outputs and cache intermediates
  • Backward pass: propagate gradients from loss to parameters
  • Each layer has local gradient formulas
  • Modern frameworks do this automatically (autograd)
  • Understanding helps with debugging and custom implementations
